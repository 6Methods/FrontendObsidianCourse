**Заголовок:** `# Подключение CSS и синтаксис`

**`## Зачем это нужно`:** Прежде чем писать стили, надо понять как подключить файл к HTML и как устроено CSS-правило. Это 5 минут теории, которые закроют «а почему у меня ничего не применяется».

**`## Главное` (~250–320 слов), подразделы `### ...`:**
- **3 способа подключения**:
    - **External (рекомендовано)** — отдельный файл `style.css`, подключается `<link rel="stylesheet" href="style.css">` в `<head>`. Стандарт индустрии, единственный правильный выбор для проектов.
    - **Internal** — `<style>...</style>` прямо в head. Годится для быстрых демо.
    - **Inline** — `style="..."` на конкретном теге. Плохая практика (кроме случаев, когда значение вычисляется в JS).
- **Синтаксис правила:** `селектор { свойство: значение; }`. Точка с запятой после каждого значения. Фигурные скобки обязательны.
- **Свойство: значение** — пары. Тысячи свойств, но 90% вёрстки — это ~30 из них (`color`, `background`, `width`, `height`, `margin`, `padding`, `display`, `font-*`, `border`, `flex`, `grid`...).
- **Комментарии** — `/* текст */`. Используй экономно, не комментируй очевидное.
- **Единицы измерения** — `px` (фиксированные), `%` (от родителя), `em`/`rem` (от размера шрифта), `vh`/`vw` (от размера окна). Пока достаточно знать `px`, `%`, `rem` — остальные по мере появления.
- **Как отлаживать** — DevTools → Elements → панель Styles. Там видно, какое правило применилось, а какое перечёркнуто (перебито каскадом). Главный инструмент CSS-разработчика.

**`## Пример кода`** — два блока: html и css.

html:
```html
<!-- index.html -->
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>CSS демо</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <h1 class="greeting">Привет, CSS!</h1>
  </body>
</html>
```

css:
```css
/* style.css */
body {
  font-family: Arial, sans-serif;
  margin: 0;
}

.greeting {
  color: #2d6cdf;
  font-size: 32px;
}
```

**`## Частые ошибки`** — 3 пункта:
- Забыть точку с запятой после значения → следующее правило в блоке игнорируется.
- Подключить `style.css` не в `<head>`, а в `<body>` в конце → страница «мигает» без стилей при загрузке.
- Использовать inline-стили везде → невозможно переиспользовать, сложно поддерживать, конфликтуют с каскадом.

**`## Задания`:**
- [ ] Создай в папке с HTML-резюме файл `style.css`.
- [ ] Подключи его через `<link>` в `<head>`.
- [ ] Задай `body` шрифт `Arial, sans-serif` и `color: #222`.
- [ ] В DevTools → Elements выбери тег `body` и убедись, что твои правила применились (видно в панели Styles).

**Заголовок:** `# Селекторы`

**`## Зачем это нужно`:** Селекторы — это то, как ты «указываешь пальцем» на элементы страницы. Без них нельзя применить стиль. Умение точно выбирать — половина CSS.

**`## Главное` (~320–400 слов):** подразделы `### ...`:
- **Селекторы по тегу** — `p`, `h1`, `a`. Применяется ко всем таким тегам. Использовать редко — обычно слишком широко.
- **Класс `.название`** — основной рабочий инструмент. `.card`, `.btn-primary`. Один класс можно навешивать на любое число элементов, один элемент — на несколько классов (через пробел в атрибуте: `class="btn btn-primary"`).
- **ID `#название`** — уникальный идентификатор, только один элемент на странице. Для стилей почти не используется — слишком специфичен (объясняется в уроке 03.3). Оставь id для якорных ссылок и JS.
- **Селектор по атрибуту** — `[type="email"]`, `input[required]`, `a[href^="https://"]`. Полезно для форм и ссылок.
- **Универсальный `*`** — все элементы. Используется в сбросах стилей (`* { box-sizing: border-box; }`).
- **Комбинаторы:**
    - `A B` (потомки, через пробел) — все `B` внутри `A` на любой глубине.
    - `A > B` (прямые дети) — только непосредственные дети.
    - `A + B` (соседний сразу после) — `B`, идущий следом за `A`.
    - `A ~ B` (все следующие соседи).
- **Псевдо-классы `:hover`, `:focus`, `:active`, `:disabled`, `:checked`, `:first-child`, `:last-child`, `:nth-child(odd/even/3n+1)`** — состояния и позиции элементов.
- **Псевдо-элементы `::before`, `::after`, `::placeholder`, `::selection`** — вставка контента до/после элемента, стилизация частей. Классика: иконка перед ссылкой через `::before`.
- **Группировка** — `.card, .tile, .block { ... }` через запятую: одинаковые стили для нескольких селекторов.

**`## Пример кода`** — блок css:
```css
/* тег */
h1 { color: navy; }

/* класс */
.card { padding: 16px; border: 1px solid #ddd; }

/* дочерние и потомки */
.card > h2 { margin-top: 0; }       /* только прямой ребёнок */
.card h2 { color: #333; }            /* любой на глубине */

/* псевдо-класс */
a:hover { color: crimson; }
input:focus { outline: 2px solid dodgerblue; }

/* чётные строки таблицы */
tr:nth-child(even) { background: #f5f5f5; }

/* псевдо-элемент */
.external-link::after {
  content: " ↗";
  color: gray;
}

/* группировка */
h1, h2, h3 { font-family: Georgia, serif; }
```

**`## Частые ошибки`** — 4 пункта:
- Ставить стили на id вместо классов → невозможно переиспользовать, ломается специфичность.
- Писать много тегов (`div p span a { ... }`) вместо нормального класса → хрупко и медленно.
- Путать `A B` и `A > B` → `A B` ловит всех потомков, `A > B` — только детей.
- Забывать `::` у псевдо-элементов (`:before` устарело, но в старом коде встречается).

**`## Задания`:**
- [ ] В `style.css` для резюме задай: цвет `h1`, стилизацию класса `.btn` (если есть), `a:hover` с изменением цвета, `input:focus` с видимой рамкой.
- [ ] Сделай `::before` у одной из ссылок с каким-то символом-иконкой (`→`, `✉`).
- [ ] Используй `nth-child(even)` на списке навыков, чтобы чётные пункты были с другим фоном.

**`## Что почитать`:**
- [MDN — CSS selectors](https://developer.mozilla.org/en-US/docs/Learn/CSS/Building_blocks/Selectors)
- [MDN — Pseudo-classes and pseudo-elements](https://developer.mozilla.org/en-US/docs/Learn/CSS/Building_blocks/Selectors/Pseudo-classes_and_pseudo-elements)

**Заголовок:** `# Каскад, специфичность, наследование`

**`## Зачем это нужно`:** «У меня правило написано, а не применяется» — вечная проблема новичков. Решение — понимать как браузер выбирает между конкурирующими правилами. Это каскад.

**`## Главное` (~300–380 слов):** подразделы `### ...`:
- **Каскад** — порядок, в котором браузер решает, какое правило применить, когда их несколько на один элемент. Три критерия в порядке важности: **источник и слой → специфичность → порядок в файле**.
- **Специфичность** — «вес» селектора. Считается как четыре числа (`a, b, c, d`):
    - `a` — inline-стили (`style="..."`) = 1, иначе 0.
    - `b` — количество id-селекторов.
    - `c` — количество классов, атрибутов, псевдо-классов.
    - `d` — количество тегов и псевдо-элементов.
    - Сравнивается лексикографически: сначала `a`, потом `b`, и т.д. Пример: `.card h2` — (0,0,1,1); `#hero h2` — (0,1,0,1). Второе сильнее.
- **Порядок в файле** — если специфичность равна, применяется то правило, что идёт **позже** в CSS.
- **`!important`** — ядерная кнопка, поднимает правило выше всех остальных (кроме другого `!important` и inline с `!important`). Почти никогда не используй — это признак, что каскад запутался. Исключения: перебить стили библиотеки или юнит-тест.
- **Наследование** — некоторые свойства автоматически наследуются от родителя (color, font-*, line-height, visibility). Другие — нет (margin, padding, border, width). Если нужно принудительно наследовать: `property: inherit`. Если сбросить к дефолту браузера: `property: initial` или `unset`.
- **Дефолт браузера** — у каждого тега есть встроенные стили (например, `h1` крупный и жирный, `ul` с отступом). Популярная практика — в самом начале сбрасывать это через reset или normalize (или `* { box-sizing: border-box; margin: 0; }`).

**`## Пример кода`** — блок css с конфликтом, объясняющим каскад:
```css
/* специфичность (0,0,1,1) */
.card h2 { color: blue; }

/* специфичность (0,0,0,1) — слабее, не применится */
h2 { color: red; }

/* inline стиль (1,0,0,0) — победит всех */
/* <h2 style="color: green;">...</h2> */

/* !important — поднимает выше обычных правил */
.card h2 { color: purple !important; }

/* наследование */
body {
  color: #222;
  font-family: Arial, sans-serif;
}
/* .card h2 унаследует font-family, если не перебить */
```

**`## Частые ошибки`** — 4 пункта:
- Использовать `!important` для «быстро пофиксить» → через неделю кода ничем не отладить.
- Писать стили через id → специфичность быстро становится неуправляемой.
- Надеяться, что новое правило «поверх старого» перебьёт, игнорируя специфичность → самая частая причина «почему не применяется».
- Накапливать глубокие селекторы (`.page .content .card .header h2`) ради специфичности — лучше переделать архитектуру классов.

**`## Задания`:**
- [ ] В `style.css` сделай два конфликтующих правила на один элемент — через разные селекторы с разной специфичностью. Убедись в DevTools → Computed, кто победил.
- [ ] Поставь одно правило через id и одно через класс — посмотри порядок применения.
- [ ] В `body` задай `font-family` и `color`. Проверь в DevTools → Computed, что `<p>`, `<h1>` и `<li>` наследуют их (и что `margin` — не наследуется).

**Заголовок:** `# Box model и позиционирование`

**`## Зачем это нужно`:** Каждый элемент страницы — это прямоугольная коробка (box). Box model объясняет из чего она состоит и почему элемент «шире, чем я задавал». Позиционирование отвечает на вопрос «как сдвинуть элемент на странице».

**`## Главное` (~320–400 слов):** подразделы `### ...`:
- **Box model — 4 слоя:**
    - **content** — то место, где лежит текст/картинка (`width`/`height` управляют этим).
    - **padding** — внутренний отступ от контента до рамки.
    - **border** — рамка.
    - **margin** — внешний отступ от других элементов.
- **`box-sizing: content-box` vs `border-box`** — по умолчанию `content-box`: `width: 200px` — это только content, реальная ширина = 200 + padding + border. Включи `border-box` глобально (`* { box-sizing: border-box; }`) — тогда `width: 200px` считает ширину **вместе с padding и border**. Делает вёрстку предсказуемой, индустрия использует именно так.
- **margin collapse** — два вертикальных margin (нижний у одного, верхний у следующего) **схлопываются** в один, равный большему. Знай — это не баг, это фича. Обход: обернуть в flex/grid или дать padding.
- **`display`** — `block`, `inline`, `inline-block`, `none`. `block` занимает всю строку (`<div>`, `<p>`). `inline` — в потоке текста, игнорирует `width`/`height` (`<span>`, `<a>`). `inline-block` — в потоке, но принимает размеры. `none` — удаляет из документа (занимает 0 места). Позже будут `flex` и `grid`.
- **Позиционирование:**
    - `static` (по умолчанию) — в потоке, `top`/`left` игнорируются.
    - `relative` — в потоке, но `top`/`left` смещают визуально (оставляя исходное место). Обычно используется как «якорь» для `absolute`-детей.
    - `absolute` — **выпадает из потока**. Привязывается к ближайшему родителю с не-`static` position. Для модалок, тултипов, абсолютного позиционирования внутри карточки.
    - `fixed` — как absolute, но привязан к viewport. Не двигается при скролле (шапка сайта, кнопка наверх).
    - `sticky` — гибрид: ведёт себя как `relative`, но «прилипает» к заданному `top` при скролле.
- **`z-index`** — только для элементов с position ≠ static (или flex/grid-детей). Чем больше число, тем выше слой.

**`## Пример кода`** — блок css:
```css
/* глобальный border-box — делаем всегда */
*,
*::before,
*::after {
  box-sizing: border-box;
}

body { margin: 0; }

.card {
  width: 300px;
  padding: 16px;
  border: 1px solid #ddd;
  margin: 24px;
  /* реальная ширина = 300px благодаря border-box */
}

/* relative-родитель для absolute-ребёнка */
.card {
  position: relative;
}
.card .badge {
  position: absolute;
  top: 8px;
  right: 8px;
  background: crimson;
  color: white;
  padding: 4px 8px;
}

/* шапка, прилипающая при скролле */
.site-header {
  position: sticky;
  top: 0;
  background: white;
  z-index: 10;
}
```

**`## Частые ошибки`** — 4 пункта:
- Не переключить `box-sizing` → каждый `padding` «ломает» ширину.
- `position: absolute` без `relative`-родителя → элемент прилипает к `<html>`, хаос.
- Использовать большие `z-index` (999, 9999) на всё подряд → потом невозможно отладить слои.
- Думать, что `margin` удаляет элемент из потока — нет, это делает только `absolute`/`fixed`/`display:none`.

**`## Задания`:**
- [ ] В глобальном CSS включи `*, *::before, *::after { box-sizing: border-box; }` и `body { margin: 0; }`.
- [ ] Открой в DevTools → Computed панель «Box model» для разных элементов, посмотри визуально слои.
- [ ] Сверстай карточку с «бейджем» в правом верхнем углу (`position: absolute` внутри `position: relative`-родителя).

**Заголовок:** `# Flexbox`

**`## Зачем это нужно`:** Flexbox — главный инструмент вёрстки на последние 10 лет. Он решает задачу «расположи элементы в ряд/колонку и распредели пространство». 80% интерфейсов сегодня — это flex-контейнеры вложенные в другие flex-контейнеры.

**`## Главное` (~320–400 слов):** подразделы `### ...`:
- **Модель — контейнер и дети.** `display: flex` ставится **на родителя** (flex-container). Все прямые дети автоматически становятся flex-items и раскладываются по правилам контейнера.
- **Оси:**
    - **main axis** — основная ось, по ней раскладываются items. По умолчанию — горизонтальная (слева направо).
    - **cross axis** — перпендикулярная ей.
    - Направление меняется через `flex-direction: row | row-reverse | column | column-reverse`.
- **Свойства контейнера:**
    - `justify-content` — раскладка по main axis: `flex-start` (слева), `center`, `flex-end`, `space-between` (по краям и равные промежутки), `space-around`, `space-evenly`.
    - `align-items` — раскладка по cross axis: `stretch` (по умолчанию — растягивает), `center`, `flex-start`, `flex-end`, `baseline`.
    - `flex-wrap: nowrap | wrap` — перенос строк, если items не влезают.
    - `gap` — отступ между items. Современный способ, заменяет костыли с `margin`.
    - `align-content` — как распределить **строки**, когда их несколько (после wrap).
- **Свойства item-а:**
    - `flex: 0 1 auto` (по умолчанию) — сокращение от `flex-grow flex-shrink flex-basis`.
    - `flex: 1` — занять всё доступное пространство. Самое частое.
    - `flex-basis` — стартовая ширина.
    - `align-self` — перебивает `align-items` у родителя для одного item-а.
    - `order` — менять порядок без перестановки HTML.
- **Типичные шаблоны:**
    - **Ряд с элементом справа:** `justify-content: space-between` + `gap`. Шапка сайта: логотип слева, навигация справа.
    - **Центр по обеим осям:** `display: flex; justify-content: center; align-items: center;` (классика «центрирование на div»).
    - **Колонка, все на всю ширину:** `display: flex; flex-direction: column; gap: 16px;`.

**`## Пример кода`** — блок css + html:
```html
<header class="site-header">
  <div class="logo">MySite</div>
  <nav class="nav">
    <a href="/">Главная</a>
    <a href="/about">О нас</a>
    <a href="/contact">Контакты</a>
  </nav>
</header>
```

```css
.site-header {
  display: flex;
  justify-content: space-between;
  align-items: center;
  gap: 16px;
  padding: 16px 24px;
}

.nav {
  display: flex;
  gap: 24px;
}

/* центрирование карточки */
.hero {
  display: flex;
  justify-content: center;
  align-items: center;
  min-height: 60vh;
}
```

**`## Частые ошибки`** — 4 пункта:
- Ставить `display: flex` на сами items вместо родителя → ничего не работает.
- Использовать `margin: auto` между items, когда есть `gap` → современно всегда `gap`.
- Забывать `flex-wrap: wrap` → на маленьком экране items выдавливают друг друга.
- Путать `justify-content` и `align-items` → запомни: justify = main axis, align = cross axis.

- [ ] Сделай центрирование блока (как `display: flex; justify-content: center; align-items: center`).


**Заголовок:** `# Grid и адаптив`

**`## Зачем это нужно`:** Flexbox — это одна ось (ряд или колонка). Grid — двумерная сетка: строки × колонки. А медиа-запросы позволяют менять раскладку в зависимости от ширины экрана. Вдвоём они закрывают всю задачу «как страница выглядит на десктопе, планшете и мобильнике».

**`## Главное` (~340–400 слов):** подразделы `### ...`:
- **Grid vs Flexbox — когда что.** Flex — раскладка по одной оси (шапка, тулбар, список карточек в ряд). Grid — сетки (страница с сайдбаром, галерея, дашборд). Правило из опыта: если чертёж напоминает таблицу — это Grid.
- **Базовый Grid:** `display: grid; grid-template-columns: 1fr 1fr 1fr; gap: 16px;` — 3 равные колонки с отступом.
- **Единица `fr`** — «доля оставшегося пространства». `1fr 2fr` — колонки в соотношении 1:2. Удобнее процентов, потому что автоматически учитывает `gap`.
- **`repeat()` и `minmax()`:**
    - `grid-template-columns: repeat(3, 1fr)` — сокращение для `1fr 1fr 1fr`.
    - `grid-template-columns: repeat(auto-fit, minmax(240px, 1fr))` — магическая формула адаптивной галереи: колонки минимум 240px, умещаются сколько влезет, на узком экране перестраиваются. Знай эту строчку наизусть.
- **Размещение элементов** — базово grid сам раскладывает в порядке HTML. Если нужно — `grid-column: 1 / 3` (с 1 по 3 линию), `grid-row: 1 / span 2`.
- **Медиа-запросы (`@media`):**
    - `@media (min-width: 768px) { ... }` — применять правила, когда экран **шире** 768px (mobile-first подход).
    - `@media (max-width: 767px) { ... }` — применять на экранах **уже** 767px (desktop-first). В современном фронте пишут mobile-first.
- **Mobile-first логика.** Сначала стили для мобилки (узкий экран), потом через `min-width` «дорисовываешь» десктопную раскладку. Почему: мобилок в мире больше, CSS получается короче и чище.
- **Популярные breakpoint-ы** — индустрия не договорилась, но часто используют: 640px (tablet), 768px, 1024px (desktop), 1280px (large). Tailwind, Bootstrap — почти все берут похожие цифры.

**`## Пример кода`** — блок css:
```css
/* адаптивная галерея карточек */
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
  gap: 16px;
}

/* лендинг с сайдбаром — mobile-first */
.layout {
  display: grid;
  grid-template-columns: 1fr;          /* на мобилке: одна колонка */
  gap: 24px;
}

@media (min-width: 768px) {
  .layout {
    grid-template-columns: 240px 1fr;  /* на планшете+: сайдбар + контент */
  }
}

@media (min-width: 1200px) {
  .layout {
    grid-template-columns: 280px 1fr 320px;  /* на большом экране: 3 колонки */
  }
}
```

**`## Частые ошибки`** — 4 пункта:
- Использовать `px` для колонок (`grid-template-columns: 200px 300px 400px`) → не адаптивно. Используй `fr` и `minmax`.
- Писать `max-width` везде вместо `min-width` → легко запутаться, когда какое правило срабатывает. Выбери один подход и держись.
- Забыть `<meta name="viewport">` в HTML → `@media` запросы на мобилке не срабатывают.
- Мешать Flex и Grid в одном месте, когда хватит одного → усложняет отладку. Выбирай по задаче.

**`## Задания`:**

- [ ] Сделай двухколонный layout страницы-резюме: слева узкая колонка с фото и контактами, справа основной контент. На экране уже 768px — в одну колонку.

- [ ] Пройди [Grid Garden](https://cssgridgarden.com/) — тренажёр, аналог Flexbox Froggy.
