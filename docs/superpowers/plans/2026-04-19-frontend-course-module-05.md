**Заголовок:** `# Переменные, типы, операторы`

**`## Зачем это нужно`:** Любая программа хранит данные и что-то с ними делает. Переменные — это «именованные коробки» для значений, типы говорят, что в коробке лежит, операторы — что можно делать. Без этого ни одна строка JS не напишется.

**`## Главное` (~300–380 слов):** подразделы `### ...`:
- **`let` и `const`, забудь про `var`.** `const` — для значения, которое не переназначается (по умолчанию выбирай его). `let` — когда значение будет меняться. `var` — устаревший, у него странные правила области видимости, в новом коде не используется.
- **Примитивные типы.** `number` (и целые, и дробные — один тип), `string`, `boolean`, `null` (явное «пусто»), `undefined` (неявное «пусто», значение по умолчанию), `bigint` (для огромных чисел, редко), `symbol` (редко, для библиотек).
- **Объекты — отдельный тип.** Всё, что не примитив — объект: `{}`, `[]`, функции. Объекты хранятся по ссылке (детали в уроке 05.5).
- **`typeof`.** Проверить тип: `typeof 42` → `"number"`. Исторический баг: `typeof null === "object"` — знай и не пугайся.
- **Арифметические операторы.** `+ - * / %` (остаток), `**` (степень). `+` с строкой — склейка: `"a" + "b" === "ab"`.
- **Сравнение.** `===` и `!==` — строгое, без преобразования типов. **Всегда используй их.** `==` и `!=` — нестрогое, делает странные вещи (`0 == ""` → `true`), не используй.
- **Логические операторы.** `&&`, `||`, `!`. Современно: `??` (nullish coalescing) — «если значение `null` или `undefined`, взять правое»: `name ?? "гость"`. И `?.` (optional chaining) — `user?.address?.city` — не упадёт, если `user` или `address` не заданы.
- **Тернарник.** `условие ? если_true : если_false`. Короткое ветвление в одну строку.
- **Преобразования типов.** `Number("42")`, `String(42)`, `Boolean(0)`. Неявные — когда JS сам приводит (`"3" - 1 === 2`). Знай о них, но явные — надёжнее.
- **Шаблонные строки.** Обратные кавычки: `` `Привет, ${name}!` `` — вставка переменной, многострочные строки из коробки.

**`## Пример кода`** — блок js:
```js
const name = "Иван";          // не меняется → const
let age = 28;                 // будет меняться → let
age += 1;                     // age === 29

const isAdult = age >= 18;    // boolean
const greeting = `Привет, ${name}! Тебе ${age}.`;

// сравнение
console.log(1 === "1");       // false — разные типы
console.log(1 == "1");        // true — нестрогое, не используй

// nullish и optional
const user = null;
const display = user?.name ?? "гость";  // "гость"

// преобразования
const num = Number("42");     // 42
const str = String(42);       // "42"
const flag = Boolean(0);      // false

// тернарник
const status = age >= 18 ? "взрослый" : "несовершеннолетний";
console.log(status);
```

**`## Частые ошибки`** — 4 пункта:
- Использовать `==` вместо `===` → нелогичные приведения и скрытые баги.
- Использовать `var` из «старых уроков» → попадёшь в проблему с областью видимости и hoisting.
- Писать `const arr = [1, 2]; arr.push(3)` и думать, что `const` запрещает это → `const` запрещает **переназначение переменной**, а не мутацию объекта/массива. Это норма.
- Конкатенировать строки через `+` (`"Привет, " + name + "!"`) вместо шаблонных — читается хуже.

**Заголовок:** `# Ветвления и циклы`

**`## Зачем это нужно`:** Программа — это не один линейный поток. В зависимости от условия она идёт разными путями (ветвления) или повторяет операцию (циклы). Без этих двух конструкций нельзя написать ничего сложнее «привет, мир».

**`## Главное` (~300–380 слов):** подразделы `### ...`:
- **`if` / `else if` / `else`.** Базовое ветвление. Условие в скобках — приводится к boolean. Обычно простое сравнение или переменная типа boolean.
- **Тернарник как одно-строчный `if`.** `const label = count > 0 ? "есть" : "пусто";`. Для простых случаев — короче `if`. Не вкладывай тернарники друг в друга — станет нечитаемо.
- **`switch`.** Альтернатива цепочке `else if`, когда сравниваешь одно значение со многими. Не забывай `break` в каждой ветке, иначе выполнение «провалится» в следующую. Современный JS часто предпочитает `if`/`else` или объект-lookup.
- **Truthy / falsy.** В JS значения приводятся к boolean автоматически. **Falsy:** `false`, `0`, `""`, `null`, `undefined`, `NaN`. Всё остальное — truthy. Поэтому работает `if (user)` и `if (array.length)`.
- **`&&` и `||` как выбор значения.** `const name = input || "гость";` — если `input` falsy, берётся «гость». Современно: для `null/undefined` используй `??` (не путать — `||` «съест» и `0`, и пустую строку).
- **Циклы `while` и `do..while`.** `while (условие) { ... }` — проверка сверху, может не выполниться ни разу. `do { ... } while (условие)` — гарантированно выполнится хотя бы раз.
- **Цикл `for`.** Классика: `for (let i = 0; i < 10; i++) { ... }`. Годится, когда нужен явный счётчик.
- **`for...of` — современный способ по массивам.** `for (const item of items) { ... }` — проще и без индекса.
- **`for...in` — по ключам объекта.** Редкий гость, и тот чаще заменяют на `Object.keys()`. На массивах `for...in` использовать **не надо** — перебирает и добавленные свойства.
- **`break` и `continue`.** `break` — выйти из цикла полностью. `continue` — прервать текущую итерацию и пойти к следующей.
- **В большинстве случаев `for...of` и `.forEach`/`.map`/`.filter` (урок 05.4) лучше голого `for`.**

**`## Пример кода`** — блок js:
```js
const age = 25;

// if / else
if (age >= 18) {
  console.log("взрослый");
} else {
  console.log("несовершеннолетний");
}

// тернарник
const label = age >= 18 ? "взрослый" : "несовершеннолетний";

// switch
const day = "пн";
switch (day) {
  case "сб":
  case "вс":
    console.log("выходной");
    break;
  default:
    console.log("будний");
}

// falsy / логические
const input = "";
const name = input || "гость";  // "гость"

// классический for
for (let i = 0; i < 3; i++) {
  console.log(i);  // 0, 1, 2
}

// современный for...of
const fruits = ["яблоко", "груша", "слива"];
for (const fruit of fruits) {
  console.log(fruit);
}

// break и continue
for (let i = 0; i < 10; i++) {
  if (i === 3) continue;   // пропускаем 3
  if (i === 7) break;      // выходим на 7
  console.log(i);
}
```

**`## Частые ошибки`** — 4 пункта:
- `switch` без `break` — выполнение «проваливается» в соседние case-ы (иногда нужно, но 99% случаев — баг).
- Писать `if (x == true)` — лишнее и опасное приведение. Просто `if (x)`.
- Использовать `for...in` для массивов → ловит прототипные свойства, порядок не гарантирован.
- Вкладывать тернарники (`a ? b ? c : d : e`) → читать невозможно, лучше обычный `if`.

**Заголовок:** `# Функции`

**`## Зачем это нужно`:** Функция — это переиспользуемый кусок логики с именем. Вместо того чтобы копировать 20 строк в пяти местах, пишешь один раз и вызываешь. Плюс функции — «первоклассные» в JS: их можно передавать в другие функции, возвращать, сохранять в переменные. Это открывает весь современный стиль (`map`, `filter`, коллбеки).

**`## Главное` (~340–420 слов):** подразделы `### ...`:
- **Function declaration.** `function greet(name) { return "Привет, " + name; }`. Поднимается в начало scope (hoisting) — можно вызывать **до** объявления в файле.
- **Function expression.** `const greet = function(name) { ... };`. Не поднимается — до этой строки переменной нет.
- **Arrow function (стрелочная).** `const greet = (name) => "Привет, " + name;`. Короче, более современный стиль, в 90% нового кода пишут так. Особенность: **нет собственного `this`** — про это подробнее в модуле 06.
- **Синтаксис стрелочной.** `(a, b) => a + b` (много аргументов). `x => x * 2` (один аргумент — скобки опциональны). `() => 42` (без аргументов). С телом в фигурных скобках — нужен `return`: `(a, b) => { return a + b; }`.
- **Параметры по умолчанию.** `function hello(name = "гость") { ... }` — если параметр не передан или `undefined`, берётся дефолт.
- **Rest-параметры.** `function sum(...nums) { ... }` — `nums` будет массивом всех переданных аргументов. Современная замена старому объекту `arguments`.
- **`return`.** Без `return` функция возвращает `undefined`. Как только встретила `return` — тело дальше не выполняется.
- **Область видимости (scope).** `let` и `const` видны только в том блоке, где объявлены (`{ ... }`). Function declaration видна во всей функции-родителе. Нет «глобальной по умолчанию» — это хорошо.
- **Замыкания (коротко).** Функция «запоминает» переменные окружения, в котором была создана. Классический пример — счётчик через внутреннюю функцию. Детально — в модуле 06.
- **Функции как значения.** Можно сохранить в переменной, передать в другую функцию (callback), вернуть из функции. Это основа для `map`, `filter`, `addEventListener`.
- **Чистые функции.** Для одних и тех же входов возвращают один и тот же выход, не меняют внешнее состояние. Чем больше таких — тем проще отлаживать. Противоположность — функции с side effects (мутация переменной, `console.log`, запрос к API).

**`## Пример кода`** — блок js:
```js
// declaration
function square(n) {
  return n * n;
}

// expression
const cube = function(n) {
  return n * n * n;
};

// arrow — современный стиль
const double = (n) => n * 2;
const sum = (a, b) => a + b;
const greet = (name = "гость") => `Привет, ${name}!`;

// rest-параметры
const total = (...nums) => nums.reduce((s, n) => s + n, 0);
console.log(total(1, 2, 3, 4));  // 10

// функция как аргумент (callback)
const items = [1, 2, 3];
const doubled = items.map(double);   // [2, 4, 6]

// замыкание
function createCounter() {
  let count = 0;
  return () => ++count;
}
const counter = createCounter();
counter();  // 1
counter();  // 2

// ранний return — выход сразу
function describe(user) {
  if (!user) return "нет пользователя";
  return `Привет, ${user.name}`;
}
```

**`## Частые ошибки`** — 4 пункта:
- Забыть `return` в стрелочной функции с телом `{ ... }` → функция тихо возвращает `undefined`.
- Мутировать входные параметры функции (`arr.push(...)` на переданный массив) → сложно отлаживать. Возвращай новое значение.
- Пытаться использовать `this` в стрелочной и удивляться, что он «не тот» → стрелки не имеют своего `this`.
- Писать огромные функции на 200 строк → разбивай на мелкие именованные. Правило большого пальца: если не можешь быстро назвать, что делает функция — она делает слишком много.

**`## Зачем это нужно`:** Массив — это упорядоченный список значений. Список задач, сообщений, товаров — всё массивы. В JS у массивов мощные методы (`map`, `filter`, `reduce`), которые заменяют циклы и делают код декларативным: «что сделать», а не «как».

**`## Главное` (~340–420 слов):** подразделы `### ...`:
- **Создание и доступ.** `const fruits = ["яблоко", "груша", "слива"];`. Доступ по индексу с нуля: `fruits[0]`. Длина: `fruits.length`.
- **Мутирующие методы — меняют оригинал.** `push` (в конец), `pop` (удалить с конца), `shift`/`unshift` (с начала), `splice` (вставить/удалить по индексу), `sort`, `reverse`. Используй, когда сознательно хочешь изменить.
- **Неизменяющие (immutable) методы — возвращают новый массив.** `slice` (копия части), `concat` (склейка). Современный стиль — больше работать с неизменяющими, чтобы не ловить баги «кто-то поменял массив где-то ещё».
- **`.forEach` — просто пройтись.** `arr.forEach(item => console.log(item))`. Замена `for...of`, когда ничего не возвращаешь, только делаешь сайд-эффект.
- **`.map` — трансформация.** Возвращает **новый массив** той же длины, где каждый элемент преобразован. `[1, 2, 3].map(n => n * 2)` → `[2, 4, 6]`. Это самый частый метод во фронте (например, превратить массив задач в массив `<li>`).
- **`.filter` — отбор.** Возвращает новый массив только с теми элементами, для которых callback вернул truthy. `users.filter(u => u.active)`.
- **`.find` — первый подходящий.** Возвращает первый элемент, удовлетворяющий условию (или `undefined`). `users.find(u => u.id === 5)`.
- **`.some` / `.every`.** Возвращают boolean. `arr.some(...)` — хоть один удовлетворяет. `arr.every(...)` — все удовлетворяют.
- **`.includes(value)`.** Проверка наличия значения. `[1, 2, 3].includes(2) === true`.
- **`.reduce(callback, initial)` — свёртка в одно значение.** Мощный и капризный: `[1, 2, 3].reduce((acc, n) => acc + n, 0)` → `6`. Используй для сумм, превращений массив → объект. Не начинай с `reduce`, если задачу решает `map`+`filter`.
- **Spread — `[...arr]`.** Копия массива. `[...a, ...b]` — склейка. `[...arr, newItem]` — добавить элемент, не мутируя.
- **Деструктуризация.** `const [first, second] = arr;`. Удобно, когда массив из фиксированного числа элементов (координаты, пара).

**`## Пример кода`** — блок js:
```js
const items = [
  { id: 1, title: "Купить хлеб", done: false },
  { id: 2, title: "Помыть кота",  done: true },
  { id: 3, title: "Сделать зарядку", done: false },
];

// map — новый массив
const titles = items.map(item => item.title);
// ["Купить хлеб", "Помыть кота", "Сделать зарядку"]

// filter — отбор
const active = items.filter(item => !item.done);

// find — первый по условию
const byId = items.find(item => item.id === 2);

// some / every
const hasDone = items.some(item => item.done);    // true
const allDone = items.every(item => item.done);   // false

// reduce — сумма / свёртка
const totalDone = items.reduce((acc, item) => acc + (item.done ? 1 : 0), 0);

// spread — иммутабельное добавление
const withNew = [...items, { id: 4, title: "Новая", done: false }];

// деструктуризация
const [firstItem, secondItem] = items;
```

**`## Частые ошибки`** — 4 пункта:
- Использовать `.map` там, где ничего не возвращаешь — нужен `.forEach`. `.map` создаёт новый массив впустую.
- Мутировать массив внутри `.map`/`.filter` (`arr.push` в callback) → хаос, массив становится непредсказуемым.
- Забывать `return` в стрелочной с фигурными: `arr.map(x => { x * 2 })` → массив `undefined`. Или без фигурных, или с `return`.
- Стартовать `.reduce` без начального значения, когда массив может быть пустой → выбросит ошибку. Всегда передавай `initial`.

**Заголовок:** `# Объекты`

**`## Зачем это нужно`:** Объект — это набор пар «ключ → значение». Всё сложнее скалярных значений в JS — это объект: пользователь, товар, задача, ответ от API. Без свободной работы с объектами нельзя ни распарсить JSON, ни вытащить поле из ответа сервера.

**`## Главное` (~320–400 слов):** подразделы `### ...`:
- **Литеральный синтаксис.** `const user = { name: "Иван", age: 28 };`. Ключи — строки (без кавычек, если валидный идентификатор). Значения — любые (число, строка, массив, функция, другой объект).
- **Доступ к полям.** `user.name` (точка — когда ключ известен и валидный). `user["name"]` или `user[key]` — когда ключ в переменной или не валидный идентификатор (`"user-id"`).
- **Изменение / добавление / удаление.** `user.name = "Пётр"` — изменить. `user.email = "..."` — добавить. `delete user.age` — удалить (редко нужно).
- **`Object.keys/values/entries`.** Получить массив ключей / значений / пар `[ключ, значение]`. Дальше на полученном массиве работают все `map`/`filter`/`reduce` из урока 05.4.
- **Вычисляемые ключи.** `const key = "name"; const obj = { [key]: "Иван" };` — когда ключ знаешь только в рантайме.
- **Shorthand properties.** Если переменная и ключ совпадают: `const name = "Иван"; const user = { name };` эквивалентно `{ name: name }`.
- **Методы.** Функция как свойство объекта. Shorthand: `{ greet() { return "hi"; } }` вместо `{ greet: function() { ... } }`.
- **Spread `{...obj}`.** Копия объекта. `{...a, ...b}` — слияние. `{...user, age: 30}` — копия с изменённым полем. Так делают иммутабельные обновления (основа Redux, React state).
- **Деструктуризация.** `const { name, age } = user;`. С переименованием: `const { name: userName } = user;`. С default: `const { age = 0 } = user;`. С остатком: `const { name, ...rest } = user;`.
- **JSON.** `JSON.stringify(obj)` → строка (для отправки в API, в `localStorage`). `JSON.parse(str)` → объект. Это абсолютный минимум для работы с API и хранилищем.
- **Объекты по ссылке.** `const a = { x: 1 }; const b = a; b.x = 2;` — тогда и `a.x === 2`. Для копии — spread или `structuredClone(obj)`. Без этого понимания — половина «почему-то меняется где-то ещё» багов.

**`## Пример кода`** — блок js:
```js
const user = {
  name: "Иван",
  age: 28,
  email: "ivan@example.com",
  greet() {
    return `Привет, я ${this.name}!`;
  },
};

// доступ
console.log(user.name);          // "Иван"
console.log(user["email"]);       // "ivan@example.com"

// изменение / добавление
user.age = 29;
user.city = "Москва";

// shorthand + вычисляемый ключ
const key = "status";
const value = "active";
const profile = { name: "Иван", [key]: value };
// { name: "Иван", status: "active" }

// деструктуризация
const { name, age, city = "неизвестно" } = user;

// spread — иммутабельное обновление
const updatedUser = { ...user, age: 30 };

// Object.keys/values/entries
const keys = Object.keys(user);       // ["name", "age", ...]
const pairs = Object.entries(user);   // [["name", "Иван"], ...]

// JSON
const json = JSON.stringify(user);    // строка
const parsed = JSON.parse(json);      // снова объект

// объект по ссылке
const a = { x: 1 };
const b = a;
b.x = 2;
console.log(a.x);  // 2 — ссылка на тот же объект
const c = { ...a }; // независимая копия
```

**`## Частые ошибки`** — 4 пункта:
- Думать, что `const obj = {...}` защищает от изменений → `const` запрещает переназначение переменной, не мутацию объекта.
- Копировать через `const copy = obj` — это не копия, это вторая ссылка на тот же объект.
- Использовать `JSON.parse(JSON.stringify(obj))` как универсальную deep-copy → падает на функциях, `Date`, `undefined`, циклах. Современный способ — `structuredClone(obj)`.
- Перебирать объекты через `for...in` без `hasOwnProperty` → может подхватить свойства прототипа. Современно — `Object.keys/entries`.

**Заголовок:** `# DOM и события`

**`## Зачем это нужно`:** До этого JS крутился в консоли. Пора подключать его к странице: находить элементы, менять текст/классы/стили, реагировать на клики, вводы, отправку форм. Это и есть «фронтенд» в узком смысле — управление HTML из кода.

**`## Главное` (~340–420 слов):** подразделы `### ...`:
- **DOM.** Document Object Model — дерево объектов, которое браузер строит по твоему HTML. Каждый тег — это объект-элемент со свойствами и методами. Всё, что ты видишь в DevTools → Elements, доступно как объекты.
- **Поиск элементов.**
    - `document.querySelector("CSS-селектор")` — первый подходящий элемент или `null`.
    - `document.querySelectorAll("CSS-селектор")` — NodeList всех подходящих (похож на массив, но неполностью — `.forEach` работает, `.map` — нет).
    - Старые `getElementById`, `getElementsByClassName` — не нужны, `querySelector` их заменяет.
- **Чтение и запись контента.**
    - `el.textContent` — текст. **Безопасно**, экранирует HTML.
    - `el.innerHTML` — HTML-строка. **Опасно** с пользовательским вводом (XSS). Используй, только если ты сам формируешь HTML.
    - `el.value` — для `<input>`, `<textarea>`, `<select>`.
- **Классы и стили.**
    - `el.classList.add("active")`, `.remove(...)`, `.toggle(...)`, `.contains(...)`. Работа с CSS-классами — обычно достаточно этого.
    - `el.style.color = "red"` — инлайн-стиль. Используй редко, в основном переключай классы.
- **Атрибуты.** `el.getAttribute("href")`, `el.setAttribute("disabled", "")`. Или напрямую через свойство: `img.src`, `input.disabled`.
- **Создание и вставка.**
    - `document.createElement("li")`.
    - `parent.appendChild(el)` или современное `parent.append(el)` (можно несколько и строки).
    - `el.remove()` — удалить из DOM.
- **События.** `el.addEventListener("click", handler)`. Частые: `click`, `input`, `change`, `submit`, `keydown`. Убрать слушатель — `removeEventListener` с той же функцией-ссылкой.
- **Объект события.** Коллбек получает `event`:
    - `event.target` — элемент, на котором сработало.
    - `event.preventDefault()` — отменить дефолтное поведение (например, submit формы).
    - `event.stopPropagation()` — остановить всплытие наверх по DOM.
- **Всплытие и делегирование.** Событие всплывает от элемента к родителям. Можно поставить один слушатель на родителя и проверять `event.target` — экономия, особенно на длинных списках.
- **DOMContentLoaded.** Если скрипт в `<head>`, элементы ещё не существуют. Варианты: переместить `<script>` в конец `<body>`, добавить атрибут `defer`, или обернуть в `document.addEventListener("DOMContentLoaded", () => { ... })`.

**`## Пример кода`** — блок html + js:
```html
<form id="todo-form">
  <input id="todo-input" type="text" placeholder="Новая задача" required>
  <button type="submit">Добавить</button>
</form>

<ul id="todo-list"></ul>
```

```js
const form = document.querySelector("#todo-form");
const input = document.querySelector("#todo-input");
const list  = document.querySelector("#todo-list");

form.addEventListener("submit", (event) => {
  event.preventDefault();                         // не перезагружать страницу

  const title = input.value.trim();
  if (!title) return;

  const li = document.createElement("li");
  li.textContent = title;
  li.classList.add("todo-item");

  const removeBtn = document.createElement("button");
  removeBtn.textContent = "✕";
  removeBtn.addEventListener("click", () => li.remove());

  li.append(removeBtn);
  list.append(li);

  input.value = "";
});

// делегирование: один слушатель на родителя
list.addEventListener("click", (event) => {
  if (event.target.tagName === "LI") {
    event.target.classList.toggle("done");
  }
});
```

**`## Частые ошибки`** — 4 пункта:
- Вставлять пользовательский ввод через `innerHTML` → XSS. Используй `textContent` или формируй DOM через `createElement`.
- Не вызывать `event.preventDefault()` на submit формы → страница перезагрузится, JS-логика сломается.
- Ставить `querySelector` до того, как DOM загружен (скрипт в `<head>` без `defer`) → `null`, ошибка.
- Подписываться на события в цикле без `removeEventListener` в SPA → утечки памяти и двойные срабатывания.

## Требования

### Функциональность
1. **Добавление задачи** — форма с `<input>` + кнопка. Пустые задачи не добавляются.
2. **Список задач** — рендерится из массива объектов (`{ id, title, done }`), а не копится в DOM напрямую.

- Единый массив `todos` — источник правды. Любое действие (добавить / удалить / переключить / отфильтровать) сначала обновляет массив, потом функция `render()` перерисовывает DOM.
- Функции чистые, где возможно. Мутацию массива оставь на один-два места (`setTodos`), остальное пусть возвращает новые массивы через `.map`, `.filter`, `spread`.

- `textContent` для пользовательского текста, НЕ `innerHTML`.
- `addEventListener`, а не `onclick="..."` в HTML.
- `id` задачи — уникальный (проще всего через `Date.now()` или `crypto.randomUUID()`).

- Библиотек (jQuery, React, Alpine — ничего).
- `innerHTML` с пользовательским вводом.

- Дублирования DOM и массива — не редактируй `<li>` напрямую, минуя массив.
