**Заголовок:** `# Структура HTML-документа`

**Раздел `## Зачем это нужно` (1–2 предложения):** Любая HTML-страница начинается с одинакового скелета — DOCTYPE, head, body. Без него браузер не понимает, что это HTML5 и как рендерить текст/кириллицу/meta-теги.

**Раздел `## Главное` (~280–350 слов):** покрой подразделами:
- **`<!DOCTYPE html>`** — первая строка любой страницы. Говорит браузеру: «это HTML5, рендери по современным правилам». Без DOCTYPE браузер включает quirks mode и верстка начинает вести себя странно.
- **Корневой `<html>` и атрибут `lang`** — контейнер всего документа. `lang="ru"` важен для поисковиков, скринридеров и автоперевода.
- **`<head>` — невидимая часть** — сюда идут meta-теги, заголовок вкладки (`<title>`), подключения CSS/JS, favicon. Всё, что не отображается на странице, но нужно браузеру.
- **`<meta charset="UTF-8">`** — без этого кириллица превратится в `�����`. Ставим первым тегом внутри head.
- **`<meta name="viewport" ...>`** — чтобы страница нормально отображалась на мобильных. Вернёмся к этому в CSS-модуле про адаптив.
- **`<title>`** — текст во вкладке браузера. И в результатах поиска Google.
- **`<body>` — видимая часть** — всё, что пользователь реально видит на странице.
- **Подключение CSS и JS** — `<link rel="stylesheet" href="style.css">` в head, `<script src="script.js" defer></script>` в конце body (или в head с `defer`). Пока не углубляемся — вернёмся в CSS- и JS-модулях.

**Раздел `## Пример кода` — блок html:** минимальный валидный HTML5-документ:
```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Моя первая страница</title>
  </head>
  <body>
    <h1>Привет, мир!</h1>
  </body>
</html>
```

**Раздел `## Частые ошибки` (3 пункта):**
- Забывать `<meta charset="UTF-8">` → кракозябры вместо кириллицы.
- Писать `<script>` в head без `defer` → браузер блокирует рендер страницы, пока не загрузит JS.
- Делать `<title>` из одного слова или оставлять дефолтный «Document» → плохо для SEO и выглядит неряшливо во вкладках.

**Раздел `## Задания`:**

**Раздел `## Зачем это нужно`:** 80% любой HTML-страницы — это одни и те же ~15 тегов: заголовки, текст, списки, ссылки, картинки. Этих тегов достаточно, чтобы сверстать большинство статичных страниц.

**Раздел `## Главное` (~300–400 слов):** подразделами:
- **Заголовки `<h1>` – `<h6>`** — иерархия. На странице должен быть один `<h1>` (главный заголовок). `<h2>` — разделы, `<h3>` — подразделы и т.д. Это важно и для SEO, и для скринридеров.
- **Параграфы `<p>`, переносы `<br>`, горизонтальная линия `<hr>`** — текстовый контент. `<br>` используй только для реальных переносов строки (адрес, стихи), не для отступов.
- **Списки `<ul>` (маркированный), `<ol>` (нумерованный), `<li>` (пункт)** — всё, что в жизни выглядит списком, в HTML делается списком, а не серией `<div>`.
- **Ссылки `<a href="">`** — `href` может быть абсолютной (`https://...`), относительной (`./about.html`), якорной (`#section-2`), или на почту (`mailto:...`). Открыть в новой вкладке — `target="_blank"` + `rel="noopener noreferrer"` (безопасность).
- **Изображения `<img src="" alt="">`** — `src` — путь к картинке, `alt` — текстовое описание для скринридеров и на случай, если картинка не загрузилась. `alt=""` (пустой) — говорит скринридеру «картинка чисто декоративная, пропусти».
- **Inline vs block элементы** — `<div>`, `<p>`, `<h1>`, `<ul>` — блочные (занимают всю строку). `<span>`, `<a>`, `<strong>`, `<em>`, `<img>` — строчные (идут в потоке текста). Полезно помнить при вёрстке, в CSS-модулях вернёмся к этому.
- **Выделение текста — `<strong>` и `<em>`** — не `<b>` и `<i>`. `<strong>` = «это важно», `<em>` = «интонационное ударение». Визуально то же самое, но смысл для поисковиков и скринридеров разный.

**Раздел `## Пример кода` — блок html:**
```html
<h1>Заголовок страницы</h1>
<p>
  Обычный параграф с <strong>важным словом</strong>
  и <a href="https://developer.mozilla.org" target="_blank" rel="noopener noreferrer">ссылкой на MDN</a>.
</p>

<h2>Список покупок</h2>
<ul>
  <li>Хлеб</li>
  <li>Молоко</li>
  <li>Кофе</li>
</ul>

<img src="./cat.jpg" alt="Рыжий кот спит на клавиатуре ноутбука">
```

**Раздел `## Частые ошибки` (4 пункта):**
- Использовать `<br>` вместо `<p>` — HTML станет нечитабельным, а стили — неуправляемыми.
- Забывать `alt` у `<img>` — страница перестаёт быть доступной и проседает по SEO.
- Два `<h1>` на одной странице — плохо для иерархии и SEO.
- Использовать `<b>` / `<i>` вместо `<strong>` / `<em>` — потеряешь смысловую нагрузку (в современной вёрстке лучше всегда `<strong>`/`<em>`).

**Раздел `## Задания`:**

- [ ] Проверь в DevTools → Elements, что иерархия заголовков выглядит логично.
- [ ] Выдели одно слово в параграфе через `<strong>` и одно — через `<em>`.

**Заголовок:** `# Семантика HTML5`

**Раздел `## Зачем это нужно`:** Теги типа `<header>`, `<main>`, `<article>` для браузера визуально = `<div>`. Но они **сообщают смысл** — поисковикам, скринридерам, будущему тебе через полгода, когда открываешь код и хочешь найти навигацию.

**Раздел `## Главное` (~280–350 слов):** подразделами:
- **Идея семантики** — один `<div>` ничем не отличается от другого. `<header>`, `<nav>`, `<article>` — говорят, что это за часть страницы. Без семантики скринридер озвучивает стену «div, div, div» — с семантикой «шапка, навигация, основное содержимое, статья, подвал».
- **`<header>`** — шапка страницы или раздела. На странице может быть несколько (одна — у страницы, одна — у каждой `<article>`).
- **`<nav>`** — блок навигации. Главное меню, футер-меню, breadcrumbs.
- **`<main>`** — основной контент страницы. На странице ровно один. Туда **не** должно попадать всё, что повторяется на других страницах (шапка, футер, сайдбар).
- **`<section>`** — тематический раздел страницы. «Блок с услугами», «блок с отзывами».
- **`<article>`** — самостоятельный кусок контента, который имеет смысл сам по себе. Пост в блоге, карточка товара, комментарий.
- **`<aside>`** — сайдбар, реклама, справочный блок — что-то сбоку основного потока.
- **`<footer>`** — подвал страницы или раздела. Контакты, копирайт, ссылки.
- **Как выбрать тег** — если это «самостоятельная публикация» → `<article>`. Если «тематический блок» → `<section>`. Если вообще непонятно и никакой смысл не добавишь → `<div>` (это нормально, не надо семантить всё подряд).

**Раздел `## Пример кода` — блок html, типовой макет страницы:**
```html
<body>
  <header>
    <h1>Мой блог</h1>
    <nav>
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">Обо мне</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <h2>Почему я учу фронтенд</h2>
      <p>Текст поста...</p>
    </article>
    <article>
      <h2>Лучшие ресурсы по HTML</h2>
      <p>Текст поста...</p>
    </article>
  </main>

  <aside>
    <h2>Популярные посты</h2>
    <ul>
      <li><a href="#">Пост 1</a></li>
    </ul>
  </aside>

  <footer>
    <p>© 2026 Мой блог</p>
  </footer>
</body>
```

**Раздел `## Частые ошибки` (3 пункта):**
- Обернуть всю страницу в `<section>` вместо `<main>` — `<section>` без заголовка внутри — уже бессмыслица.
- Сделать несколько `<main>` на странице — валидатор HTML ругается, логика ломается.
- Использовать `<article>` для каждого `<div>` — семантика ≠ обязаловка. Там, где смысла нет, `<div>` вполне ок.

**Раздел `## Задания`:**

**Заголовок:** `# Формы`

**Раздел `## Зачем это нужно`:** Формы — главный способ получить ввод от пользователя. Регистрация, вход, обратная связь, поиск, фильтры. HTML даёт довольно много из коробки: валидацию, типы полей, доступность — без единой строчки JS.

**Раздел `## Главное` (~300–400 слов):** подразделами:
- **`<form>` — обёртка.** Обычно с атрибутами `action` (куда отправить) и `method` (`GET` или `POST`). В ранних модулях не шлём никуда реально — просто создаём структуру.
- **`<input>` — главный рабочий элемент.** Это не один тег, а целая семья — тип задаётся атрибутом `type`. Основные: `text`, `email`, `password`, `tel`, `number`, `url`, `date`, `checkbox`, `radio`, `file`, `hidden`, `submit`.
- **`<label>` — подпись к полю.** Связывается с `<input>` через `for="id-инпута"` или оборачивая инпут. Клик по label фокусит инпут — базовая юзабилити.
- **`<textarea>`** — многострочное поле.
- **`<select>` + `<option>`** — выпадающий список.
- **`<button>`** — кнопки. `type="submit"` — отправляет форму (по умолчанию), `type="button"` — ничего не делает без JS, `type="reset"` — сбрасывает форму.
- **Базовая HTML-валидация без JS:**
    - `required` — обязательное поле.
    - `type="email"` — браузер сам проверит, что это похоже на email.
    - `type="url"` — что это URL.
    - `minlength` / `maxlength` — длина строки.
    - `min` / `max` — границы чисел и дат.
    - `pattern="..."` — регулярка для более сложных правил.
- **`placeholder` ≠ `<label>`.** Плейсхолдер — это подсказка внутри поля, она исчезает при вводе. Подписью к полю он не является — скринридеры его часто пропускают, а пользователь забывает, что надо ввести.

**Раздел `## Пример кода` — блок html:**
```html
<form action="/submit" method="POST">
  <div>
    <label for="name">Имя</label>
    <input id="name" name="name" type="text" required minlength="2">
  </div>

  <div>
    <label for="email">Email</label>
    <input id="email" name="email" type="email" required>
  </div>

  <div>
    <label for="age">Возраст</label>
    <input id="age" name="age" type="number" min="0" max="120">
  </div>

  <div>
    <label for="message">Сообщение</label>
    <textarea id="message" name="message" rows="5" required></textarea>
  </div>

  <button type="submit">Отправить</button>
</form>
```

**Раздел `## Частые ошибки` (4 пункта):**
- `placeholder` вместо `<label>` → форма нерабочая для скринридеров, пользователь забывает, что было за поле.
- Не связывать `<label>` с `<input>` через `for`/`id` → клик по подписи не фокусит поле, доступность нарушена.
- `<button>` без атрибута `type` внутри `<form>` → по умолчанию ведёт себя как `submit` и сабмитит форму, даже если ты этого не хотел.
- Полагаться только на HTML-валидацию в серьёзном приложении → её легко обойти, серверная валидация всё равно нужна (разбор — в модуле 13).

**Раздел `## Зачем это нужно`:** Доступность (a11y) — это про то, чтобы сайтом мог пользоваться любой человек: с плохим зрением, через скринридер, с клавиатуры без мыши, с медленным интернетом. В фрилансе и на заказных сайтах всё чаще спрашивают Lighthouse-score ≥ 90 по a11y — без этого работу могут не принять.

**Раздел `## Главное` (~270–350 слов):** подразделами:
- **Семантика — основа a11y.** Всё, что ты выучил в уроке 02.3, уже работает на доступность: правильные заголовки, `<nav>`, `<main>`, `<button>` вместо кликабельных `<div>`. Начни с семантики — она закрывает 50% задач a11y.
- **`alt` у изображений.** Описывай, что на картинке, если картинка несёт смысл. Если картинка чисто декоративная (разделитель, фон) — `alt=""` (пустой), так скринридер её пропустит.
- **`<label>` у форм.** Каждый `<input>` должен иметь связанную подпись через `for`/`id`. Без этого скринридер не знает, что это за поле.
- **`aria-label` — когда подписи не видно.** Если иконка-кнопка без текста (например, крестик закрытия), дай ей `aria-label="Закрыть"`. Это текст «для скринридера», визуально его нет.
- **Клавиатурная навигация.** По Tab должны обходиться все интерактивные элементы (ссылки, кнопки, инпуты) в логичном порядке. Enter и Space работают на кнопках. Если пришлось делать кликабельный `<div>` — лучше не делай, используй `<button>` или `<a>`.
- **Контраст и фокус.** Это больше про CSS, но пока запомни: должен быть видимый `:focus` (рамка вокруг элемента при Tab), контраст текста к фону не ниже 4.5:1 для обычного текста. К этому вернёмся в CSS-модулях.
- **Lighthouse.** Встроенный в Chrome DevTools аудит. Вкладка Lighthouse → Accessibility → Analyze → получаешь score и список конкретных проблем. Удобно проверять себя.

**Раздел `## Пример кода` — блок html, иконка-кнопка с aria-label и картинка с правильным alt:**
```html
<!-- иконка-кнопка без видимого текста -->
<button type="button" aria-label="Закрыть окно">
  <svg aria-hidden="true" width="16" height="16">...</svg>
</button>

<!-- смысловая картинка -->
<img src="./team-photo.jpg" alt="Команда из 6 человек сидит в офисе за одним столом">

<!-- декоративный разделитель -->
<img src="./divider.svg" alt="">
```

**Раздел `## Частые ошибки` (4 пункта):**
- `<div onclick="...">` вместо `<button>` → нет фокуса по Tab, не работает с клавиатуры, скринридер не понимает, что это кнопка.
- Пустой `alt` у смысловой картинки или, наоборот, `alt="picture"` / `alt="image"` → мусор для скринридера.
- Форма без `<label>` (одни placeholder-ы) → скринридер зачитывает «edit, blank».
- Отключать `outline` у `:focus` без замены → клавиатурники теряют видимость фокуса.
