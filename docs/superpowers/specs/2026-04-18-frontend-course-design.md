# Frontend-курс для ученика (mentorship) — design-документ

- **Дата:** 2026-04-18
- **Автор:** ментор (Иван) + Claude
- **Статус:** проект утверждён, готов к плану реализации

---

## 1. Контекст и цель

Ментор ведёт одного ученика один-на-один. Ученик — полный ноль в программировании. Цель курса — довести его до уровня, когда он уверенно делает **pet-проекты** и берёт **фриланс-заказы** по Frontend. Подготовка к собеседованиям в найм **не** входит в задачи.

**Ключевые ограничения и принципы:**
- Темп свободный — нет дедлайнов и «недельной программы».
- Теория — самостоятельное чтение в Obsidian; практика и ревью — на созвонах.
- Прокачивается навык чтения официальной документации.
- Стек задаётся личным опытом ментора: база → React → Next.js → библиотеки / БД / auth.

---

## 2. Формат работы

| Слой | Где | Кто делает |
|---|---|---|
| Теория | Obsidian vault (читает сам) | Claude наполняет, ученик читает |
| Практика (задания, проекты) | репозиторий ученика на GitHub | Ученик кодит сам |
| Ревью и разбор сложностей | Созвоны 1:1 | Ментор |
| Трекинг прогресса | roadmap.sh (официальный Frontend-roadmap) | Ученик отмечает узлы |

### 3.3. Рекомендуемые плагины Obsidian

- **Templater** — вставка шаблона урока одной горячей клавишей.
- **Dataview** — авто-списки «to-read / in-progress / done» и общий прогресс-бар по модулям (через поле `статус` в frontmatter).
- **Tasks** — централизованный список всех домашек с дедлайнами.
- **Kanban** — доска ученика: To-read / In-progress / Done (визуальный дубль roadmap.sh).

| 11 | TypeScript | Базовые типы, интерфейсы, дженерики, utility-types, типизация событий/props/API, **перевод React-проекта на TS** |
| 12 | Build tools и тестирование | Vite глубже, npm/pnpm, env vars, package.json, scripts, лёгкое тестирование: Vitest (unit) + Playwright (e2e, 2-3 примера) |
| 13 | Next.js и backend | App Router, Server Components, routing, data fetching, БД (Supabase или Drizzle + Postgres), auth (NextAuth/Clerk), API routes |

**Развилки, зафиксированные в обсуждении:**
- **TypeScript — после React** (модуль 11), не до.
- **State management** — TanStack Query + Context как основной, Redux Toolkit только обзорно.
- **Тестирование** — лёгкое (вариант B), не отдельный большой модуль.
- **Accessibility / SEO / performance** — в модуле 07 основы + возврат в модуле 14 на прод-уровне.

## 5. План проектов

### 5.1. Мини-проекты (после коротких модулей)
Назначение — закрепить синтаксис. Длительность — 1–2 дня.

| После модуля | Мини-проект |
|---|---|
| 02 HTML | Статичная страница-резюме с семантикой |
| 03 CSS база | Адаптивная карточка товара + страница каталога на Flexbox/Grid |
| 05 JS база | TODO-лист на vanilla JS (DOM + localStorage) |
| 06 JS продвинутый | Погода-апп с внешним API (fetch + обработка ошибок) |
| 08 Git + DevTools | Пара PR в чужой публичный репо (issue first timers + пара своих фиксов) |
| 09 React база | Переписать TODO на React |
| 11 TypeScript | Перевод React-TODO/погоды на TS |

### 5.2. Крупные проекты-вехи (3 шт.)

**P1 — Лендинг по Figma** (после модуля 04)
- Адаптивный, точный пиксель-перфект, семантика, доступность.
- Критерий сдачи: верстается без костылей, выглядит одинаково в Chrome/Firefox/Safari, проходит Lighthouse ≥ 90 по a11y/best practices.

**P2 — React SPA** (после модуля 10, до TS)
- Пример: трекер привычек, доска задач или мини-CRM.
- Стек: React + Router + TanStack Query + RHF + zod + Tailwind.
- Источник данных: публичный API или мок-бэкенд (json-server).
- Критерий сдачи: чистая архитектура папок, разделение UI/логики, рабочий production-билд.

**P3 — Fullstack на Next.js** (после модуля 13, финальный)
- Пример: мини-соцсеть / блог-платформа / маркетплейс pet-объявлений.
- Стек: Next.js App Router + Server Components + Supabase или Drizzle+Postgres + NextAuth/Clerk + TS.
- Критерий сдачи: развёрнут на Vercel, работают регистрация/авторизация и базовый CRUD, прод-сборка без ошибок.

## 6. Процесс работы по циклу модуля

1. **Старт модуля**: ученик открывает `00-Обзор-модуля.md`, отмечает соответствующий узел в roadmap.sh как «in progress».
2. **Теория**: читает уроки по порядку, отмечает статус `done` в frontmatter заметок.
3. **Практика**: выполняет задания и мини-проект, пушит на GitHub в ветку `module-NN`.
4. **Созвон**: ментор ревьюит код, разбирает сложности, даёт следующий шаг.
5. **Закрытие модуля**: ученик отмечает узел в roadmap.sh как done, двигает карточки в Kanban.
6. **После крупной вехи (P1/P2/P3)**: ретро в `Ресурсы/Прогресс.md` (что зашло, что сложно, план на следующий блок).
