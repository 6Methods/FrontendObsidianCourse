# Playwright

## Зачем это нужно

Юнит- и компонентные тесты запускают кусочки приложения в jsdom — это не настоящий браузер, а имитация DOM поверх Node. E2E проверяет, что приложение работает целиком: фронт плюс роуты плюс сеть плюс `localStorage`. Playwright запускает реальный Chromium/Firefox/WebKit и кликает по твоему сайту как живой пользователь.

## Главное

### Что такое E2E
«End-to-end» — тест запускает полное приложение, эмулирует пользователя (клики, ввод, навигация) и проверяет, что он видит на экране. Такие тесты долгие (секунды, а не миллисекунды), но зато ловят интеграционные баги: неправильный роут, сломанный запрос, разошедшиеся версии фронта и бэка.

### Playwright vs Cypress
Playwright быстрее, работает с тремя движками из коробки (Chromium, Firefox, WebKit), лучше параллелится и официально поддерживается Microsoft. Cypress — проще в старте и с красивой UI, но медленнее и ограничен Chromium/Chrome-family. В новых проектах бери Playwright.

### Установка
`npm init playwright@latest` создаёт скелет: конфиг, пример-тест, папку `tests/` и пример workflow. Отвечай: TypeScript, папка `tests/e2e`, GitHub Actions — да. Дальше уже дотачиваешь под себя: браузеры, reporter, baseURL.

### Браузеры
Playwright качает свои бинарники: `npx playwright install`. В CI — `npx playwright install --with-deps chromium`, флаг `--with-deps` доставляет системные библиотеки, без которых браузер не стартанёт в Ubuntu-раннере.

### Тест
Структура теста: `test("что проверяем", async ({ page }) => { await page.goto("/"); await page.getByRole("button", { name: "Войти" }).click(); await expect(page.getByText("Привет")).toBeVisible(); })`. Читается как сценарий пользователя.

### Locators
Предпочитай семантические локаторы: `getByRole`, `getByText`, `getByLabel`, `getByPlaceholder`. Избегай CSS-селекторов типа `.btn.primary` — они хрупкие и ломаются от любого твика стилей. Семантика отражает то, что видит пользователь.

### Assertions
`expect(locator).toBeVisible()`, `toHaveText()`, `toHaveURL()`. Ключевая фича — auto-retry: Playwright сам ждёт появления или изменения элемента до таймаута, не нужно вручную писать ожидания.

### Codegen
`npx playwright codegen http://localhost:5173` — запускает реальный браузер, записывает твои действия и генерит готовый тест-код. Быстрый способ набросать новый сценарий и потом его причесать.

### webServer
В конфиге: `webServer: { command: "npm run dev", url: "http://localhost:5173", reuseExistingServer: !process.env.CI }`. Playwright сам поднимет dev-сервер, дождётся, прогонит тесты и в CI уронит его после.

### Traces, screenshots, video
При падении Playwright сохраняет trace (таймлайн со снимками DOM и сетью) и screenshot. В CI подключай `actions/upload-artifact` на папку `playwright-report/`. Локально открывай через `npx playwright show-report` — интерактивный отчёт, видно каждый шаг.

### Когда E2E окупается
Happy-path критических сценариев: логин, checkout, оплата. Не пиши E2E на каждую мелочь — дорого и долго. Правило большого пальца: «если это сломается — юзер заметит первым», значит, пиши E2E.

## Пример кода

```ts
// playwright.config.ts
import { defineConfig, devices } from "@playwright/test";

export default defineConfig({
  testDir: "./tests/e2e",
  fullyParallel: true,
  retries: process.env.CI ? 2 : 0,
  reporter: [["html"], ["list"]],
  use: {
    baseURL: "http://localhost:5173",
    trace: "on-first-retry",
    screenshot: "only-on-failure",
  },
  projects: [
    { name: "chromium", use: { ...devices["Desktop Chrome"] } },
  ],
  webServer: {
    command: "npm run dev",
    url: "http://localhost:5173",
    reuseExistingServer: !process.env.CI,
  },
});

// tests/e2e/catalog.spec.ts
import { test, expect } from "@playwright/test";

test("добавление товара в корзину", async ({ page }) => {
  await page.goto("/");
  await page.getByRole("link", { name: /каталог/i }).click();
  await expect(page).toHaveURL(/\/products/);

  const firstCard = page.getByRole("article").first();
  await firstCard.getByRole("button", { name: /в корзину/i }).click();

  const cartBadge = page.getByTestId("cart-count");
  await expect(cartBadge).toHaveText("1");
});

// tests/e2e/checkout.spec.ts
test("checkout happy path", async ({ page }) => {
  await page.goto("/products");
  await page.getByRole("button", { name: /в корзину/i }).first().click();
  await page.getByRole("link", { name: /корзина/i }).click();
  await page.getByRole("link", { name: /оформить/i }).click();

  await page.getByLabel(/имя/i).fill("Иван");
  await page.getByLabel(/email/i).fill("ivan@example.com");
  await page.getByLabel(/телефон/i).fill("+71234567890");
  await page.getByLabel(/город/i).fill("Москва");
  await page.getByLabel(/адрес/i).fill("ул. Пушкина, 1");

  await page.getByRole("button", { name: /оформить/i }).click();
  await expect(page).toHaveURL(/\/success/);
  await expect(page.getByText(/спасибо/i)).toBeVisible();
});

// package.json scripts
// "test:e2e": "playwright test",
// "test:e2e:headed": "playwright test --headed",
// "test:e2e:ui": "playwright test --ui"
```

## Частые ошибки

- Хардкод CSS-селекторов типа `.card > div:nth-child(2)` — хрупко, ломается от любого твика вёрстки. Используй семантические локаторы.
- `waitForTimeout(5000)` — плохая привычка. Playwright авто-ждёт, используй `toBeVisible`/`toHaveText` вместо слепых пауз.
- Писать E2E на всё подряд — 100+ тестов прогоняются полчаса, CI становится невыносимым, и их перестают ждать.
- Игнорировать flaky-тесты. Один нестабильный тест ломает доверие ко всему пайплайну — чини сразу, а не затыкай `test.skip`.
