# Vite в деталях

## Зачем это нужно

До сих пор ты запускал `npm run dev` и не думал, что под капотом. В прод-проекте это перестаёт работать: нужно понимать, как отдавать env-переменные, как поднять прокси к бэку, как выжать быстрый HMR, что летит в `dist/` и чем `public/` отличается от `src/assets`. Vite — дефолт-сборщик в 2026, знать его обязательно.

## Главное

### Зачем вообще сборщик

Браузеры давно понимают ES-модули, и в dev-режиме можно грузить файлы напрямую. Но в проде проект на 300-500 файлов, каждый из которых тянет соседа через `import` — это сотни сетевых запросов и медленный старт. Сборщик объединяет модули в чанки, минифицирует, режет на куски по динамическим импортам (code splitting) и добавляет хеши в имена для кэширования.

### Vite = ESM в dev + Rollup в prod

В dev Vite почти ничего не собирает: он отдаёт твои `.ts`/`.tsx` файлы как нативные ES-модули, компилирует их на лету через esbuild. Поэтому старт мгновенный даже на большом проекте. На `vite build` включается Rollup и делает честную прод-сборку с tree-shaking, минификацией и чанками.

### Команды

`vite` (или `npm run dev`) запускает dev-сервер с HMR. `vite build` собирает прод-сборку и складывает результат в `dist/`. `vite preview` поднимает локальный сервер поверх `dist/`, чтобы глазами проверить, что билд работает. Это не замена прод-серверу, а санити-чек.

### HMR и React Fast Refresh

Ты правишь компонент — в браузере обновляется только он, а state формы или счётчика сохраняется. Это Hot Module Replacement. Для React оно работает через плагин `@vitejs/plugin-react` (Fast Refresh). Без плагина страница будет перезагружаться целиком.

### Env-переменные

Файлы: `.env`, `.env.local`, `.env.development`, `.env.production`. Префикс `VITE_` обязателен, без него переменная в клиентский бандл не попадёт. Читать через `import.meta.env.VITE_API_URL`. `.env.local` добавь в `.gitignore`, это локальные оверрайды.

### Aliases

`resolve.alias: { "@": path.resolve(__dirname, "./src") }` — и вместо `../../../components/Button` ты пишешь `@/components/Button`. Если используешь TypeScript, продублируй те же пути в `tsconfig.json` в `compilerOptions.paths`, иначе редактор не подсветит.

### Proxy к бэку

`server.proxy: { "/api": "http://localhost:3000" }` — dev-сервер перенаправит все запросы на `/api/*` на бэкенд. В коде просто `fetch("/api/users")`: никакого CORS, никаких абсолютных URL.

### Ассеты

`src/assets/` — импортируешь через `import img from "./logo.png"`, Vite обрабатывает: хеширует имя, оптимизирует, встраивает мелкие картинки как base64. `public/` копируется в `dist/` как есть, доступ по абсолютному пути `/icon.svg`. Туда кладут то, что нельзя трогать: `robots.txt`, `favicon`, `og-image`.

### Output

`dist/` — готовая статика: `index.html` плюс хешированные `.js` и `.css`. Chunk splitting по динамическим импортам (`import("./Heavy.tsx")`) работает из коробки, роуты лениво подгружаются отдельными файлами.

### Плагины

Минимум: `@vitejs/plugin-react` (обязателен для React). Полезно: `vite-tsconfig-paths` сам читает `paths` из `tsconfig.json`, не надо дублировать алиасы в конфиге Vite.

## Пример кода

```ts
// vite.config.ts
import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";
import path from "node:path";

export default defineConfig({
  plugins: [react()],
  resolve: {
    alias: {
      "@": path.resolve(__dirname, "./src"),
    },
  },
  server: {
    port: 5173,
    proxy: {
      "/api": {
        target: "http://localhost:3000",
        changeOrigin: true,
      },
    },
  },
  build: {
    outDir: "dist",
    sourcemap: true,
  },
});

// .env
// VITE_API_URL=https://api.example.com

// src/api.ts
const baseUrl = import.meta.env.VITE_API_URL;

export async function getUsers() {
  const r = await fetch(`${baseUrl}/users`);
  return r.json();
}

// src/App.tsx
import logo from "@/assets/logo.png";      // через alias + обработка Vite
import "@/styles/global.css";

export function App() {
  return (
    <div>
      <img src={logo} alt="logo" />
      <img src="/favicon.svg" alt="" />      {/* из public/ */}
    </div>
  );
}
```

## Частые ошибки

- Забыть префикс `VITE_` у env: переменная не попадёт в клиент, на фронте будет `undefined`.
- Класть секреты в `.env` — он бандлится в клиент, всё видно через DevTools. Для secrets нужна серверная среда.
- Путать `public/` и `src/assets/`: в `public/` картинку не оптимизируют и не хешируют, кэш будет кривой.
- Импортить `process.env`. В Vite это `import.meta.env`, старая привычка из CRA/Webpack.

## Задания

- [ ] Добавь alias `@` → `src` в `vite.config.ts`. Замени 3-5 относительных импортов на alias.
- [ ] Создай `.env.local` с `VITE_API_URL`. Выведи значение в консоль, убедись, что подхватилось.
- [ ] Подними proxy к публичному API (например, `jsonplaceholder.typicode.com`) через `server.proxy`.
- [ ] Собери прод: `npm run build`, посмотри `dist/`, запусти `npm run preview` и открой в браузере.
