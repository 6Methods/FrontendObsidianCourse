# Fetch и работа с API

## Зачем это нужно

Именно здесь проходит граница между «статическим сайтом» и «приложением»: как только страница начинает общаться с сервером, появляется динамика. `fetch` — встроенный в браузер способ отправить HTTP-запрос и получить ответ. Почти вся ежедневная работа фронтендера — это разные способы ходить в API: загрузить данные, отправить форму, обновить состояние.

## Главное

### fetch(url, options)

`fetch` возвращает промис, который резолвится объектом `Response`. Важный нюанс: **`fetch` не падает на HTTP-ошибках** вроде 404 или 500. Такой ответ он считает успешным и просто отдаёт `Response`. Промис реджектится только на сетевых сбоях: нет интернета, DNS, CORS. Поэтому после каждого запроса проверяй `res.ok` вручную и сам кидай исключение, если что-то не так.

### GET-запрос и параметры

GET — метод по умолчанию, `options` можно не передавать. Для query-параметров используй `new URL(url)` и `url.searchParams.set("q", value)` — это корректно экранирует пробелы и спецсимволы. Руками клеить строки через `?` и `&` — путь к багам.

### POST/PUT/DELETE

Для изменения данных передавай объект опций: `{ method, headers, body: JSON.stringify(data) }`. Тело запроса — **строка**, поэтому объект сериализуй через `JSON.stringify`. Обязательно добавь заголовок `Content-Type: application/json`, иначе сервер не поймёт формат.

### Чтение ответа

У `Response` есть методы `res.json()`, `res.text()`, `res.blob()`, и все возвращают промис. Тело ответа можно прочитать **только один раз**: вторая попытка кинет `TypeError: body used already`.

### Статусы и заголовки

`res.status` — число (200, 404), `res.ok` — `true` для 200–299. Типовой подход: `if (!res.ok) throw new Error(...)`, дальше ловим в `catch`. Заголовки ответа — `res.headers.get("content-type")`; свои заголовки кладутся в `{ headers: {...} }`.

### CORS в двух словах

Браузер блокирует запросы к чужому домену, если сервер явно не разрешил их специальным заголовком. В публичных API (GitHub, open-meteo) CORS уже открыт, можно ходить с фронтенда. В свой бэкенд настраивай разрешения на сервере, не на фронте.

### AbortController

Позволяет отменить уже запущенный запрос: создаёшь `const ctrl = new AbortController()`, передаёшь `signal: ctrl.signal` в `fetch`, а потом в нужный момент вызываешь `ctrl.abort()`. Полезно, когда пользователь быстро вводит текст в поиске: старый запрос отменяем, оставляем только последний.

### Типичный паттерн

Показать loader, дальше `await fetch`, дальше `if (!res.ok) throw`, дальше `await res.json()`, обновить UI, в `catch` показать ошибку, в `finally` убрать loader. Эта четвёрка (loader/try/catch/finally) повторяется почти в каждой фиче.

### Таймауты

Нативно у `fetch` таймаута нет, и это частая ловушка для новичков: без явного ограничения запрос может висеть десятки секунд. Раньше делали вручную через `AbortController` и `setTimeout`, который дёргает `abort` по истечении времени. Сейчас есть более короткий вариант — `AbortSignal.timeout(5000)`: одна строка, и запрос сам отменится через пять секунд.

## Пример кода

```js
async function searchUser(login) {
  const url = new URL("https://api.github.com/search/users");
  url.searchParams.set("q", login);

  const res = await fetch(url, {
    headers: { Accept: "application/vnd.github+json" },
  });

  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  return res.json();
}

async function createPost(title, body) {
  const res = await fetch("https://jsonplaceholder.typicode.com/posts", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ title, body, userId: 1 }),
  });

  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  return res.json();
}

async function renderUser(login) {
  const loader = document.querySelector("#loader");
  const error  = document.querySelector("#error");
  const view   = document.querySelector("#view");

  loader.hidden = false;
  error.hidden = true;

  try {
    const res = await fetch(`https://api.github.com/users/${login}`);
    if (!res.ok) throw new Error(`Не найден: ${res.status}`);
    const data = await res.json();
    view.textContent = data.name || data.login;
  } catch (err) {
    error.textContent = err.message;
    error.hidden = false;
  } finally {
    loader.hidden = true;
  }
}

const ctrl = new AbortController();
fetch("https://api.github.com/users/octocat", { signal: ctrl.signal });
ctrl.abort();
```

## Частые ошибки

- Считать, что `fetch` кинет исключение на 404. Нет, проверяй `res.ok` руками.
- Забыть `Content-Type: application/json` при POST с JSON — сервер не распарсит тело.
- Читать `res` дважды — `TypeError: body used already`.
- Не чистить loader в `finally` — при ошибке крутилка висит вечно.
