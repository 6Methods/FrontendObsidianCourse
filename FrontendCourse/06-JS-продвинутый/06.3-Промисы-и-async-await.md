# Промисы и async/await

## Зачем это нужно

Всё, что занимает время (сетевой запрос, `setTimeout`, чтение файла) в JavaScript асинхронно: результат приходит не сразу. Промисы — стандартный способ представить «значение, которое будет позже», а `async/await` — синтаксический сахар, который делает асинхронный код похожим на обычный пошаговый. Без этих вещей невозможно нормально работать с `fetch` и любыми API.

## Главное

### Что такое Promise

`Promise` — объект-обёртка над будущим значением. У него ровно три состояния: `pending` (ожидание), `fulfilled` (успех, есть значение) и `rejected` (ошибка, есть причина). Переход происходит один раз и навсегда: как только промис стал `fulfilled` или `rejected`, изменить его нельзя. Это упрощает рассуждения о коде: результат гарантированно придёт один раз.

### Создание промиса

Вручную промисы создают редко — чаще их возвращают готовые API: `fetch`, обёртки над таймерами, SDK библиотек. Синтаксис: `new Promise((resolve, reject) => { ... })`. Внутри колбэка вызываешь `resolve(value)` при успехе и `reject(error)` при ошибке. Типичный самописный пример — обёртка над `setTimeout`, превращающая его в ожидание.

### `.then` / `.catch` / `.finally`

`.then(onFulfilled)` получает значение успешного промиса. `.catch(onRejected)` ловит ошибки (эквивалент `then(null, onRejected)`). `.finally(cb)` выполняется в любом случае — удобно для очистки, например, спрятать спиннер.

### Цепочки

Каждый `then` возвращает новый промис. Значение из одного `then` попадает в следующий, а ошибка пролетает по цепочке до ближайшего `.catch`. Это позволяет выстраивать шаги обработки данных один за другим.

### `async/await`

`async function` всегда возвращает промис, даже если внутри `return 42` — снаружи это будет `Promise<number>`. `await` ставит функцию на паузу, пока промис не завершится, и возвращает его значение. Код читается сверху вниз, без вложенных `then`.

### Обработка ошибок

Вокруг `await` удобно ставить обычный `try/catch`: в `catch` прилетит то, чем был отклонён промис. Такой код чище цепочек `.catch`, особенно когда ошибка может случиться на нескольких шагах подряд.

### `Promise.all`

Принимает массив промисов и ждёт, пока завершатся все. Возвращает массив значений в том же порядке. Если хоть один промис отклонён, `Promise.all` сразу падает с этой ошибкой, остальные результаты теряются.

### `Promise.allSettled`

Тоже ждёт все, но не падает. Возвращает массив объектов `{status: "fulfilled", value}` или `{status: "rejected", reason}`. Нужен, когда важны и успешные, и неуспешные результаты — например, несколько независимых запросов.

### `Promise.race`

Возвращает результат первого завершившегося промиса — успех или ошибку. Классика применения — таймауты: гоняем запрос против `wait(5000)` и ловим то, что сработает раньше.

### Последовательно vs параллельно

`await a(); await b();` запускает `b` только после `a`. Это медленно, если задачи независимы. `await Promise.all([a(), b()])` стартует обе сразу и ждёт обе — заметно быстрее. Последовательность нужна только тогда, когда результат `a` требуется для `b`.

## Пример кода

```js
const wait = (ms) => new Promise(resolve => setTimeout(resolve, ms));

fetch("https://api.github.com/users/octocat")
  .then(res => res.json())
  .then(data => console.log(data.name))
  .catch(err => console.error("Ошибка:", err));

async function loadUser(login) {
  try {
    const res = await fetch(`https://api.github.com/users/${login}`);
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    const data = await res.json();
    return data;
  } catch (err) {
    console.error("Ошибка:", err);
    return null;
  }
}

loadUser("octocat").then(user => console.log(user?.name));

async function loadMany() {
  const [a, b] = await Promise.all([
    loadUser("octocat"),
    loadUser("torvalds"),
  ]);
  return [a, b];
}

async function loadSlow() {
  const a = await loadUser("octocat");
  const b = await loadUser("torvalds");
  return [a, b];
}
```

## Частые ошибки

- Забыть `await` перед вызовом `async`-функции — в переменной окажется сам промис, а не данные.
- Не обрабатывать ошибки (нет `.catch` и нет `try/catch`) — получаешь unhandled promise rejection и молчаливые баги.
- Последовательные `await` там, где запросы независимы: теряешь секунды на ровном месте, используй `Promise.all`.
- `throw` внутри колбэка `.then` без последующего `.catch` — ошибка «съедается» и никуда не попадает.
