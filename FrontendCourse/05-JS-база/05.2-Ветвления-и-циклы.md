# Ветвления и циклы

## Зачем это нужно

Программа почти никогда не выполняется строго сверху вниз. Нужно принимать решения («если пользователь вошёл — покажи профиль, иначе — форму логина») и повторять действия над коллекциями данных. Ветвления и циклы — это базовый инструмент управления потоком, без которого не обойдётся ни один реальный фронтенд.

## Главное

### if / else if / else

Условие внутри `if (...)` приводится к boolean по правилам truthy/falsy. Пишешь `if (user)` — проверяешь, что переменная не пустая и не `null`. Цепочка `else if` — когда вариантов несколько, `else` — если ни одна ветка не сработала. Фигурные скобки ставь всегда, даже для одной строки: меньше шансов выстрелить в ногу при правках.

### Тернарный оператор

`условие ? а : б` — короткая запись однострочного `if/else`, обычно для выбора значения. Отлично подходит для JSX и присваиваний. Не вкладывай тернарники друг в друга: получается нечитаемо. Если вариантов больше двух — разбей на нормальный `if` или вынеси в функцию.

### switch

Сравнивает значение с каждым `case` через строгое равенство (`===`). Каждый `case` обязан заканчиваться `break`, иначе начнётся fall-through — выполнение «провалится» в следующий кейс. На практике `switch` часто заменяют объектом-словарём: `const label = { ru: "Привет", en: "Hi" }[lang]` — короче, гибче и проще расширять.

### Truthy / falsy

Falsy-значений всего шесть: `false`, `0`, `""`, `null`, `undefined`, `NaN`. Всё остальное — truthy, включая `"0"`, `[]` и `{}`. Поэтому `if (arr.length)` надёжнее, чем `if (arr)` — пустой массив всё равно truthy. Привыкай думать категориями truthy/falsy, это сильно сокращает код.

### Логические операторы как выбор

`a && b` вернёт `a`, если оно falsy, иначе `b`. `a || b` — наоборот: первое truthy. Удобно для дефолтов: `const name = input || "гость"`. Но `||` съест `0` и `""` — если это валидные значения, используй `??`, который срабатывает только для `null` и `undefined`.

### Циклы

- `while (cond)` — крутится, пока условие истинно; `do..while` — то же, но тело выполнится хотя бы один раз.
- Классический `for (let i = 0; i < n; i++)` — когда реально нужен индекс или идёшь с шагом.
- `for...of` — современный перебор по элементам массива, строки, `Set`, `Map`. Читается как «для каждого элемента».
- `for...in` — по ключам объекта. На массивах НЕ использовать: подхватит прототипные свойства и не гарантирует порядок обхода.

### break и continue

`break` полностью выходит из цикла, `continue` пропускает текущую итерацию и идёт к следующей. В большинстве случаев `for...of` и методы массивов `.forEach`/`.map`/`.filter` (см. 05.4) читабельнее голого `for` и лучше выражают намерение.

## Пример кода

```js
const age = 25;
if (age >= 18) { console.log("взрослый"); } else { console.log("несовершеннолетний"); }

const label = age >= 18 ? "взрослый" : "несовершеннолетний";

const day = "пн";
switch (day) {
  case "сб":
  case "вс":
    console.log("выходной");
    break;
  default:
    console.log("будний");
}

const input = "";
const name = input || "гость";

for (let i = 0; i < 3; i++) { console.log(i); }

const fruits = ["яблоко", "груша", "слива"];
for (const fruit of fruits) { console.log(fruit); }

for (let i = 0; i < 10; i++) {
  if (i === 3) continue;
  if (i === 7) break;
  console.log(i);
}
```

## Частые ошибки

- `switch` без `break` — молчаливый fall-through в следующий кейс.
- `if (x == true)` — лишнее приведение, пиши просто `if (x)`.
- `for...in` по массиву — подхватит прототипные свойства и даст неверный порядок.
- Вложенные тернарники — нечитаемо, замени на `if/else` или вынеси в функцию.
