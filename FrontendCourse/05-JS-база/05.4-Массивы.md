# Массивы

## Зачем это нужно

Массив — это упорядоченный список значений. Во фронтенде массивы буквально везде: задачи в todo, сообщения в чате, товары в каталоге, пункты меню. А ещё у массивов есть мощные методы, которые делают код коротким и декларативным: ты описываешь, что хочешь получить, а не как это сделать вручную.

## Главное

### Создание и доступ, `.length`

Массив создают литералом: `const arr = [1, 2, 3]`. Доступ по индексу идёт с нуля: `arr[0]` даёт первый элемент, `arr[arr.length - 1]` — последний. Свойство `.length` это количество элементов, его можно читать и даже присваивать (обрезать массив). В массиве могут лежать значения любых типов: числа, строки, булевы, другие массивы и объекты. Массив это объект под капотом, поэтому сравнение `===` работает по ссылке, а не по содержимому.

### Мутирующие методы

Они меняют оригинальный массив и возвращают либо удалённый элемент, либо новую длину. `push` добавляет в конец, `pop` забирает с конца, `unshift` добавляет в начало, `shift` забирает с начала. `splice(start, count, ...items)` вырезает кусок и/или вставляет новые элементы по индексу. `sort` сортирует массив на месте (по умолчанию как строки — для чисел обязательно передавай компаратор `(a, b) => a - b`), `reverse` переворачивает. Все они меняют исходный массив. Забегая вперёд: в React (модуль 09) это станет источником багов, потому что ссылка на массив не меняется и компонент не перерисовывается. Поэтому уже сейчас полезно привыкать к иммутабельному стилю.

### Неизменяющие методы

`slice(start, end)` возвращает копию части массива, не трогая оригинал. `concat` склеивает массивы и тоже возвращает новый. Современный стиль во фронте — максимум иммутабельности: не менять оригинал, а возвращать новый массив с изменениями. Так проще отслеживать состояние и не ловить неожиданные баги.

### Итерация и преобразования

- `.forEach(cb)` просто проходит по элементам и ничего не возвращает. Годится только для побочных эффектов, например логирования.
- `.map(cb)` возвращает новый массив той же длины, где каждый элемент преобразован функцией. Это главный метод во фронте: массив данных → массив `<li>`-тегов (а позже, в React, — массив компонентов).
- `.filter(cb)` — новый массив только с теми элементами, для которых callback вернул truthy-значение.
- `.find(cb)` возвращает первый подходящий элемент или `undefined`, если ничего не нашлось.
- `.some(cb)` и `.every(cb)` возвращают boolean: «есть хотя бы один» и «подходят все».
- `.includes(value)` — простая проверка наличия значения, удобна для примитивов (чисел, строк).
- `.reduce(cb, initial)` — свёртка массива в одно значение: сумма, объект-индекс, группировка. Начинай с `map` и `filter`, они читаются проще. `reduce` доставай, только когда они реально не справляются.

### Spread и деструктуризация

`[...arr]` — быстрая поверхностная копия массива. `[...a, ...b]` — склейка двух массивов. `[...arr, newItem]` — иммутабельное добавление в конец без мутации. Деструктуризация `const [a, b] = arr` достаёт элементы по позиции в отдельные переменные. Пригодится для любых пар значений, а в модуле 09 встретим её в хуках React (например, `const [count, setCount] = useState(0)`).

## Пример кода

```js
const items = [
  { id: 1, title: "Купить хлеб", done: false },
  { id: 2, title: "Помыть кота",  done: true },
  { id: 3, title: "Сделать зарядку", done: false },
];

const titles = items.map(item => item.title);
const active = items.filter(item => !item.done);
const byId = items.find(item => item.id === 2);
const hasDone = items.some(item => item.done);
const allDone = items.every(item => item.done);
const totalDone = items.reduce((acc, item) => acc + (item.done ? 1 : 0), 0);
const withNew = [...items, { id: 4, title: "Новая", done: false }];
const [firstItem, secondItem] = items;
```

## Частые ошибки

- Использовать `.map` там, где ничего не возвращаешь — тогда нужен `.forEach`. `.map` без return даёт массив `undefined`.
- Мутировать массив внутри `.map` или `.filter` (callback делает `push` во внешний массив) — получаются неожиданные побочные эффекты и хаос.
- Забыть `return` в стрелочной функции с фигурными скобками: `arr.map(x => { x * 2 })` вернёт `[undefined, undefined, ...]`. Либо пиши `x => x * 2`, либо добавляй `return`.
- Вызывать `.reduce` без initial на массиве, который может оказаться пустым — будет `TypeError`. Всегда передавай начальное значение.
