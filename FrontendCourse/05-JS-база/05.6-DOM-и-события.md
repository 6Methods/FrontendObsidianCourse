# DOM и события

## Зачем это нужно

До сих пор JS жил в консоли: считал, проверял, логировал. Пора подключать его к странице: находить элементы, менять текст и классы, реагировать на клики и ввод. Вот это и есть фронтенд в узком смысле слова.

## Главное

### DOM это дерево объектов

Когда браузер загружает HTML, он строит по нему дерево объектов — DOM (Document Object Model). Всё, что ты видишь во вкладке DevTools → Elements, доступно из JS как обычные объекты со свойствами и методами. Корень дерева это глобальный объект `document`, от него ты и ходишь по веткам.

### Поиск элементов

Основной инструмент это `document.querySelector("sel")`: принимает любой CSS-селектор и возвращает первый подходящий элемент или `null`, если ничего не нашлось. Если нужны все совпадения — `document.querySelectorAll("sel")`, он вернёт `NodeList`. По нему можно идти `.forEach`, но `.map` не сработает, сначала сделай `Array.from(...)`. Древний `getElementById` помнить не обязательно: `querySelector("#id")` делает то же самое.

### Контент и значения

Текст читай и пиши через `textContent` — он безопасный, всё экранирует. `innerHTML` вставляет HTML как разметку, поэтому с данными от пользователя это прямой путь к XSS. Для `<input>`, `<textarea>` и `<select>` текущее значение живёт не в `textContent`, а в `value`.

### Классы, стили и атрибуты

Под классы есть `classList`: `add`, `remove`, `toggle`, `contains`. Прямой `style.color = "red"` тоже работает, но использовать его стоит редко — почти всегда лучше переключить CSS-класс. Атрибуты читаются через `getAttribute`/`setAttribute` или напрямую как свойства: `img.src = "..."`, `input.disabled = true`.

### Создание и вставка

Новый элемент создаётся через `document.createElement("li")`, пока он просто висит в памяти. Чтобы он появился на странице, вставь его методом `parent.append(el)`: современный метод, принимает сразу несколько узлов и даже строки. Удалить элемент — `el.remove()`.

### События

Слушатель вешается через `element.addEventListener("click", handler)`. Самые частые события: `click`, `input`, `change`, `submit`, `keydown`. Отписаться можно `removeEventListener`, но только с той же самой ссылкой на функцию — анонимка не подойдёт.

### Объект события

Хендлер получает `event`. Важное: `event.target` — конкретный элемент, по которому кликнули; `event.preventDefault()` — отменить поведение по умолчанию (например, перезагрузку на submit формы); `event.stopPropagation()` — остановить всплытие.

### Всплытие и делегирование

События всплывают от конкретного элемента к корню. Это даёт удобный трюк под названием делегирование: вместо ста слушателей на каждый `<li>` вешаешь один на `<ul>` и проверяешь `event.target`. Меньше подписок, работает и для элементов, добавленных позже.

### DOMContentLoaded

Если `<script>` стоит в `<head>` без атрибута `defer`, он выполнится раньше, чем браузер прочитает разметку, и `querySelector` вернёт `null`. Решений три: добавь `defer`, перенеси `<script>` в конец `<body>` или оберни код в `document.addEventListener("DOMContentLoaded", ...)`.

## Пример кода

```html
<form id="todo-form">
  <input id="todo-input" type="text" placeholder="Новая задача" required>
  <button type="submit">Добавить</button>
</form>

<ul id="todo-list"></ul>
```

```js
const form = document.querySelector("#todo-form");
const input = document.querySelector("#todo-input");
const list  = document.querySelector("#todo-list");

form.addEventListener("submit", (event) => {
  event.preventDefault();
  const title = input.value.trim();
  if (!title) return;

  const li = document.createElement("li");
  li.textContent = title;
  li.classList.add("todo-item");

  const removeBtn = document.createElement("button");
  removeBtn.textContent = "✕";
  removeBtn.addEventListener("click", () => li.remove());

  li.append(removeBtn);
  list.append(li);

  input.value = "";
});

list.addEventListener("click", (event) => {
  if (event.target.tagName === "LI") {
    event.target.classList.toggle("done");
  }
});
```

## Частые ошибки

- Вставлять пользовательский ввод через `innerHTML` → XSS. Используй `textContent` или `createElement`.
- Забыть `event.preventDefault()` на submit формы → страница перезагрузится и состояние потеряется.
- Вызвать `querySelector` до готовности DOM (скрипт в `<head>` без `defer`) → вернётся `null`.
- Подписываться в цикле и не снимать слушатели `removeEventListener` в SPA → утечки памяти и двойные срабатывания.

## Задания

- [ ] Через `querySelector` найди элемент, поменяй его `textContent` и добавь класс `active`.
- [ ] Повесь обработчик `click` на кнопку и выведи `event.target` в консоль.
- [ ] Сделай форму: по submit добавляет `<li>` в `<ul>`, страница не перезагружается.
- [ ] Используй делегирование: один `click` на `<ul>` — определи, по какому `<li>` кликнули.
- [ ] Положи скрипт в `<head>` без `defer` — убедись, что `querySelector` возвращает `null`, и почини через `DOMContentLoaded`.
