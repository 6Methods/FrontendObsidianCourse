# Функции

## Зачем это нужно

Функция — это переиспользуемый кусок логики, у которого есть имя и, как правило, параметры на вход и результат на выходе. В JavaScript функции это «первоклассные» значения: их можно хранить в переменных, передавать в другие функции и возвращать из них. Именно это открывает работу с `map`, `filter`, коллбеками и всеми «функциональными» приёмами, которые ты будешь встречать каждый день.

## Главное

### Function declaration

Классическая запись через ключевое слово `function`: `function greet() {}`. Такие функции «поднимаются» (hoisting), их можно вызвать в файле ещё до строки объявления. Удобно, когда хочется описать вспомогательные функции ниже, а использовать их сверху.

### Function expression

Функция, записанная как значение переменной: `const greet = function() {};`. В отличие от declaration, она не поднимается: до строки объявления переменной использовать её нельзя. Зато её легко передавать в другие места — это обычное значение.

### Arrow function

Стрелочные функции — современный и самый короткий синтаксис. Главное отличие в том, что у них нет собственного `this`: он берётся из окружающего кода (детали разберём в модуле 06, когда дойдём до объектов и методов).

### Синтаксис стрелок

Для двух и более параметров — скобки: `(a, b) => a + b`. Для одного параметра скобки можно опустить: `x => x * 2`. Без параметров — пустые скобки: `() => 42`. Если тело в `{}`, возвращать значение нужно явно через `return`: `x => { return x * 2; }`.

### Параметры по умолчанию

Если аргумент не передан, подставится значение по умолчанию: `function hello(name = "гость") {}`. Работает и в declaration, и в arrow-функциях.

### Rest-параметры

`function sum(...nums)` собирает все переданные аргументы в массив `nums`. Это современная замена устаревшему `arguments`: работает предсказуемо и сразу даёт массив со всеми методами.

### `return` и выход из функции

Без `return` функция возвращает `undefined`. После `return` тело не выполняется, поэтому это удобный способ делать ранний выход из функции по условию.

### Область видимости

`let` и `const` видны только внутри блока, где объявлены. Function declaration видна во всей родительской функции (или в модуле), независимо от того, внутри какого блока она написана.

### Замыкания

Функция запоминает окружение, в котором была создана. Классический пример это счётчик: внешняя функция хранит `count`, внутренняя его увеличивает и возвращает. Подробнее — в модуле 06.

### Функции как значения

Функции можно передавать как аргументы и возвращать из других функций: `items.map(double)`, `button.addEventListener("click", handler)`. Это и есть «коллбеки».

### Чистые функции

Один и тот же вход всегда даёт один и тот же выход, без изменений внешнего мира (side effects). Такие функции проще тестировать, отлаживать и переиспользовать.

## Пример кода

```js
function square(n) { return n * n; }

const cube = function(n) { return n * n * n; };

const double = (n) => n * 2;
const sum = (a, b) => a + b;
const greet = (name = "гость") => `Привет, ${name}!`;

const total = (...nums) => nums.reduce((s, n) => s + n, 0);
console.log(total(1, 2, 3, 4));

const items = [1, 2, 3];
const doubled = items.map(double);

function createCounter() {
  let count = 0;
  return () => ++count;
}
const counter = createCounter();
counter(); counter();

function describe(user) {
  if (!user) return "нет пользователя";
  return `Привет, ${user.name}`;
}
```

## Частые ошибки

- Забыть `return` в стрелочной функции с `{}` — получишь `undefined` и будешь долго искать, почему.
- Мутировать входные параметры (например, `arr.push(...)` по переданному массиву) — ломает предсказуемость и усложняет отладку.
- Использовать `this` внутри стрелочной функции, рассчитывая на привязку к объекту — у стрелок своего `this` нет.
- Писать огромные функции «на всё сразу» — разбивай на мелкие с говорящими именами, код станет читабельнее.
