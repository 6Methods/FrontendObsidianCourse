# Переменные, типы, операторы

## Зачем это нужно

Любая программа по сути делает одно: хранит какие-то данные и что-то с ними делает. Переменные — это именованные коробки, куда ты складываешь значения. Типы говорят, что именно лежит в коробке. Операторы определяют, что с этим можно сделать. Без этих трёх штук ты физически не напишешь ни одной осмысленной строки JavaScript.

## Главное

### let и const, и забудь про var

Для объявления переменных используй `const`, если значение не меняется, и `let`, если меняется. Старый `var` не трогай: он всплывает в начало функции (hoisting) и игнорирует блочную область видимости, поэтому за пределами `if` или `for` он остаётся живым и ломает логику. `let` и `const` живут только внутри `{}`, где их объявили. Это предсказуемо.

### Примитивные типы

В JS семь примитивов: `number` (и целые, и дробные — всё одно), `string` (текст), `boolean` (`true`/`false`), `null` (пустота явная), `undefined` (пустота по умолчанию), `bigint` (огромные целые с суффиксом `n`), `symbol` (уникальный идентификатор, в начале не пригодится).

### Объекты

Всё, что не примитив, — объект: обычные `{}`, массивы `[]`, функции. Объекты хранятся по ссылке, и это принципиально отличает их от примитивов. Подробно разберём в 05.5.

### typeof и его баг

Оператор `typeof x` возвращает строку с именем типа. Главная ловушка: `typeof null === "object"`. Это исторический баг языка, который никогда не починят. Запомни и живи с этим.

### Арифметика

Работают привычные `+`, `-`, `*`, `/`, `%` (остаток от деления), `**` (степень). Но `+` со строкой — это не сложение, а склейка: `"2" + 2` даёт `"22"`.

### Сравнение

Всегда пиши `===` и `!==`: они сравнивают и значение, и тип. От `==` и `!=` держись подальше: они приводят типы молча, и `0 == ""` вдруг становится `true`.

### Логика

`&&` это «и», `||` это «или», `!` это «не». Современный `??` (nullish coalescing) возвращает правое значение только если левое — `null` или `undefined`. Оператор `?.` (optional chaining) безопасно достаёт поле из, возможно, пустого объекта: `user?.name`.

### Тернарник

Короткий `if`: `условие ? a : b`. Возвращает `a`, если условие истинно, иначе `b`. Удобно для маленьких выборов в одну строку.

### Преобразования типов

Явно: `Number("42")`, `String(42)`, `Boolean(0)`. Неявно JS приводит сам и часто неожиданно: `"3" - 1 === 2`, потому что минус заставляет строку стать числом. Лучше приводить руками.

### Шаблонные строки

Обратные кавычки `` ` `` позволяют вставлять переменные через `${...}` и делать многострочный текст без `\n`: `` `Привет, ${name}!` ``. Чисто, читаемо, современно.

## Пример кода

```js
const name = "Иван";
let age = 28;
age += 1;

const isAdult = age >= 18;
const greeting = `Привет, ${name}! Тебе ${age}.`;

console.log(1 === "1");  // false
console.log(1 == "1");   // true — не используй

const user = null;
const display = user?.name ?? "гость";  // "гость"

const num = Number("42");
const str = String(42);
const flag = Boolean(0);

const status = age >= 18 ? "взрослый" : "несовершеннолетний";
console.log(status);
```

## Частые ошибки

- Пишут `==` вместо `===` и получают скрытые баги типа `0 == ""` или `null == undefined`, которые всплывают в самый неподходящий момент.
- Тянут `var` из старых туториалов, ломают область видимости и ловят странные баги с hoisting. В 2026 году `var` не пишут.
- Думают, что `const arr = [1, 2]; arr.push(3)` выдаст ошибку. Нет: `const` запрещает переназначать саму переменную, но содержимое объекта или массива мутировать можно.
- Клеят строки через `+`: `"Привет, " + name + "!"`. Работает, но шаблонные строки через `` ` `` читаются в разы лучше.
