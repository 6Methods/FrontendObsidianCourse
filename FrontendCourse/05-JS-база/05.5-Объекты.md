# Объекты

## Зачем это нужно

Объект — это структура из пар «ключ → значение», базовый способ группировать связанные данные под одной переменной. Пользователь, товар в корзине, задача в todo-листе, ответ от API — всё это объекты. Без свободной работы с объектами не распарсить JSON, не сохранить ничего в `localStorage` и не работать со state в React (познакомимся с ним в модуле 09).

## Главное

### Литеральный синтаксис

Чаще всего объект создают литералом через фигурные скобки: `const user = { name: "Иван", age: 28 }`. Ключи это строки (кавычки не обязательны, если имя — валидный идентификатор), значения — что угодно: число, строка, массив, другой объект, функция.

### Доступ к свойствам

Через точку: `user.name` — самый частый способ. Через квадратные скобки: `user["name"]` — нужен, когда ключ лежит в переменной (`user[key]`) или когда имя ключа невалидно для точки (например, содержит пробел или дефис).

### Изменение, добавление, удаление

`user.name = "Пётр"` — меняем. `user.city = "Москва"` — добавляем новое поле прямо на лету. `delete user.age` — удаляем (используется редко, чаще просто перезаписывают).

### Object.keys / values / entries

`Object.keys(user)` вернёт массив ключей, `Object.values(user)` — значений, `Object.entries(user)` — массив пар `[ключ, значение]`. После этого работают привычные методы массива: `map`, `filter`, `reduce`, `forEach`.

### Вычисляемые ключи и shorthand

Если имя ключа хранится в переменной, его берут в скобки: `const key = "name"; const obj = { [key]: "Иван" }`. Если переменная и ключ называются одинаково, пишут коротко: `const name = "Иван"; const user = { name }` вместо `{ name: name }`.

### Методы

Функцию-свойство можно писать коротко: `{ greet() { return "hi"; } }`. Это то же самое, что `greet: function() {...}`.

### Spread и иммутабельное обновление

`{...obj}` создаёт поверхностную копию. `{...user, age: 30}` — копия, в которой `age` перезаписан. Этот паттерн (иммутабельное обновление вместо мутации) станет фундаментом работы со state в React — увидим в модуле 09.

### Деструктуризация

`const { name, age } = user` — вытащить поля в переменные. С переименованием: `const { name: userName } = user`. С дефолтом: `const { age = 0 } = user`. С остатком: `const { name, ...rest } = user` — `rest` заберёт всё, кроме `name`.

### JSON

`JSON.stringify(obj)` превращает объект в строку, `JSON.parse(str)` — строку обратно в объект. Абсолютный минимум для работы с API, `fetch` и `localStorage`.

### Объекты по ссылке

`const a = {x:1}; const b = a; b.x = 2;` — теперь `a.x === 2`, потому что `a` и `b` указывают на один и тот же объект в памяти. Чтобы получить независимую копию, используют spread `{...a}` или `structuredClone(a)` для глубокого копирования.

## Пример кода

```js
const user = {
  name: "Иван",
  age: 28,
  email: "ivan@example.com",
  greet() { return `Привет, я ${this.name}!`; },
};

console.log(user.name);
console.log(user["email"]);

user.age = 29;
user.city = "Москва";

const key = "status";
const value = "active";
const profile = { name: "Иван", [key]: value };

const { name, age, city = "неизвестно" } = user;

const updatedUser = { ...user, age: 30 };

const keys = Object.keys(user);
const pairs = Object.entries(user);

const json = JSON.stringify(user);
const parsed = JSON.parse(json);

const a = { x: 1 };
const b = a;
b.x = 2;
console.log(a.x);  // 2
const c = { ...a }; // независимая копия
```

## Частые ошибки

- Думать, что `const obj = {...}` запрещает изменения — нет, `const` запрещает только переназначение самой переменной, а поля объекта менять можно свободно.
- `const copy = obj` это не копия, а вторая ссылка на тот же объект; изменение через `copy` изменит и `obj`.
- `JSON.parse(JSON.stringify(obj))` как deep-copy падает на функциях, `Date`, `undefined` и циклических ссылках; современный способ — `structuredClone(obj)`.
- `for...in` по объекту без `hasOwnProperty` подхватит свойства из прототипа; проще и безопаснее использовать `Object.keys` или `Object.entries`.
