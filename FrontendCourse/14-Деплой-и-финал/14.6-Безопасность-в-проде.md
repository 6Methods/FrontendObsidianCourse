# 14.6 — Безопасность в проде

## Зачем это нужно

«Локально работает» не значит «можно в прод». Забыл префикс у env-переменной → `service_role` key утёк на клиент, и любой юзер может удалить все записи. Не поставил CSP → XSS-скрипт уводит сессии твоих пользователей. Не лимитируешь форму → бот спамит 10 000 запросов за минуту, и ты должен Vercel $200. Безопасность в проде это не paranoid-уровень и не «нанимаем аудитора», а базовый чеклист, который должен быть пройден до того, как ссылка уйдёт публично. Пропустишь шаг — узнаешь о нём по счёту или по твиту «смотрите, какой дырявый сайт».

## Главное

### Env vars и секреты

Правило одно: всё с префиксом `NEXT_PUBLIC_*` уходит на клиент, попадает в JS-бандл, и любой через DevTools его прочитает. Всё остальное — только на сервере. Supabase `anon key` можно класть в `NEXT_PUBLIC_`, он и рассчитан на публичное использование (работает вместе с RLS). А вот `SUPABASE_SERVICE_ROLE_KEY`, OAuth `client_secret`, Sentry auth token — никогда. Проверяй перед пушем: `grep -r "NEXT_PUBLIC_" .env.production`. В списке должно быть только то, что тебе не страшно показать миру.

### Security headers

`Content-Security-Policy` (CSP) запрещает браузеру грузить XSS-скрипты с чужих доменов. `Strict-Transport-Security` (HSTS) — только HTTPS, браузер сам редиректит. `X-Frame-Options: DENY` — нельзя встраивать твой сайт в `<iframe>` (защита от clickjacking). `X-Content-Type-Options: nosniff` — браузер не угадывает MIME-типы. `Referrer-Policy` контролирует, что уходит в `Referer`. Всё это настраивается в `next.config.ts` через функцию `headers()` — один раз написал, работает глобально.

### Content-Security-Policy

CSP нельзя скопировать у соседа, его надо настроить под свой стек. Пример: `default-src 'self'; script-src 'self' 'unsafe-inline' vercel.live; img-src 'self' data: https://storage.supabase.co`. Проверить результат — [Mozilla Observatory](https://observatory.mozilla.org). В Next.js плюс App Router часто нужен `'unsafe-inline'` для стилей: прими компромисс, nonce-подход сложнее и для учебного проекта избыточен.

### CORS в Route Handlers

Если твой API вызывается с другого домена (мобильное приложение, внешний партнёр), ставь конкретное значение `Access-Control-Allow-Origin: https://your-partner.com`, не `*`. Звёздочка означает «пускаю кого угодно», и это почти всегда ошибка. Для своего же фронта на том же домене CORS не нужен вообще, это same-origin, браузер не блокирует.

### Rate limiting

Любая публичная форма или API без лимита это будущий DDoS и/или счёт за трафик. **Upstash Ratelimit** — бесплатный tier, Redis на edge, пример конфига: 10 запросов/минуту с одного IP. В Server Action пишешь `await limiter.limit(ip); if (!success) throw new Error(...)`. На Pro-плане Vercel есть нативный Vercel WAF, настраивается в дашборде без кода.

### Vercel BotID

Защита от ботов на критичных формах: логин, регистрация, создание записей. GA с июня 2025. Подключается одним компонентом в форму плюс server-side проверка токена. Блокирует headless-браузеры и работает по репутации IP, не ломает UX для живых людей, как капча.

### RLS в Supabase

Никогда не выключай RLS в проде. Политика «читать только свои строки» это не фронтенд-фильтр и не условие в запросе, а защита на уровне БД. Даже если ты забыл `.eq('user_id', user.id)` в запросе, RLS не отдаст чужие данные. Это твой последний рубеж.

### OAuth redirect URLs

В Google Cloud Console / GitHub OAuth Apps добавь прод-домен в список разрешённых redirect URL. Иначе после логина пользователя редиректнёт на `localhost`, и всё сломается. Это типичный баг первого деплоя, который ловят сами пользователи.

### Dependency security

`npm audit` в CI это минимум. Dependabot на GitHub сам шлёт PR с обновлениями уязвимых пакетов. Не игнорируй алерты: половина прод-уязвимостей прилетает из зависимостей, а не из твоего кода.

## Пример кода

`next.config.ts` с security-заголовками:

```ts
import type { NextConfig } from 'next';

const securityHeaders = [
  {
    key: 'Content-Security-Policy',
    value:
      "default-src 'self'; " +
      "script-src 'self' 'unsafe-inline' https://vercel.live; " +
      "style-src 'self' 'unsafe-inline'; " +
      "img-src 'self' data: https://*.supabase.co; " +
      "connect-src 'self' https://*.supabase.co;",
  },
  { key: 'Strict-Transport-Security', value: 'max-age=63072000; includeSubDomains; preload' },
  { key: 'X-Frame-Options', value: 'DENY' },
  { key: 'X-Content-Type-Options', value: 'nosniff' },
  { key: 'Referrer-Policy', value: 'strict-origin-when-cross-origin' },
];

const nextConfig: NextConfig = {
  async headers() {
    return [{ source: '/:path*', headers: securityHeaders }];
  },
};

export default nextConfig;
```

Upstash Ratelimit в Server Action:

```ts
'use server';

import { Ratelimit } from '@upstash/ratelimit';
import { Redis } from '@upstash/redis';
import { headers } from 'next/headers';

const limiter = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(10, '60 s'),
  analytics: true,
});

export async function createNote(formData: FormData) {
  const h = await headers();
  const ip = h.get('x-forwarded-for')?.split(',')[0] ?? 'anonymous';

  const { success } = await limiter.limit(ip);
  if (!success) {
    throw new Error('Слишком много запросов, подожди минуту');
  }

  // дальше обычная логика создания заметки
}
```

`SUPABASE_SERVICE_ROLE_KEY` идёт без `NEXT_PUBLIC_` и используется только в Server Actions/Route Handlers.

## Частые ошибки

- `SUPABASE_SERVICE_ROLE_KEY` попал в `NEXT_PUBLIC_*` — кто угодно может удалить твою БД через DevTools. Всегда без префикса, читается только на сервере.
- CSP написан «по туториалу» без проверки, блокирует свои же inline-скрипты, и страница становится белой. Прогони через Mozilla Observatory и исправь до публичного деплоя.
- Нет rate limit на Server Action создания: бот за ночь создаст 100 000 записей, и тебе придёт счёт за Supabase/Vercel. Ставь `Ratelimit.slidingWindow(10, '60 s')` минимум.
- RLS выключен «для простоты», и из инкогнито видны приватные заметки всех юзеров. Включай и явно проверяй через второй аккаунт, а не «на слово».

## Задания

1. В `next.config.ts` настрой security headers: CSP, HSTS, X-Frame-Options, X-Content-Type-Options, Referrer-Policy. Задеплой и проверь через [securityheaders.com](https://securityheaders.com), цель A или A+.
2. Подключи Upstash Ratelimit (бесплатный tier) к Server Action создания заметки из M13. Лимит: 10 запросов/минуту с одного IP. Проверь через цикл в DevTools Console: на 11-м запросе должна прилетать ошибка.
3. В Supabase проверь RLS: залогинься как юзер А, создай приватную заметку. Открой инкогнито, залогинься как юзер B, попробуй получить список заметок А через прямой запрос. Не должно вернуть приватных.
4. Запусти `npm audit` в своём проекте. Исправь все `high`/`critical` через `npm audit fix` или ручное обновление. Включи Dependabot в Settings → Security → Dependabot alerts.
