# 14.7 — Финал: резюме и что дальше

## Зачем это нужно

Код написан, проекты работают, но работодатель их не увидит, если ты не упакуешь. Половина оффера — это как ты показываешь свою работу: pinned repos, чистый README, портфолио-сайт, LinkedIn с проектами, резюме. Вторая половина — что ты учишь дальше, потому что фронтенд не стоит на месте. Этот урок закрывает курс: три проекта в портфолио, конкретный план подготовки к собесу и направления роста. Получать оффер нужно уметь так же, как писать код.

## Главное

### Портфолио-сайт

Одна страница, этого достаточно. Структура: кто ты (1–2 строки), стек (React, Next.js, TypeScript), три проекта (P1 — лендинг, P2 — React SPA, P3 — fullstack Next.js), контакты (email, GitHub, LinkedIn). Делаешь на том же `Next.js + Tailwind`, в котором уже уверен — это и есть демонстрация навыков. Деплой на Vercel. Домен купи на Namecheap (~$10/год) или оставь `username.vercel.app`. Цель: HR открыл ссылку → за 30 секунд понял, что ты умеешь и как с тобой связаться.

### README проектов

Золотой стандарт README: заголовок плюс одна строка описания, скриншот или gif главной, **live URL** (жирным, чтобы не искать), стек списком, фичи булитами, «как запустить локально» (`git clone` → `npm install` → `npm run dev`), env-vars (имена, не значения!), credits. README — первое, что открывает ревьюер. Плохой README: «автор не умеет общаться, не умеет документировать, не ценит своё время». Хороший: «с этим человеком приятно работать» ещё до первого созвона.

### Pinned repos на GitHub

Зайди в свой профиль → кнопка Customize your pins → выбери три лучших (P1, P2, P3). Остальные репозитории никто не будет листать. В профиле GitHub сделай жирный README (`github.com/username/username`): кто ты, фото или эмодзи, стек, ссылки на projects, статистика активности. Это твоё лицо разработчика, работает 24/7, пока ты спишь.

### LinkedIn

Заголовок «Frontend Engineer» (без «aspiring», «junior wannabe» и прочей неуверенности). В About — ключевые технологии: React, Next.js, TypeScript. Секция Projects со ссылками на портфолио и P3. Включи «Open to work» badge либо напиши в About «Open to junior/middle frontend roles». Рекрутеры ищут именно по этим фильтрам, без них ты невидимка.

### Резюме

Одна страница, **PDF**. Структура: контакты → краткая сводка (2–3 строки) → навыки (ключевые технологии) → проекты (P1/P2/P3 с live URL и stack) → образование. Никаких «водяных» фраз вроде «ответственный, коммуникабельный, стрессоустойчивый». Цифры там, где можно: «Lighthouse 95», «500 юзеров/мес», «деплой за 40 секунд». Шаблон [rezi.ai](https://rezi.ai) или просто чистый Google Docs.

### Подготовка к собесу

Не LeetCode в первую очередь. Знай свой стек: что такое `useMemo` и когда он нужен, как работает CSR vs SSR, чем Server Component отличается от Client. Открой свой P3 и в голове пройди: «вот тут я сделал X, потому что Y». Junior-собес это 70% «расскажи про проекты». Для senior добавляй system design и углубление в производительность.

### Что учить дальше

По приоритету. **1. Backend**: Node.js глубже, Postgres, REST и GraphQL — фуллстек-джуниоров берут быстрее. **2. System design**: как устроены большие фронты (SSR, кеш, CDN), базовые архитектурные решения. **3. DevOps basics**: Docker, основы Linux, CI pipelines подробнее. **4. Open-source**: первый PR в React или Next — огромный буст для резюме. **5. Английский** обязателен для оффера в международной компании.

## Частые ошибки

- **Одно предложение в README**: «My React project». Никакого live URL, никакого стека, никакого скрина. Ревьюер закрыл вкладку через 3 секунды, второго шанса не будет.
- **15 pinned repos** с туториалами типа `todo-app`, `react-hooks-practice`, `another-todo`. Выбери 3 лучших и спрячь остальное. Меньше = сильнее: лучше три вылизанных проекта, чем пятнадцать сырых.
- **Ждать идеального проекта** перед выходом в поиск. Твой P3 достаточен для junior-оффера уже сейчас. Подавайся параллельно с доработкой: первые отказы дадут больше фидбэка, чем ещё месяц полировки в одиночку.
- **Гриндить LeetCode 3 месяца** до интервью. Junior-собесы идут по проектам и базе стека. Грамотный разговор про свой P3 важнее 50 решённых Easy-задач: рекрутер слушает, как ты думаешь, а не сколько ты задротил.

## Задания

1. Сделай портфолио-сайт (`Next.js + Tailwind`, одна страница). Задеплой на Vercel. Заполни всеми тремя P-проектами: скрин, описание, стек, ссылки live плюс repo.
2. Перепиши README в P1, P2, P3 по шаблону из примера: скрин, live URL, стек, «как запустить локально», env-переменные.
3. Обнови LinkedIn: заголовок «Frontend Engineer», About с ключевыми технологиями, Projects-секция со ссылками на P3 и портфолио-сайт. Включи «Open to work».
4. Собери резюме на одну страницу (PDF). Проекты с live URL, без воды. Пошли мне на ревью или покажи 2–3 знакомым разработчикам.
