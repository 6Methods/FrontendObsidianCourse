# 14.2 — next/image и next/font

## Зачем это нужно

Картинки и шрифты — главные виновники медленного LCP и плохого CLS. Картинка без размера прыгает при загрузке, гигантский JPEG грузит мобильник 5 секунд, кастомный шрифт с CDN делает FOIT (невидимый текст). Next.js решает это двумя встроенными компонентами: `next/image` оптимизирует изображение (формат, размер, lazy), `next/font` самостоятельно хостит шрифты и убирает сетевые запросы. Это не «приятно иметь» — это разница между Lighthouse 60 и 95.

## Главное

### next/image — что делает

`next/image` — это обёртка над `<img>`, которая делает за тебя четыре вещи: автоматически выбирает формат (AVIF, потом WebP, потом исходник), генерит responsive `srcset` под разные размеры экрана, включает lazy-loading по умолчанию (кроме картинок с `priority`) и рисует **блюр-плейсхолдер**, пока грузится оригинал. На Vercel это всё идёт через Image Optimization API — бесплатно до 5000 трансформаций в месяц на Hobby-плане. В селф-хосте — через встроенный оптимизатор на Node.js runtime.

### Props, которые надо знать

`src` принимает либо локальный `import`, либо URL-строку. `alt` обязателен, иначе линт ругается. `width` и `height` обязательны — кроме случая `fill`, когда картинка растягивается на родитель с `position: relative`. `priority` снимает lazy и добавляет preload — ставь **только** на LCP-картинку. `sizes` — это hint для браузера, какой размер картинка займёт на экране в разных breakpoint. `quality` по умолчанию 75, можно снизить до 50 для не-LCP. `placeholder="blur"` вместе с `blurDataURL` даёт размытый тизер вместо пустого места.

### Local vs remote images

С локальным импортом — `import hero from '@/public/hero.png'` — Next сам прочитает `width`, `height` и сгенерит `blurDataURL` на билде. Ничего настраивать не надо. Remote-картинка (`src="https://..."`) требует явного разрешения через `remotePatterns` в `next.config.ts`: перечисляешь `protocol` и `hostname` для каждого допустимого источника. Без этого — 400 Bad Request на проде. Это защита от того, чтобы через твой оптимизатор не гоняли чужой трафик.

### next/font — zero runtime

`import { Inter } from 'next/font/google'` — на билде Next скачивает файлы шрифта с Google, кладёт их в `.next/static`, и раздаёт с твоего же домена. Никаких запросов к `fonts.googleapis.com` в рантайме — ни у тебя, ни у пользователя. Это и быстрее, и приватнее (GDPR радуется). Для своего `.woff2` есть `next/font/local` — подключается похоже.

### Как подключить

В `app/layout.tsx` создаёшь объект шрифта один раз на уровне модуля: `const inter = Inter({ subsets: ['latin', 'cyrillic'] })`. Потом либо `<body className={inter.className}>` — и шрифт применится глобально, либо `inter.variable` как CSS-переменная (`--font-inter`) — удобно для Tailwind через `fontFamily.sans`.

### display: 'swap' vs 'optional'

`display: 'swap'` — браузер сразу показывает fallback, потом меняет его на кастомный шрифт, когда тот придёт. Безопасно для читаемости, но даёт CLS на свопе. `display: 'optional'` — если шрифт не успел за ~100ms, fallback остаётся навсегда до следующего визита. Жёстче для дизайна, но **лучше для Core Web Vitals**.

### CLS и reserve space

`<Image>` с явными `width`/`height` держит место через `aspect-ratio` — страница не прыгает. Для шрифта Next сам подбирает `size-adjust` и `ascent-override` через `adjustFontFallback` — fallback-шрифт занимает ровно столько же места, сколько кастомный.

### priority на LCP

Одна картинка на страницу — та, что видна сразу без скролла (обычно hero) — получает `priority`. Next добавит `fetchpriority="high"` и `<link rel="preload">`. Все остальные картинки — без `priority`, пусть грузятся лениво.

## Пример кода

```tsx
// app/layout.tsx
import { Inter } from 'next/font/google';
import type { ReactNode } from 'react';
import './globals.css';

const inter = Inter({
  subsets: ['latin', 'cyrillic'],
  display: 'swap',
  variable: '--font-inter',
});

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="ru" className={inter.variable}>
      <body className={inter.className}>{children}</body>
    </html>
  );
}
```

```tsx
// app/page.tsx
import Image from 'next/image';
import hero from '@/public/hero.png';

const products = [
  { id: 1, title: 'Кружка', cover: 'https://storage.example/mug.jpg' },
  { id: 2, title: 'Футболка', cover: 'https://storage.example/tee.jpg' },
];

export default function HomePage() {
  return (
    <main>
      <Image
        src={hero}
        alt="Главный баннер магазина"
        priority
        placeholder="blur"
        sizes="100vw"
      />

      <ul>
        {products.map((p) => (
          <li key={p.id}>
            <Image
              src={p.cover}
              alt={p.title}
              width={400}
              height={300}
              sizes="(max-width: 768px) 100vw, 400px"
            />
            <span>{p.title}</span>
          </li>
        ))}
      </ul>
    </main>
  );
}
```

```ts
// next.config.ts
import type { NextConfig } from 'next';

const config: NextConfig = {
  images: {
    remotePatterns: [
      { protocol: 'https', hostname: 'storage.example' },
    ],
  },
};

export default config;
```

## Частые ошибки

- **`<img>` вместо `<Image>`** — теряешь формат-негошиэйшен, srcset, lazy и blur. Всё сразу. Lint-правило `@next/next/no-img-element` это ловит — не выключай его.
- **Remote-картинка без `remotePatterns`** — в деве может работать, на проде получишь **400** на `/_next/image`. Добавь домен в `next.config.ts` и перезапусти.
- **`priority` на всех картинках** — если всё приоритетное, то ничего не приоритетное. Должна быть **ровно одна** LCP-картинка на страницу, остальное — lazy.
- **Шрифт через `@font-face` с чужого CDN** — лишний DNS, лишний TCP, FOIT, и CLS на свопе. Вместо этого — только `next/font/google` или `next/font/local`.

## Задания

1. В любом проекте (P2 или M13) замени все `<img>` на `<Image>`. Добавь `width`/`height` либо оберни в контейнер с `fill`. Проверь, что Lighthouse перестал ругаться на `image-size-responsive`.
2. Подключи `next/font/google` для `Inter` с сабсетами `['latin', 'cyrillic']`. Сделай CSS-переменную через `inter.variable` и подцепи её в `tailwind.config.ts` через `fontFamily.sans`.
3. Добавь `priority` на LCP-картинку главной (обычно hero). Открой Lighthouse в incognito — метрика **LCP** должна упасть, **Performance** подняться хотя бы на 5–10 пунктов.
4. Добавь в `next.config.ts` `remotePatterns` для аватарок с Supabase Storage (`*.supabase.co`). Проверь негативный кейс: картинка с `unsplash.com` должна давать **400** через `/_next/image?url=...` — это правильно, так и задумано.
