# 14.7 — Accessibility 2.0 и i18n

## Зачем это нужно

В 2026 финтех, госсектор и крупные B2C требуют соответствия **WCAG 2.2 AA** — это уже не гигиена, а юридическая обязанность (EU Accessibility Act действует с июня 2025). Если инвалид по зрению с NVDA не может заполнить твою форму — клиента нет, а компанию ждёт штраф. Международный продукт без i18n теряет до 80% рынка. Эти навыки редко встречаются у джунов, поэтому кандидат, который умеет делать по-настоящему доступные формы и базовую локализацию, на голову выше средней волны резюме.

## Главное

### Semantic first, ARIA потом

Правило №1: семантический HTML побеждает любой ARIA. `<button>`, `<nav>`, `<label for>` уже умеют всё, что тебе нужно: фокус, клавиатуру, озвучку screen reader-ом. Конструкция `<div role="button" tabIndex={0}>` — провал: не срабатывает `Enter`, не отправляет форму, легко забыть `onKeyDown`. Используй ARIA, только когда нативного элемента не существует (сложный `role="tablist"`, `role="dialog"`).

### ARIA, которую надо знать

Минимальный набор: `aria-label` (название, когда нет видимого текста), `aria-labelledby` (ссылка на `id` заголовка), `aria-describedby` (доп.описание, например ошибка поля), `aria-invalid` (поле с ошибкой), `aria-live` (динамический контент), `aria-expanded` (dropdown, accordion), `aria-hidden` (скрыть от reader-а). Правило: **не пиши ARIA, если не понимаешь, что оно делает** — пустой `<button>` без атрибутов чаще лучше, чем «украшенный» неправильно.

### Focus management

После закрытия модалки верни фокус на кнопку, которая её открыла. После удаления строки таблицы — на соседнюю. В SPA после навигации — на `<h1>` новой страницы. Без этого пользователь screen reader-а просто «теряется» в пустоте. В React это `useRef` + `.focus()` в `useEffect`, плюс focus trap внутри модалки.

### Live regions

Контент, который меняется после взаимодействия (toast, результаты поиска, сообщения валидации), объявляй через `aria-live="polite"` — мягко, или `"assertive"` — срочно, для ошибок. Screen reader сам озвучит изменение, не перебивая пользователя.

### Keyboard navigation

Пройди весь флоу только клавиатурой: `Tab`, `Shift+Tab`, `Enter`, `Space`, `Esc`, стрелки. Кастомные компоненты (меню, табы, комбобоксы) требуют своих клавиш по [ARIA Authoring Practices Guide](https://www.w3.org/WAI/ARIA/apg/). **Никогда не убирай `:focus-visible` стили** — это единственный визуальный сигнал keyboard-пользователю, где он сейчас находится.

### Проверка: Axe, Lighthouse, screen reader

**axe DevTools** (Chrome-экстеншн, бесплатно) находит около 40% проблем автоматически. **Lighthouse** accessibility score ≥ 95 — это минимум, не цель. Оставшиеся 60% ловятся только руками: пройди всю страницу `Tab`-ом и послушай через VoiceOver (macOS, `Cmd+F5`) или NVDA (Windows, бесплатно). Десять минут с включенным screen reader-ом меняют мышление навсегда.

### i18n в Next.js

Для чисел и дат используй **Intl API** браузера: `new Intl.NumberFormat('ru-RU').format(1234.5)`, `new Intl.DateTimeFormat(...)`, `Intl.RelativeTimeFormat`. Для строк — библиотека **next-intl**: JSON-файлы переводов по локалям, `useTranslations('Namespace')` в компонентах, routing `/en/about` и `/ru/about`. Она из коробки работает с App Router и поддерживает серверные компоненты.

## Пример кода

Доступная модалка на React — фокус на первом элементе при открытии, trap внутри, `Esc` закрывает, возврат фокуса при закрытии:

```tsx
'use client';
import { useEffect, useRef } from 'react';

export function Modal({ open, onClose, children }: {
  open: boolean;
  onClose: () => void;
  children: React.ReactNode;
}) {
  const dialogRef = useRef<HTMLDivElement>(null);
  const openerRef = useRef<HTMLElement | null>(null);

  useEffect(() => {
    if (!open) return;
    openerRef.current = document.activeElement as HTMLElement;
    const first = dialogRef.current?.querySelector<HTMLElement>(
      'button, [href], input, [tabindex]:not([tabindex="-1"])'
    );
    first?.focus();

    const onKey = (e: KeyboardEvent) => {
      if (e.key === 'Escape') onClose();
    };
    document.addEventListener('keydown', onKey);
    return () => {
      document.removeEventListener('keydown', onKey);
      openerRef.current?.focus(); // возврат фокуса
    };
  }, [open, onClose]);

  if (!open) return null;
  return (
    <div
      ref={dialogRef}
      role="dialog"
      aria-modal="true"
      aria-labelledby="modal-title"
    >
      <h2 id="modal-title">Подтвердите действие</h2>
      {children}
    </div>
  );
}
```

Поле формы с ошибкой — связка `label` + `aria-describedby` + `aria-invalid`:

```tsx
<label htmlFor="email">Email</label>
<input
  id="email"
  type="email"
  aria-describedby="email-error"
  aria-invalid={Boolean(error)}
/>
{error && <p id="email-error" role="alert">{error}</p>}
```

Живой счётчик корзины через live region:

```tsx
<div aria-live="polite">В корзине: {count}</div>
```

i18n на **next-intl** — файл `messages/ru.json` и использование:

```tsx
// messages/ru.json → { "Home": { "greeting": "Привет, {name}!" } }
import { useTranslations } from 'next-intl';

export default function Home() {
  const t = useTranslations('Home');
  return <h1>{t('greeting', { name: 'Иван' })}</h1>;
}
```

## Частые ошибки

- **`<div onClick={...}>` вместо `<button>`.** Не срабатывает `Enter`/`Space`, элемент не попадает в tab-order, screen reader его игнорирует. Всегда бери `<button type="button">`.
- **`aria-hidden="true"` на интерактивной кнопке**, «чтобы спрятать от reader-а». Результат: кнопка видна глазу, но не озвучивается — юзер кликает в пустоту.
- **Убрать `outline: none` без замены на `:focus-visible`.** Keyboard-пользователь полностью теряется на странице. Если не нравится дефолт — перерисуй, но **не удаляй**.
- **Писать `aria-label` на нативном `<button>Click me</button>`** — видимый текст уже озвучивается, `aria-label` его **перезаписывает**. Либо убирай `aria-label`, либо используй его только когда кнопка без текста (иконка).

## Задания

1. Открой свой **P3**. Пройди всю страницу только `Tab`/`Shift+Tab`/`Enter`/`Esc`. Почини все места, где фокус теряется или не виден.
2. Установи **axe DevTools**, прогони главную. Исправь все нарушения с уровнем `serious` и `critical`.
3. Включи **VoiceOver** (macOS, `Cmd+F5`) или **NVDA** (Windows) на 10 минут. Прослушай свою форму логина. Заметь, где звучит странно или непонятно — почини.
4. Подключи **`next-intl`** к проекту **M13**. Две локали (`ru`, `en`), переключатель в хедере. Переведи главную страницу и форму создания заметки.
