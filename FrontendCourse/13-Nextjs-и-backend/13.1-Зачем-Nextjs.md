# Зачем Next.js

## Зачем это нужно

Ты умеешь писать React-SPA, но это только браузерная часть истории. Настоящие продукты требуют SEO, серверного рендера, работы с базой, авторизации и быстрого деплоя. Next.js собирает всё это в один фреймворк и даёт zero-config деплой на Vercel. В 2026 это индустриальный стандарт для fullstack на React.

## Главное

### Что даёт Next.js поверх React

React — только UI-библиотека: компоненты и реактивность. Next.js — полноценный фреймворк: роутинг, серверный рендер, API-роуты, Server Components, оптимизированные `Image` и `Font`, build-пайплайн. Одна команда `create-next-app`, и у тебя production-ready проект без ручной сборки конфигов.

### SSR, SSG, ISR, CSR — одним абзацем

CSR — JS рендерит всё в браузере (классический React). SSR — HTML генерится на каждый запрос (dynamic). SSG — HTML генерится при билде (static). ISR — это SSG плюс фоновая регенерация по расписанию. App Router в Next.js делает выбор автоматически, исходя из того, что ты дёргаешь в компоненте (`cookies`, `fetch` с `cache: 'no-store'` и так далее).

### SEO и метаданные

При CSR Google-бот видит пустую страницу с `<div id="root">`. При SSR/SSG он получает полный HTML с текстом и мета-тегами. Для контентных сайтов, маркетинга и e-commerce серверный рендер обязателен.

### File-based routing

Папка `app/blog/page.tsx` автоматически становится роутом `/blog`. Никаких ручных `<Route>` и конфигов: структура директорий и есть схема URL.

### App Router vs Pages Router

App Router — новый (с Next 13+), дефолт с 2024. Pages Router — legacy, всё ещё поддерживается для старых проектов. В этом модуле работаем только с App Router.

### Server Components и Actions

Server Components выполняются на сервере и отдают готовый HTML, в JS-бандл они не попадают. Server Actions — серверные функции, вызываемые из форм без отдельного API-роута. React 19 + Next 15 сделали оба механизма GA.

### Performance из коробки

Streaming, partial prerendering, automatic code splitting по роутам, image optimization, font optimization. Принцип «быстро по умолчанию»: не надо вручную настраивать lazy-loading и сжатие картинок.

### Vercel — нативный хостинг

Один `git push` → прод-URL. Preview deployments на каждый PR. Zero-config env-переменные. Fluid Compute для серверной части. Хостить можно и на других провайдерах, но на Vercel проще всего: платформу делают те же, кто пишет Next.js.

### Когда Next.js НЕ нужен

Чистый SPA без SEO (админка, дашборды за логином), десктоп на Electron, блог на Astro/Hugo. Для лендинга в 5 страниц Next.js тяжеловат. Инструмент подбирай под задачу.

### Setup

`npx create-next-app@latest my-app` → интерактивные вопросы (TS, App Router, Tailwind, ESLint). 30 секунд, и у тебя боевой каркас.

## Пример кода

```bash
# Новый проект
npx create-next-app@latest my-app
# ответы: TypeScript — yes, ESLint — yes, Tailwind — на вкус,
# src/ dir — yes, App Router — yes, Turbopack — yes

cd my-app
npm run dev
# http://localhost:3000
```

```tsx
// app/page.tsx — Server Component по умолчанию
export default async function HomePage() {
  const posts = await fetch("https://api.example.com/posts").then(r => r.json());

  return (
    <main>
      <h1>Блог</h1>
      <ul>
        {posts.map((p: { id: number; title: string }) => (
          <li key={p.id}>{p.title}</li>
        ))}
      </ul>
    </main>
  );
}

// app/layout.tsx — корневой layout (обязателен)
export const metadata = {
  title: "Мой блог",
  description: "Примеры на Next.js",
};

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="ru">
      <body>{children}</body>
    </html>
  );
}
```

## Частые ошибки

- Путать React и Next.js: React — библиотека UI, Next.js — фреймворк поверх React. Для продукта нужен фреймворк.
- Использовать Pages Router в новых проектах: API иной, legacy-путь. Начинай с App Router.
- Ждать, что Next.js автомагически ускорит CSR-приложение. Без правильного использования Server Components преимущества не получишь.
- Тащить Next.js туда, где достаточно SPA. Инструмент под задачу: дашборд за логином без SEO — Vite SPA хватает.
