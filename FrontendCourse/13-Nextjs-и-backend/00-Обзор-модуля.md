# Модуль 13 — Next.js и backend

**roadmap.sh:** [Next.js roadmap](https://roadmap.sh/nextjs)

Ты уже умеешь писать React-SPA и типизировать его. Но настоящие продуктовые приложения требуют серверной части: SEO-дружественный рендер, авторизация, база данных, мутации без лишних API. Next.js (App Router) в 2026 — индустриальный стандарт для fullstack-React: Server Components из коробки, Server Actions вместо Express-роутов, Vercel как нативный хостинг. Этот модуль делает из тебя fullstack-разработчика.

## Что изучаем
- **Зачем Next.js**: SSR/SSG/ISR, SEO, file-based routing, zero-config деплой.
- **App Router**: `page`, `layout`, `loading`, `error`, dynamic segments, metadata.
- **Server vs Client Components**: когда какой, где ставить `"use client"`, сериализация props.
- **Data fetching**: async Server Components, `fetch` с кешем, revalidation, streaming.
- **Route Handlers**: API-endpoints в `app/api/*/route.ts`.
- **Server Actions**: серверные функции из форм, `revalidatePath`, `useActionState`.
- **БД и Auth**: Supabase + Drizzle, auth.js (NextAuth v5), RLS, session на сервере.
