# Route Handlers

## Зачем это нужно

Route Handlers это HTTP API внутри App Router. Если тебе нужен внешний endpoint (мобильный клиент, webhook от Stripe, публичное API), пишешь `route.ts` в нужной папке и получаешь обычный HTTP-метод. Для внутренних мутаций фронта (формы, кнопки в своём же приложении) лучше Server Actions, см. 13.6.

## Главное

### Что это

Файл `app/api/.../route.ts` это серверный endpoint. Из него ты экспортируешь функции-методы с именами HTTP-глаголов: `GET`, `POST`, `PUT`, `PATCH`, `DELETE`, `OPTIONS`, `HEAD`. Каждая такая функция становится обработчиком соответствующего запроса по пути, который совпадает со структурой папок. Важно: в одной и той же папке нельзя держать `route.ts` и `page.tsx`, это конфликт.

### Web Fetch API

Сигнатура функции — стандартный Web API: `(request: Request) => Response | Promise<Response>`. Никаких Express-style `res.json()` или `res.send()`. Если ты работал с Web Workers, Service Workers или Deno, всё знакомо: принимаешь `Request`, возвращаешь `Response`.

### `NextRequest` и `NextResponse`

Next.js расширяет стандартные типы своими helper-ами. `NextResponse.json(data, { status })` короче, чем вручную городить `Response`. `NextRequest.nextUrl.searchParams` удобнее, чем парсить URL. Импортируешь из `next/server`.

### Параметры роута

Для `app/api/posts/[id]/route.ts` сигнатура такая: `export async function GET(req, ctx: { params: Promise<{ id: string }> })`. В Next 15+ `params` это Promise, обязательно распакуй через `await ctx.params`.

### Query-параметры

Берёшь либо `const { searchParams } = new URL(req.url)`, либо через `req.nextUrl.searchParams`. Второй вариант короче. Чтение: `searchParams.get("page")`. Возвращает строку или `null`, так что приводи тип руками и подставляй дефолт.

### Тело запроса

`await req.json()` для JSON, `await req.formData()` для form-data (файлы сюда), `await req.text()` для plain text или raw body при проверке подписи.

### Ответ

`NextResponse.json({ id: 1 }, { status: 201 })` — стандартный JSON-ответ. Пустой ответ: `new Response(null, { status: 204 })`. Редирект: `NextResponse.redirect(new URL("/", req.url))`.

### Кеш и revalidation

По умолчанию Route Handlers — dynamic, без кеша. Чтобы включить кеш, добавь `export const dynamic = "force-static"` и внутри используй `fetch` с кешем. В 90% случаев оставляй dynamic: API чаще отдаёт актуальные данные.

### Cookies и headers

`cookies()` и `headers()` из `next/headers` работают. Чтобы установить cookie в ответе: `NextResponse.json(...).cookies.set("token", value)`.

### Когда handler, а когда Server Action

Handler — для внешних клиентов, webhook-ов, file upload, публичных API. Server Action — для мутаций состояния из формы внутри того же приложения: проще, меньше boilerplate, не надо руками делать `fetch` с фронта.

### CORS

По умолчанию закрыт для cross-origin. Если делаешь публичный API, добавь заголовки `Access-Control-Allow-*` в ответ или в middleware. Не открывай `*` везде подряд.

## Пример кода

```ts
// app/api/posts/route.ts
import { NextRequest, NextResponse } from "next/server";
import { z } from "zod";

// GET /api/posts?limit=10
export async function GET(req: NextRequest) {
  const limit = Number(req.nextUrl.searchParams.get("limit") ?? "20");
  const posts = await db.query.posts.findMany({ limit });
  return NextResponse.json(posts);
}

// POST /api/posts
const CreatePostSchema = z.object({
  title: z.string().min(1),
  body: z.string().min(1),
});

export async function POST(req: NextRequest) {
  const raw = await req.json();
  const parsed = CreatePostSchema.safeParse(raw);
  if (!parsed.success) {
    return NextResponse.json(
      { error: parsed.error.issues },
      { status: 400 }
    );
  }
  const post = await db.insert("posts", parsed.data);
  return NextResponse.json(post, { status: 201 });
}

// app/api/posts/[id]/route.ts
export async function GET(
  req: NextRequest,
  ctx: { params: Promise<{ id: string }> }
) {
  const { id } = await ctx.params;
  const post = await db.query.posts.findFirst({ where: { id } });
  if (!post) {
    return NextResponse.json({ error: "Not found" }, { status: 404 });
  }
  return NextResponse.json(post);
}

export async function DELETE(
  req: NextRequest,
  ctx: { params: Promise<{ id: string }> }
) {
  const { id } = await ctx.params;
  await db.delete("posts", { id });
  return new Response(null, { status: 204 });
}

// Cookies в ответе
export async function POST(req: NextRequest) {
  const res = NextResponse.json({ ok: true });
  res.cookies.set("session", "abc", { httpOnly: true, secure: true });
  return res;
}

// Webhook c проверкой подписи
export async function POST(req: NextRequest) {
  const raw = await req.text(); // важно: raw body, НЕ json
  const signature = req.headers.get("stripe-signature");
  // verify signature here...
  return new Response("ok");
}
```

## Частые ошибки

- Писать Route Handler для внутренней формы: получается два слоя (client fetch → handler). Для такого нужен Server Action.
- Забыть `await params` в Next 15+: TS-ошибка или `undefined` в рантайме.
- Парсить body через `await req.json()`, а потом проверять подпись (webhook). Подпись проверяется на raw text.
- Открытый CORS `*` на всё API. Не раздавай доступ всем подряд, только нужные origin.
