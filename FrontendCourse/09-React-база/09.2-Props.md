# Props

## Зачем это нужно

Компонент без пропсов — статичная картинка. Пропсы работают как аргументы функции: передаёшь данные сверху вниз, компонент рендерит то, что ты дал. Без понимания пропсов невозможна переиспользуемость.

## Главное

### Что такое props

Props — первый аргумент функции-компонента. Обычный JS-объект, в котором лежат все данные, переданные тебе родителем. Выглядит примерно так: `function Button(props) { return <button>{props.label}</button>; }`, а вызывается — `<Button label="Ок" />`. Всё, что ты написал атрибутом в JSX, попадёт внутрь компонента как поле этого объекта.

### Деструктуризация

На практике почти никто не пишет `props.label` — сразу разбирают объект: `function Button({ label, onClick }) {...}`. Так короче, и по сигнатуре сразу видно, какие пропсы компонент реально ждёт. Это просто синтаксис, под капотом всё те же props.

### Типы значений

В проп можно положить что угодно: строки, числа, boolean, массивы, объекты, функции и даже JSX-элементы. Всё, кроме строк, передаётся через фигурные скобки: `<User age={25} isAdmin={true} items={[1,2,3]} />`. Строку можно писать и в кавычках, и в `{}`, но кавычки короче.

### Значения по умолчанию

Если проп может быть не передан, задавай дефолт прямо в деструктуризации: `function Button({ label = "OK", size = "md" }) {...}`. Родитель не передал — возьмётся значение по умолчанию, передал — подставится его. Это заменяет старый `Button.defaultProps`.

### children — особый prop

Всё, что ты написал между открывающим и закрывающим тегом, автоматически попадает в `props.children`: `<Card>содержимое</Card>`. Это позволяет делать «контейнеры» (Card, Modal, Section), которые оборачивают произвольный контент. Очень частый приём в React.

### Props read-only

Props менять внутри компонента НЕЛЬЗЯ. Строчка вроде `props.name = "Иван"` — антипаттерн и сломает реактивность. Если тебе нужно динамическое значение, которое меняется со временем, это state, и живёт он либо здесь, либо у родителя (поднимешь в 09.7).

### Передача обработчиков

Функции — тоже обычные пропсы. Пишешь `<Button onClick={handleClick} />`, и Button внутри дёргает эту функцию на клике. Стандартный паттерн: «Button не знает, что произойдёт, решает родитель». Так компоненты общаются снизу вверх.

### Спред пропсов

Если у тебя есть объект со свойствами, его можно раскидать как props одним жестом: `<input {...inputProps} />`. Удобно для обёрток над нативными input/button, но теряется прозрачность: по коду непонятно, что именно прилетело внутрь. Используй осознанно.

### Именование

Пропсы всегда `camelCase`: `onClick`, `isLoading`, `maxWidth`. Булевы пропсы можно писать без значения: `<Button disabled />` — это ровно то же самое, что `<Button disabled={true} />`. Короче и читабельнее.

### Композиция через props

React не любит наследование. Он любит композицию. Вместо «Button extends BaseButton» ты просто передаёшь элементы через props: `<Layout header={<Header />} sidebar={<Nav />}>...</Layout>`. Компоненты складываются как кубики, это и есть React-подход.

## Пример кода

```jsx
function Card({ title, children, footer }) {
  return (
    <div className="card">
      <h3>{title}</h3>
      <div>{children}</div>
      {footer && <div className="footer">{footer}</div>}
    </div>
  );
}

function Button({ label = "OK", onClick, disabled = false }) {
  return (
    <button onClick={onClick} disabled={disabled}>
      {label}
    </button>
  );
}

function App() {
  const user = { name: "Иван", age: 25 };

  return (
    <>
      <Card title="Профиль" footer={<small>обновлено сегодня</small>}>
        <p>Имя: {user.name}</p>
        <p>Возраст: {user.age}</p>
        <Button
          label="Редактировать"
          onClick={() => console.log("edit")}
        />
      </Card>

      <Card title="Другая карточка">
        <p>Без футера.</p>
        <Button label="OK" disabled />
      </Card>
    </>
  );
}
```

## Частые ошибки

- Писать в props: `props.count++`. Не работает и плохая идея. Props read-only.
- Забывать `{}` для не-строк: `<Button disabled="true" />` — строка `"true"` вместо булева. Правильно: `<Button disabled={true} />` или `<Button disabled />`.
- Пропуск `onClick={handleClick}` vs `onClick={handleClick()}` — второе ВЫЗОВЕТ функцию при рендере, не при клике. Нужно передавать ссылку.
- Сотни props на один компонент — знак, что компонент перегружен. Разбей на части или используй children.
