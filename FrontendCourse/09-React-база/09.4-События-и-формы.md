# События и формы

## Зачем это нужно

Пользователь взаимодействует с интерфейсом через клики, ввод текста, submit формы. Без этого твоё приложение просто картинка. В React всё это делается через пропсы-обработчики: `onClick`, `onChange`, `onSubmit`. Отдельная большая тема — controlled inputs: React полностью владеет содержимым `<input>` через state, и это основа абсолютно всех форм.

## Главное

### Синтаксис обработчика

На элемент вешаешь обработчик через проп: `<button onClick={handleClick}>`. Имя пропа всегда в `camelCase`: `onClick`, `onChange`, `onSubmit`, `onMouseEnter`. Не `onclick` как в HTML. Значение — это **функция**, а НЕ её вызов: пиши `onClick={handle}`, а не `onClick={handle()}`. Иначе функция отработает сразу при рендере.

### Inline-функции

Часто удобно написать обработчик прямо в пропсе: `onClick={() => setCount(c => c + 1)}` — коротко, видно что происходит, не надо прыгать по файлу. Для сложной логики (пара условий, несколько setState) лучше вынести в отдельную функцию. Типовой паттерн с параметром: `onClick={() => remove(id)}` — когда нужно передать id из `.map`.

### Событие (event object)

В обработчик React передаёт `SyntheticEvent` — свою обёртку над нативным DOM-событием (чтобы работало одинаково во всех браузерах). Самое полезное: `e.target.value` — что сейчас в input, `e.currentTarget` — элемент с которого висит обработчик, `e.preventDefault()` — отменить стандартное поведение браузера (сабмит формы с перезагрузкой страницы, переход по ссылке).

### Controlled input

Базовый паттерн: `<input value={text} onChange={e => setText(e.target.value)} />`. Содержимое input полностью управляется state. При каждом нажатии клавиши срабатывает `onChange`, обновляет state, React ре-рендерит input с новым `value`. Плюсы: ты в любой момент знаешь текущее значение, можешь валидировать на лету, очищать через `setText("")`, подставлять значение из API.

### Uncontrolled — кратко

Альтернатива controlled — uncontrolled input: `useRef` + `defaultValue`, и читаешь значение только когда нужно (например, в onSubmit через `ref.current.value`). Уместно для файловых полей (`<input type="file">` — там только uncontrolled), или когда форма большая и не хочется ре-рендерить на каждый keystroke. По умолчанию выбирай controlled, так проще и привычнее в React-экосистеме.

### Form + onSubmit

Оборачиваешь поля в `<form onSubmit={handleSubmit}>` — теперь форма сабмитится и по клику на `<button type="submit">`, и по Enter в input. Внутри `handleSubmit` **первой строкой** пишешь `e.preventDefault()`. Иначе браузер по умолчанию перезагрузит страницу, и весь твой state улетит в ноль. После этого валидация, отправка на сервер, очистка полей.

### Разные типы input

Для текстовых полей (`text`, `email`, `number`, `password`, `url`, `tel`) — всё через `value`/`onChange`. Для `checkbox` и `radio` — через `checked` (не `value`!): `<input type="checkbox" checked={agreed} onChange={e => setAgreed(e.target.checked)} />`. Для `<select>` — `value` ставится на сам `<select>`, а опции генерируются через `.map`. Для `<textarea>` в React `value` пропс (в отличие от HTML, где контент пишется между тегами).

### Важность `name`

Атрибут `name` нужен для логической группировки (особенно критично для `radio` — все кнопки одной группы должны иметь одинаковый `name`), а также если ты собираешься использовать браузерный `FormData(e.target)` для сбора данных. Правило: каждый input в форме с уникальным `name` (кроме radio внутри одной группы).

### Один обработчик для многих полей

Не плоди 10 отдельных функций для 10 полей. Паттерн: `onChange={e => setForm({...form, [e.target.name]: e.target.value})}` — через компьютед-ключ по `name` обновляешь нужное поле. Одна функция, универсально, расширяется добавлением новых полей без новой логики.

### Клавиатура

События `onKeyDown` и `onKeyUp` дают доступ к `e.key` — название нажатой клавиши: `"Enter"`, `"Escape"`, `"ArrowUp"`. Типичный кейс: поле поиска с ручным сабмитом по Enter — `onKeyDown={e => { if (e.key === "Enter") search() }}`. Ещё `Escape` для закрытия модалок.

### Фокус и blur

`onFocus` срабатывает когда поле получает фокус, `onBlur` — когда теряет. Часто используется для валидации: показываем ошибку «невалидный email» только после `onBlur`, а не при каждом keystroke, чтобы не пугать пользователя красным текстом пока он ещё печатает.

## Пример кода

```jsx
import { useState } from "react";

function SearchBar() {
  const [q, setQ] = useState("");

  function handleSubmit(e) {
    e.preventDefault();
    console.log("искать:", q);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input
        type="text"
        value={q}
        onChange={e => setQ(e.target.value)}
        placeholder="Поиск..."
      />
      <button type="submit">Найти</button>
    </form>
  );
}

function SignupForm() {
  const [form, setForm] = useState({
    name: "",
    email: "",
    subscribe: false,
    plan: "free",
  });

  function handleChange(e) {
    const { name, value, type, checked } = e.target;
    setForm(prev => ({
      ...prev,
      [name]: type === "checkbox" ? checked : value,
    }));
  }

  function handleSubmit(e) {
    e.preventDefault();
    if (!form.email.includes("@")) return;
    console.log("регистрация:", form);
  }

  return (
    <form onSubmit={handleSubmit}>
      <input name="name" value={form.name} onChange={handleChange} />
      <input name="email" value={form.email} onChange={handleChange} />
      <label>
        <input
          name="subscribe"
          type="checkbox"
          checked={form.subscribe}
          onChange={handleChange}
        />
        Подписка
      </label>
      <select name="plan" value={form.plan} onChange={handleChange}>
        <option value="free">Free</option>
        <option value="pro">Pro</option>
      </select>
      <button type="submit">OK</button>
    </form>
  );
}
```

## Частые ошибки

- Забыть `e.preventDefault()` в `onSubmit` — страница перезагружается, state теряется, ты недоумеваешь почему «ничего не работает».
- `onClick={handleClick()}` — функция выполняется при рендере, а не при клике. Правильно: `onClick={handleClick}` или `onClick={() => handleClick(arg)}`.
- Input без `onChange`, только с `value` — поле становится read-only (React выкинет warning в консоли). Нужен либо `onChange`, либо явный `readOnly`.
- Controlled + `defaultValue` одновременно — React ругается. Выбирай один подход: либо `value` + `onChange`, либо `defaultValue` + `ref`.
