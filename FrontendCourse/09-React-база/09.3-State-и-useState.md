# State и useState

## Зачем это нужно

Props приходят извне и статичны для компонента. State — переменная, которой компонент владеет и может менять. Без state UI не интерактивен. `useState` — первый и главный хук React, с него начинается реальная работа.

## Главное

### Синтаксис

Базовая строка выглядит так: `const [count, setCount] = useState(0)`. Первое значение — текущее состояние, второе — функция обновления, `0` — начальное значение. Деструктуризация массива — чисто соглашение, имена выбираешь ты сам: хоть `[value, setValue]`, хоть `[open, setOpen]`.

### Что делает `useState`

Хук сохраняет значение между рендерами. Обычная `let count = 0` внутри компонента сбрасывалась бы при каждом вызове функции-компонента; `useState` же запоминает значение и отдаёт его при следующем рендере. Это «память» компонента, которую React держит за тебя.

### Обновление и ре-рендер

Вызов `setCount(1)` помечает компонент как «нужен новый рендер» → React вызывает функцию-компонент снова → `count` внутри уже равен `1`. Это **не** меняет существующую переменную. Создаётся новый рендер с новыми значениями. В старом рендере `count` навсегда остаётся прежним.

### Асинхронность setState

Если написать `setCount(count + 1); console.log(count);` — выведет СТАРОЕ значение. Обновление применится к следующему рендеру, не сразу. Текущий рендер уже зафиксировал свои значения, изменить их внутри него нельзя.

### Функциональная форма

Если новое состояние зависит от старого, пиши `setCount(prev => prev + 1)`. Это гарантирует, что ты возьмёшь актуальное значение, даже если несколько `setCount` вызваны подряд. React передаст в колбэк самое свежее состояние.

### Иммутабельность

React сравнивает ссылки, чтобы понять, нужно ли перерендерить. Для объектов и массивов создавай новые, не мутируй старые. НЕЛЬЗЯ: `arr.push(x); setArr(arr)` — ссылка та же, React ничего не заметит. МОЖНО: `setArr([...arr, x])` — новый массив, новая ссылка.

### Правила обновления объекта

Копия плюс перезапись поля: `setUser({ ...user, name: "Иван" })`. Удаление из массива: `setItems(items.filter(i => i.id !== id))`. Правка одного элемента в массиве: `setItems(items.map(i => i.id === id ? {...i, done: true} : i))`. Паттерн один и тот же: spread, map, filter, и никакого `.push`/`.splice`.

### Несколько useState vs один объект

Лучше несколько отдельных `useState` для независимых значений: счётчик и модалка это разные вещи, не слепляй их. Один объект уместен, когда поля связаны логически: форма с 5 полями, координаты мыши `{x, y}`. Независимые куски стейта — отдельными хуками.

### Правила хуков

Хуки вызываются **только на верхнем уровне функции** и **только в React-компонентах** (или внутри других хуков). Никаких `if`/`for`/`while` вокруг `useState`. Порядок вызовов должен быть стабильным между рендерами, React опознаёт хуки по порядку, не по имени.

### Ленивая инициализация

Если начальное значение требует тяжёлых вычислений, передавай функцию: `useState(() => expensiveCompute())`. Функция вызывается ОДИН раз, при первом рендере, дальше игнорируется. Без неё `expensiveCompute()` отрабатывал бы при каждом рендере впустую.

## Пример кода

```jsx
import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Значение: {count}</p>
      <button onClick={() => setCount(count + 1)}>+1</button>
      <button onClick={() => setCount(prev => prev - 1)}>-1</button>
      <button onClick={() => setCount(0)}>Сбросить</button>
    </div>
  );
}

function UserForm() {
  const [user, setUser] = useState({ name: "", age: 0 });

  return (
    <>
      <input
        value={user.name}
        onChange={e => setUser({ ...user, name: e.target.value })}
      />
      <input
        type="number"
        value={user.age}
        onChange={e => setUser({ ...user, age: Number(e.target.value) })}
      />
      <pre>{JSON.stringify(user)}</pre>
    </>
  );
}

function Todos() {
  const [todos, setTodos] = useState([]);

  function addTodo(text) {
    setTodos(prev => [...prev, { id: Date.now(), text, done: false }]);
  }

  function toggle(id) {
    setTodos(prev =>
      prev.map(t => (t.id === id ? { ...t, done: !t.done } : t))
    );
  }

  function remove(id) {
    setTodos(prev => prev.filter(t => t.id !== id));
  }

  return null; // заглушка для краткости
}
```

## Частые ошибки

- Мутировать массив/объект: `items.push(x); setItems(items)`. React не увидит изменений, ссылка та же. Всегда создавай новую.
- Ожидать синхронности: `setCount(1); console.log(count)` — всё ещё старое значение. Новое будет в следующем рендере.
- Вызывать хук внутри условия: `if (cond) useState(0)`. Сломает порядок хуков, React упадёт с ошибкой.
- Использовать `setCount(count + 1)` несколько раз подряд — получишь `+1`, не `+3`. Для таких случаев функциональная форма `setCount(prev => prev + 1)`.
