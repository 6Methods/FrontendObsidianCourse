# JSX и компоненты

## Зачем это нужно

React построен на одной идее: UI — это функция от state. Компонент принимает данные, возвращает разметку. JSX — синтаксис, который позволяет писать HTML прямо внутри JS. Без понимания JSX и компонента как единицы переиспользования — никуда.

## Главное

### Функциональный компонент

Это обычная JS-функция, которая возвращает JSX. Имя пиши с большой буквы (`Button`, не `button`) — иначе React сочтёт за HTML-тег и всё сломается. Функция должна вернуть ОДИН корневой элемент, либо Fragment `<>...</>`, если элементов несколько.

### Что такое JSX

JSX — это синтаксический сахар над `React.createElement`. Под капотом `<h1>Привет</h1>` превращается в `React.createElement("h1", null, "Привет")`. Браузер такое не понимает — компилирует Vite/Babel ещё до того, как код попадёт в рантайм.

### HTML vs JSX — отличия

`class` становится `className`, `for` — `htmlFor`, все атрибуты пишутся в camelCase (`onClick`, `tabIndex`, `readOnly`). Само-закрывающиеся теги обязательны: `<img />`, `<br />`, `<input />`. Числовые значения передаются через фигурные скобки (`tabIndex={0}`), строковые можно просто в кавычках.

### Выражения в JSX через `{}`

Внутри фигурных скобок можно писать любое JS-**выражение**: `{user.name}`, `{count + 1}`, `{items.map(...)}`, вызов функции. А вот `if`, `for`, `switch` — это не выражения, они туда не лезут. Для условий используй тернарник или `&&`, для списков — `.map`.

### Множественные элементы и Fragment

Функция возвращает ОДИН элемент. Если нужно несколько — оберни в `<div>` или в Fragment `<>...</>`. Fragment хорош тем, что не создаёт лишнего DOM-узла и не ломает вёрстку (flex, grid).

### Импорты и экспорты

Один компонент — один файл, это базовое соглашение. Экспорт обычно через `export default function Button() {...}`, а импорт — `import Button from "./Button.jsx"`. Файл называй как компонент: `Button.jsx`, `UserCard.jsx`.

### Vite + React

Стандартный старт проекта: `npm create vite@latest my-app -- --template react`. Получаешь готовую структуру, hot reload, быстрый бандлер из коробки. В 99% случаев современного React-проекта стартуешь именно с Vite — CRA мёртв.

### Что запускается, когда React рендерит

Первый рендер: компонент вызывается, возвращает дерево элементов, React кладёт это дерево в DOM. При изменении state функция вызывается ЗАНОВО, React сравнивает старое дерево с новым (diff) и обновляет в DOM только то, что реально изменилось.

### Привычки

Один компонент — одна ответственность, не пихай всё в `App.jsx`. Файлы — `.jsx` (или `.tsx` на TypeScript). Именование: `PascalCase` для компонентов, `camelCase` для пропсов, переменных и функций.

## Пример кода

```jsx
// Button.jsx
function Button() {
  return <button className="btn">Нажми</button>;
}

export default Button;

// App.jsx
import Button from "./Button.jsx";

function App() {
  const user = { name: "Иван", age: 25 };
  const isAdmin = true;

  return (
    <>
      <h1 className="title">Привет, {user.name}!</h1>
      <p>Возраст: {user.age}</p>
      {isAdmin && <span>Ты админ</span>}
      <Button />
    </>
  );
}

export default App;

// main.jsx — точка входа (Vite генерирует автоматически)
import { createRoot } from "react-dom/client";
import App from "./App.jsx";

createRoot(document.getElementById("root")).render(<App />);
```

## Частые ошибки

- Писать компонент с маленькой буквы (`button`) — React решит, что это HTML-тег, и ничего не отрендерит.
- Возвращать несколько элементов без Fragment: `return <h1>...</h1><p>...</p>` — синтаксическая ошибка.
- Путать `class` и `className` — стили не применятся.
- Пихать `if` внутрь JSX: `<div>if (x) ...</div>` — не работает. Условия — через тернарник `{cond ? a : b}` или `&&`.
