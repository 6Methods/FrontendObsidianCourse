# Списки и условный рендеринг

## Зачем это нужно

В реальном UI почти всегда есть два паттерна: «отрендерь массив» (список постов, задач, товаров) и «покажи это, если…» (loader, пустое состояние, ошибка). Оба делаются JS-выражениями внутри JSX, без специальных директив, как во Vue или Angular. Если понимаешь `.map`, тернарник и `&&` — считай, 80% шаблонов UI ты уже умеешь собирать.

## Главное

### Списки через `.map`

Когда нужно отрендерить массив, берёшь его и вызываешь `.map`, который возвращает массив JSX-элементов: `items.map(item => <li>{item.name}</li>)`. React спокойно принимает массив детей и рендерит каждый. Никаких `for`, `v-for` или `*ngFor` — только обычный JS внутри фигурных скобок.

### Атрибут `key`

Каждому элементу списка нужен `key`, обязательно: `items.map(item => <li key={item.id}>...</li>)`. React использует его, чтобы понимать, какой элемент в новом массиве соответствует старому, и эффективно обновлять DOM при добавлении, удалении или перестановке.

### Как выбирать key

`key` должен быть уникальным и стабильным: `id` из базы, `Date.now()` при создании, `crypto.randomUUID()`. НЕ используй `index` массива, если список меняется (сортировка, фильтрация, удаление). Это приводит к багам: состояние внутренних компонентов (значения в input, фокус, анимации) привязывается к позиции, а не к данным, и начинает «прыгать».

### Пустой список

Если массив пустой, покажи заглушку, а не пустое место: `{items.length === 0 ? <Empty /> : items.map(...)}`. Это часть UX, не забывай про неё.

### Условный рендеринг: `&&`

Самый короткий способ: `{isLoading && <Spinner />}`. Если `isLoading` истинно, React отрендерит `<Spinner />`, иначе ничего. Удобно, когда нужна одна ветка «показать или не показать».

### Подводный камень `&&` с числом

`{items.length && <List />}` — ловушка. Если `items.length === 0`, то `0 && <List />` возвращает `0`, и React отрендерит в DOM цифру `0`! Правильно: `items.length > 0 && <List />` или тернарник. Всегда приводи к `boolean`, когда используешь `&&`.

### Тернарник `? :`

Когда нужны две ветки — тернарник: `{isLoggedIn ? <Profile /> : <LoginForm />}`. Читается как «если залогинен — профиль, иначе форма входа». Удобно, когда два разных варианта UI.

### Ранний возврат

Если ветвление большое, не пихай всё в JSX. Вынеси в `if` перед `return`: `if (!user) return <LoginForm />`. Это разгружает разметку и делает код понятнее, особенно для состояний «загрузка / ошибка / данные».

### Фильтр + map

Цепочки работают так же, как в обычном JS: `items.filter(i => !i.done).map(i => <Item ... />)`. Сначала отфильтровали, потом превратили в JSX. Ничего магического.

### Стили в условии

Классы через тернарник: `<div className={isActive ? "tab active" : "tab"}>`. Если классов много, бери `clsx`/`classnames`, это удобнее. Не лепи всё через inline `style={{...}}`. Стили должны жить в CSS.

### Вынесение в переменную

Если JSX в условии громоздкий, вынеси: `const content = isLoading ? <Spinner /> : <List .../>`, а потом `return <>{content}</>`. Читается намного приятнее, чем пятиэтажные тернарники прямо в `return`.

## Пример кода

```jsx
function TodoList({ todos, onToggle, onDelete }) {
  if (todos.length === 0) {
    return <p className="empty">Задач пока нет 🌱</p>;
  }

  return (
    <ul>
      {todos.map(todo => (
        <li key={todo.id} className={todo.done ? "done" : ""}>
          <input
            type="checkbox"
            checked={todo.done}
            onChange={() => onToggle(todo.id)}
          />
          <span>{todo.text}</span>
          <button onClick={() => onDelete(todo.id)}>✕</button>
        </li>
      ))}
    </ul>
  );
}

function App({ user, isLoading, error, todos }) {
  if (isLoading) return <Spinner />;
  if (error) return <ErrorBox message={error} />;
  if (!user) return <LoginForm />;

  const activeTodos = todos.filter(t => !t.done);
  const stats = `${activeTodos.length} из ${todos.length} в работе`;

  return (
    <div>
      <header>
        Привет, {user.name}!
        {activeTodos.length > 0 && <span> У тебя {activeTodos.length} дел.</span>}
      </header>
      <TodoList
        todos={todos}
        onToggle={/* ... */}
        onDelete={/* ... */}
      />
      <footer>{stats}</footer>
    </div>
  );
}
```

## Частые ошибки

- `key={index}` в динамическом списке. При сортировке/удалении React путает состояние внутренних компонентов (фокус, input-значения сбиваются).
- `{items.length && <List />}` вместо `items.length > 0 && <List />` → на пустом списке увидишь `0` в UI.
- Не обрабатывать пустое состояние — пользователь видит просто пустоту, не понимая, что всё в порядке.
- Логика в JSX через `map` + `if` + `else if` — разбей на функцию/переменную: JSX должен быть читаемым.
