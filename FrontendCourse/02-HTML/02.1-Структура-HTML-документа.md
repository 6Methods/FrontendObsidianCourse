# Структура HTML-документа

## Зачем это нужно

Любая HTML-страница начинается с одинакового скелета: DOCTYPE, head, body. Без него браузер не понимает, что это HTML5 и как правильно рендерить кириллицу и meta-теги.

## Главное

### `<!DOCTYPE html>`

Первая строка любой страницы. По сути это записка браузеру: «это HTML5, рендери по современным правилам». Если DOCTYPE забыть, браузер включает так называемый quirks mode, режим совместимости со старыми сайтами девяностых. В этом режиме отступы, размеры блоков и вообще вся вёрстка ведут себя непредсказуемо. Пишется один раз в самом верху, без закрывающего тега.

### Корневой `<html>` и атрибут `lang`

Контейнер всего документа: внутри него лежат head и body. Атрибут `lang="ru"` говорит браузеру и внешним сервисам, на каком языке написана страница. Это важно для поисковиков (лучше ранжируется), скринридеров (читают с правильным акцентом) и автоперевода в Chrome.

### `<head>` — невидимая часть

Сюда идёт всё, что не отображается на странице, но нужно браузеру: meta-теги, `<title>`, подключения CSS и JS, favicon. Представь это как паспорт документа: сам паспорт никто не видит, но без него ты никуда не пройдёшь.

### `<meta charset="UTF-8">`

Указывает кодировку. Без этого тега кириллица с высокой вероятностью превратится в `�����` или кракозябры. Ставим самым первым тегом внутри head, до `<title>` и всего остального.

### `<meta name="viewport" ...>`

Отвечает за нормальное отображение на мобильных. Без него сайт будет выглядеть как уменьшенная десктопная версия. Подробнее в CSS-модуле про адаптив, пока просто копируй как есть.

### `<title>`

Текст, который виден во вкладке браузера и в результатах Google. Первое, что пользователь видит о странице. Не ленись писать осмысленно.

### `<body>` — видимая часть

Всё, что пользователь реально видит на странице: заголовки, текст, картинки, кнопки. Весь контент идёт сюда.

### Подключение CSS и JS

CSS подключаем в head: `<link rel="stylesheet" href="style.css">`. JS подключаем через `<script src="script.js" defer></script>` либо в конце body, либо в head с атрибутом `defer`, чтобы скрипт не блокировал загрузку страницы. Глубже разберём в следующих модулях.

## Пример кода

```html
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Моя первая страница</title>
  </head>
  <body>
    <h1>Привет, мир!</h1>
  </body>
</html>
```

## Частые ошибки

- Забывать `<meta charset="UTF-8">` → кракозябры вместо кириллицы.
- Писать `<script>` в head без `defer` → браузер блокирует рендер, пока не загрузит JS.
- Оставлять дефолтный `<title>Document</title>` → плохо для SEO и выглядит неряшливо во вкладках.
