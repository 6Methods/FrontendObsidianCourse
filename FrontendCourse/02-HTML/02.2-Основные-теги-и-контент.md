# Основные теги и контент

## Зачем это нужно

80% любой HTML-страницы — это одни и те же ~15 тегов: заголовки, текст, списки, ссылки, картинки. Этих тегов достаточно, чтобы сверстать большинство статичных страниц.

## Главное

### Заголовки `<h1>` – `<h6>`

Заголовки задают иерархию, как оглавление в книге. На странице должен быть ровно один `<h1>`: главный заголовок, о чём вообще страница. `<h2>` — это разделы, `<h3>` — подразделы внутри них, и так далее до `<h6>`. Это важно не только визуально. Поисковики по заголовкам понимают структуру страницы, а скринридеры (программы для незрячих) позволяют пользователю прыгать по ним, как по оглавлению. Если налепить пять `<h1>`, и SEO просядет, и навигация сломается.

### Параграфы `<p>`, переносы `<br>`, горизонтальная линия `<hr>`

`<p>` — обычный абзац текста, основа основ. `<br>` принудительно переносит строку, но использовать его стоит только там, где перенос важен по смыслу: адрес, стихи, подпись. Для отступов между абзацами `<br>` не годится: для этого есть `<p>` и CSS. `<hr>` — горизонтальная линия, смысловой разделитель между блоками.

### Списки `<ul>`, `<ol>`, `<li>`

`<ul>` — маркированный список (точки), `<ol>` — нумерованный (1, 2, 3), а `<li>` — пункт внутри любого из них. Правило простое: всё, что по жизни выглядит списком (меню навигации, пункты товаров, шаги), в HTML делается списком, а не серией `<div>`. Так понятнее и людям, и машинам.

### Ссылки `<a href="">`

Атрибут `href` может быть абсолютным (`https://...`), относительным (`./about.html`), якорным (`#section-2` — прыжок внутри страницы) или на почту (`mailto:hi@example.com`). Чтобы ссылка открылась в новой вкладке, добавь `target="_blank"`, но обязательно с `rel="noopener noreferrer"` — иначе открытая страница получит доступ к твоей через `window.opener`, а это небезопасно.

### Изображения `<img src="" alt="">`

`src` — путь к картинке, `alt` — текстовое описание. Скринридер зачитает `alt`, поисковик по нему поймёт, что на картинке, а если файл не загрузится, пользователь увидит текст вместо битой иконки. Для чисто декоративных картинок ставь `alt=""` (пустая строка), скринридер их пропустит.

### Inline vs block элементы

Блочные (`<div>`, `<p>`, `<h1>`, `<ul>`) занимают всю ширину строки и встают друг под другом. Строчные (`<span>`, `<a>`, `<strong>`, `<em>`, `<img>`) идут в потоке текста, как слова в предложении. Подробнее разберём в CSS-модулях.

### Выделение текста — `<strong>` и `<em>`, не `<b>` и `<i>`

`<strong>` = «это важное слово», `<em>` = «интонационное ударение». Визуально получится то же самое (жирный и курсив), но смысл разный. Поисковики и скринридеры обрабатывают `<strong>`/`<em>` как семантическое выделение, а `<b>`/`<i>` — просто как оформление.

## Пример кода

```html
<h1>Заголовок страницы</h1>
<p>
  Обычный параграф с <strong>важным словом</strong>
  и <a href="https://developer.mozilla.org" target="_blank" rel="noopener noreferrer">ссылкой на MDN</a>.
</p>

<h2>Список покупок</h2>
<ul>
  <li>Хлеб</li>
  <li>Молоко</li>
  <li>Кофе</li>
</ul>

<img src="./cat.jpg" alt="Рыжий кот спит на клавиатуре ноутбука">
```

## Частые ошибки

- Использовать `<br>` вместо `<p>` — HTML станет нечитабельным, а стили неуправляемыми.
- Забывать `alt` у `<img>` — страница перестаёт быть доступной и проседает по SEO.
- Два `<h1>` на одной странице — плохо для иерархии и SEO.
- Использовать `<b>` / `<i>` вместо `<strong>` / `<em>` — теряешь смысловую нагрузку.
