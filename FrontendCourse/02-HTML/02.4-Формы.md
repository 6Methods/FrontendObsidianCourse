# Формы

## Зачем это нужно

Формы — главный способ получить ввод от пользователя. Регистрация, вход, обратная связь, поиск, фильтры. HTML даёт довольно много из коробки: валидацию, типы полей, доступность — без единой строчки JS.

## Главное

### `<form>` — обёртка

Это контейнер, внутри которого живут поля и кнопки. Обычно у формы два ключевых атрибута: `action` (куда браузер отправит данные) и `method` (`GET` — данные в URL, для поиска и фильтров; `POST` — данные в теле запроса, для регистрации и отправки чего-то чувствительного). В ранних модулях мы никуда реально не шлём данные — просто создаём структуру и смотрим, как она ведёт себя в браузере.

### `<input>` — главный рабочий элемент

Это не один тег, а целая семья. Тип задаётся атрибутом `type`, и от него зависит и внешний вид, и клавиатура на телефоне, и встроенная валидация. Основные типы: `text`, `email`, `password`, `tel`, `number`, `url`, `date`, `checkbox`, `radio`, `file`, `hidden`, `submit`. У каждого `<input>` почти всегда должен быть `name` — именно под этим ключом данные уедут на сервер.

### `<label>` — подпись к полю

Связывается с `<input>` двумя способами: через `for="id-инпута"` или оборачиванием инпута внутрь label. Клик по label фокусит поле — это базовая юзабилити и must-have для доступности. Скринридер читает подпись вслух, когда пользователь попадает на инпут.

### `<textarea>`

Многострочное поле. Размер регулируется атрибутами `rows` и `cols` или через CSS.

### `<select>` + `<option>`

Выпадающий список. `<select>` — обёртка, `<option>` — пункты внутри.

### `<button>`

Кнопки. `type="submit"` (по умолчанию) — отправляет форму. `type="button"` — ничего не делает без JS, просто кликабельный элемент. `type="reset"` — сбрасывает все поля формы к исходным значениям.

### Базовая HTML-валидация без JS

Браузер умеет проверять поля сам, если правильно расставить атрибуты:

- `required` — обязательное поле.
- `type="email"` — браузер сам проверит формат.
- `type="url"` — URL.
- `minlength` / `maxlength` — длина строки.
- `min` / `max` — границы чисел и дат.
- `pattern="..."` — регулярка для сложных правил.

### `placeholder` ≠ `<label>`

Плейсхолдер — серая подсказка внутри поля, исчезает при вводе. Подписью он не является: скринридеры его часто пропускают, а пользователь, начав печатать, забывает, что именно надо ввести. Плейсхолдер — это пример или уточнение, а не замена label.

## Пример кода

```html
<form action="/submit" method="POST">
  <div>
    <label for="name">Имя</label>
    <input id="name" name="name" type="text" required minlength="2">
  </div>

  <div>
    <label for="email">Email</label>
    <input id="email" name="email" type="email" required>
  </div>

  <div>
    <label for="age">Возраст</label>
    <input id="age" name="age" type="number" min="0" max="120">
  </div>

  <div>
    <label for="message">Сообщение</label>
    <textarea id="message" name="message" rows="5" required></textarea>
  </div>

  <button type="submit">Отправить</button>
</form>
```

## Частые ошибки

- `placeholder` вместо `<label>` → форма нерабочая для скринридеров, пользователь забывает, что было за поле.
- Не связывать `<label>` с `<input>` через `for`/`id` → клик по подписи не фокусит поле, доступность нарушена.
- `<button>` без атрибута `type` внутри `<form>` → по умолчанию ведёт себя как `submit` и отправляет форму, даже если ты этого не хотел.
- Полагаться только на HTML-валидацию в серьёзном приложении → её легко обойти, серверная валидация всё равно нужна (разбор — в модуле 13).
