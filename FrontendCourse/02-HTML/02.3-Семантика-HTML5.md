# Семантика HTML5

## Зачем это нужно

Теги типа `<header>`, `<main>`, `<article>` для браузера визуально равны `<div>`. Но они **сообщают смысл** поисковикам, скринридерам, будущему тебе через полгода, когда откроешь код и захочешь найти навигацию.

## Главное

### Идея семантики

Один `<div>` не отличается от другого. Это пустая коробка без этикетки. `<header>`, `<nav>`, `<article>` говорят, что это за часть страницы. Без семантики скринридер озвучивает стену «div, div, div». С семантикой звучит «шапка, навигация, основное содержимое, статья, подвал». Поисковики тоже понимают структуру лучше, а значит, страница индексируется качественнее.

### `<header>`

Шапка страницы или раздела: логотип, название, основная навигация. Важный момент: `<header>` может быть несколько на странице. Один у всей страницы, и ещё по одному у каждой `<article>` (там обычно заголовок поста и метаданные).

### `<nav>`

Блок навигации. Главное меню сайта, футер-меню со ссылками, breadcrumbs («хлебные крошки»). Не каждый список ссылок — это `<nav>`. Если это просто набор ссылок внутри текста, `<nav>` не нужен.

### `<main>`

Основной контент страницы. На странице ровно один `<main>`. Туда **не** должно попадать то, что повторяется на других страницах: шапка, футер, сайдбар остаются снаружи. Это как главная полка в магазине: только то, за чем пришли.

### `<section>`

Тематический раздел страницы: «блок с услугами», «блок с отзывами», «блок FAQ». У `<section>` почти всегда есть заголовок внутри, без него смысла в теге нет.

### `<article>`

Самостоятельный кусок контента, который имеет смысл сам по себе, если вырвать его из страницы. Пост в блоге, карточка товара, комментарий под постом, письмо в списке писем.

### `<aside>`

Сайдбар, реклама, справочный блок — что-то сбоку основного потока. Например, блок «Популярные посты» рядом со статьёй или всплывающая сноска.

### `<footer>`

Подвал страницы или раздела. Контакты, копирайт, ссылки на соцсети, служебные ссылки. Как и `<header>`, может быть у всей страницы и у отдельных `<article>`.

### Как выбрать тег

Если это «самостоятельная публикация» → `<article>`. Если «тематический блок с заголовком» → `<section>`. Если это что-то сбоку → `<aside>`. Если вообще непонятно и никакой смысл не добавишь → `<div>` (это нормально, не надо семантить всё подряд).

## Пример кода

```html
<body>
  <header>
    <h1>Мой блог</h1>
    <nav>
      <ul>
        <li><a href="/">Главная</a></li>
        <li><a href="/about">Обо мне</a></li>
      </ul>
    </nav>
  </header>

  <main>
    <article>
      <h2>Почему я учу фронтенд</h2>
      <p>Текст поста...</p>
    </article>
    <article>
      <h2>Лучшие ресурсы по HTML</h2>
      <p>Текст поста...</p>
    </article>
  </main>

  <aside>
    <h2>Популярные посты</h2>
    <ul>
      <li><a href="#">Пост 1</a></li>
    </ul>
  </aside>

  <footer>
    <p>© 2026 Мой блог</p>
  </footer>
</body>
```

## Частые ошибки

- Обернуть всю страницу в `<section>` вместо `<main>`. `<section>` без заголовка внутри уже бессмыслица.
- Несколько `<main>` на странице. Валидатор HTML ругается, логика ломается.
- Использовать `<article>` для каждого `<div>`. Семантика ≠ обязаловка. Там, где смысла нет, `<div>` ок.
