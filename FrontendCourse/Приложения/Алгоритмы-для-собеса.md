# Алгоритмы для собеса

## Зачем это тебе

LeetCode-гринд фронту не нужен — у тебя не Google и не конкурсы по олимпиадному программированию. Но на живом собесе **30-50% компаний** дают одну разминочную задачу минут на 20: перевернуть строку, найти дубли, `flatten` массив, сгруппировать объекты по ключу. Это проверка не на IQ, а на «умеет ли кандидат думать под давлением и чисто писать код». Что реально надо знать: строки, массивы, `Map`/`Set`, простые хеш-таблицы, двухуказательники, базовая рекурсия и интуиция по `O(n)`. Этого минимума хватит, чтобы не развалиться на **90% фронтовых собесов** уровня junior/middle. Этот файл — тот самый минимум, без воды.

## Интуиция по сложности

Коротко, чтобы говорить с интервьюером на одном языке:

- **O(1)** — доступ по индексу (`arr[i]`), `Map.get`, `Set.has`, `obj.key`. Мгновенно, не зависит от размера данных.
- **O(n)** — один проход по массиву/строке. `forEach`, `map`, `filter`, `reduce`, один `for`.
- **O(n log n)** — сортировка (`Array.prototype.sort`). Это дорого, но это базовый инструмент.
- **O(n²)** — вложенный цикл. Появляется, когда ты внутри цикла вызываешь `.includes()`, `.indexOf()`, `.find()`. Избегай где можно, замени на `Map`/`Set`.
- **O(2ⁿ)** — наивная рекурсия без мемоизации (классика: `fib(n) = fib(n-1) + fib(n-2)` без кеша). Почти всегда плохо.

**Главное правило:** если внутри цикла ты ищешь через `.includes()` / `.indexOf()` / `.find()` — это вложенность `O(n²)`. Замени на `Set`/`Map` и получи `O(n)`. На собесе это почти всегда тот самый ожидаемый «скачок оптимизации».

## Методология решения

Пошагово, как действовать у доски / на Codeshare:

1. **Переспроси условие.** Что на входе, что на выходе? Пустой массив допустим? Отрицательные числа? `null`/`undefined`? Дубли разрешены? — интервьюеры любят, когда уточняют.
2. **Примеры от руки.** Напиши 2-3 примера комментарием: обычный кейс, граничный (пустой массив, один элемент), с повтором. Это помогает и тебе, и интервьюеру.
3. **Наивное решение вслух.** Скажи прямо: «Сначала решу в лоб через вложенный цикл, `O(n²)`, потом оптимизирую». Это уже лучше молчания.
4. **Оптимизация.** Что ускорит? `Map`/`Set` для `O(1)` lookup? Сортировка перед двухуказательниками? Префиксная сумма?
5. **Пиши код.** Имена переменных осмысленные (`count`, `seen`, `result`, а не `a`/`b`/`x`). Комментарии только там, где не очевидно.
6. **Прогон на примерах.** Вслух, step-by-step, на входе типа `[2, 7, 11, 15]`. Интервьюер должен слышать, как ты ходишь по коду.
7. **Граничные случаи.** Пустой массив? `null`? Один элемент? Все одинаковые? Отрицательные?

### Строки (6 задач)

### 1. Развернуть строку

**Условие:** вернуть строку в обратном порядке.

**Пример:**
```
Вход: "hello"
Выход: "olleh"
```

**Подсказка:** `split` + `reverse` + `join`, или цикл с конца.

**Решение:**
```javascript
const reverseString = (str) => str.split('').reverse().join('');

// Альтернатива без встроенных методов
const reverseManual = (str) => {
  let result = '';
  for (let i = str.length - 1; i >= 0; i--) {
    result += str[i];
  }
  return result;
};
```

**Сложность:** O(n).

### 2. Палиндром (игнор регистра и не-букв)

**Условие:** проверить, что строка читается одинаково в обе стороны. Игнорируй регистр и всё, что не буквы/цифры.

**Пример:**
```
Вход: "A man, a plan, a canal: Panama"
Выход: true
```

**Подсказка:** нормализуй строку, затем двухуказательник или сравнение с reverse.

**Решение:**
```javascript
const isPalindrome = (str) => {
  const clean = str.toLowerCase().replace(/[^a-z0-9]/g, '');
  let left = 0;
  let right = clean.length - 1;
  while (left < right) {
    if (clean[left] !== clean[right]) return false;
    left++;
    right--;
  }
  return true;
};
```

**Сложность:** O(n).

---

### 3. Анаграммы

**Условие:** два слова — анаграммы, если состоят из одних и тех же букв в разном порядке.

**Пример:**
```
Вход: "listen", "silent"
Выход: true
```

**Подсказка:** отсортируй обе строки и сравни, или считай частоты через `Map`.

**Решение:**
```javascript
const isAnagram = (a, b) => {
  if (a.length !== b.length) return false;
  const count = new Map();
  for (const ch of a) count.set(ch, (count.get(ch) ?? 0) + 1);
  for (const ch of b) {
    if (!count.get(ch)) return false;
    count.set(ch, count.get(ch) - 1);
  }
  return true;
};
```

**Сложность:** O(n).

### 4. Первый не повторяющийся символ

**Условие:** вернуть первый символ строки, который встречается ровно один раз. Если нет — вернуть `null`.

**Пример:**
```
Вход: "leetcode"
Выход: "l"
```

**Подсказка:** два прохода через `Map`: первый считает частоты, второй ищет первую с частотой 1.

**Решение:**
```javascript
const firstUniqueChar = (str) => {
  const count = new Map();
  for (const ch of str) count.set(ch, (count.get(ch) ?? 0) + 1);
  for (const ch of str) {
    if (count.get(ch) === 1) return ch;
  }
  return null;
};
```

**Сложность:** O(n).

### 5. Подсчитать гласные

**Условие:** посчитать, сколько гласных в строке (a, e, i, o, u, y).

**Пример:**
```
Вход: "hello world"
Выход: 3
```

**Подсказка:** `Set` гласных + один проход.

**Решение:**
```javascript
const countVowels = (str) => {
  const vowels = new Set(['a', 'e', 'i', 'o', 'u', 'y']);
  let count = 0;
  for (const ch of str.toLowerCase()) {
    if (vowels.has(ch)) count++;
  }
  return count;
};
```

**Сложность:** O(n).

### 6. Сжать строку

**Условие:** сжать строку по подсчёту подряд идущих одинаковых символов. Если сжатая версия не короче — вернуть оригинал.

**Пример:**
```
Вход: "aaabb"
Выход: "a3b2"
```

**Подсказка:** один проход, счётчик текущего символа.

**Решение:**
```javascript
const compress = (str) => {
  if (!str) return str;
  let result = '';
  let count = 1;
  for (let i = 1; i <= str.length; i++) {
    if (str[i] === str[i - 1]) {
      count++;
    } else {
      result += str[i - 1] + count;
      count = 1;
    }
  }
  return result.length < str.length ? result : str;
};
```

**Сложность:** O(n).

### 7. Найти максимум

**Условие:** найти максимальное число в массиве одним проходом.

**Пример:**
```
Вход: [3, 1, 7, 2, 9, 4]
Выход: 9
```

**Подсказка:** храни текущий максимум в переменной, проходи один раз.

**Решение:**
```javascript
const findMax = (arr) => {
  if (arr.length === 0) return null;
  let max = arr[0];
  for (let i = 1; i < arr.length; i++) {
    if (arr[i] > max) max = arr[i];
  }
  return max;
};
```

**Сложность:** O(n).

### 8. Two Sum

**Условие:** найти индексы двух элементов массива, которые в сумме дают `target`. Каждый элемент используется один раз.

**Пример:**
```
Вход: nums = [2, 7, 11, 15], target = 9
Выход: [0, 1]
```

**Подсказка:** `Map` со значением `{число: индекс}`. Для каждого `x` проверяй, лежит ли `target - x`.

**Решение:**
```javascript
const twoSum = (nums, target) => {
  const seen = new Map();
  for (let i = 0; i < nums.length; i++) {
    const complement = target - nums[i];
    if (seen.has(complement)) return [seen.get(complement), i];
    seen.set(nums[i], i);
  }
  return null;
};
```

**Сложность:** O(n).

### 9. Удалить дубли (сохранив порядок)

**Условие:** из массива убрать повторы, оставив первое вхождение каждого значения.

**Пример:**
```
Вход: [1, 2, 2, 3, 1, 4]
Выход: [1, 2, 3, 4]
```

**Подсказка:** `Set` для отслеживания виденного + собираем результат в массив.

**Решение:**
```javascript
const removeDuplicates = (arr) => {
  const seen = new Set();
  const result = [];
  for (const item of arr) {
    if (!seen.has(item)) {
      seen.add(item);
      result.push(item);
    }
  }
  return result;
};

// Одной строкой через Set (уместно, если порядок не критичен)
const dedupe = (arr) => [...new Set(arr)];
```

**Сложность:** O(n).

### 10. Развернуть массив in-place

**Условие:** перевернуть массив, не создавая новый (без дополнительной памяти O(n)).

**Пример:**
```
Вход: [1, 2, 3, 4, 5]
Выход: [5, 4, 3, 2, 1]
```

**Подсказка:** двухуказательник: меняй местами `left` и `right`, двигайся к центру.

**Решение:**
```javascript
const reverseInPlace = (arr) => {
  let left = 0;
  let right = arr.length - 1;
  while (left < right) {
    [arr[left], arr[right]] = [arr[right], arr[left]];
    left++;
    right--;
  }
  return arr;
};
```

**Сложность:** O(n), память O(1).

### 11. Flatten вложенный массив

**Условие:** разгладить массив. Сначала — один уровень, потом — глубокий (любой уровень вложенности).

**Пример:**
```
Вход: [1, [2, [3, [4]]]]
Выход (deep): [1, 2, 3, 4]
```

**Подсказка:** для одного уровня — `reduce` + `concat`. Для глубокого — рекурсия или `Array.prototype.flat(Infinity)`.

**Решение:**
```javascript
// Один уровень
const flattenOne = (arr) => arr.reduce((acc, item) => acc.concat(item), []);

// Глубокий flatten, рекурсивно
const flattenDeep = (arr) => {
  const result = [];
  for (const item of arr) {
    if (Array.isArray(item)) {
      result.push(...flattenDeep(item));
    } else {
      result.push(item);
    }
  }
  return result;
};
```

**Сложность:** O(n), где n — общее число элементов во всех уровнях.

### 12. Missing number

**Условие:** в массиве числа от 1 до n, одно пропущено. Найти пропущенное.

**Пример:**
```
Вход: [1, 2, 4, 5], n = 5
Выход: 3
```

**Подсказка:** сумма 1..n равна `n*(n+1)/2`. Вычти сумму массива — получишь пропущенное.

**Решение:**
```javascript
const findMissing = (arr, n) => {
  const expected = (n * (n + 1)) / 2;
  let actual = 0;
  for (const num of arr) actual += num;
  return expected - actual;
};
```

**Сложность:** O(n), память O(1).

### 13. Частоты слов

**Условие:** по массиву строк построить объект `{слово: количество}`.

**Пример:**
```
Вход: ["apple", "banana", "apple", "orange", "banana", "apple"]
Выход: { apple: 3, banana: 2, orange: 1 }
```

**Подсказка:** `reduce` + объект-аккумулятор.

**Решение:**
```javascript
const wordFrequency = (words) =>
  words.reduce((acc, word) => {
    acc[word] = (acc[word] ?? 0) + 1;
    return acc;
  }, {});
```

**Сложность:** O(n).

### 14. Group by key

**Условие:** массив объектов сгруппировать по значению указанного ключа.

**Пример:**
```
Вход: [{ age: 20, name: "A" }, { age: 30, name: "B" }, { age: 20, name: "C" }], key = "age"
Выход: { 20: [{...A}, {...C}], 30: [{...B}] }
```

**Подсказка:** `reduce`, создавай массив-корзину под каждый новый ключ.

**Решение:**
```javascript
const groupBy = (arr, key) =>
  arr.reduce((acc, item) => {
    const groupKey = item[key];
    if (!acc[groupKey]) acc[groupKey] = [];
    acc[groupKey].push(item);
    return acc;
  }, {});
```

**Сложность:** O(n).

### 15. Пересечение массивов

**Условие:** вернуть уникальные элементы, которые есть в обоих массивах.

**Пример:**
```
Вход: [1, 2, 2, 3], [2, 3, 4]
Выход: [2, 3]
```

**Подсказка:** один массив → `Set`, по второму фильтруй.

**Решение:**
```javascript
const intersection = (a, b) => {
  const setA = new Set(a);
  const result = new Set();
  for (const item of b) {
    if (setA.has(item)) result.add(item);
  }
  return [...result];
};
```

**Сложность:** O(n + m).

### 16. Deep merge двух объектов

**Условие:** слить два объекта. Если по ключу в обоих — объекты, слить рекурсивно. Иначе — значение из второго перекрывает первое.

**Пример:**
```
Вход: { a: 1, b: { x: 10 } }, { b: { y: 20 }, c: 3 }
Выход: { a: 1, b: { x: 10, y: 20 }, c: 3 }
```

**Подсказка:** рекурсия + проверка `typeof === 'object'` и `!Array.isArray`.

**Решение:**
```javascript
const isPlainObject = (v) =>
  v !== null && typeof v === 'object' && !Array.isArray(v);

const deepMerge = (target, source) => {
  const result = { ...target };
  for (const key of Object.keys(source)) {
    if (isPlainObject(result[key]) && isPlainObject(source[key])) {
      result[key] = deepMerge(result[key], source[key]);
    } else {
      result[key] = source[key];
    }
  }
  return result;
};
```

**Сложность:** O(n), где n — общее число ключей во всех уровнях.

### 17. First duplicate

const firstDuplicate = (arr) => {

### Двухуказательники (4 задачи)

### 19. Слить два отсортированных массива

**Условие:** объединить два отсортированных массива в один отсортированный.

**Пример:**
```
Вход: [1, 3, 5], [2, 4, 6]
Выход: [1, 2, 3, 4, 5, 6]
```

**Подсказка:** два указателя `i`, `j` на начало каждого массива, бери меньший элемент.

**Решение:**
```javascript
const mergeSorted = (a, b) => {
  const result = [];
  let i = 0;
  let j = 0;
  while (i < a.length && j < b.length) {
    if (a[i] <= b[j]) result.push(a[i++]);
    else result.push(b[j++]);
  }
  while (i < a.length) result.push(a[i++]);
  while (j < b.length) result.push(b[j++]);
  return result;
};
```

**Сложность:** O(n + m).

### 20. Удалить дубли из отсортированного массива in-place

**Условие:** массив отсортирован. Удалить дубли на месте, вернуть новую длину.

**Пример:**
```
Вход: [1, 1, 2, 2, 3]
Выход: 3, массив: [1, 2, 3, _, _]
```

**Подсказка:** два указателя: `slow` — позиция для записи, `fast` — сканирует.

**Решение:**
```javascript
const removeDuplicatesSorted = (arr) => {
  if (arr.length === 0) return 0;
  let slow = 0;
  for (let fast = 1; fast < arr.length; fast++) {
    if (arr[fast] !== arr[slow]) {
      slow++;
      arr[slow] = arr[fast];
    }
  }
  return slow + 1;
};
```

**Сложность:** O(n), память O(1).

### 21. Container with Most Water (упрощённая)

**Условие:** дан массив положительных чисел — «высоты». Найди пару индексов `(i, j)`, у которых `min(h[i], h[j]) * (j - i)` максимально.

**Пример:**
```
Вход: [1, 8, 6, 2, 5, 4, 8, 3, 7]
Выход: 49
```

**Подсказка:** двухуказательник с концов. Двигай тот, у кого высота меньше.

**Решение:**
```javascript
const maxArea = (heights) => {
  let left = 0;
  let right = heights.length - 1;
  let max = 0;
  while (left < right) {
    const area = Math.min(heights[left], heights[right]) * (right - left);
    if (area > max) max = area;
    if (heights[left] < heights[right]) left++;
    else right--;
  }
  return max;
};
```

**Сложность:** O(n).

### 22. Проверить подстроку (без indexOf)

**Условие:** содержит ли строка `haystack` подстроку `needle`. `indexOf`/`includes` использовать нельзя.

**Пример:**
```
Вход: haystack = "hello world", needle = "world"
Выход: true
```

**Подсказка:** внешний цикл по `haystack`, внутренний — посимвольная проверка с `needle`.

**Решение:**
```javascript
const contains = (haystack, needle) => {
  if (needle.length === 0) return true;
  for (let i = 0; i <= haystack.length - needle.length; i++) {
    let match = true;
    for (let j = 0; j < needle.length; j++) {
      if (haystack[i + j] !== needle[j]) {
        match = false;
        break;
      }
    }
    if (match) return true;
  }
  return false;
};
```

**Сложность:** O(n * m) в худшем случае.

### 23. Факториал

**Условие:** вычислить `n!` рекурсивно. `0! = 1`.

**Пример:**
```
Вход: 5
Выход: 120
```

**Подсказка:** базовый случай `n <= 1`, рекурсивный `n * factorial(n - 1)`.

```javascript
const factorial = (n) => {
  if (n <= 1) return 1;
  return n * factorial(n - 1);
};
```

**Сложность:** O(n), стек O(n).

### 24. Фибоначчи с мемоизацией

**Условие:** `n`-й член последовательности Фибоначчи. `fib(0) = 0`, `fib(1) = 1`.

**Пример:**
```
Вход: 10
Выход: 55
```

**Подсказка:** наивная рекурсия — `O(2ⁿ)`. С кешем — `O(n)`.

**Решение:**
```javascript
const fib = (n, memo = new Map()) => {
  if (n < 2) return n;
  if (memo.has(n)) return memo.get(n);
  const result = fib(n - 1, memo) + fib(n - 2, memo);
  memo.set(n, result);
  return result;
};
```

**Сложность:** O(n) по времени и памяти.

### 25. Обход объекта-дерева

**Условие:** собрать все значения заданного поля во вложенной структуре.

**Пример:**
```
Вход:
{
  id: 1,
  children: [
    { id: 2, children: [{ id: 4, children: [] }] },
    { id: 3, children: [] }
  ]
}
Поле: "id"
Выход: [1, 2, 4, 3]
```

**Подсказка:** рекурсия: если текущий узел — объект, проверь поле, затем пройдись по всем значениям.

**Решение:**
```javascript
const collectField = (node, field) => {
  const result = [];
  const walk = (current) => {
    if (current === null || typeof current !== 'object') return;
    if (field in current) result.push(current[field]);
    for (const value of Object.values(current)) {
      if (Array.isArray(value)) value.forEach(walk);
      else if (typeof value === 'object') walk(value);
    }
  };
  walk(node);
  return result;
};
```

**Сложность:** O(n), где n — число узлов.

### 26. Power set

**Условие:** вернуть все подмножества массива уникальных элементов.

**Пример:**
```
Вход: [1, 2, 3]
Выход: [[], [1], [2], [1,2], [3], [1,3], [2,3], [1,2,3]]
```

**Подсказка:** для каждого элемента либо берём его, либо нет. Рекурсия или итерация «удваиванием».

**Решение:**
```javascript
const powerSet = (arr) => {
  const result = [[]];
  for (const item of arr) {
    const len = result.length;
    for (let i = 0; i < len; i++) {
      result.push([...result[i], item]);
    }
  }
  return result;
};
```

**Сложность:** O(2ⁿ * n), потому что подмножеств `2ⁿ` и каждое копируется.

### 27. FizzBuzz

**Условие:** числа от 1 до n. Кратные 3 — `"Fizz"`, кратные 5 — `"Buzz"`, кратные и 3 и 5 — `"FizzBuzz"`, иначе само число.

**Пример:**
```
Вход: 5
Выход: [1, 2, "Fizz", 4, "Buzz"]
```

**Подсказка:** сначала проверка на 15 (или на оба), потом 3, потом 5.

**Решение:**
```javascript
const fizzBuzz = (n) => {
  const result = [];
  for (let i = 1; i <= n; i++) {
    if (i % 15 === 0) result.push('FizzBuzz');
    else if (i % 3 === 0) result.push('Fizz');
    else if (i % 5 === 0) result.push('Buzz');
    else result.push(i);
  }
  return result;
};
```

**Сложность:** O(n).

### 28. Debounce

**Условие:** написать функцию, которая откладывает вызов: после каждого вызова таймер сбрасывается, реальный вызов — только через `delay` мс тишины.

**Пример:**
```
const log = debounce(() => console.log("go"), 300);
log(); log(); log(); // через 300мс от последнего вызова — один "go"
```

**Подсказка:** замыкание, `setTimeout` + `clearTimeout`.

**Решение:**
```javascript
const debounce = (fn, delay) => {
  let timerId;
  return function (...args) {
    clearTimeout(timerId);
    timerId = setTimeout(() => fn.apply(this, args), delay);
  };
};
```

**Сложность:** O(1) на каждый вызов обёртки.

### 29. Throttle

**Условие:** написать функцию, которая вызывает `fn` не чаще одного раза в `limit` мс. Все вызовы внутри «окна» — игнорируются.

**Пример:**
```
const onScroll = throttle(() => console.log("s"), 200);
// 10 вызовов подряд → 1 вывод
```

**Подсказка:** флаг или отметка времени последнего вызова.

**Решение:**
```javascript
const throttle = (fn, limit) => {
  let lastCall = 0;
  return function (...args) {
    const now = Date.now();
    if (now - lastCall >= limit) {
      lastCall = now;
      fn.apply(this, args);
    }
  };
};
```

**Сложность:** O(1) на каждый вызов обёртки.

### 30. Мемоизация универсальная

**Условие:** написать обёртку `memoize(fn)`, кеширующую результат для тех же аргументов. Для простоты — аргументы примитивы.

**Пример:**
```
const slow = (a, b) => a + b;
const fast = memoize(slow);
fast(1, 2); // считает и кеширует
fast(1, 2); // из кеша
```

**Подсказка:** `Map`, ключ — сериализация аргументов (например, `JSON.stringify`).

**Решение:**
```javascript
const memoize = (fn) => {
  const cache = new Map();
  return function (...args) {
    const key = JSON.stringify(args);
    if (cache.has(key)) return cache.get(key);
    const result = fn.apply(this, args);
    cache.set(key, result);
    return result;
  };
};
```

**Сложность:** O(1) на попадание в кеш, плюс стоимость сериализации ключа.

## Частые ошибки на собесе

- **Молчишь и думаешь в голове.** Интервьюер должен слышать твой ход мысли — иначе он видит кандидата, который не умеет коммуницировать.
- **Сразу кидаешься в код без примеров на бумаге.** В результате получаешь баг на пустом массиве и тратишь 10 минут на отладку.
- **Пишешь `O(n²)` и не озвучиваешь** «знаю, можно улучшить через `Map`, но сначала напишу в лоб». Интервьюер ждёт именно этой фразы — это сигнал, что ты понимаешь сложность.
- **Не тестируешь на граничных случаях** (пустой массив, один элемент, повторы, отрицательные числа). Граничные — половина оценки.
- **Используешь `forEach` там, где нужен `for` с возможностью `break`.** `forEach` нельзя прервать — если нужно выйти раньше, бери обычный `for` или `for..of`.
- **Не уточняешь, можно ли мутировать вход.** Это часть контракта. Спроси — и интервьюер поставит плюсик за вдумчивость.
