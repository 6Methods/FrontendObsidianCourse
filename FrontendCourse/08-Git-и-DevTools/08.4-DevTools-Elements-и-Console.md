# DevTools — Elements и Console

## Зачем это нужно

DevTools — это главный рентген фронтендера. Открыл страницу и можешь прочитать её DOM, править CSS в реалтайме, выполнять произвольный JS, смотреть, что именно сломалось и почему. Без DevTools ты работаешь вслепую: меняешь код, перезагружаешь, гадаешь. С ними видишь результат сразу и лезешь прямо в живую страницу. Два первых таба (Elements и Console) покрывают 80% ежедневного использования, с них и начинаем.

## Главное

### Как открыть

`Cmd+Opt+I` на Mac, `F12` или `Ctrl+Shift+I` на Windows/Linux. Ещё вариант: правой кнопкой по любому месту страницы → **Inspect** (откроется сразу на выбранном элементе). Панель DevTools можно пристыковать справа, снизу, слева или выкинуть в отдельное окно: кнопка с тремя точками в углу панели → Dock side. Для узких макетов удобнее снизу, для широких — справа.

### Elements — DOM в реальном времени

Показывает текущий DOM, а не исходный HTML с сервера. Это критически важно: React, Vue, Svelte вставляют элементы динамически, и в Elements ты видишь результат работы JS, а не то, что пришло по сети. Слева — дерево DOM, справа — панель стилей и Computed. Можно редактировать прямо здесь: двойной клик по тегу, тексту или атрибуту, меняешь значение, страница обновляется мгновенно. Полезно для быстрой проверки гипотез.

### Styles + Computed

**Styles** показывает все CSS-правила, которые применяются к элементу: с селекторами, именами файлов и номерами строк. Зачёркнутое свойство = перебито более специфичным правилом ниже. **Computed** — финальные значения всех свойств, как их реально применил браузер (после каскада, наследования и резолва единиц измерения). Когда не понимаешь, почему `color` не тот — смотри Computed.

### Force state (:hover, :focus)

В Styles есть кнопка `:hov`, она позволяет принудительно включить состояния `:hover`, `:focus`, `:active`, `:visited`. Удобно тестировать ховер-эффекты, когда не хочется держать мышь на элементе, или смотреть `:focus`, не теряя фокус при клике в DevTools.

### Box model

В Computed снизу есть квадратная схема — **box model**: content, padding, border, margin. Сразу видно, какого отступа не хватает, где лишний margin, почему элемент «съезжает». Можно кликнуть на любое значение и отредактировать прямо в схеме.

### Console — твой REPL

Выполняет любой JS в контексте текущей страницы. `document.querySelectorAll('a').length` — сколько ссылок на странице. Доступны все глобальные переменные, функции и объекты, определённые на странице. Отличный способ быстро что-то проверить, не добавляя код в файл.

### Полезные console-методы

- `console.log` — обычный вывод.
- `console.warn` / `console.error` — разные уровни важности (подсветка и иконки).
- `console.table(arr)` — массив объектов как настоящая таблица со столбцами.
- `console.group('имя')` / `console.groupEnd()` — вложенные, сворачиваемые блоки логов.
- `console.time('x')` / `console.timeEnd('x')` — замер времени выполнения куска кода.

### Shortcut $

В консоли DevTools есть удобные алиасы: `$('.selector')` = `document.querySelector`, `$$('.selector')` = `document.querySelectorAll` (возвращает массив, а не NodeList). `$0` — последний выбранный элемент в Elements. Выделил элемент в DOM-дереве → в консоли сразу `$0.click()` или `$0.style.color = 'red'`. Экономит тонну времени.

### Ошибки и источник

Красные ошибки в консоли — кликни по имени файла справа, откроется Sources с точным местом падения. Трассировку читай снизу вверх: сверху — где именно упало, ниже — кто вызвал эту функцию, ещё ниже — кто вызвал того.

### Live expression

Иконка «глаза» над консолью создаёт «висячую» переменную, значение которой обновляется в реальном времени. Удобно следить за `window.scrollY`, `document.activeElement`, шириной окна и т. п. без спама `console.log`.

### Preserve log и Clear

По умолчанию консоль очищается при перезагрузке страницы. Чекбокс **Preserve log** — не чистить (важно при отладке ошибок, после которых идёт редирект). Кнопка ⊘ (или `Cmd+K`) — очистить консоль вручную прямо сейчас.

## Пример кода

```js
// быстрый DOM-аудит
$$('a').length                        // сколько ссылок
$$('img:not([alt])')                  // картинки без alt (a11y)

// потыкать элемент
$0.style.outline = '2px solid red'    // подсветить выбранный элемент
$0.click()                            // программный клик

// массив объектов — как таблица
const users = [{id:1,name:"Иван"},{id:2,name:"Анна"}];
console.table(users);

// замер времени
console.time('calc');
for (let i = 0; i < 1e6; i++) Math.sqrt(i);
console.timeEnd('calc');              // "calc: 12.3ms"

// условный breakpoint через консоль
// выбрал кнопку в Elements → правой кнопкой → Break on → subtree modifications
```

## Частые ошибки

- **Думать, что в Elements — исходный HTML.** Это актуальный DOM, который мог быть изменён JS-ом после загрузки. Исходник смотри в Sources или через View Source (`Cmd+U`).
- **Пытаться отредактировать `node_modules/...` или компилированный бандл в Sources вместо исходника.** Нужны source maps — настройка в следующем уроке.
- **Не использовать `$0`** — каждый раз копировать селектор или писать `document.querySelector('...')`. Выдели элемент в Elements и работай через `$0`.
- **Оставить `console.log`-и в проде.** Убирай перед коммитом или настрой линтер/плагин (`no-console` в ESLint), чтобы ловил автоматически.

## Задания

- [ ] На любом сайте: открой DevTools, выдели заголовок, в Styles поменяй `color: red` — посмотри, как меняется.

- [ ] В Console выполни `$$('img').forEach(img => console.log(img.src))` — получи список всех картинок.
