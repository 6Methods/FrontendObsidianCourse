# useMemo, useCallback, useRef

## Зачем это нужно

Компоненты ре-рендерятся чаще, чем хотелось бы. Три хука дают точечную оптимизацию и работу с не-реактивными данными. `useMemo` кеширует значение, `useCallback` — функцию, `useRef` хранит ссылку между рендерами без ре-рендера при изменении.

## Главное

### useMemo: синтаксис

`const value = useMemo(() => compute(a, b), [a, b])`. React запоминает результат вычисления и на следующем рендере смотрит в массив зависимостей. Если `a` и `b` не изменились, возвращает старое закешированное значение, не вызывая `compute` заново. Если хоть одна зависимость изменилась, пересчитывает и кеширует новое.

### useMemo: когда

Два сценария, когда `useMemo` реально нужен. Первый: дорогое вычисление — фильтр большого массива, парсинг JSON, сложная математика. Второй: стабильная ссылка, которую ты передаёшь в `useEffect` deps или в props `React.memo`-дочки. Для банального `x + 1` или конкатенации строк `useMemo` не нужен — сам хук тоже стоит процессорного времени, и ты только сделаешь хуже.

### useCallback: синтаксис

`const handle = useCallback(() => {...}, [deps])`. Возвращает ту же самую функцию между рендерами, пока `deps` не изменились. Без `useCallback` каждая перерисовка создаёт новый объект-функцию, у него другой reference, даже если тело идентичное.

### useCallback: когда

Когда функцию передаёшь в `React.memo`-компонент (чтобы shallow compare не сработал как «prop изменился») или в deps другого хука (`useEffect`, `useMemo`, кастомный хук). В остальных случаях не нужен: обычная inline-функция пересоздаётся бесплатно, это не та оптимизация, за которую стоит бороться.

### React.memo

Обёртка компонента: `export default memo(Child)`. Компонент ре-рендерится только если его props изменились по shallow compare. Вместе с `useCallback` (для функций-props) и `useMemo` (для объектов/массивов-props) защищает от лишних рендеров в дочках. Без `memo` сам по себе `useCallback` бесполезен.

### useRef: синтаксис

`const ref = useRef(initial)`. Возвращает объект `{ current: initial }`. Ты читаешь и пишешь в `ref.current` напрямую, это мутабельно. Изменение `ref.current` НЕ вызывает ре-рендер — это принципиально отличает его от `useState`.

### useRef для DOM

Самый частый кейс: `<input ref={inputRef} />` плюс `inputRef.current.focus()`. Прямой доступ к DOM-элементу: для фокуса, скролла, измерений (`getBoundingClientRect`), управления `<video>`/`<canvas>`. Работает после маунта, до него `ref.current === null`.

### useRef для «переменных между рендерами»

Всё, что нужно сохранить между рендерами, но не нужно показывать в UI: ID таймера от `setTimeout`, предыдущее значение prop для сравнения, флаг «первый рендер», ссылка на WebSocket. Если бы ты использовал `useState`, каждая запись триггерила бы рендер, а тебе это не надо.

### Не оптимизируй заранее

React быстрый. В 90% случаев `useMemo`/`useCallback` не нужны — они только делают код сложнее для чтения. Добавляй оптимизации, когда замерил реальные тормоза через React DevTools Profiler, а не «на всякий случай».

### Правильные deps

Линтер `react-hooks/exhaustive-deps` всегда указывает правильные зависимости для хуков. Не обманывай его пустым массивом — получишь stale closure: функция будет видеть старые значения state/props, и ты словишь неуловимые баги. Слушай линтер или вытаскивай значение из замыкания через ref.

## Пример кода

```jsx
import { useState, useMemo, useCallback, useRef, memo, useEffect } from "react";

// Дорогая фильтрация — через useMemo
function ProductList({ products, query }) {
  const filtered = useMemo(
    () => products.filter(p => p.name.toLowerCase().includes(query.toLowerCase())),
    [products, query]
  );
  return <ul>{filtered.map(p => <li key={p.id}>{p.name}</li>)}</ul>;
}

// Стабильная функция для memo-дочки
const ChildButton = memo(function ChildButton({ onClick, label }) {
  console.log("render ChildButton");
  return <button onClick={onClick}>{label}</button>;
});

function Parent() {
  const [count, setCount] = useState(0);
  const handleClick = useCallback(() => setCount(c => c + 1), []);
  return (
    <>
      <p>{count}</p>
      <ChildButton onClick={handleClick} label="+" />
    </>
  );
}

// useRef: DOM и таймер
function SearchInput() {
  const inputRef = useRef(null);
  const timerRef = useRef(null);
  const [text, setText] = useState("");

  useEffect(() => {
    inputRef.current.focus();
  }, []);

  function handleChange(e) {
    setText(e.target.value);
    clearTimeout(timerRef.current);
    timerRef.current = setTimeout(() => console.log("search:", e.target.value), 500);
  }

  return <input ref={inputRef} value={text} onChange={handleChange} />;
}
```

## Частые ошибки

- Использовать `useMemo` для простого выражения: `useMemo(() => a + b, [a, b])` — оверхед самого хука больше, чем вычисление. Только для дорогих операций.
- `useCallback` без `React.memo` на дочернем — бесполезен: child всё равно ре-рендерится от родителя, стабильность ссылки никому не нужна.
- Писать в `ref.current` и ждать ре-рендер: нет, `useRef` не триггерит рендер. Для реактивных значений, которые должны отражаться в UI, бери `useState`.
- Пустой массив deps в `useEffect`/`useCallback`, когда внутри используется state/props: stale closure, функция бьёт всегда старыми значениями. Слушай линтер `react-hooks/exhaustive-deps`.
