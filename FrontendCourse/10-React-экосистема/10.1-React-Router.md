# React Router

## Зачем это нужно

Без Router SPA это одна страница, всё внутри неё. С Router ты получаешь настоящие URL: `/products`, `/products/42`, `/cart`. Переходы без перезагрузки, но с полноценной историей, закладками и кнопкой «Назад». Стандарт для любого React-приложения больше одного экрана.

## Главное

### Установка и версия

Ставишь пакет: `npm install react-router-dom`. Актуальная версия v6/v7, синтаксис отличается от v5. Будь внимателен: старые гайды на Stack Overflow могут путать. Проверяй дату и версию.

### BrowserRouter

Это обёртка вокруг всего приложения, живёт в `main.jsx`: `<BrowserRouter><App /></BrowserRouter>`. Она подключается к History API браузера и даёт всем компонентам доступ к текущему URL. Без неё хуки `useNavigate`, `useParams` и компонент `<Routes>` просто не работают.

### Routes и Route

`<Routes>` это контейнер, внутри которого ты описываешь все маршруты. Один маршрут — `<Route path="/" element={<Home />} />`, где `path` это паттерн URL, а `element` — компонент, который рендерится при совпадении. Работает принцип «первый совпавший рендерится», остальные игнорируются.

### Link vs `<a>`

Для переходов внутри приложения всегда используй `<Link to="/products">Каталог</Link>`. Обычный `<a href="...">` делает полную перезагрузку страницы, теряется весь state в памяти. `<NavLink>` это то же самое, но автоматически ставит `className` активной ссылке. Удобно для подсветки текущего пункта меню.

### useNavigate

Когда переход нужно сделать программно, бери хук `useNavigate`: `const navigate = useNavigate(); navigate("/cart")`. Типичные места: после успешного сабмита формы, после логина, после удаления товара. `navigate(-1)` уводит назад по истории.

### useParams

Динамический сегмент в пути объявляется через двоеточие: `path="/products/:id"`. Внутри компонента ты достаёшь значение: `const { id } = useParams();`. Важно: параметр всегда строка, даже если в URL число.

### useSearchParams

Это query string, то, что после `?`: `?page=2&sort=price`. Работаешь как с хуком: `const [params, setParams] = useSearchParams();`, читаешь через `params.get("page")`. Удобно для фильтров, сортировки, пагинации: состояние живёт в URL и переживает перезагрузку.

### Вложенные роуты

Роуты можно вкладывать: `<Route path="/dashboard" element={<Layout />}> <Route path="stats" element={<Stats />} /> </Route>`. В родительском `Layout` ставишь `<Outlet />` — точка, куда React Router подставит дочерний роут. Идеально для общих сайдбаров и шапок.

### Защищённые маршруты

Делается через компонент-обёртку: `if (!user) return <Navigate to="/login" replace />`. Оборачиваешь нужные роуты в `<Protected>...</Protected>`, и неавторизованный пользователь сразу улетает на логин. `replace` нужен, чтобы страница не копилась в истории.

### 404 страница

Последний роут со звёздочкой ловит всё несовпавшее: `<Route path="*" element={<NotFound />} />`. Ставь его в самый конец списка, иначе перекроет остальные роуты и пользователь всегда будет видеть 404. Обязательная штука для любого проекта, иначе неверный URL даст пустой экран без фидбэка.

## Пример кода

```jsx
// main.jsx
import { BrowserRouter } from "react-router-dom";
createRoot(document.getElementById("root")).render(
  <BrowserRouter>
    <App />
  </BrowserRouter>
);

// App.jsx
import { Routes, Route, Link, NavLink, useParams, useNavigate, Navigate, Outlet } from "react-router-dom";

function Nav() {
  return (
    <nav>
      <NavLink to="/" end>Главная</NavLink>
      <NavLink to="/products">Каталог</NavLink>
      <NavLink to="/cart">Корзина</NavLink>
    </nav>
  );
}

function ProductPage() {
  const { id } = useParams();
  const navigate = useNavigate();
  return (
    <>
      <h2>Товар {id}</h2>
      <button onClick={() => navigate(-1)}>Назад</button>
    </>
  );
}

function Protected({ user, children }) {
  if (!user) return <Navigate to="/login" replace />;
  return children;
}

function Layout() {
  return (
    <>
      <Nav />
      <main><Outlet /></main>
    </>
  );
}

function App() {
  const user = null; // из Context в реальном проекте
  return (
    <Routes>
      <Route path="/" element={<Layout />}>
        <Route index element={<h1>Главная</h1>} />
        <Route path="products" element={<h1>Каталог</h1>} />
        <Route path="products/:id" element={<ProductPage />} />
        <Route path="cart" element={<Protected user={user}><h1>Корзина</h1></Protected>} />
        <Route path="*" element={<h1>404</h1>} />
      </Route>
    </Routes>
  );
}
```

## Частые ошибки

- Использовать `<a href="...">` вместо `<Link to="...">`: полная перезагрузка страницы, теряется state.
- Забыть `BrowserRouter` в `main.jsx`: `useNavigate`/`useParams` упадут с ошибкой «must be used within a Router».
- `useParams` ожидает число: параметр всегда строка. `Number(id)` — руками.
- Порядок роутов: более общие сверху перекрывают конкретные. В v6 это решено, но в именованных сегментах (`:id` vs `new`) всё равно следи.
