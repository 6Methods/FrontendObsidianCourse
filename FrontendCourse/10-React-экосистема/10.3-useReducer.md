# useReducer

## Зачем это нужно

Когда state сложный (много полей, разная логика обновления) — 10 разных `useState` + обработчиков превращаются в кашу. `useReducer` собирает всё в одну функцию: «на такое событие — такое новое состояние». Паттерн Redux, но встроенный в React, без библиотек.

## Главное

### Синтаксис

`const [state, dispatch] = useReducer(reducer, initialState)`. Из хука получаешь текущий `state` и функцию `dispatch`. Вызываешь `dispatch(action)` → React сам дёргает `reducer(state, action)` → берёт то, что reducer вернул, и кладёт в state. Ты не трогаешь state напрямую — только через dispatch.

### Reducer — чистая функция

```js
function reducer(state, action) {
  switch (action.type) { ... }
}
```

Чистая — значит на одни и те же входы всегда возвращает один и тот же результат. Не мутирует старый state, а возвращает новый объект/массив. Не делает side-effects: никаких fetch, setTimeout, console.log с побочным смыслом. Только «было такое → стало такое».

### Action — обычный объект

Минимум — `{ type: "ADD_TODO" }`. Чаще с данными: `{ type: "ADD_TODO", payload: { text: "..." } }`. Соглашение по `type` — строка в SCREAMING_SNAKE_CASE, чтобы в switch сразу было видно. `payload` — название для данных, тоже общепринятое.

### Когда useReducer, а когда useState

Один-два независимых значения (число, строка, флажок) → `useState`. Много связанных полей, сложные переходы (форма со всеми полями, корзина, undo/redo, wizard) → `useReducer`. Правило простое: если обработчиков больше 3-4 и они работают с одним и тем же state — пора в reducer.

### Иммутабельность

Как и с `useState`, всегда возвращай новый объект/массив. `{ ...state, count: state.count + 1 }`, `[...state.items, newItem]`, `state.items.filter(...)`. Ни в коем случае не `state.items.push(x)` — React сравнивает по ссылке, увидит ту же ссылку и не перерендерит.

### default в switch

Всегда пиши `default: return state;` — защита от опечатки в `action.type`. Или строже — `throw new Error("Unknown action: " + action.type)`, чтобы ошибка всплыла сразу, а не тихо обнулила state.

### Инициализация

Третий аргумент — `useReducer(reducer, initialArg, init)`. Функция `init(initialArg)` вызовется один раз при маунте. Полезно, когда начальное значение считается дорого (например, парсится из localStorage) — чтобы не делать это на каждом рендере.

### Reducer + Context

Классический паттерн: reducer живёт в Context, через провайдер раздаёшь `state` и `dispatch` вниз по дереву. Получается «маленький Redux» — глобальный state, доступный везде, но без самой библиотеки Redux.

### Почему не всегда reducer

Для простых случаев — оверхед: писать отдельную функцию ради одного счётчика глупо. Но если у тебя 4 обработчика для todos (add/remove/toggle/clear) — reducer делает код в разы чище.

### Отладка

Все переходы state централизованы в одной функции. Легко логировать (`console.log(action, state)`), ставить breakpoint на нужный `case`, писать юнит-тесты: reducer — чистая функция, вход → выход, без моков.

## Пример кода

```jsx
import { useReducer } from "react";

// Reducer
function todosReducer(state, action) {
  switch (action.type) {
    case "ADD":
      return [...state, {
        id: crypto.randomUUID(),
        text: action.payload,
        done: false,
      }];
    case "TOGGLE":
      return state.map(t =>
        t.id === action.payload ? { ...t, done: !t.done } : t
      );
    case "REMOVE":
      return state.filter(t => t.id !== action.payload);
    case "CLEAR_DONE":
      return state.filter(t => !t.done);
    default:
      throw new Error("Unknown action: " + action.type);
  }
}

function TodoApp() {
  const [todos, dispatch] = useReducer(todosReducer, []);

  return (
    <>
      <button onClick={() => dispatch({ type: "ADD", payload: "New" })}>
        Добавить
      </button>
      <ul>
        {todos.map(t => (
          <li key={t.id}>
            <span onClick={() => dispatch({ type: "TOGGLE", payload: t.id })}>
              {t.done ? "✓" : "○"} {t.text}
            </span>
            <button onClick={() => dispatch({ type: "REMOVE", payload: t.id })}>
              ✕
            </button>
          </li>
        ))}
      </ul>
      <button onClick={() => dispatch({ type: "CLEAR_DONE" })}>
        Очистить завершённые
      </button>
    </>
  );
}
```

## Частые ошибки

- Мутировать state в reducer: `state.items.push(x); return state` — React не увидит изменений (та же ссылка).
- Делать side-effects в reducer (fetch, setTimeout) — reducer должен быть чистым. Side-effects — в `useEffect` или обработчиках.
- Забыть `default` в switch — опечатка в `action.type` тихо вернёт `undefined`, и state станет `undefined`.
- Использовать reducer для одной-двух переменных — усложнение без пользы. Начни с `useState`.
