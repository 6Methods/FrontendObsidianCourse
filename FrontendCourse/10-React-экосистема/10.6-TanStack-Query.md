# TanStack Query

## Зачем это нужно

Fetch руками через `useEffect` — это всегда boilerplate: loading, error, отмена, cache, refetch, синхронизация между компонентами. TanStack Query (бывший React Query) решает всё это одной строкой `useQuery`. Это индустриальный стандарт работы с серверным state в React.

## Главное

### Серверный vs клиентский state

Клиентский state — это твой UI: открыта ли модалка, какой таб выбран, что в инпуте. Серверный state — это копия данных с бэка, которые в любой момент могут устареть (кто-то другой их поменял). Это принципиально разные штуки, и `useState` — не лучший инструмент для серверных: тебе придётся руками делать cache, invalidation, refetch. TanStack Query специально под серверный state и заточен.

### Установка

```bash
npm install @tanstack/react-query
```

В `main.jsx` оборачиваешь приложение в `<QueryClientProvider client={queryClient}>` и создаёшь инстанс `new QueryClient()`. Без этого хуки работать не будут.

### useQuery

Базовый хук для чтения данных:

```js
const { data, isLoading, error } = useQuery({
  queryKey: ["users"],
  queryFn: fetchUsers,
});
```

Всё — cache, автоматический refetch при фокусе окна, дедупликация одинаковых запросов уже работает из коробки.

### queryKey

Массив, уникально идентифицирующий запрос. Зависит от параметров: `["user", userId]`, `["posts", { page: 2 }]`. Как только ключ изменился — TanStack считает это новым запросом и идёт в сеть.

### Состояния

- `isLoading` — первичная загрузка (когда в кеше пусто).
- `isFetching` — любой fetch, включая background-refetch.
- `isError` — флаг ошибки.
- `error` — объект ошибки.
- `data` — результат запроса.

### Cache и stale time

По умолчанию данные считаются устаревшими сразу (`staleTime: 0`), и при фокусе окна будет refetch. Ставишь `staleTime: 60_000` — минуту данные «свежие», refetch не происходит. `gcTime` (бывший `cacheTime`) — сколько держать данные в памяти после того, как последний подписчик отмонтировался.

### useMutation

Для POST/PUT/DELETE:

```js
const { mutate, isPending } = useMutation({
  mutationFn: (data) => fetch("/api/todos", { method: "POST", body: JSON.stringify(data) }),
});
```

Дальше в обработчике — `mutate(payload)`.

### Инвалидация

После успешной mutation обнови связанные запросы:

```js
queryClient.invalidateQueries({ queryKey: ["todos"] });
```

TanStack сам перезапросит свежие данные для всех компонентов, которые этот ключ слушают.

### Optimistic updates

Можно сразу обновить кеш до ответа сервера (`onMutate`), а при ошибке откатить (`onError`). Юзер видит результат мгновенно — отличный UX для лайков, чекбоксов, простых форм.

### DevTools

`@tanstack/react-query-devtools` — overlay в углу экрана, где видишь все запросы, их состояние, данные, ключи. Обязательно поставь в dev-сборке, без них отлаживать cache — мучение.

### Что даёт из коробки

Cache, дедупликация, автоматический refetch при фокусе и реконнекте, background refetch, отмена устаревших запросов, retry при ошибке. Всё то, что руками пришлось бы писать неделю — здесь работает сразу.

## Пример кода

```jsx
// main.jsx
import { QueryClient, QueryClientProvider } from "@tanstack/react-query";
const queryClient = new QueryClient();

createRoot(document.getElementById("root")).render(
  <QueryClientProvider client={queryClient}>
    <App />
  </QueryClientProvider>
);

// hooks/useUsers.js
import { useQuery, useMutation, useQueryClient } from "@tanstack/react-query";

export function useUsers() {
  return useQuery({
    queryKey: ["users"],
    queryFn: () => fetch("/api/users").then(r => r.json()),
    staleTime: 60_000, // свежие минуту
  });
}

export function useCreateUser() {
  const qc = useQueryClient();
  return useMutation({
    mutationFn: (user) =>
      fetch("/api/users", {
        method: "POST",
        headers: { "Content-Type": "application/json" },
        body: JSON.stringify(user),
      }).then(r => r.json()),
    onSuccess: () => qc.invalidateQueries({ queryKey: ["users"] }),
  });
}

// components/UsersList.jsx
function UsersList() {
  const { data, isLoading, error } = useUsers();
  const createUser = useCreateUser();

  if (isLoading) return <p>Загрузка…</p>;
  if (error) return <p>Ошибка: {error.message}</p>;

  return (
    <>
      <ul>
        {data.map(u => <li key={u.id}>{u.name}</li>)}
      </ul>
      <button
        onClick={() => createUser.mutate({ name: "Новый" })}
        disabled={createUser.isPending}
      >
        Добавить
      </button>
    </>
  );
}
```

## Частые ошибки

- Забыть `queryKey` — без него TanStack не может кешировать и дедуплицировать запросы.
- Менять `queryKey` через объект без стабильной структуры — кеш не попадает, каждый раз новый запрос.
- Делать fetch в `useEffect` и дублировать с `useQuery` — выбирай что-то одно, синхронизация сломается.
- Не инвалидировать запросы после mutation — UI показывает старые данные до ручного refresh.
