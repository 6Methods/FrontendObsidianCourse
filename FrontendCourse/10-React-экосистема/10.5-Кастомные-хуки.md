# Кастомные хуки

## Зачем это нужно

Если в двух компонентах одинаковая логика (подписка на resize, чтение localStorage, fetch-обвязка) — вынеси в функцию `useSomething`. Получишь переиспользование без копипасты. Кастомные хуки — главный способ делиться логикой между компонентами, заменяющий HOC и render-props.

## Главное

### Правило одно
Кастомный хук — это функция с именем на `use`, которая внутри вызывает другие хуки. Всё — это и есть кастомный хук. Не магия, не отдельная сущность React — просто соглашение об именовании, которое понимает линтер.

### Возвращать можно что угодно
Объект `{ data, loading, error }`, массив `[value, setValue]` (как у `useState`), одно значение, функцию — что угодно. Выбирай так, как будет логично использовать. Для пары «значение + сеттер» — массив (можно переименовать при деструктуризации). Для трёх+ полей — объект.

### Типичный пример: useLocalStorage
`const [val, setVal] = useLocalStorage("key", defaultVal)`. Внутри — `useState` с ленивой инициализацией из `localStorage.getItem` и `useEffect` для синхронизации обратно при изменении. Получается API как у обычного `useState`, только с автосохранением.

### useFetch
Базовый учебный хук: берёт URL, возвращает `{ data, loading, error }`. Внутри — `useState` + `useEffect` с `AbortController`, чтобы отменять старый запрос при смене URL. В реальном проекте бери `TanStack Query` (см. 10.6) — там кэш, ретраи, dedup, — но свой `useFetch` писал каждый, полезно понимать как.

### useDebounce
Значение, которое обновляется с задержкой. Полезно для поиска: пользователь печатает, а запрос уходит только когда он остановился. `const debounced = useDebounce(query, 300)` — и в `useEffect` реагируешь на `debounced`.

### useMediaQuery
`const isMobile = useMediaQuery("(max-width: 768px)")`. Подписка на `window.matchMedia`, ре-рендер при изменении. Удобно для условного рендера мобильной/десктопной вёрстки, когда CSS-медиа недостаточно.

### useToggle
Утилитарный: `const [isOpen, toggle] = useToggle(false)`. Минимальный хук, но удобный для модалок, дропдаунов, аккордеонов — избавляет от `setOpen(v => !v)` в каждом компоненте.

### Правила хуков в своих хуках
Внутри кастомного хука действуют те же правила: вызывай хуки только на верхнем уровне функции, без условий и циклов. Линтер `react-hooks/rules-of-hooks` не делает исключений — кастомный хук для него такой же, как встроенный.

### Когда выносить в хук
Если одинаковая логика повторилась ≥2 раз — пора выносить. Или если один компонент перегружен `useEffect`/`useState` и ты хочешь сфокусировать его на UI — вынеси логику в хук, компонент станет «тонким». Не выноси «на всякий случай» — преждевременное обобщение тоже вредит.

### Именование
Всегда `use<Name>`: `useUser`, `useAuth`, `useCart`. Линтер React хуков ищет именно префикс `use` — по нему он понимает, что внутри могут быть вызовы хуков, и проверяет правила. Без префикса он решит, что это обычная функция, и пропустит проверки.

### Кастомный хук — не общий state
Каждый вызов хука создаёт свой state. `useLocalStorage("x")` в двух компонентах — это два независимых `useState`, которые просто синхронизируются через localStorage (и то не автоматически между вкладками). Это не единый source of truth. Если нужен общий state между компонентами — бери Context (см. 10.2) или стейт-менеджер.

## Пример кода

```jsx
import { useState, useEffect, useCallback } from "react";

// useLocalStorage
export function useLocalStorage(key, initial) {
  const [val, setVal] = useState(() => {
    const raw = localStorage.getItem(key);
    return raw ? JSON.parse(raw) : initial;
  });
  useEffect(() => {
    localStorage.setItem(key, JSON.stringify(val));
  }, [key, val]);
  return [val, setVal];
}

// useDebounce
export function useDebounce(value, delay = 300) {
  const [debounced, setDebounced] = useState(value);
  useEffect(() => {
    const id = setTimeout(() => setDebounced(value), delay);
    return () => clearTimeout(id);
  }, [value, delay]);
  return debounced;
}

// useToggle
export function useToggle(initial = false) {
  const [on, setOn] = useState(initial);
  const toggle = useCallback(() => setOn(v => !v), []);
  return [on, toggle, setOn];
}

// useFetch (учебный, боевой — TanStack Query)
export function useFetch(url) {
  const [state, setState] = useState({ data: null, loading: true, error: null });
  useEffect(() => {
    const ctrl = new AbortController();
    setState({ data: null, loading: true, error: null });
    fetch(url, { signal: ctrl.signal })
      .then(r => r.json())
      .then(data => setState({ data, loading: false, error: null }))
      .catch(err => {
        if (err.name !== "AbortError") {
          setState({ data: null, loading: false, error: err });
        }
      });
    return () => ctrl.abort();
  }, [url]);
  return state;
}

// Использование
function Profile({ userId }) {
  const { data, loading, error } = useFetch(`/api/users/${userId}`);
  if (loading) return <p>Загрузка…</p>;
  if (error) return <p>Ошибка: {error.message}</p>;
  return <h2>{data.name}</h2>;
}
```

## Частые ошибки

- Имя без `use`: `function getLocalStorage()` — линтер не проверит, React-правила нарушатся тихо.
- Вызывать обычную функцию так же, как хук (или наоборот) — это разные вещи. Хук — содержит хуки, обычная функция — нет.
- Ожидать общего state между вызовами одного хука — каждый вызов независим. Для общего — Context.
- Копировать логику между двумя компонентами «ещё разок» — уже на 2-й раз время выносить в хук.
