# TS и React

## Зачем это нужно

React + TS — индустриальный стандарт. Props типизированы — нельзя забыть обязательное поле, IDE подсветит ошибку прямо при вызове компонента. Events типизированы — сразу видно, что `e.target.value` это строка, а state-хуки типа `useState<User | null>(null)` сами подсказывают, где нужна проверка на null.

## Главное

### Типизация props

Самое частое. Пишешь `type Props = { title: string; onClick: () => void }` и в компоненте `function Button({ title, onClick }: Props) { ... }`. `type` или `interface` — разница косметическая, см. [[11.3-Aliases-interfaces-unions]].

### Опциональные props

Ставь `?` и задавай дефолт:

`type Props = { label: string; variant?: "primary" | "secondary" }`.

Внутри: `variant ?? "primary"` или дефолт в деструктуризации `{ variant = "primary" }`.

### children

`children: ReactNode` — любой валидный JSX (строка, число, элемент, массив, `null`). Импорт: `import { ReactNode } from "react"`. Иногда нужно строже: `children: string` — только текст, `children: ReactElement` — только один React-элемент.

### Props для HTML-элементов

`ComponentPropsWithoutRef<"button">` — все стандартные props нативного `<button>` (onClick, disabled, type и т.д.). Удобно при обёртке нативных элементов:

`type Props = ComponentPropsWithoutRef<"button"> & { variant: string }`.

Так твой `Button` принимает и свои `variant`, и все нативные атрибуты.

### useState

`useState<User | null>(null)` — явный тип. Без generic TS выводит из initial: `useState("")` это `string`, `useState(0)` — `number`. Уточняй, когда initial это `null` или пустой массив (иначе TS выведет `null` или `never[]`).

### useEffect

Хук сам типизирован, типов писать не надо. Главное — cleanup: возвращай либо `void`, либо функцию очистки. TS проверит, что ты не возвращаешь Promise (частая ошибка с async).

### useRef

`useRef<HTMLInputElement>(null)`. Для DOM-рефа `ref.current: HTMLInputElement | null`. Всегда проверяй на null перед использованием: `ref.current?.focus()`.

### useReducer

`useReducer<Reducer<State, Action>>(reducer, initial)` или тип выводится из типизированного reducer. Типизируй `State` и `Action`. `Action` часто — discriminated union (см. [[11.3-Aliases-interfaces-unions]]), и TS сам сужает по `type`.

### События

`onChange: (e: ChangeEvent<HTMLInputElement>) => void`. Важные типы: `ChangeEvent`, `FormEvent`, `MouseEvent`, `KeyboardEvent`. Все импортируются из `react`.

### useContext

Типизируй через дефолт или generic: `const Ctx = createContext<Theme | null>(null)`. В кастомном хуке `useTheme` — narrow через проверку плюс throw, чтобы наружу возвращать строго `Theme`, а не `Theme | null`.

### Generic-компоненты

`function List<T>({ items, render }: { items: T[]; render: (t: T) => ReactNode })`. При использовании TS сам подставит `T` по переданному массиву, явно указывать не нужно.

### Типы от библиотек

React Router, TanStack Query, RHF — все ship с типами. Например, `useQuery<User[]>(...)`, `UseFormReturn<FormData>`. Импортируй из пакетов, не пиши сам.

## Пример кода

```tsx
import {
  useState, useRef, useEffect, ChangeEvent, FormEvent,
  ReactNode, ComponentPropsWithoutRef
} from "react";

// Простой компонент с props
type ButtonProps = {
  label: string;
  variant?: "primary" | "secondary";
  onClick: () => void;
};

function Button({ label, variant = "primary", onClick }: ButtonProps) {
  return (
    <button className={variant} onClick={onClick}>
      {label}
    </button>
  );
}

// Обёртка нативного <input>
type InputProps = ComponentPropsWithoutRef<"input"> & {
  label: string;
};

function Field({ label, ...rest }: InputProps) {
  return (
    <label>
      {label}
      <input {...rest} />
    </label>
  );
}

// useState + events
type User = { id: number; name: string };

function UserForm() {
  const [user, setUser] = useState<User | null>(null);
  const [name, setName] = useState("");
  const inputRef = useRef<HTMLInputElement>(null);

  useEffect(() => {
    inputRef.current?.focus();
  }, []);

  function handleChange(e: ChangeEvent<HTMLInputElement>) {
    setName(e.target.value);
  }

  function handleSubmit(e: FormEvent<HTMLFormElement>) {
    e.preventDefault();
    setUser({ id: Date.now(), name });
  }

  return (
    <form onSubmit={handleSubmit}>
      <input ref={inputRef} value={name} onChange={handleChange} />
      <Button label="OK" onClick={() => {}} />
      {user && <p>Добавлен: {user.name}</p>}
    </form>
  );
}

// children
function Card({ title, children }: { title: string; children: ReactNode }) {
  return (
    <div className="card">
      <h3>{title}</h3>
      {children}
    </div>
  );
}

// Generic-компонент
function List<T>({
  items,
  renderItem,
}: {
  items: T[];
  renderItem: (item: T) => ReactNode;
}) {
  return <ul>{items.map((item, i) => <li key={i}>{renderItem(item)}</li>)}</ul>;
}
```

## Частые ошибки

- `React.FC<Props>` — устаревший паттерн, неявно добавляет `children`. Современный React: просто `function X(props: Props)`.
- `useState([])` без generic: TS выведет `never[]`. Нужен `useState<User[]>([])`.
- `e: any` в обработчике — привет, весь смысл TS пропал. Используй правильный `ChangeEvent<...>`.
- `useRef(null)` без generic: `ref.current: null`. Укажи тип элемента: `useRef<HTMLInputElement>(null)`.
