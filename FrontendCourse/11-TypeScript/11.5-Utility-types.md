# Utility types

## Зачем это нужно

У тебя есть тип `User`, а нужен такой же, но со всеми полями опциональными (для формы редактирования), или только 2 поля (для превью), или без пароля (для ответа API). Вручную переписывать — копипаста, которая рассыпется при первом же изменении базового типа. Utility types — готовые генераторы типов из других типов, встроенные в TS.

## Главное

### Partial<T>
Делает все поля опциональными. `Partial<User>` превращает `{ id: number; name: string }` в `{ id?: number; name?: string }`. Используется для форм редактирования и patch-запросов, где прилетают только изменённые поля, а не весь объект целиком.

### Required<T>
Противоположность `Partial`: все опциональные поля становятся обязательными. Полезно, когда ты постепенно «дособрал» объект и хочешь гарантировать на уровне типов, что все поля заполнены и ниже по коду можно их не проверять.

### Readonly<T>
Все поля становятся `readonly`, TS запретит присваивание. Защита от случайной мутации, используется в `Object.freeze`-подобных паттернах и при работе с конфигами.

### Pick<T, K>
Выбирает подмножество полей. `Pick<User, "id" | "name">` = `{ id: number; name: string }`. Нужен, когда API возвращает только часть полей, когда ты собираешь тип превью, или когда компоненту нужны ровно два поля из большого типа.

### Omit<T, K>
Убирает указанные поля. `Omit<User, "password">` — всё, кроме `password`. Типичный паттерн для Response-типов, где надо скрыть чувствительные данные, и для пропсов, где один конкретный ключ лишний.

### Record<K, V>
Объект-словарь. `Record<string, number>` = `{ [key: string]: number }`. Если ключи известны заранее, `Record<"en" | "ru", string>` даст гарантию, что оба ключа на месте, и TS заругается, если ты забудешь один.

### ReturnType<F>
Тип того, что возвращает функция. `type Result = ReturnType<typeof fetchUser>`. Полезно, когда не хочется дублировать тип вручную, особенно если функция чужая или её сигнатура ещё меняется.

### Parameters<F>
Кортеж типов параметров функции. `Parameters<typeof fetchUser>` — массив типов аргументов. Удобно для обёрток, декораторов и мемоизации, где ты хочешь принять те же аргументы, что и оригинал.

### Awaited<T>
Разворачивает `Promise`. `Awaited<Promise<User>>` = `User`. Для `ReturnType<typeof asyncFn>` обязательно оборачивай в `Awaited`, иначе получишь `Promise<User>` вместо `User` и потом будешь гадать, почему не видно полей.

### Exclude<U, E> и Extract<U, E>
Работают с union. `Exclude<"a" | "b" | "c", "a">` = `"b" | "c"`. `Extract` наоборот — оставляет только указанные члены union. Пригождается, когда из большого набора статусов надо оставить «только успешные» или «все, кроме отменённых».

### NonNullable<T>
Убирает `null` и `undefined`. `NonNullable<string | null>` = `string`. Пригождается после narrowing и в утилитах, где ты уже точно знаешь, что значение есть, но TS пока об этом не догадался.

### typeof и keyof
`keyof User` = `"id" | "name" | "email"` — все ключи типа как union строк. `typeof user` забирает тип реального значения, чтобы не описывать его руками. Часто комбинируется с остальными утилитами: `Pick<User, keyof Partial<User>>`, `Record<keyof User, string>`.

## Пример кода

```ts
type User = {
  id: number;
  name: string;
  email: string;
  password: string;
  role: "admin" | "user";
};

// Partial — для формы редактирования
type UserEdit = Partial<User>;
// { id?: number; name?: string; email?: string; password?: string; role?: ... }

// Pick — для превью
type UserPreview = Pick<User, "id" | "name">;

// Omit — для ответа API (без пароля)
type UserResponse = Omit<User, "password">;

// Readonly — защита от мутации
type FrozenUser = Readonly<User>;

// Record — словарь переводов
type Lang = "en" | "ru";
type Translations = Record<Lang, string>;
const t: Translations = { en: "Hello", ru: "Привет" };

// ReturnType и Awaited
async function fetchUser(id: number): Promise<User> {
  const r = await fetch(`/api/users/${id}`);
  return r.json();
}
type User2 = Awaited<ReturnType<typeof fetchUser>>; // User

// keyof
type UserKey = keyof User; // "id" | "name" | "email" | "password" | "role"

function getField<K extends UserKey>(user: User, key: K): User[K] {
  return user[key];
}

// Exclude
type PublicKey = Exclude<UserKey, "password">; // всё кроме password

// NonNullable
type Name = NonNullable<string | null>; // string

// Комбинация: «тип User без пароля, где все поля опциональные»
type EditForm = Partial<Omit<User, "id" | "password">>;
```

## Частые ошибки

- Дублировать тип вручную вместо `Partial`/`Omit`: при изменении базового типа забудешь обновить копию.
- `ReturnType<asyncFn>` без `Awaited` — получишь `Promise<User>` вместо `User`.
- `Pick<User, "some_string">` — ключ должен совпадать с полем. TS заругается, читай ошибку.
- Utility types не работают в рантайме, это только типы. Для получения полей объекта в коде — `Object.keys()`.
