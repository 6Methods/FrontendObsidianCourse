# Что нам понадобится — список и зачем

## Зачем это нужно

Прежде чем написать первую строчку кода, нужно собрать рабочее место. Это как перед началом работы в Figma убедиться, что Figma вообще установлена, а не пытаться рисовать в Paint. У каждого инструмента своя роль — сейчас разберём, что и для чего.

## Главное

### Полный список (спойлер: нас всего четыре вещи)

| Инструмент | Зачем |
|---|---|
| **VS Code** | Редактор, в котором ты будешь писать код |
| **Node.js + npm** | Позволяет запускать инструменты разработки на твоём компьютере |
| **Git + GitHub** | Сохраняет историю изменений и хранит проекты в облаке |
| **Браузер** (Chrome или Firefox) | Показывает, как выглядит твой сайт; содержит DevTools |

Это весь стандартный набор frontend-разработчика. Больше ничего ставить не нужно — по крайней мере, сейчас.

---

### VS Code — редактор кода

Думай о VS Code как о Figma, только для кода. Ты открываешь файлы, пишешь в них текст (HTML, CSS, JS), и VS Code помогает: подсвечивает ошибки, подсказывает, как закончить слово, автоматически форматирует. Бесплатный, работает на Mac и Windows. Скачать: [code.visualstudio.com](https://code.visualstudio.com/).

Что важно понять: VS Code — это просто текстовый редактор. Он не запускает твой код, не открывает браузер. Это просто место, где ты пишешь.

---

### Node.js и npm — «движок за кулисами»

Котёнок, вот важный момент: JavaScript изначально работал только в браузере. Node.js — это способ запустить JavaScript прямо на твоём компьютере, без браузера.

Тебе не нужно писать серверный код. Node нужен для инструментов:

- Запустить локальный сервер (чтобы открыть свой сайт через `http://localhost:3000`, а не двойным кликом по файлу).
- Установить библиотеки и фреймворки (React, Tailwind, Vite и всё, что будет дальше в курсе).
- Запустить линтер, сборщик, тесты.

**npm** — это менеджер пакетов, который идёт вместе с Node. Им ты будешь устанавливать чужой код в свои проекты одной командой. Скачать Node: [nodejs.org](https://nodejs.org/) → всегда LTS-версия.

---

### Git и GitHub — сохранение и облако

**Git** — инструмент, который запоминает все изменения в твоих файлах. Как «история версий» в Google Docs, только гораздо мощнее. Ты говоришь Git'у «запомни текущее состояние» — он запоминает. Облажался и сломал всё? Можно откатиться к предыдущей версии.

**GitHub** — это облачный сервис, где хранятся Git-репозитории. Туда пушат (отправляют) свои проекты, чтобы:
- Не потерять при поломке компьютера.
- Показывать работодателю и заказчикам.
- Работать в команде.

С Git и GitHub мы разберёмся подробно в модуле 08. Сейчас — просто зарегистрируйся на [github.com](https://github.com/).

---

### Браузер — не только «где открыть сайт»

Chrome или Firefox. Лучше иметь оба — разные сайты могут отображаться чуть по-разному, и это нужно замечать.

Главная суперсила браузера для разработчика — встроенные **DevTools**. Нажми `F12` на любом сайте, и откроется панель, где видно весь HTML, CSS, ошибки в коде, сетевые запросы. Это твой второй главный инструмент после VS Code.

---

### Порядок установки

1. Скачай и установи **VS Code**.
2. Скачай и установи **Node.js LTS** (npm идёт в комплекте).
3. Зарегистрируйся на **GitHub**.
4. Убедись, что стоит **Chrome или Firefox**.

Git мы поставим отдельно в модуле 08, когда придёт время. Сейчас хватит этих четырёх вещей.

## Пример кода

```bash
# После установки Node.js открой терминал и проверь:
node -v   # выведет что-то вроде v22.x.x
npm -v    # выведет что-то вроде 10.x.x
```

Если оба вывели версии — всё установлено правильно. Если видишь ошибку «command not found» — Node не установился или нужно перезапустить терминал.

Если мы говорим про macOS то терминал у тебя установлен уже.
Либо переходим в VsCode и ищем терминал там

## Частые ошибки

- **Установить Node не с официального сайта** — установщики с других сайтов могут дать старую или сломанную версию. Только [nodejs.org](https://nodejs.org/).
- **Взять «Current»-версию вместо «LTS»** — Current — экспериментальная. LTS — стабильная, для работы.
- **Не перезапустить терминал после установки Node** — без этого `node -v` даст ошибку, даже если всё установилось правильно.
