# Что нам понадобится: список и зачем

## Зачем это нужно

Прежде чем написать первую строчку кода, нужно собрать рабочее место. Это как перед тем как сесть в Figma убедиться, что Figma вообще установлена, а не пытаться рисовать в Paint. У каждого инструмента своя роль, сейчас разберёмся, зачем нам каждый.

## Главное

### Полный список (всего четыре пункта)

| Инструмент | Зачем |
|---|---|
| VS Code | Редактор, в котором ты будешь писать код |
| Node.js + npm | Запускает инструменты разработки на твоём компьютере |
| Git + GitHub | Хранит историю изменений и держит проекты в облаке |
| Браузер (Chrome или Firefox) | Показывает, как выглядит сайт, и содержит DevTools |

Это весь стандартный набор фронтендера. Больше пока ничего не нужно.

### VS Code, редактор кода

Думай о VS Code как о Figma, только для текста. Открываешь файлы, пишешь в них код, редактор подсвечивает синтаксис, подсказывает, как закончить слово, форматирует. Бесплатный, есть под Mac и Windows. Ставим с [code.visualstudio.com](https://code.visualstudio.com/).

Важное уточнение: VS Code это просто текстовый редактор. Он не запускает твой код, не открывает браузер, не разворачивает сервер. Он просто место, где ты пишешь. Запуском будет заниматься Node.js и сам браузер.

### Node.js и npm, движок за кулисами

Котёнок, тут важный момент. JavaScript изначально жил только в браузере. Node.js это способ запускать тот же JavaScript уже на твоём компьютере, без браузера.

Тебе при этом не надо писать серверный код. Node нам нужен ради инструментов:

- поднять локальный сервер, чтобы открыть свой сайт по адресу `http://localhost:3000`, а не через двойной клик по файлу;
- ставить библиотеки и фреймворки (React, Tailwind, Vite — всё это придёт позже в курсе);
- запускать линтер, сборщик, тесты.

npm это менеджер пакетов, который ставится вместе с Node. Им мы будем устанавливать чужой код в свои проекты одной командой. Скачивается Node с [nodejs.org](https://nodejs.org/), всегда бери LTS-версию (расшифровывается как Long Term Support, то есть стабильная).

### Git и GitHub, сохранение и облако

Git это инструмент, который запоминает все изменения в твоих файлах. Похоже на «историю версий» в Google Docs, только мощнее. Ты говоришь Git'у «запомни текущее состояние», он запоминает. Если потом что-то сломал, откатываешься на любой шаг назад.

GitHub это облачный сервис, где Git-репозитории хранятся. Туда пушат проекты, чтобы:

- не потерять при поломке ноутбука;
- показывать заказчикам и работодателям;
- работать в команде с другими разработчиками.

С Git и GitHub подробно разберёмся в модуле 08. Сейчас просто зарегистрируйся на [github.com](https://github.com/), этого достаточно.

### Браузер

Chrome или Firefox. Лучше иметь оба, потому что один и тот же сайт в них может выглядеть слегка по-разному, и эти расхождения нужно замечать.

Главная сила браузера для разработчика это встроенные DevTools. Нажимаешь F12 на любом сайте, и открывается панель, в которой видно весь HTML, все CSS-стили, все ошибки и все сетевые запросы. Это второй главный инструмент после VS Code.

### Порядок установки

1. Скачать и поставить VS Code.
2. Скачать и поставить Node.js LTS (npm идёт в комплекте).
3. Зарегистрироваться на GitHub.
4. Убедиться, что стоит Chrome или Firefox.

Сам Git ставим отдельно в модуле 08, когда дойдём. Сейчас четырёх вещей выше достаточно.

## Пример кода

```bash
# После установки Node.js открой терминал и проверь:
node -v   # выведет что-то вроде v22.x.x
npm -v    # выведет что-то вроде 10.x.x
```

Если оба вывели версии, всё прошло успешно. Если видишь `command not found`, значит Node либо не установился, либо нужно перезапустить терминал.

На macOS терминал у тебя уже стоит из коробки, искать в Applications → Utilities → Terminal. На Windows проще всего открывать встроенный терминал прямо в VS Code — туда мы и придём в следующих уроках.

## Частые ошибки

- Поставить Node не с официального сайта. На сторонних сайтах часто висят старые или нестандартные сборки. Только [nodejs.org](https://nodejs.org/).
- Выбрать «Current» вместо «LTS». Current это экспериментальная версия для тех, кто хочет потестить новые фичи. Для работы и обучения — LTS.
- Не перезапустить терминал после установки Node. Терминал «видит» новые программы только после перезапуска. Если `node -v` ругается — закрой терминал и открой заново.
