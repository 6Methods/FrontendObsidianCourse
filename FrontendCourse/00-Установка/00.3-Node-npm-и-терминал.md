# Node, npm и первый запуск терминала

## Зачем это нужно

Без Node.js не получится поставить ни один современный фреймворк, ни React, ни Vite, ни Tailwind. Все они ставятся через npm, а npm работает в терминале. Поэтому три темы упакованы в один урок: что такое терминал, как поставить Node, как проверить, что всё работает.

## Главное

### Терминал это просто другой способ говорить с компьютером

Котёнок, терминал не страшный. Это альтернатива клику мышкой: вместо того чтобы тыкать в иконки и меню, ты пишешь команды текстом.

Сравни с Finder на Mac или Проводником на Windows. Там ты тоже ходишь по папкам, создаёшь и переименовываешь файлы. Терминал делает то же самое, только текстом.

Зачем разработчики используют его, если есть Finder?

- Быстрее. Одна команда заменяет десять кликов.
- Мощнее. Часть действий через интерфейс просто не сделать (поставить npm-пакет, запустить локальный сервер).
- Единый стандарт. Все инструкции в документации написаны под терминал, а не «нажми правой кнопкой».

И ещё важное. Случайно сломать систему обычными командами навигации и установки нельзя. Удалить что-то можно только если явно вызвать `rm` и указать путь. Простые команды — безопасны, экспериментируй.

### Как открыть терминал

На Mac есть два пути:

- `Cmd + Space`, набираешь «Terminal», Enter.
- Прямо в VS Code: `` Ctrl+` `` (это та же клавиша, что и `~`). Внизу окна откроется терминал. Самый удобный вариант, потому что не нужно переключаться между приложениями.

На Windows:

- Поиск → «Windows Terminal» или «PowerShell».
- В VS Code так же: `` Ctrl+` ``.

Для курса используй встроенный терминал VS Code. Он сам открывается в папке текущего проекта, и это удобно.

### Из чего состоит командная строка

Открыл терминал, видишь что-то такое:

```
имя@компьютер папка %
```

Например:

```
katerina@MacBook-Pro my-project %
```

По частям.

| Часть | Что означает |
|---|---|
| `katerina` | Имя пользователя на компьютере |
| `@MacBook-Pro` | Имя компьютера |
| `my-project` | Папка, в которой ты сейчас находишься |
| `%` или `$` | Приглашение ввести команду (символ, не часть команды) |

Знак `%` (на современном Mac) или `$` (Linux и старый bash) означает «готов принять команду». Сам по себе ничего не значит, его не надо вводить, просто пиши команду рядом и жми Enter.

Тильда `~` означает домашнюю папку пользователя (на Mac это `/Users/имя`). Видишь `~` в строке — значит, ты «дома».

### Пять команд, которых хватит на старте

```bash
pwd        # покажи, в какой папке я сейчас
ls         # покажи файлы в текущей папке
cd папка   # войди в папку
cd ..      # поднимись на уровень выше (.. это родительская папка)
mkdir имя  # создай новую папку
```

Этого набора хватит на первые недели. Остальное добавим, когда понадобится.

### Установить Node.js

1. Открой [nodejs.org](https://nodejs.org/).
2. На главной увидишь две кнопки: LTS и Current. Всегда LTS, это стабильная версия.
3. Скачай установщик под свою систему: `.pkg` для Mac, `.msi` для Windows.
4. Запусти, на каждом шаге жми «Continue» / «Next», все галочки оставляй по умолчанию.
5. После установки перезапусти терминал. Закрой и открой заново или пересоздай вкладку в VS Code.

### Проверить, что всё встало

В новом терминале:

```bash
node -v
```

Должно вывести что-то вроде `v22.14.0`. Цифры могут отличаться, главное — версия выводится без ошибки.

Дальше:

```bash
npm -v
```

Тоже должна показаться версия, например `10.9.2`.

Видишь обе версии — Node и npm у тебя стоят. Котёнок, поздравляю, самая страшная часть установочного модуля позади.

Если получаешь `command not found`, перезапусти терминал и повтори. Если и после этого не работает, удали Node и поставь заново с официального сайта.

## Пример кода

```bash
# Навигация по папкам — попробуй прямо сейчас
pwd                    # где я?
ls                     # что здесь есть?
cd Desktop             # войти в папку Desktop
mkdir test-project     # создать новую папку
cd test-project        # войти в неё
pwd                    # убедиться, что мы там
cd ..                  # выйти обратно на Desktop
```

```bash
# Проверка, что Node и npm установились
node -v     # v22.x.x — OK
npm -v      # 10.x.x — OK
```

## Частые ошибки

- Не перезапустить терминал после установки Node. Терминал считывает доступные программы один раз при запуске. Новая программа становится «видна» только после перезапуска.
- Скачать Current вместо LTS. Current это нестабильная ветка. Для работы и обучения нужна LTS.
- Паниковать при виде красного текста. Любая команда, которая чего-то не нашла, ругается красным. Прочитай ошибку: обычно прямо в ней написано, что не так («command not found», «no such file or directory»). Это диагностика, а не катастрофа.
- Скопировать команду вместе с символом `%` или `$`. Это приглашение командной строки, его в команду включать не надо.
