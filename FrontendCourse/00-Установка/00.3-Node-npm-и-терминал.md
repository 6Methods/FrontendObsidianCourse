# Node, npm и первый запуск терминала

## Зачем это нужно

Без Node.js не получится установить ни один современный фреймворк — ни React, ни Vite, ни Tailwind. Все эти инструменты ставятся через npm, который идёт вместе с Node. А для работы с npm нужен терминал. Поэтому три темы — в одном уроке: поставили Node, разобрались с терминалом, проверили, что всё работает.

## Главное

### Часть 1: Что такое терминал (и почему это не страшно)

Котёнок, терминал — это просто **другой способ общаться с компьютером**. Вместо того чтобы кликать мышкой по иконкам, ты пишешь команды текстом.

Представь Finder (Mac) или Проводник (Windows) — программу, где ты ходишь по папкам, создаёшь файлы, переименовываешь их. Терминал делает ровно то же самое, только текстом. Никакой магии.

Почему разработчики используют терминал, а не Finder?
- Быстрее: одна команда делает то, на что ушло бы 10 кликов.
- Мощнее: некоторые действия вообще нет смысла делать через интерфейс (установка npm-пакетов, запуск сервера).
- Стандарт: все инструкции в документации написаны для терминала.

Ты **не можешь случайно сломать компьютер** обычными командами навигации и установки. Удалить важный файл можно только если написать `rm` и явно указать его путь. Простые команды — безопасны.

---

### Часть 2: Как открыть терминал

**На Mac:**
- Способ 1: `Cmd + Space` → напечатай «Terminal» → Enter. Откроется приложение Terminal.
- Способ 2: Прямо в VS Code. Нажми `` Ctrl+` `` (обратный апостроф, та же клавиша что `~`). Внизу откроется встроенный терминал. Это удобнее всего — не нужно переключаться между окнами.

**На Windows:**
- Способ 1: Поиск → «Windows Terminal» или «PowerShell».
- Способ 2: В VS Code: `` Ctrl+` `` — так же, как на Mac.

Для курса используй **терминал внутри VS Code** — он всегда открывается в папке текущего проекта, что очень удобно.

---

### Часть 3: Анатомия командной строки

Когда откроешь терминал, увидишь примерно такую строку:

```
имя@компьютер папка %
```

Например:

```
katerina@MacBook-Pro my-project %
```

Разберём по частям:

| Часть | Что означает |
|---|---|
| `katerina` | Имя пользователя на компьютере |
| `@MacBook-Pro` | Имя компьютера |
| `my-project` | Папка, в которой ты сейчас находишься |
| `%` или `$` | Приглашение ввести команду (просто символ, не часть команды) |

Символ `%` (Mac/zsh) или `$` (Linux/старый bash) означает «готов принять команду». Его не нужно вводить — просто пиши команду и нажимай Enter.

Тильда `~` означает домашнюю папку пользователя (на Mac это `/Users/твоёимя`). Когда видишь `~` — ты дома.

### Часть 4: Пять команд, которые нужно знать прямо сейчас

```bash
pwd        # покажи, в какой папке я сейчас нахожусь
ls         # покажи список файлов в текущей папке
cd название-папки   # войди в папку
cd ..      # выйди на уровень выше (.. = родительская папка)
mkdir название      # создай новую папку
```

Этих пяти хватит на первое время. Всё остальное — по необходимости.

---

### Часть 5: Установить Node.js

1. Открой [nodejs.org](https://nodejs.org/).
2. Ты увидишь две кнопки: **LTS** и **Current**. Всегда выбирай **LTS** — это «Long Term Support», стабильная версия для работы.
3. Скачай установщик под свою систему (`.pkg` для Mac, `.msi` для Windows).
4. Запусти установщик, нажимай «Continue» / «Next» на всех шагах — всё оставляй по умолчанию.
5. После установки **перезапусти терминал** (закрой и открой заново, или закрой вкладку в VS Code и открой новую).

---

### Часть 6: Проверить, что всё установилось

После перезапуска терминала напечатай:

```bash
node -v
```

Должно вывести что-то вроде `v22.14.0` (цифры могут отличаться — это нормально, главное что версия есть).

Потом проверь npm:

```bash
npm -v
```

Должно вывести что-то вроде `10.9.2`.

Видишь версии? Отлично — Node и npm установлены и готовы к работе. Котёнок, поздравляю: самая страшная часть позади.

Если видишь `command not found` — перезапусти терминал и попробуй ещё раз. Если не помогло — удали Node и установи заново с официального сайта.

## Пример кода

```bash
# Навигация по папкам — попробуй прямо сейчас
pwd                    # где я?
ls                     # что здесь есть?
cd Desktop             # войти в папку Desktop
mkdir test-project     # создать новую папку
cd test-project        # войти в неё
pwd                    # убедиться, что мы там
cd ..                  # выйти обратно на Desktop
```

```bash
# Проверка установки Node.js и npm
node -v     # v22.x.x — OK
npm -v      # 10.x.x — OK
```

## Частые ошибки

- **Не перезапустить терминал после установки Node** — терминал загружается один раз при открытии. Новые программы он «видит» только при перезапуске.
- **Выбрать Current вместо LTS** — Current может содержать нестабильные фичи. Для обучения и работы нужна LTS.
- **Паника при виде ошибки в терминале** — любая команда, которая не нашла нужного файла, выдаёт красный текст. Прочитай ошибку: обычно там написано, что именно не так (`command not found`, `no such file or directory`). Это не значит «что-то сломалось», это значит «проверь путь или имя команды».
- **Вводить `%` или `$` как часть команды** — это приглашение командной строки, его не нужно копировать вместе с командой.
