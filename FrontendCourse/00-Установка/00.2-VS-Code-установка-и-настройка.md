# VS Code: установка и настройка

## Зачем это нужно

В VS Code ты проведёшь тысячи часов. Один раз настроить его в начале означает не отвлекаться на неудобства потом. Технически это похоже на первый запуск Figma: выбрал тему, поставил плагины, поехали.

## Главное

### Скачать и поставить

Открой [code.visualstudio.com](https://code.visualstudio.com/). Сайт сам определит твою систему и предложит правильный установщик. Скачается `.dmg` для Mac или `.exe` для Windows.

На Mac открываешь `.dmg` и перетаскиваешь VS Code в папку Applications. Всё.

На Windows запускаешь `.exe`, в установщике оставляешь все галочки по умолчанию. Одну дополнительную поставить полезно: «Add to PATH». Она позволит открывать VS Code из терминала командой `code .`. Если забудешь — не критично, переживаемо.

### Где что лежит в интерфейсе

Первое окно после запуска выглядит примерно так. Вот карта.

```
┌─────────────────────────────────────────────────────────┐
│  Файловое дерево    │  Редактор (где пишешь код)         │
│  (список файлов     │                                    │
│   твоего проекта)   │                                    │
│                     │                                    │
│                     ├────────────────────────────────────│
│                     │  Терминал (внизу, открывается      │
│                     │  через Ctrl+` или Cmd+`)           │
└─────────────────────────────────────────────────────────┘
  ↑ Иконки слева: файлы, поиск, Git, расширения
```

Файловое дерево слева показывает все файлы проекта. Открыл папку — увидел структуру.

Редактор по центру это место, где ты пишешь код. Кликаешь на файл слева, он открывается здесь.

Терминал внизу открывается сочетанием `` Ctrl+` `` (Windows) или `` Cmd+` `` (Mac). Сюда мы будем вводить команды в следующем уроке.

### Тема

Меню Code → Settings → Color Theme на Mac или File → Preferences → Color Theme на Windows. Сочетание клавиш: `Cmd+K` потом `Cmd+T`.

Из тёмных стандарт — One Dark Pro, Dracula, Tokyo Night. Из светлых — GitHub Light, Quiet Light. Это вкусовщина и на обучение никак не влияет, ставь то, на что приятно смотреть.

### Расширения

Иконка квадратиков в левой панели или сочетание `Cmd+Shift+X`. Это магазин расширений.

Сейчас ставим только два.

Prettier — автоматически форматирует код при сохранении файла. Без него HTML с неровными отступами, с ним всегда опрятно. В поиске ищи `Prettier - Code formatter`, автор Prettier, кнопка Install.

ESLint — подчёркивает ошибки в JavaScript ещё до запуска. Похоже на проверку орфографии, только для кода. Ищи `ESLint`, автор Microsoft.

Больше пока ничего не ставим. Остальные расширения добавим, когда они понадобятся.

### Одна важная настройка: Format on Save

Открой настройки сочетанием `Cmd+,` или `Ctrl+,`. В поиске введи `format on save`. Поставь галочку у пункта «Editor: Format On Save».

Теперь при каждом сохранении (`Cmd+S` / `Ctrl+S`) Prettier приводит код в порядок автоматически. Очень удобно: про отступы и пробелы можно вообще не думать.

### Открыть первый проект

File → Open Folder, либо просто перетащи папку в окно VS Code. Один важный нюанс: VS Code работает не с отдельными файлами, а с папками целиком. Всегда открывай папку проекта, тогда слева видна вся структура и автодополнение работает нормально.

## Пример кода

```html
<!-- Создай файл index.html в любой папке, открой в VS Code -->
<!-- Напечатай ! и нажми Tab — VS Code сам вставит шаблон: -->

<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8" />
    <title>Мой первый файл</title>
  </head>
  <body>
    <h1>Привет!</h1>
  </body>
</html>
```

Эта магия называется Emmet. Она встроена в VS Code и расширяет короткие аббревиатуры в полный код. `!` превращается в HTML-шаблон. `div.card` превращается в `<div class="card"></div>`. Очень экономит время на вёрстке.

## Частые ошибки

- Открывать отдельные файлы вместо папки. В таком режиме не видно дерево, автодополнение работает хуже. Всегда открывай папку проекта.
- Поставить десяток расширений сразу. VS Code начинает тормозить, и непонятно, какое из них что делает. Двух-трёх для старта достаточно.
- Не включить Format on Save. Потом будешь руками поправлять отступы, и это будет раздражать. Включи сейчас, пока не забыл.
- Забывать сохранять файл. По умолчанию VS Code не сохраняет автоматически. Можно включить Auto Save в настройках, но проще выработать рефлекс `Cmd+S` / `Ctrl+S`.
