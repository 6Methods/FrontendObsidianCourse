# VS Code — установка и настройка

## Зачем это нужно

VS Code — это редактор, в котором ты проведёшь тысячи часов. Правильно настроить его один раз в начале означает не отвлекаться на неудобства потом. Тут нет ничего сложного — по сути, это как первый запуск Figma: выбрать тему, поставить нужные плагины.

## Главное

### Шаг 1: Скачать и установить

Перейди на [code.visualstudio.com](https://code.visualstudio.com/).

Сайт автоматически определит твою операционную систему — просто нажми большую синюю кнопку «Download». Скачается `.dmg` (Mac) или `.exe` (Windows).

**Mac:** открой `.dmg`, перетащи VS Code в папку Applications. Готово.

**Windows:** запусти `.exe`, пройди через установщик, оставь все галочки по умолчанию. Одну важную галочку можно поставить дополнительно: «Add to PATH» — она позволит открывать VS Code из терминала командой `code .`. Если не поставил — не критично, можно без неё.

---

### Шаг 2: Первый запуск — ориентация в интерфейсе

Когда откроешь VS Code, увидишь несколько зон. Котёнок, вот карта:

```
┌─────────────────────────────────────────────────────────┐
│  Файловое дерево    │  Редактор (где пишешь код)         │
│  (список файлов     │                                    │
│   твоего проекта)   │                                    │
│                     │                                    │
│                     ├────────────────────────────────────│
│                     │  Терминал (внизу, открывается      │
│                     │  через Ctrl+` или Cmd+`)           │
└─────────────────────────────────────────────────────────┘
  ↑ Иконки: файлы, поиск, Git, расширения
```

Три части, которые используются каждый день:
- **Файловое дерево** (слева) — список всех файлов проекта. Открываешь папку проекта, видишь её структуру.
- **Редактор** (центр) — там пишешь код. Открываешь файл — он появляется здесь.
- **Терминал** (внизу) — встроенный терминал. Открывается через `` Ctrl+` `` (Windows) или `` Cmd+` `` (Mac). Сюда мы будем вводить команды.

---

### Шаг 3: Выбрать тему

Меню → **Code → Settings → Color Theme** (Mac) или **File → Preferences → Color Theme** (Windows). Или нажми `Cmd+K → Cmd+T`.

Популярные тёмные темы: **One Dark Pro**, **Dracula**, **Tokyo Night**. Светлые: **GitHub Light**, **Quiet Light**. Это вкусовщина — поставь, что нравится. На обучение не влияет.

---

### Шаг 4: Нужные расширения (Extensions)

Нажми иконку квадратиков в левой панели (или `Cmd+Shift+X`) — откроется магазин расширений.

Поставь эти два сейчас:

**Prettier — Code formatter**
> Автоматически форматирует твой код при сохранении файла. Без него HTML с неровными отступами, с ним — всё опрятно.
Найди в поиске: `Prettier - Code formatter` (автор: Prettier). Нажми Install.

**ESLint**
> Подчёркивает ошибки в JavaScript-коде ещё до запуска. Как проверка орфографии, только для кода.
Найди: `ESLint` (автор: Microsoft). Нажми Install.

Больше ничего ставить сейчас не нужно. Остальные расширения будем добавлять по мере необходимости.

---

### Шаг 5: Одна важная настройка — «сохранять и форматировать»

Открой настройки: `Cmd+,` (Mac) или `Ctrl+,` (Windows).

В поиске введи `format on save`. Увидишь чекбокс **Editor: Format On Save** — поставь галочку.

Теперь каждый раз при сохранении файла (`Cmd+S` / `Ctrl+S`) Prettier будет автоматически приводить код в порядок. Очень удобно — не нужно думать об отступах и пробелах вручную.

---

### Шаг 6: Открыть первый проект

В VS Code: **File → Open Folder** (или перетащи папку прямо в окно). VS Code работает с папками, а не с отдельными файлами — открывай всегда папку проекта целиком, чтобы видеть всю структуру в дереве слева.

## Пример кода

```html
<!-- Создай файл index.html в любой папке, открой в VS Code -->
<!-- Напечатай ! и нажми Tab — VS Code автоматически вставит шаблон: -->

<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8" />
    <title>Мой первый файл</title>
  </head>
  <body>
    <h1>Привет!</h1>
  </body>
</html>
```

Это называется Emmet — встроенный в VS Code инструмент, который расширяет короткие аббревиатуры в полный код. `!` → полный HTML-шаблон, `div.card` → `<div class="card"></div>`. Очень экономит время.

## Частые ошибки

- **Открывать отдельные файлы вместо папки** — тогда не видно дерево файлов, автодополнение работает хуже. Всегда открывай папку целиком.
- **Ставить десятки расширений сразу** — VS Code замедляется, и непонятно, какое расширение что делает. Два-три — достаточно для старта.
- **Не включить Format On Save** — потом будешь мучаться с отступами вручную. Включи сейчас, пока не забыл. 
- **Забывать сохранять файл** — VS Code не сохраняет автоматически (но ты можешь включить Auto Save в настройках, если хочешь). `Cmd+S` / `Ctrl+S` — рефлекс, который нужно выработать.
