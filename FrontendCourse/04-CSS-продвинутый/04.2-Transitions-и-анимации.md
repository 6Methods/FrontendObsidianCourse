# Transitions и анимации

## Зачем это нужно
Кнопки, мгновенно меняющие цвет при hover, выглядят дёшево и напоминают сайт из нулевых. Плавный переход в 150мс — и интерфейс сразу ощущается качественным и «живым», хотя ты не добавил ни одной строчки JS. А `@keyframes` позволяют делать спиннеры, появление модалок, скролл-эффекты и даже целые мини-иллюстрации без тяжёлых JS-библиотек. По сути, это бесплатный способ поднять воспринимаемое качество продукта.

## Главное

### Transition — переход между состояниями
`transition` — это обещание браузеру: если значение свойства изменится, не меняй его скачком, а плавно интерполируй за указанное время. Синтаксис: `transition: свойство длительность easing задержка;`, например `transition: background 200ms ease;`. Применяется автоматически, как только значение меняется: через `:hover`, `:focus` или когда JS добавляет класс. Никакого ручного запуска — ты описываешь «до» и «после».

### Easing (функция кривой)
Easing отвечает за характер движения. Встроенные варианты — `ease` (по умолчанию), `linear`, `ease-in`, `ease-out`, `ease-in-out`, и для тонкой настройки — `cubic-bezier(...)` из онлайн-редактора. Для UI приятнее всего `ease-out`: быстрый старт и мягкое завершение, как будто элемент подъезжает и паркуется. `linear` оставляй для вращений и бесконечных анимаций.

### Что анимировать дёшево
`transform` (translate, rotate, scale) и `opacity` — браузер считает их на GPU отдельным слоем, не трогая раскладку. Это даёт стабильные 60 fps даже на слабом ноуте. Главное правило: для движения используй `transform: translate(...)`, а не `top`/`left`/`margin`. Для появления — `opacity`, а не `display`.

### Что анимировать дорого
`width`, `height`, `margin`, `top`/`left` заставляют браузер пересчитывать раскладку всей страницы — это reflow, и он каскадом бьёт по соседним элементам. На десктопе ты можешь этого не заметить, а на мобиле фризы гарантированы. Если очень хочется поменять размер — почти всегда можно заменить это на `transform: scale(...)`.

### `@keyframes` — многошаговые анимации
Transition умеет только «из A в B», а `@keyframes` позволяет описать последовательность кадров по процентам: `0%`, `50%`, `100%` (или короткие `from`/`to`). Запускаешь через свойство `animation: имя длительность easing iteration direction fill-mode;`. В отличие от transition, анимация стартует сразу, как только правило применяется к элементу — никакого триггера не нужно.

### `animation-iteration-count`, `animation-fill-mode`
`iteration-count: infinite` — анимация крутится вечно, классика для спиннеров. `fill-mode: forwards` — после окончания элемент остаётся в конечном состоянии, иначе он скакнёт обратно к исходному виду и появится неприятный «щелчок». Для появления модалок и тостов `forwards` — обязательный спутник.

### `prefers-reduced-motion`
В настройках macOS, iOS, Windows и Android есть пункт «Уменьшить движение». Пользователи с вестибулярными расстройствами или мигренями его реально включают, уважай это через `@media (prefers-reduced-motion: reduce)`. Самый простой подход — в этом медиа-запросе сократить длительность всех анимаций почти до нуля.

### `will-change`
`will-change: transform` — подсказка браузеру: «готовься, этот элемент будет анимироваться, вынеси его на отдельный слой заранее». Полезно, когда анимация подтормаживает на старте. Но используй точечно: браузер выделяет под каждый слой память, и если навесить на всё подряд — получишь общий тормоз вместо оптимизации.

## Пример кода
```css
/* плавная кнопка */
.btn {
  background: #2d6cdf;
  color: white;
  padding: 12px 20px;
  border: none;
  border-radius: 8px;
  transition: background 150ms ease-out, transform 150ms ease-out;
}

.btn:hover {
  background: #1e4fa0;
  transform: translateY(-2px);
}

.btn:active {
  transform: translateY(0);
}

/* спиннер */
@keyframes spin {
  from { transform: rotate(0deg); }
  to   { transform: rotate(360deg); }
}

.spinner {
  width: 32px;
  height: 32px;
  border: 3px solid #eee;
  border-top-color: #2d6cdf;
  border-radius: 50%;
  animation: spin 800ms linear infinite;
}

/* появление модалки */
@keyframes fade-in {
  from { opacity: 0; transform: translateY(-10px); }
  to   { opacity: 1; transform: translateY(0); }
}

.modal {
  animation: fade-in 200ms ease-out forwards;
}

@media (prefers-reduced-motion: reduce) {
  *, *::before, *::after {
    animation-duration: 0.01ms !important;
    transition-duration: 0.01ms !important;
  }
}
```

## Частые ошибки
- Анимировать `width`/`height` вместо `transform: scale(...)` — фризы на мобиле и дёрганая раскладка из-за reflow.
- Transition длиннее 300мс для простых UI-переходов — интерфейс ощущается заторможенным и ленивым. 100–200мс — норма для hover и смен состояний.
- Не учитывать `prefers-reduced-motion` — для людей с вестибулярными нарушениями скачущий интерфейс это реальная физическая боль, а не эстетический вкус.
- Навешивать `will-change: transform` на все элементы «профилактически» — браузер выделяет память под каждый слой, и вместо ускорения получаешь общий тормоз страницы.
