# CSS-переменные (custom properties)

## Зачем это нужно

Представь: проект рос-рос, и фирменный синий цвет в итоге оказался прописан в 30 местах по всему CSS. Теперь дизайнер хочет чуть другой оттенок — и ты идёшь ловить эти 30 строк глазами через Find & Replace, молясь, чтобы не задеть похожий синий у ссылок. CSS-переменные убирают эту боль: значение живёт в одном месте, а используется везде через `var(...)`. Плюс бонусом открываются тематизация (светлая/тёмная), динамика из JS и основа для полноценной дизайн-системы.

## Главное

### Синтаксис

Объявление выглядит так: `--имя: значение;` — пишется внутри любого селектора, чаще всего это `:root`. Чтобы прочитать переменную, используется функция `var(--имя)`: например, `color: var(--text);`. Имя может быть любым, но обязательно начинается с двух дефисов — это и есть признак custom property. Регистр имеет значение: `--Primary` и `--primary` — две разные переменные.

### `:root` и scope

`:root` — это корень документа, по сути тот же `<html>`, только со специфичностью повыше. Если объявить переменную там, она будет доступна абсолютно везде — во всех селекторах страницы. Но CSS-переменные можно объявлять и в любом другом селекторе — тогда их увидит только сам элемент и его потомки, потому что custom properties наследуются. Это удобно, когда какая-то переменная имеет смысл только внутри компонента: например, `--card-padding` внутри `.card`.

### Fallback

У `var()` есть второй аргумент — значение по умолчанию: `var(--primary, #2d6cdf)`. Если переменная не задана или не видна в этом месте, браузер подставит `#2d6cdf`. Это спасает при постепенной миграции: можно сначала добавить `var()` с fallback по всему коду, а переменные в `:root` вводить потом — ничего не сломается.

### Тематизация (light/dark)

Классический приём: базовые значения лежат в `:root`, а тёмная тема переопределяет нужные переменные в отдельном селекторе, например `[data-theme="dark"] { --bg: #111; --text: #eee; }`. Переключение темы — это один JS-вызов, который меняет атрибут `data-theme` на `<html>`. Никаких больше перегенераций CSS или ручной перекраски каждого блока — стили переключаются сами благодаря наследованию.

### Динамика из JS

Переменные можно менять прямо в рантайме: `element.style.setProperty('--x', '200px')`. Стили, которые эту переменную используют, пересчитаются на лету. Так делают слайдеры громкости, drag-n-drop с подсветкой, эффекты, следящие за курсором — JS обновляет переменную, а CSS сам отрисовывает результат.

### Отличие от препроцессорных переменных

Sass-переменные (`$primary`) — это чистый compile-time: препроцессор подставляет их значения ещё до того, как CSS попадёт в браузер. CSS custom properties живут в рантайме: их видит браузер, DevTools и JS, их можно менять, переопределять в media queries и привязывать к атрибутам. Это принципиально разные инструменты, и `$primary` не заменяет `var(--primary)` в задачах, где нужна динамика.

### Дизайн-токены

В реальных проектах переменные не валят в кучу, а группируют по смыслу: цвета (`--color-primary`, `--color-text`), отступы (`--space-sm`, `--space-md`, `--space-lg`), шрифты (`--font-body`, `--font-heading`), радиусы и тени. Такой набор токенов и становится основой дизайн-системы — единым языком между дизайном и кодом. Позже поверх этих токенов легко строить темы, адаптив и интеграции с Tailwind.

## Пример кода

```css
:root {
  --color-primary: #2d6cdf;
  --color-text: #222;
  --color-bg: #ffffff;
  --space-md: 16px;
  --radius: 8px;
}

[data-theme="dark"] {
  --color-text: #eee;
  --color-bg: #111;
}

body {
  color: var(--color-text);
  background: var(--color-bg);
  font-family: system-ui, sans-serif;
}

.btn {
  background: var(--color-primary);
  padding: var(--space-md);
  border-radius: var(--radius);
  color: white;
}

/* локальный scope — переменная только внутри .card */
.card {
  --card-padding: 24px;
  padding: var(--card-padding);
}
```

## Частые ошибки

- Забыть два дефиса при объявлении (`-primary` вместо `--primary`) — переменная не работает, при этом никаких ошибок в консоли нет, и ты долго ищешь, почему ничего не применяется.
- Пытаться использовать Sass-переменную (`$primary`) там, где нужна рантайм-переменная — не сработает в `calc()` с JS-значениями, в медиа-запросах на лету и вообще нигде, где нужен живой браузер: Sass отработал и ушёл.
- Сваливать всё подряд в `:root` без группировки и префиксов — через полгода палитра превращается в кашу из `--blue`, `--blue2`, `--new-blue`, в которой уже никто не разберётся.
- Переопределять переменные на каждом селекторе «на всякий случай» — это ломает наследование, делает темы непредсказуемыми и сводит на нет всю пользу от единой точки правды.

## Задания

- [ ] В `style.css` резюме вынеси палитру (минимум 4 цвета) и отступы в переменные на `:root`.
- [ ] Добавь переключение `data-theme="dark"` на `<html>` — через кнопку или вручную через DevTools. Проверь, что все цвета переключаются одним свойством.
- [ ] Через DevTools → Elements → Styles → `:root` отредактируй переменную вживую и посмотри, как обновляется страница.
- [ ] Проверь `element.style.setProperty('--color-primary', 'crimson')` в консоли — убедись, что работает.
