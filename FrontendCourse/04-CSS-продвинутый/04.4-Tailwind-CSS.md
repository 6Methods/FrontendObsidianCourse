# Tailwind CSS

## Зачем это нужно
Писать CSS по-старому — это постоянно придумывать классы, держать файл стилей рядом с HTML и следить, чтобы `.card` у тебя и `.card` у коллеги не конфликтовали. Tailwind переворачивает подход: вместо `.card { padding: 16px; }` ты пишешь `<div class="p-4">` прямо в разметке. Звучит странно и даже немного дико на первый раз, но это самый популярный CSS-инструмент последних трёх лет. Его любят за скорость разработки и консистентность результата. Для pet-проектов, MVP и React-приложений Tailwind — стандарт де-факто.

## Главное

### Utility-first идея
Основная мысль Tailwind простая: каждый класс отвечает ровно за одно CSS-свойство. `p-4` превращается в `padding: 16px;`, `flex` — в `display: flex;`, `text-lg` — в `font-size: 18px;`. Ты не пишешь CSS в отдельном файле, а комбинируешь готовые утилиты прямо в HTML. Имена классов придумывать больше не нужно: всё уже названо за тебя.

### Почему это не плохо, хотя выглядит ужасно
На первый взгляд длинная строка классов пугает, но за ней прячется четыре плюса. Ты не тратишь энергию на имена вроде `.user-card__title--highlighted`, стили не «утекают» на соседние компоненты, при удалении компонента его стили исчезают вместе с ним, а `p-4`, `p-6`, `p-8` дают согласованную шкалу вместо «вчера 14px, сегодня 15px».

### Установка
Для проекта с Vite или Next.js (стек разберём в модулях 09 и 10) всё делается за пару команд: `npm i -D tailwindcss`, затем `npx tailwindcss init`. В главный CSS добавляешь три директивы: `@tailwind base; @tailwind components; @tailwind utilities;`. После этого классы начинают работать сразу.

### `tailwind.config.js` — твоя дизайн-система
Этот файл — сердце проекта. Здесь ты переопределяешь палитру, брейкпоинты, шрифты, радиусы. Написал `theme.extend.colors.primary = "#2d6cdf"`, и заработают `bg-primary`, `text-primary`, `border-primary`. По сути это и есть дизайн-токены, о которых ты уже знаешь по CSS-переменным.

### Responsive префиксы
`md:flex` означает «применить стиль начиная с брейкпоинта `md` (768px) и выше». Tailwind устроен mobile-first: сначала описываешь вид для узкого экрана, а префиксы `md:` и `lg:` дорисовывают поведение на больших. Пример: `<div class="flex-col md:flex-row">` — колонка на мобилке, ряд на десктопе.

### State-префиксы
Всё, что раньше было псевдо-классами, в Tailwind превращается в префикс. `hover:bg-blue-600`, `focus:ring-2`, `disabled:opacity-50`, `dark:bg-slate-900` читается на лету и не требует отдельного файла. Dark mode, наведение, фокус, `disabled` — всё рядом с остальными классами.

### `@apply` — когда очень хочется именованный класс
Иногда всё-таки хочется вернуть старый добрый `.btn-primary`. Для этого есть директива `@apply`: `.btn-primary { @apply px-4 py-2 bg-blue-500 text-white rounded; }`. Пользуйся ей редко и только для реально повторяющихся сложных комбинаций. Если заворачиваешь в `@apply` всё подряд — это просто старый CSS в обёртке.

### Когда использовать
Pet-проекты, MVP, React- и Next.js-приложения, landing-страницы с жёстким дедлайном. Идеальные кандидаты. В модулях 10 и 13 будем делать React-приложения именно с Tailwind.

### Когда НЕ использовать
Сильно кастомный дизайн, где утилиты не ложатся на макет, но чаще это сигнал настроить config. Корпоративные дизайн-системы с жёсткими токенами тоже не противопоказание: их можно описывать в `tailwind.config.js`.

## Пример кода

```html
<!-- карточка товара на Tailwind -->
<article class="max-w-sm rounded-lg bg-white p-6 shadow-md hover:shadow-lg transition-shadow dark:bg-slate-800">
  <h2 class="text-xl font-semibold text-slate-900 dark:text-white">
    Название товара
  </h2>
  <p class="mt-2 text-sm text-slate-600 dark:text-slate-300">
    Короткое описание в две строки.
  </p>
  <div class="mt-4 flex items-center justify-between">
    <span class="text-lg font-bold text-blue-600">1 990 ₽</span>
    <button class="rounded-md bg-blue-600 px-4 py-2 text-white hover:bg-blue-700 focus:outline-none focus:ring-2 focus:ring-blue-400">
      Купить
    </button>
  </div>
</article>

<!-- адаптивная сетка -->
<div class="grid gap-4 grid-cols-1 md:grid-cols-2 lg:grid-cols-3">
  <!-- карточки -->
</div>
```

```js
// tailwind.config.js — пример кастомизации
/** @type {import('tailwindcss').Config} */
export default {
  content: ["./index.html", "./src/**/*.{js,ts,jsx,tsx}"],
  theme: {
    extend: {
      colors: {
        primary: "#2d6cdf",
        accent: "#ffb347",
      },
      fontFamily: {
        sans: ["Inter", "system-ui", "sans-serif"],
      },
    },
  },
  plugins: [],
};
```

## Частые ошибки
- Использовать `@apply` везде подряд → возвращаешься к старому CSS, теряешь преимущества Tailwind.
- Не настраивать `tailwind.config.js` под свой дизайн → используешь дефолтные цвета и получаешь «bootstrap-look».
- Забывать про `content: [...]` в config → Tailwind не сканирует твои файлы и в prod-билде классы выпиливаются как «неиспользуемые».
- Писать «магические» строки типа `class="w-[237px] text-[13.5px]"` всегда → arbitrary-значения норм для исключений, но если ты их ставишь постоянно, твоя шкала не настроена.

## Задания
- [ ] Создай отдельную папку, поставь Tailwind по [официальному гайду](https://tailwindcss.com/docs/installation). Убедись, что `<h1 class="text-4xl font-bold text-blue-600">Hello</h1>` работает.
- [ ] Свёрстай одну карточку (как в примере выше) и одну адаптивную сетку карточек (`grid-cols-1 md:grid-cols-2 lg:grid-cols-3`).
- [ ] В `tailwind.config.js` добавь кастомный цвет `primary` и используй `bg-primary text-primary` в разметке.
- [ ] Пройди официальный [Tailwind Play](https://play.tailwindcss.com/) — sandbox, можно без установки потренироваться.
