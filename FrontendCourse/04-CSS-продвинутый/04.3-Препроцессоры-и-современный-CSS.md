# Препроцессоры и современный CSS

## Зачем это нужно

Исторически CSS был бедным языком: ни переменных, ни вложенности, ни функций, нельзя было разбить стили на модули по-человечески. Чтобы выжить, фронтенд придумал препроцессоры — Sass, Less, Stylus. Они позволяют писать свой расширенный синтаксис, а потом компилируют его в обычный CSS, который уже понимает браузер. Сегодня многое из того, ради чего их придумывали, завезли прямо в нативный CSS, и роль препроцессоров постепенно убывает. Но они всё ещё живут в легаси-кодобазах и больших дизайн-системах, так что натыкаться на `.scss`-файлы ты будешь регулярно. Урок обзорный: чтобы ты не пугался чужого кода и понимал, когда препроцессор реально нужен, а когда это дань привычке.

## Главное

### Что такое препроцессор

Препроцессор — это инструмент, который позволяет писать «расширенный CSS» на своём синтаксисе, а потом билдит его в обычный `.css`, который понимает браузер. По сути, это ещё один шаг сборки: ты пишешь `.scss`, запускаешь компилятор, получаешь `.css`. Самый популярный сегодня — **Sass** (расширение `.scss`).

### Что даёт Sass

Набор возможностей, которых либо не было в CSS, либо они появились недавно: переменные (`$primary: blue;`), вложенность селекторов (nesting), миксины (переиспользуемые блоки стилей, похожие на функции), наследование через `@extend`, разбиение на файлы через partials и `@use`, математические операции. Всё это компилируется в плоский CSS — браузер ничего не знает про Sass, ему прилетает обычный результат.

### Синтаксис SCSS — быстрый обзор

SCSS выглядит почти как обычный CSS, только с фичами сверху. Самое частое, что ты увидишь:

- Nesting: `.card { h2 { color: blue; } &:hover { ... } }`. Селекторы можно вкладывать друг в друга, а `&` означает «родительский селектор».
- Mixin: `@mixin button($color) { background: $color; ... }`, а потом `@include button(red);`. Это как вызов функции, которая разворачивается в набор правил.
- Partial: файл `_variables.scss` не компилируется в отдельный CSS, а подключается через `@use "variables";`. Так удобно разбивать проект на модули.

### CSS Nesting нативный

С 2023 года все современные браузеры (Chrome 112+, Safari 16.5+) умеют вложенность прямо в CSS, без препроцессоров. Синтаксис почти идентичен Sass. Это серьёзно снижает мотивацию тащить Sass в новый проект: одна из главных фич теперь есть «из коробки».

### PostCSS и autoprefixer

PostCSS — другая категория инструментов: он не придумывает новый синтаксис, а берёт обычный CSS и прогоняет его через цепочку плагинов. Самый известный плагин — **autoprefixer**: сам дорисовывает вендорные префиксы (`-webkit-`, `-moz-`) там, где они нужны. Почти во всех современных сборках (Vite, Next.js) PostCSS и autoprefixer встроены по умолчанию: ты про них можешь даже не знать.

### Когда реально нужен Sass

Ситуаций не так много. Легаси-проект, где кодобаза уже написана на Sass. Большая дизайн-система с кучей mixin-логики и генерации стилей: тут Sass всё ещё вне конкуренции. Команда привыкла к `.scss` и не хочет мигрировать. Для новых pet-проектов связка CSS custom properties + нативный nesting + Tailwind закрывает 99% задач.

### Не путай с CSS-in-JS

`styled-components`, `Emotion`, CSS Modules — это уже не препроцессоры, а отдельный мир, связанный с React. Стили пишутся прямо в JS/JSX и привязаны к компоненту. Подробно разберём в модуле 10. Сейчас просто запомни, что это другая категория.

## Пример кода

Два блока: сначала SCSS, потом тот же смысл на современном нативном CSS.

```scss
// variables.scss
$primary: #2d6cdf;
$space-md: 16px;

@mixin card {
  padding: $space-md;
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
}

.article {
  @include card;
  color: $primary;

  h2 {
    margin-top: 0;

    &:hover {
      color: darken($primary, 10%);
    }
  }

  .badge {
    background: $primary;
    color: white;
  }
}
```

```css
:root {
  --primary: #2d6cdf;
  --space-md: 16px;
}

.article {
  padding: var(--space-md);
  border-radius: 8px;
  box-shadow: 0 2px 8px rgba(0, 0, 0, 0.1);
  color: var(--primary);

  & h2 {
    margin-top: 0;

    &:hover {
      color: color-mix(in srgb, var(--primary) 80%, black);
    }
  }

  & .badge {
    background: var(--primary);
    color: white;
  }
}
```

## Частые ошибки

- Начинать новый pet-проект в 2026 с Sass «по привычке». Потеряешь время на настройку билда, а выгоды почти нет, если ты не делаешь большую дизайн-систему с сотнями mixin.
- Путать Sass-переменные и CSS custom properties. Sass-переменные — это compile-time, они превращаются в значения ещё до браузера, их не видно в DevTools и нельзя менять из JS. Custom properties (`--primary`) — настоящие переменные рантайма.
- Вкладывать селекторы на 5 уровней вглубь просто потому, что Sass позволяет: специфичность выстреливает в ногу, код становится хрупким и нечитаемым. Правило: не больше 2–3 уровней вложенности.
