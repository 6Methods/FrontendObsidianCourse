# Что мы вообще строим — контекст для дизайнеров

## Зачем это нужно

Ты уже умеешь делать красиво. Теперь тебе нужно научиться делать красиво **и работающее** — чтобы макет из Figma не умирал в папке «финал_финал_v3», а открывался в браузере и жил на настоящем сайте.

## Главное

### Ты не становишься разработчиком — ты становишься сильнее как дизайнер

Дизайнеры, которые понимают код, делают макеты, которые можно построить. Они не просят «тень 20px» там, где это технически невозможно в данном контексте. Они знают, почему кнопка ведёт себя именно так при наведении. Они говорят с разработчиком на одном языке — и их уважают.

Котёнок, тебе не нужно становиться программистом. Нужно понять, что происходит между Figma и браузером.

### Из чего состоит любой сайт: три слоя

Представь, что ты строишь комнату:

- **HTML — это каркас и мебель.** Стены, окна, стол, стул — что вообще есть в комнате и в каком порядке. Это структура. В Figma-аналогии — это твои фреймы и слои: «вот тут заголовок, вот тут картинка, вот тут кнопка».

- **CSS — это дизайн интерьера.** Цвет стен, шрифт на табличке, размер подушек, расположение всего по комнате. Именно CSS отвечает за то, что сайт выглядит так, а не иначе. В Figma ты уже делаешь CSS — просто руками мышкой, а не строчками текста.

- **JavaScript — это поведение.** Дверь открывается при нажатии, свет включается по расписанию, форма проверяет, заполнено ли поле. JS делает сайт живым и интерактивным. Для начала это не главное — сначала HTML и CSS.

### В этом курсе мы идём именно в этом порядке

Сначала HTML — разбираемся со структурой. Потом CSS — стилизуем. Потом JS — добавляем жизнь. Потом React — строим большие приложения из маленьких кусочков. Каждый шаг опирается на предыдущий.

Хорошая новость: ты уже думаешь правильно. Когда ты делаешь в Figma фрейм с заголовком, текстом и кнопкой внутри — ты уже думаешь HTML-структурой. Когда ты задаёшь padding, gap и border-radius — ты уже пишешь CSS, только визуально.

### Чего ожидать от первых недель

Поначалу будет ощущение «я всё понял, но руки не слушаются». Это нормально — так работает любой новый инструмент. Figma тоже не казалась простой в первые дни. Главное — не читать пассивно, а открывать VS Code параллельно и пробовать сразу.

## Пример кода

```html
<!-- Вот простейшая страница. Сравни с фреймом в Figma -->

<!DOCTYPE html>
<html>
  <head>
    <title>Моя первая страница</title>
  </head>
  <body>
    <h1>Привет, я заголовок</h1>
    <p>А я — абзац с текстом.</p>
    <button>Нажми меня</button>
  </body>
</html>
```

Открой этот код в браузере — ты увидишь три элемента без стилей. Некрасиво? Да. Но структура уже есть — и это уже HTML.

## Частые ошибки

- **«Мне не нужен код, я дизайнер»** — дизайнеры без понимания кода рисуют неосуществимые макеты и теряют в споре с разработчиком. Даже базовые знания меняют всё.
- **«Сначала изучу всё, потом начну делать»** — так не работает. Нужно сразу открыть файл и написать хоть одну строчку.
- **«Это слишком сложно, я не технический человек»** — HTML — это не математика и не алгоритмы. Это теги с текстом. Серьёзно.

## Задания

- [ ] Открой любой понравившийся сайт, нажми правой кнопкой → «Просмотр кода страницы» (или «View Page Source»). Просто посмотри — не нужно понимать всё сразу. Найди слово `<body>` и `<h1>`. Почувствуй, что это не магия.
- [ ] Создай пустой файл `index.html` на рабочем столе, вставь туда пример кода выше и открой в браузере двойным кликом.
