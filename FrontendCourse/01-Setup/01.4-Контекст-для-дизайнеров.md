# Что мы вообще строим — контекст для дизайнеров

## Зачем это нужно

Ты уже умеешь делать красиво. Дальше нужно научиться делать красиво и работающее: чтобы макет из Figma не умирал в папке «финал_финал_v3», а открывался в браузере у живых людей.

## Главное

### Ты не становишься разработчиком, ты становишься сильнее как дизайнер

Когда дизайнер хотя бы немного понимает код, ему перестают возвращать макеты с пометкой «нереализуемо». Он не просит тень и анимацию там, где они сломают вёрстку, а если просит, то осознанно. На созвоне с разработчиком они говорят на одном языке, и компромиссы рождаются за пять минут вместо двух часов.

Котёнок, тебе не нужно становиться программистом. Нужно понять, что происходит в промежутке между Figma и браузером.

### Из чего состоит любой сайт

Представь, что ты строишь комнату.

HTML — это каркас и мебель. Что вообще стоит в комнате и в каком порядке: стены, окна, стол, стул. В терминах Figma это панель слоёв: вот заголовок, вот картинка, вот кнопка.

CSS — это интерьер. Цвет стен, обивка стула, расположение всего на своих местах. Сайт выглядит так, а не иначе, именно из-за CSS. В Figma ты уже делаешь CSS, просто мышкой: задал fill, padding, gap — это и есть стили.

JavaScript — это поведение. Дверь открывается при нажатии, по таймеру включается свет, форма проверяет, что email написан правильно. JS оживляет страницу. Для старта он не нужен, до него мы дойдём ближе к середине курса.

### В этом порядке мы и идём

HTML, потом CSS, потом JavaScript, потом React. Каждый следующий слой опирается на предыдущий. Без HTML стилизовать нечего, без CSS не выглядит, без JS ничего не реагирует, а React это просто способ собирать большой интерфейс из маленьких повторяющихся блоков.

Хорошая новость: ты уже думаешь правильно. Когда ты в Figma складываешь фрейм из заголовка, текста и кнопки, ты буквально проектируешь HTML-структуру. Когда настраиваешь padding и border-radius, пишешь CSS, просто визуальным редактором.

### Чего ждать от первых недель

Будет состояние «я всё понял, но руки не слушаются». Это нормально для любого нового инструмента. Figma тоже не казалась лёгкой в первый день. Помогает простое правило: не читай курс пассивно, держи открытым VS Code и сразу пробуй то, что только что прочитал. Десять минут практики после параграфа дают больше, чем час чтения подряд.

## Пример кода

```html
<!-- Простейшая страница. Сравни с любым фреймом в Figma -->

<!DOCTYPE html>
<html>
  <head>
    <title>Моя первая страница</title>
  </head>
  <body>
    <h1>Привет, я заголовок</h1>
    <p>А я абзац с текстом.</p>
    <button>Нажми меня</button>
  </body>
</html>
```

Открой этот файл в браузере. Увидишь три голых элемента без всякого дизайна. Некрасиво? Очень. Но структура уже есть, и она и называется HTML.

## Частые ошибки

- «Мне не нужен код, я дизайнер». Дизайнеры без понимания кода рисуют макеты, которые невозможно собрать в разумные сроки, а потом проигрывают любой технический спор с разработчиком. Даже базовый уровень кардинально меняет ситуацию.
- «Сначала изучу всё, потом начну делать». В коде так не работает. Открывай файл и пиши, даже если непонятно. Понимание приходит после, а не до.
- «Это слишком сложно, я не технический человек». HTML это не математика. Это теги, в которых лежит текст. Без шуток.

## Задания

- [ ] Открой любой сайт, который тебе нравится. Правый клик → «Просмотр кода страницы» (View Page Source). Не пытайся понять всё, просто найди слова `<body>` и `<h1>`. Это снимет страх.
- [ ] Создай файл `index.html` на рабочем столе, вставь туда пример из урока, открой в браузере двойным кликом. Поздравляю, твой первый сайт.
