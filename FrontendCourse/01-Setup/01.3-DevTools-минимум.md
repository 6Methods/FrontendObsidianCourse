# DevTools — минимум, чтобы начать

## Зачем это нужно

DevTools — это второй редактор фронтендера после VS Code. Если VS Code — это место, где ты пишешь код, то DevTools — это место, где ты смотришь, что этот код делает в живом браузере. Без них ты в принципе не сможешь отладить ни стили, которые вдруг поехали, ни JS, который молча ничего не делает. Сейчас разбираемся на базовом уровне — просто чтобы ты не боялся открывать эту панель. Глубже (Performance, Sources, брейкпоинты, снапшоты памяти) — в модуле 08.

## Главное

### 1. Как открыть DevTools

Самый быстрый способ — нажать `F12`. На маке работает сочетание `Cmd + Opt + I`. Ещё вариант для тех, кто забыл шорткаты: правый клик по любому месту страницы → «Inspect» или «Проверить» (зависит от языка браузера). Откроется большая панель — либо снизу, либо справа. Её положение можно менять в настройках самой панели, если мешает.

### 2. Вкладка Elements

Это дерево DOM — то самое HTML-представление страницы, только живое и актуальное на текущий момент. Рядом обычно окошко со стилями: там видно, какие CSS-правила применились к выбранному элементу и откуда они пришли. Фишка в том, что всё это можно править прямо на странице: поменять цвет кнопки, увеличить шрифт, убрать отступ. Важно понимать — после перезагрузки (`F5`) все твои правки пропадут, они живут только в памяти браузера. Но для быстрых экспериментов это бесценно: подкрутил — посмотрел — понял, что хочешь получить, и только потом идёшь менять в коде. Ещё полезная мелочь: когда ты кликаешь на элемент в дереве, он подсвечивается на странице. Так легко понять, что за блок за что отвечает.

### 3. Вкладка Console

Сюда JS выводит сообщения и ошибки. Если страница ведёт себя странно — загляни в Console первым делом, красные строчки обычно сразу объясняют проблему. Плюс это маленькая площадка для JS: любую строчку можно набрать и сразу выполнить. Попробуй `console.log('hello')`, `1 + 1`, `document.title` — увидишь, как консоль отвечает. По сути это мини-песочница внутри уже открытой страницы.

### 4. Вкладка Network

Тут видно все запросы, которые страница шлёт: картинки, шрифты, данные от серверов. По каждому запросу можно посмотреть URL, статус (`200` — ок, `404` — не найдено, `500` — сервер сломался) и сколько времени он занял. Сейчас задача — просто открыть, удивиться количеству запросов и закрыть. Разбор по косточкам — в модуле 08.

## Пример кода

```javascript
// Открой DevTools → Console и вставь:
console.log('Привет, DevTools!');
document.title = 'Поменяли заголовок вкладки';
```

## Частые ошибки

- Думать, что правки в Elements сохраняются — они пропадут после F5.
- Игнорировать красные сообщения в Console — это ошибки, их надо читать.
- Путать DevTools разных браузеров — в Chrome/Firefox/Safari немного разные, но принципы одинаковые.

## Задания

- [ ] Открыть любой знакомый сайт и поменять в Elements цвет любого заголовка на красный.
- [ ] В Console вывести `console.log('hello')` и получить ответ.
- [ ] В Console поменять `document.title` на своё имя, убедиться, что вкладка в браузере переименовалась.
- [ ] Зайти на вкладку Network, обновить страницу и найти запрос с расширением `.html` (это главный документ).
