# Инструменты frontend-разработчика

## Зачем это нужно

До того как писать первую строчку кода, нужно поставить на компьютер рабочее окружение — без него курс физически не пройти. Всё, что мы перечислим ниже, — это стандарт индустрии: примерно такой же набор у джуна в студии, у сеньора в продукте и у фрилансера, который клепает лендинги из дома.

## Главное

### Редактор кода — VS Code

VS Code — это бесплатный редактор от Microsoft, в котором пишет фронтенд код примерно полмира. Можно думать о нём как о «Word для кода»: подсветка синтаксиса, автодополнение, поиск по проекту, встроенный терминал и магазин расширений. Скачать: [code.visualstudio.com](https://code.visualstudio.com/).

По ходу курса мы поставим пару полезных расширений: **Prettier** (автоформатирование кода, чтобы он выглядел опрятно), **ESLint** (подсвечивает ошибки и плохие паттерны до запуска) и **GitLens** (делает историю Git-а нагляднее). Сейчас их не настраиваем — просто запомни названия, вернёмся к ним позже.

### Браузер — Chrome или Firefox

Браузер для разработчика — это не просто «где открывать сайт». Это инструмент отладки: внутри каждого нормального браузера есть DevTools, где можно ковыряться в HTML, смотреть CSS, ловить JS-ошибки и замерять скорость. Хорошо иметь оба — Chrome и Firefox: один и тот же сайт в них может выглядеть чуть по-разному, и это как раз то, что называется **кроссбраузерностью**. Лучше ловить такие расхождения у себя на ноутбуке, чем читать про них в сообщении от заказчика.

### Node.js и npm

Исторически JavaScript жил только в браузере. **Node.js** — это рантайм, который позволяет запускать JS ещё и на твоём компьютере, как обычную программу. Фронтендеру он нужен не для того, чтобы писать бэкенд, а для инструментов: сборщиков, линтеров, локальных серверов, CLI-утилит, скриптов для проекта. Скачивай **LTS-версию** (long-term support, самая стабильная) с [nodejs.org](https://nodejs.org/).

Вместе с Node автоматически ставится **npm** — менеджер пакетов. Через него ты будешь подтягивать чужой код (библиотеки, фреймворки) в свои проекты одной командой.

### Терминал

Терминал — это тот самый «чёрный экран», которого все боятся в фильмах про хакеров. На macOS есть встроенный Terminal, но дальше в курсе мы будем пользоваться терминалом прямо внутри VS Code — удобнее, всё в одном окне. Не пугайся: 90% работы — это 10 одних и тех же команд. Набрать их с клавиатуры быстрее, чем кликать мышкой по десятку меню.

### GitHub (аккаунт)

[GitHub](https://github.com/) — это, если совсем грубо, «облако для кода»: место, где лежат твои проекты и где их видят другие люди, в том числе потенциальные заказчики. Пока что просто зарегистрируйся — придумай вменяемый никнейм, по нему тебя будут находить. Полноценно с Git и GitHub разберёмся в модуле 08.

## Пример кода

```bash
# проверить, что Node и npm поставились
node -v   # должно вывести что-то вроде v24.x.x
npm -v    # 10.x.x или свежее
```

## Частые ошибки

- Ставить Node не с официального сайта → непонятные версии и конфликты.
- Игнорировать терминал и делать всё кликами → потом больно.
- Ставить десятки расширений сразу → замедляет VS Code, не понимаешь, что за что отвечает.
