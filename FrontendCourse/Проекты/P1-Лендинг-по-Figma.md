# P1 — Лендинг по Figma

Первый крупный проект курса. Вехa после модулей 01–04: HTML, CSS база, CSS продвинутый + Tailwind. Цель — собрать всё в руках на реальной задаче: взять дизайн-макет и превратить его в адаптивный задеплоенный сайт.

Это не упражнение, а **портфолио-работа**. По её итогам будет что показать в резюме, добавить в GitHub-профиль и отправить как пример кода.

## Что делаем

Берёшь **любой готовый макет с [Figma Community](https://www.figma.com/community/)** и один-в-один верстаешь его в код. Потом деплоишь в интернет.

### Правила выбора макета

1. **Одна страница (landing).** Если в файле 5 экранов — делаешь только главный. Цель — лендинг, а не мульти-страничный сайт.
2. **Минимум 5 логических секций.** Обычно это Hero, Features / Benefits, About или Product, Testimonials или Pricing, CTA-форма, Footer. Чем больше — тем лучше тренировка.
3. **Есть мобильная версия в макете** — или ты готов сам адаптировать, если в макете только десктоп.
4. **Не выбирай очень «простые» макеты** (2 секции, один Hero с кнопкой) — потеряешь смысл проекта.
5. **Не бери макеты с анимациями, требующими JS-библиотек** (Framer-сцены, 3D-модели и т.п.) — это выходит за рамки проекта.
6. **Согласуй выбор со мной на созвоне** до старта вёрстки. Я подскажу, если макет слишком амбициозный или слишком мелкий.

Где искать: [Figma Community → Templates → Landing](https://www.figma.com/community/tag/landing%20page/files), [Figma Community → Websites](https://www.figma.com/community/websites).

## Стек

**На выбор** — но выбрать нужно до старта и не менять по ходу:

### Вариант A — чистый HTML + CSS
- Закрепляет ручной навык, полностью по модулям 02–03.
- CSS-переменные для палитры обязательно (из урока 04.1).
- Анимации на `transition` + `@keyframes` из урока 04.2.
- Дольше — зато понятно, что именно ты пишешь.

### Вариант B — Tailwind CSS
- Продолжение модуля 04 (урок 04.4).
- Быстрее и ближе к тому, как делают в реальных проектах.
- Кастомный `tailwind.config.js` с цветами и шрифтами из макета.
- Собираем через **Tailwind CLI** (не CDN): `npx tailwindcss -i input.css -o style.css --watch`. Сборщик Vite изучим в модуле 12, здесь он не нужен.

**Если колеблешься — бери Tailwind.** Ближе к реальной индустрии, быстрее до рабочего результата.

**Фреймворки НЕ используем** — никакого React, Next.js, Vue и т.д. Это модули 09+. Здесь — статический сайт.

## Технические требования

### HTML
- Валидный HTML5 (DOCTYPE, charset, viewport, lang, title).
- Семантика (из урока 02.3): `<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`.
- Один `<h1>` (обычно заголовок Hero). Иерархия `h2`–`h3` по секциям.
- `alt` у всех изображений (описательный, не пустой — кроме декоративных с `alt=""`).
- Если есть форма — `<label>`-ы связаны с `<input>`, `type="email"` где надо, базовая HTML-валидация (`required`, `type`).

### CSS / стили
- Mobile-first: базовые стили — для мобилки, `@media (min-width: ...)` дорисовывает широкие экраны.
- Работает на **375 px / 768 px / 1440 px** без поломок и горизонтального скролла.
- Box-sizing глобальный `border-box` + `body { margin: 0 }`.
- Если чистый CSS — палитра и отступы в `:root` как переменные.
- Если Tailwind — кастомные токены (цвета, шрифт) в `tailwind.config.js`, не дефолтный Tailwind-look.
- Плавные hover/focus на интерактивных элементах (`transition` 100–200мс).
- `prefers-reduced-motion` учтён (анимации отключаются).

### Accessibility
- Контраст текста к фону ≥ **4.5:1**.
- `:focus-visible` виден на всех интерактивных элементах (клавиатура).
- Навигация клавиатурой (Tab / Shift+Tab) проходит логично.
- Форма (если есть) — с `<label>` на каждом поле.
- Lighthouse → **Accessibility ≥ 95**.

### Performance
- Изображения оптимизированы: WebP или AVIF, ширина под реальное отображение, не 4K-PNG на 300-пиксельную карточку.
- Шрифты подключены корректно (`font-display: swap`, не блокируют рендер).
- Lighthouse → **Performance ≥ 90** на прод-деплое (не на `file://`, а на URL).

### Вёрстка «по пиксель в пиксель» — нет
Задача не скопировать каждый отступ до пикселя, а **передать дух макета**: та же иерархия, те же пропорции, те же акценты. Если в макете `margin-top: 73px`, а у тебя `72px` — ок. Если секция зеркально перевёрнута — не ок.

## Этапы работы

Примерный план, по которому удобно двигаться:

### Этап 1. Подготовка (1–2 дня)
- [ ] Выбрать макет, согласовать со мной на коротком созвоне.
- [ ] Форкнуть макет в свой Figma-аккаунт.
- [ ] Разметить в Figma: сколько секций, где какие шрифты, цвета, отступы. Подсмотреть `inspect` в правой панели Figma — там готовые CSS-значения.
- [ ] Создать новый GitHub-репозиторий `landing-<короткое-имя>`.
- [ ] Инициализировать проект: `index.html`, `style.css` (или Vite + Tailwind).
- [ ] Настроить базу: reset, box-sizing, `:root` с палитрой / `tailwind.config.js` с токенами.

### Этап 2. Статика (3–5 дней)
- [ ] Сверстать каждую секцию на **мобильную** ширину (375 px) — структура и стили для узкого экрана.
- [ ] Семантика, заголовки, контент, кнопки, формы.
- [ ] Отдельные коммиты по секциям: `feat: hero section`, `feat: features section`, `feat: pricing`, и т.д.

### Этап 3. Адаптив (2–3 дня)
- [ ] Добавить `@media (min-width: 768px)` и `@media (min-width: 1200px)` (или Tailwind-префиксы `md:`, `lg:`) — раскладка на планшете и десктопе.
- [ ] DevTools → Device Mode — проверить 375 / 768 / 1024 / 1440.
- [ ] Проверить на реальном телефоне через локальную сеть (`npm run dev -- --host` в Vite, или `python -m http.server --bind 0.0.0.0`).

### Этап 4. Доводка (1–2 дня)
- [ ] Hover / focus состояния на всех интерактивных элементах.
- [ ] Анимации: плавное появление Hero, fade на секциях при скролле (опционально через `IntersectionObserver` — но если залезаешь в JS, не увлекайся).
- [ ] Изображения оптимизированы (через [Squoosh](https://squoosh.app/) или [tinypng.com](https://tinypng.com/)).
- [ ] Lighthouse — если Accessibility < 95 или Performance < 90, смотрим отчёт, правим.

### Этап 5. Деплой (0.5 дня)

Для статичного HTML/CSS-сайта самый простой и логичный способ — **GitHub Pages**: бесплатно, встроено в GitHub, никаких конфигов и аккаунтов на сторонних сервисах.

- [ ] В репозитории: **Settings → Pages → Source → Deploy from branch → main → / (root) → Save**.
- [ ] Подождать 1–2 минуты — GitHub соберёт сайт и покажет URL вида `username.github.io/landing-название`.
- [ ] Если выбирал Tailwind CLI — сначала собери финальный CSS: `npx tailwindcss -i input.css -o style.css --minify`, закоммить результат, потом пуш.
- [ ] Открой URL в браузере, убедись что всё работает.
- [ ] Lighthouse на этом URL — чтобы цифры были «честные», не на `localhost`.

> **Альтернативы:** [Timeweb Cloud](https://timeweb.cloud/) (хостинг с русской поддержкой, есть бесплатный тариф для статики), Netlify (удобен, но потребует регистрации). Vercel — для фреймворков (React, Next.js), будем использовать его в P2/P3, сейчас избыточен.

### Этап 6. README и сдача
- [ ] README-файл в репозитории: описание проекта, стек, ссылка на прод-деплой, скриншоты 375/768/1440.
- [ ] Ссылку на макет Figma (с правами view).
- [ ] Список фич: семантика, адаптив, анимации, accessibility.
- [ ] Прислать мне ссылку на репо и на прод в Telegram / на созвон.

## Чего НЕ должно быть

- React, Vue, фреймворков.
- jQuery.
- CDN-версия Tailwind в проде (работает, но огромный бандл).
- Библиотек типа AOS, Bootstrap, Foundation.
- Захардкоженных цветов в HTML (`style="color: #..."`).
- Единого файла `style.css` на 2000 строк «в одну кучу» (если чистый CSS — разбей по секциям или по смыслу).
- `!important` без обоснования.
- Таблиц для вёрстки. Никогда.

## Чего можно, но осторожно

- **Немного JS** для мобильного меню-бургера, переключателя темы, плавного скролла по якорям. Это 10–30 строк — без библиотек. Если ты пока не писал JS — можно обойтись CSS-only solution (checkbox hack для бургера).
- **Google Fonts / Fontsource** для шрифтов.
- **SVG-иконки** — свои или из [Heroicons](https://heroicons.com/), [Lucide](https://lucide.dev/).
- **Pexels / Unsplash** для фото.

## Критерии приёмки ревью

- [ ] Макет выбран осознанно, согласован, достаточно сложный.
- [ ] Сайт задеплоен, работает по HTTPS-URL.
- [ ] Lighthouse (прод-URL, desktop и mobile):
    - Accessibility ≥ 95
    - Performance ≥ 90
- [ ] Адаптив: 375 / 768 / 1440 — без горизонтального скролла и поломок.
- [ ] Семантика не формальна (не обёртка одного `<section>` на всё).
- [ ] CSS / Tailwind-токены настроены, не дефолтные.
- [ ] `:focus-visible` виден, Tab-навигация проходит логично.
- [ ] Изображения оптимизированы.
- [ ] История коммитов осмысленная (не один коммит «done»).
- [ ] README полный: описание, стек, ссылки, скриншоты.

## Сдача и ревью

1. Залей всё в публичный GitHub-репозиторий.
2. Задеплой через **GitHub Pages** (Settings → Pages) — получишь URL вида `username.github.io/repo`.
3. Пришли мне в Telegram: ссылку на репо, ссылку на прод, ссылку на Figma.
4. На созвоне — вместе проходимся по критериям приёмки. Если что-то не проходит — отмечаем фиксы и короткая итерация.
5. После успешного ревью — пишу короткую ретру в [[Прогресс]]: что было сложно, что получилось, что узнал нового.

## Ориентир по времени
