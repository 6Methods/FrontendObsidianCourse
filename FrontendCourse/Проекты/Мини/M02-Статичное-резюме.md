# M02 — Статичное резюме

Мини-проект после модуля 02 HTML. Цель — собрать всё, что выучил в модуле, на одной реальной странице.

## Задача

Сверстать страницу-резюме на **чистом HTML**, без единой строчки CSS и JS.

Требования:
1. Валидный HTML5 (DOCTYPE, charset, lang="ru", title, viewport).
2. Семантика: `<header>` с именем и коротким био, `<main>` с секциями (опыт / навыки / проекты), `<footer>` с контактами.
3. Один `<h1>` (твоё имя), подзаголовки `<h2>`–`<h3>` для секций.
4. Маркированный список навыков, нумерованный список — если нужна приоритизация.
5. Ссылки на GitHub / LinkedIn / Telegram с `target="_blank"` и `rel="noopener noreferrer"`.
6. Фото с адекватным `alt`. Если нет фото — можно дескриптивный `<svg>` или пропустить.
7. Форма обратной связи: имя (required), email (type="email", required), сообщение (textarea, required), submit-кнопка.
8. Lighthouse → Accessibility ≥ 95.
9. Валидация на [validator.w3.org](https://validator.w3.org/) без ошибок.

## Чего НЕ должно быть

- `<div>`-супа вместо семантики.
- `<br>` вместо абзацев.
- `placeholder` как замена `<label>`.
- `<b>`/`<i>` вместо `<strong>`/`<em>`.
- Пустых/мусорных `alt`.

## Сдача

1. Создать публичный репозиторий на GitHub, например `resume-html`.
2. Запушить туда `index.html`.
3. В README.md — короткое описание «Мой первый HTML-проект в рамках курса».
4. Прислать ментору ссылку на репо в созвон.

## Критерии приёмки ревью
- [ ] HTML валидируется на validator.w3.org без ошибок.
- [ ] Lighthouse Accessibility ≥ 95.
- [ ] Семантика использована по смыслу, не формально.
- [ ] Все ссылки рабочие.
- [ ] Форма имеет все `<label>`, работает HTML-валидация при пустом submit.
- [ ] Навигация с клавиатуры проходит логично.
