# M05 — ToDo-лист

Мини-проект после модуля 05. Классическая учебная задача — но не просто «сделать», а собрать всё из модуля в одном приложении: массивы, объекты, функции, DOM, события, `localStorage`.

## Задача

Одностраничное приложение без фреймворков: `index.html` + `style.css` + `main.js`. Стили — любые (чистый CSS или Tailwind — на твой выбор).

## Требования

### Функциональность
1. **Добавление задачи** — форма с `<input>` + кнопка. Пустые задачи не добавляются.
2. **Список задач** — рендерится из массива объектов (`{ id, title, done }`), а не копится в DOM напрямую.
3. **Переключение «выполнено»** — клик по задаче переключает `done`, визуально перечёркивается.
4. **Удаление задачи** — кнопка ✕ у каждой задачи.
5. **Фильтр:** «Все / Активные / Выполненные» — три кнопки, показывают соответствующую подвыборку.
6. **Счётчик:** «Осталось: N» — сколько невыполненных.
7. **Очистить выполненные** — кнопка, удаляет все `done: true`.
8. **Сохранение в `localStorage`** — при каждом изменении массив сериализуется через `JSON.stringify`, при загрузке страницы — читается и парсится.

### Технические правила
- Единый массив `todos` — источник правды. Любое действие (добавить / удалить / переключить / отфильтровать) сначала обновляет массив, потом функция `render()` перерисовывает DOM.
- Функции чистые, где возможно. Мутацию массива оставь на один-два места (`setTodos`), остальное пусть возвращает новые массивы через `.map`, `.filter`, `spread`.
- Делегирование событий на списке (один слушатель на `<ul>`, внутри определяешь `event.target`), а не 100 слушателей на каждой `<li>`.
- `textContent` для пользовательского текста, НЕ `innerHTML`.
- `addEventListener`, а не `onclick="..."` в HTML.
- `id` задачи — уникальный (проще всего через `Date.now()` или `crypto.randomUUID()`).

## Чего НЕ должно быть

- Библиотек (jQuery, React, Alpine — ничего).
- `innerHTML` с пользовательским вводом.
- Глобальных переменных «везде» — старайся инкапсулировать в функции.
- Дублирования DOM и массива — не редактируй `<li>` напрямую, минуя массив.

## Этапы

1. **Разметка.** `index.html` — форма, список, кнопки фильтров, счётчик. Валидный HTML, семантика.
2. **Стили.** Минимальные, но аккуратные. Можно из M03/M04 палитру взять.
3. **Состояние.** Массив `todos`, функции `addTodo`, `removeTodo`, `toggleTodo`, `clearDone`, `setFilter`.
4. **Рендер.** Функция `render()` — из `todos + currentFilter` строит HTML. Вызывается после каждого изменения.
5. **События.** `submit` на форме, `click` на списке (через делегирование), `click` на кнопках фильтров и «очистить».
6. **`localStorage`.** При каждом изменении — `localStorage.setItem("todos", JSON.stringify(todos))`. При загрузке — `JSON.parse(localStorage.getItem("todos") ?? "[]")`.

## Критерии приёмки ревью

- [ ] Все 8 функциональных пунктов работают.
- [ ] `todos` — единый источник правды, рендер полностью от него.
- [ ] После `F5` задачи остаются (localStorage).
- [ ] Нет `innerHTML` с пользовательским вводом.
- [ ] Используются `.map` / `.filter` / spread, а не ручное мутирование.
- [ ] Делегирование событий настроено.
- [ ] Код разбит на функции с говорящими именами, не одна лапша на 200 строк.
- [ ] В DevTools → Console — нет ошибок и warning-ов.
