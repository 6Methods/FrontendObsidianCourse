# M03 — Стилизованное резюме

Мини-проект после модуля 03. Берём HTML-резюме из [[M02-Статичное-резюме]] и превращаем его в адаптивный презентабельный сайт. Без фреймворков, на чистом CSS.

## Задача

Файл `style.css` в той же папке, подключён через `<link>`. HTML можно чуть поправить, но без радикальных переделок.

## Требования

1. **Типографика:** веб-безопасный шрифт или подключённый с Google Fonts. `body` с `line-height: 1.5`, иерархия заголовков визуально читается.
2. **Box-sizing:** глобальный `border-box` плюс `body { margin: 0 }`.
3. **Цвет:** продуманная палитра — фон, акцент, основной и вторичный текст. Минимум 3 цвета, не стоковая серость. Контраст текста к фону ≥ 4.5:1 (проверить Lighthouse).
4. **Layout:**
    - Шапка (`<header>`) на Flex: имя/аватар слева, контакты или nav справа.
    - Основной контент (`<main>`) на Grid: на широком экране две-три колонки (например, фото и контакты слева, опыт и навыки справа). На мобилке — одна колонка.
    - Секция «Навыки» на Grid с `repeat(auto-fit, minmax(...))`.
5. **Адаптив:** работает на 375px, 768px, 1440px. Mobile-first: базовые стили рассчитаны на узкий экран, `@media (min-width: ...)` дорисовывает остальное.
6. **Интерактив:** у ссылок и кнопок есть `:hover` и видимый `:focus`. Форма выглядит адекватно, инпуты с отступами и рамкой, кнопка submit заметна.
7. **Sticky-шапка** или плавный скролл по якорям — опционально, как дополнительный штрих.
8. **Lighthouse:** Accessibility ≥ 95, Performance ≥ 90.

## Чего НЕ должно быть

- Библиотек типа Bootstrap/Tailwind (Tailwind будет в модуле 04).
- `!important` без обоснования.
- Фиксированных `px` ширин на главных блоках — только `%`, `fr`, `minmax`.
- `position: absolute` на всё подряд без надобности.

## Сдача

1. Закоммить изменения в репозиторий `resume-html` (тот же, что в M02). Отдельной веткой `css-styling` или сразу в main.
2. Дополни README: «Стилизовал в рамках модуля 03».
3. Скриншоты страницы на 375px, 768px, 1440px положи в папку `/screenshots/` и покажи на созвоне.

## Критерии приёмки ревью
- [ ] Lighthouse Accessibility ≥ 95, Performance ≥ 90.
- [ ] Раскладка корректна на трёх разрешениях (mobile / tablet / desktop).
- [ ] Flexbox и Grid использованы по смыслу, не просто «везде flex».
- [ ] `:focus` у интерактивных элементов заметен.
- [ ] Код читаемый: классы по смыслу, селекторы без избыточных id, без `!important` без причины.
