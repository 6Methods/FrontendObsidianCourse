# Box model и позиционирование

## Зачем это нужно

Любой элемент на странице — это прямоугольная коробка (box), даже если визуально он круглый или треугольный. Box model объясняет, из каких слоёв эта коробка состоит и почему элемент часто оказывается «шире, чем я задавал». А позиционирование отвечает на вопрос «как сдвинуть элемент туда, куда мне надо»: прижать шапку сверху, поставить бейдж в угол карточки или зафиксировать кнопку на экране.

## Главное

### Box model — 4 слоя

У каждой коробки четыре слоя, вложенные как матрёшка. `content` — область, где живёт сам контент: текст, картинка, другой элемент; именно им управляют `width` и `height`. `padding` — внутренний отступ между контентом и рамкой, фон заходит и сюда. `border` — сама рамка (толщина, стиль, цвет). `margin` — внешний отступ, пустое пространство между коробкой и соседями.

### `box-sizing: content-box` vs `border-box`

По умолчанию действует `content-box`: `width: 200px` — это ширина только content, padding и border прибавляются сверху. В итоге реальная ширина = 200 + padding + border, задаёшь 200px, получаешь 242px. Решение — включить `border-box` глобально через `*, *::before, *::after { box-sizing: border-box; }`, тогда `width: 200px` считается вместе с padding и border. Так делает вся индустрия, вёрстка становится предсказуемой.

### margin collapse

Если у одного блока снизу `margin: 20px`, а у следующего сверху `margin: 30px`, между ними будет не 50px, а 30px — вертикальные margin схлопываются в больший. Это не баг, а фича для красивой типографики, но если ты не в курсе, она выносит мозг. Обход: обернуть блоки в flex/grid контейнер либо использовать padding у родителя.

### `display`

Свойство определяет поведение в потоке. `block` занимает всю ширину и начинается с новой строки (`<div>`, `<p>`). `inline` идёт в потоке текста, `width`/`height` игнорируются (`<span>`, `<a>`). `inline-block` — в потоке, но принимает размеры. `none` полностью удаляет элемент из документа, он не занимает места. Позже познакомишься с `flex` и `grid`.

### Позиционирование

Свойство `position` меняет способ размещения. `static` — по умолчанию, в потоке, `top`/`left` игнорируются. `relative` — в потоке, но `top`/`left` смещают визуально (исходное место остаётся); обычно используется как «якорь» для absolute-детей. `absolute` выпадает из потока и привязывается к ближайшему родителю с не-`static` position; используется для модалок, тултипов, бейджей внутри карточки. `fixed` — как absolute, но привязан к viewport, не двигается при скролле (шапка, кнопка «наверх»). `sticky` — гибрид: ведёт себя как relative, но «прилипает» к заданному `top` при скролле.

### `z-index`

Управляет слоями по оси Z. Работает только для элементов с position ≠ static (а также flex/grid-детей). Чем больше число, тем выше слой.

## Пример кода

```css
/* глобальный border-box — делаем всегда */
*,
*::before,
*::after {
  box-sizing: border-box;
}

body { margin: 0; }

.card {
  width: 300px;
  padding: 16px;
  border: 1px solid #ddd;
  margin: 24px;
  /* реальная ширина = 300px благодаря border-box */
}

/* relative-родитель для absolute-ребёнка */
.card {
  position: relative;
}
.card .badge {
  position: absolute;
  top: 8px;
  right: 8px;
  background: crimson;
  color: white;
  padding: 4px 8px;
}

/* шапка, прилипающая при скролле */
.site-header {
  position: sticky;
  top: 0;
  background: white;
  z-index: 10;
}
```

## Частые ошибки

- Не переключить `box-sizing` на `border-box`. Каждый `padding` ломает заданную ширину, вёрстка едет.
- `position: absolute` без `relative`-родителя. Элемент прилипает к `<html>`, и начинается хаос.
- Шлёпать большие `z-index` (999, 9999, 99999) на всё подряд. Через неделю невозможно отладить, какой слой над каким.
- Думать, что `margin` удаляет элемент из потока. Нет, это делает только `position: absolute`/`fixed` или `display: none`.

## Задания

- [ ] В глобальном CSS включи `*, *::before, *::after { box-sizing: border-box; }` и `body { margin: 0; }`.
- [ ] Открой в DevTools → Computed панель «Box model» для разных элементов, посмотри визуально слои content/padding/border/margin.
- [ ] Сверстай карточку с «бейджем» в правом верхнем углу (`position: absolute` внутри `position: relative`-родителя).
- [ ] Сделай sticky-шапку для своего резюме, чтобы она прилипала при скролле.
