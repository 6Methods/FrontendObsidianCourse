# Селекторы

## Зачем это нужно
Селекторы — это то, как ты «указываешь пальцем» на элементы страницы. Без них браузер просто не поймёт, к чему именно ты хочешь применить стиль. Умение точно и аккуратно выбирать нужные элементы — это, считай, половина всего CSS.

## Главное

### Селекторы по тегу
Самые простые: пишешь `p`, `h1` или `a`, и стиль применяется ко всем таким тегам. Подходят для базовых настроек, чтобы задать шрифт всем параграфам, например. Точечно использовать их не стоит: слишком широко бьёт и можно случайно зацепить лишнее.

### Класс `.название`
Твой основной рабочий инструмент. 90% стилей будут на классах. Точка перед именем говорит браузеру: «ищи элементы с таким классом». Один класс можно навесить на любое число элементов, а на один элемент — сразу несколько классов через пробел: `class="btn btn-primary"`. Примеры: `.card`, `.btn-primary`, `.nav-link`.

### ID `#название`
Уникальный идентификатор, встречается на странице только один раз. В CSS пишется через решётку: `#header`, `#main`. Для стилей его почти не используют — он слишком «тяжёлый» по специфичности (подробнее в уроке 03.3). Оставь id для якорных ссылок и работы с JavaScript.

### Селектор по атрибуту
В квадратных скобках указываешь HTML-атрибут и его значение: `[type="email"]`, `input[required]`, `a[href^="https://"]`. Удобно для форм (обязательные поля, email) и ссылок (внешние, на якорь). Символы `^=`, `$=`, `*=` означают «начинается с», «заканчивается на», «содержит».

### Универсальный `*`
Звёздочка выбирает все элементы на странице. Чаще всего её применяют в сбросах стилей: например, `* { box-sizing: border-box; }`, чтобы размеры у всех считались одинаково. В обычных стилях — почти не нужна.

### Комбинаторы
`A B` (потомки, через пробел) ловит все `B` внутри `A` на любой глубине, даже если между ними ещё пять других тегов. `A > B` (прямые дети) — только те `B`, которые лежат непосредственно внутри `A`, без промежуточных оболочек. `A + B` (соседний сразу после) — один элемент `B`, идущий прямо следом за `A`. `A ~ B` (все следующие соседи) — все `B`, идущие после `A` на том же уровне.

### Псевдо-классы
Состояния и позиции элементов, цепляются через двоеточие: `:hover` (наведение), `:focus` (фокус в поле), `:active` (клик), `:disabled`, `:checked`, `:first-child`, `:last-child`, `:nth-child(odd/even/3n+1)`. С их помощью стилизуешь интерактив и «каждый второй» элемент в списке.

### Псевдо-элементы
Пишутся через двойное двоеточие: `::before`, `::after`, `::placeholder`, `::selection`. Первые два вставляют контент до или после элемента прямо из CSS, остальные стилизуют отдельные части. Классика — иконка-стрелочка перед внешней ссылкой через `::before`.

### Группировка
Если нескольким селекторам нужны одинаковые стили, перечисли их через запятую: `.card, .tile, .block { ... }`. Избавляет от копипасты и делает файл короче. Работает с любыми селекторами.

## Пример кода
```css
/* тег */
h1 { color: navy; }

/* класс */
.card { padding: 16px; border: 1px solid #ddd; }

/* дочерние и потомки */
.card > h2 { margin-top: 0; }       /* только прямой ребёнок */
.card h2 { color: #333; }            /* любой на глубине */

/* псевдо-класс */
a:hover { color: crimson; }
input:focus { outline: 2px solid dodgerblue; }

/* чётные строки таблицы */
tr:nth-child(even) { background: #f5f5f5; }

/* псевдо-элемент */
.external-link::after {
  content: " ↗";
  color: gray;
}

/* группировка */
h1, h2, h3 { font-family: Georgia, serif; }
```

## Частые ошибки
- Ставить стили на id вместо классов. Невозможно переиспользовать, ломается специфичность.
- Писать много тегов (`div p span a { ... }`) вместо нормального класса. Хрупко и медленно.
- Путать `A B` и `A > B`. `A B` ловит всех потомков, `A > B` — только детей.
- Забывать `::` у псевдо-элементов (`:before` устарело, но в старом коде встречается).

## Задания
- [ ] В `style.css` для резюме задай: цвет `h1`, стилизацию класса `.btn` (если есть), `a:hover` с изменением цвета, `input:focus` с видимой рамкой.
- [ ] Сделай `::before` у одной из ссылок с каким-то символом-иконкой (`→`, `✉`).
- [ ] Используй `nth-child(even)` на списке навыков, чтобы чётные пункты были с другим фоном.
