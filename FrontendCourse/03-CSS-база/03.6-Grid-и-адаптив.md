# Grid и адаптив

## Зачем это нужно
Flexbox — это одна ось: элементы выстраиваются либо в ряд, либо в колонку. Grid работает иначе, это полноценная двумерная сетка, где ты одновременно управляешь и строками, и колонками. А медиа-запросы позволяют менять раскладку в зависимости от ширины экрана: на мобилке одно, на десктопе другое. Вдвоём Grid и media queries закрывают всю задачу «как страница должна выглядеть на десктопе, планшете и мобильнике».

## Главное

### Grid vs Flexbox — когда что
Flex нужен, когда элементы раскладываются по одной оси: шапка с логотипом и меню, тулбар с кнопками, список карточек в ряд. Grid нужен, когда у тебя сетка: страница с сайдбаром, галерея, дашборд. Рабочее правило: если чертёж раскладки напоминает таблицу со строками и колонками, это Grid. Если всё тянется вдоль одной линии — Flex.

### Базовый Grid
Минимальный рецепт выглядит так: `display: grid; grid-template-columns: 1fr 1fr 1fr; gap: 16px;`. Эти три строчки создают контейнер с тремя равными колонками и отступом 16px между ними. Всё, что лежит внутри как прямой потомок, автоматически разложится по сетке, никаких доп. классов на детях не нужно.

### Единица `fr`
`fr` расшифровывается как «fraction» — доля свободного пространства. Запись `1fr 2fr` означает, что первая колонка получит одну долю, вторая две, то есть соотношение 1:2. Большой плюс `fr` перед процентами: она сама учитывает `gap` и не ломает раскладку при появлении отступов.

### `repeat()` и `minmax()`
`grid-template-columns: repeat(3, 1fr)` — короткая запись для `1fr 1fr 1fr`. А сочетание `repeat(auto-fit, minmax(240px, 1fr))` — магическая формула адаптивной галереи: колонки минимум 240px, браузер умещает столько, сколько влезет, а на узком экране перестраивает сетку. Запомни эту строчку наизусть, она покрывает 80% задач с сетками карточек.

### Размещение элементов
По умолчанию Grid сам раскладывает детей в порядке их следования в HTML, слева направо, сверху вниз. Если нужно что-то растянуть или переместить, используй `grid-column: 1 / 3` (с 1 по 3 линию, то есть две колонки) или `grid-row: 1 / span 2` (растянуть на две строки). Для старта этого хватит.

### Медиа-запросы (`@media`)
Запись `@media (min-width: 768px) { ... }` означает «применять правила, когда экран шире 768px» — это mobile-first подход. Зеркальная `@media (max-width: 767px) { ... }` срабатывает на экранах уже 767px, это desktop-first. В современном фронте стандартом стал mobile-first.

### Mobile-first логика
Сначала пишешь стили для самого узкого экрана — мобилки, без всяких медиа-запросов. Потом через `@media (min-width: ...)` «дорисовываешь» раскладку для планшета и десктопа. Так делают потому, что мобилок в мире больше, и CSS получается короче и чище: ты не переопределяешь правила, а наращиваешь их.

### Популярные breakpoint-ы
Индустрия не договорилась о едином стандарте, но большинство команд использует близкие значения: 640px (small), 768px (tablet), 1024px (desktop), 1280px (large). Tailwind, Bootstrap и другие фреймворки берут почти одинаковые цифры, можно смело ориентироваться на них. Для учебного проекта хватит двух breakpoint-ов, например 768px и 1200px.

## Пример кода
```css
/* адаптивная галерея карточек */
.gallery {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(240px, 1fr));
  gap: 16px;
}

/* лендинг с сайдбаром — mobile-first */
.layout {
  display: grid;
  grid-template-columns: 1fr;          /* на мобилке: одна колонка */
  gap: 24px;
}

@media (min-width: 768px) {
  .layout {
    grid-template-columns: 240px 1fr;  /* на планшете+: сайдбар + контент */
  }
}

@media (min-width: 1200px) {
  .layout {
    grid-template-columns: 280px 1fr 320px;  /* на большом экране: 3 колонки */
  }
}
```

## Частые ошибки
- Использовать `px` для колонок (`grid-template-columns: 200px 300px 400px`). Раскладка перестаёт быть адаптивной и ломается на узких экранах. Используй `fr` и `minmax`.
- Писать `max-width` везде вместо `min-width`. Легко запутаться, когда какое правило срабатывает, особенно когда их несколько. Выбери один подход (лучше mobile-first с `min-width`) и держись его по всему проекту.
- Забыть `<meta name="viewport" content="width=device-width, initial-scale=1">` в HTML. `@media` запросы на мобилке не сработают, и страница будет показываться как уменьшенная десктопная версия.
- Мешать Flex и Grid в одном месте, когда хватит одного. Раскладка становится непредсказуемой и трудно отлаживается. Выбирай инструмент по задаче: сетка — Grid, ось — Flex.
