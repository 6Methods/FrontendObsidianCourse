# Подключение CSS и синтаксис

## Зачем это нужно
Прежде чем рисовать красивые кнопки и раскрашивать заголовки, надо понять две базовые вещи: как файл со стилями вообще попадает на страницу и как выглядит одно CSS-правило изнутри. Это буквально 5 минут теории, которые закрывают самый частый вопрос новичка: «почему у меня ничего не применяется». Разберёмся — и дальше CSS перестанет быть магией.

## Главное

### 3 способа подключения
В CSS три способа «прикрутить» стили к HTML, и они не равнозначны.

**External** — отдельный файл `style.css`, который подключается одной строкой `<link rel="stylesheet" href="style.css">` внутри `<head>`. Это стандарт индустрии: стили отделены от разметки, переиспользуются между страницами, браузер кэширует файл. Для любого нормального проекта — единственный правильный выбор.

**Internal** — `<style>...</style>` прямо в `<head>`, внутри тега живёт CSS. Годится для быстрых демо, одностраничных примеров или писем. Для реальных проектов не подходит.

**Inline** — атрибут `style="..."` на конкретном теге. Плохая практика: нельзя переиспользовать, тяжело поддерживать, конфликтует с каскадом. Исключение — когда значение вычисляется в JS на лету.

### Синтаксис правила
CSS-правило выглядит так: `селектор { свойство: значение; }`. Селектор говорит, к чему применить (`h1`, `.button`), а в фигурных скобках перечисляем пары «свойство: значение» через точку с запятой. Фигурные скобки обязательны, точку с запятой после каждого значения лучше ставить всегда — привычка сэкономит часы отладки.

### Свойство: значение
Всё внутри фигурных скобок — пары «свойство и его значение». Свойств в CSS буквально тысячи, но 90% реальной вёрстки делается примерно тридцатью: `color`, `background`, `width`, `height`, `margin`, `padding`, `display`, группа `font-*`, `border`, `flex`, `grid` и ещё немного. Их и будем учить.

### Комментарии
Комментарий в CSS — `/* текст */`, работает и на одну строку, и на несколько. Используй экономно: подписывай секции и объясняй неочевидное, но не комментируй `color: red; /* красный */` — это шум.

### Единицы измерения
`px` — фиксированные пиксели. `%` — от размера родителя. `em` и `rem` — от размера шрифта (em — родителя, rem — корня документа). `vh` и `vw` — от высоты и ширины окна. Пока достаточно уверенно чувствовать `px`, `%`, `rem`. Остальные подключим по мере задач.

### Как отлаживать
Главный инструмент CSS-разработчика — DevTools, вкладка Elements и панель Styles справа. Там видно все правила, которые применились к элементу, и какие были перебиты каскадом (они перечёркнуты). Если стиль не применяется, ответ почти всегда там.

## Пример кода

```html
<!-- index.html -->
<!DOCTYPE html>
<html lang="ru">
  <head>
    <meta charset="UTF-8">
    <title>CSS демо</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <h1 class="greeting">Привет, CSS!</h1>
  </body>
</html>
```

```css
/* style.css */
body {
  font-family: Arial, sans-serif;
  margin: 0;
}

.greeting {
  color: #2d6cdf;
  font-size: 32px;
}
```

## Частые ошибки
- Забыть точку с запятой после значения. Следующее правило в блоке молча игнорируется, и ты полчаса ищешь, почему не работает.
- Подключить `style.css` не в `<head>`, а в конце `<body>`. Страница «мигает» без стилей в момент загрузки (FOUC, flash of unstyled content).
- Использовать inline-стили везде. Невозможно переиспользовать, тяжело поддерживать, постоянные конфликты с каскадом и специфичностью.

## Задания
- [ ] Создай в папке с HTML-резюме файл `style.css`.
- [ ] Подключи его через `<link>` в `<head>`.
- [ ] Задай `body` шрифт `Arial, sans-serif` и `color: #222`.
- [ ] В DevTools → Elements выбери тег `body` и убедись, что твои правила применились (видно в панели Styles).
