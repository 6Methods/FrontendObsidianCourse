# Каскад, специфичность, наследование

## Зачем это нужно

«У меня правило написано, а не применяется» — это боль буквально каждого новичка в CSS. Ты пишешь `color: red`, сохраняешь, обновляешь страницу, а текст всё равно синий. Магии тут нет: браузер выбирает между несколькими конкурирующими правилами по чётким правилам. Этот механизм называется каскад. Если его понять, ты перестанешь воевать со стилями и начнёшь ими управлять.

## Главное

### Каскад

Каскад — это алгоритм, по которому браузер выбирает, какое правило применить, если их несколько. Он смотрит три критерия строго по важности. Сначала источник и слой (`@layer`, стили браузера/автора), потом специфичность селектора, и только в конце — порядок в файле. Если правило сильнее по специфичности, порядок уже не важен. Поэтому «я написал стиль ниже, а он не применился» — это почти всегда про специфичность.

### Специфичность

Специфичность — это «вес» селектора, четыре числа `(a, b, c, d)`:

- `a` — есть ли inline-стиль (`style="..."`): 1 или 0.
- `b` — количество id-селекторов (`#hero`).
- `c` — количество классов, атрибутов, псевдо-классов (`:hover`).
- `d` — количество тегов и псевдо-элементов (`h2`, `::before`).

Сравнивают лексикографически: сначала по `a`, при равенстве — по `b`, и так далее. Пример: `.card h2` даёт `(0,0,1,1)` — класс и тег. А `#hero h2` даёт `(0,1,0,1)` — id и тег. Второе сильнее: на позиции `b` у него 1 против 0, и классы это уже не перебьют.

### Порядок в файле

Если специфичность полностью совпала, побеждает то правило, что идёт позже в CSS (или позже подключено). Это единственный случай, когда «написать ниже» реально работает. Поэтому свои стили подключают после стилей библиотеки.

### `!important`

`!important` — ядерная кнопка. Правило с ним поднимается выше всех обычных, даже выше inline. Перебить его может только другой `!important` или inline с `!important`. Почти никогда не используй: если он понадобился, значит каскад запутался. Исключения — перебить стиль библиотеки или зафиксировать что-то в тесте.

### Наследование

Некоторые свойства автоматически передаются от родителя к детям: `color`, `font-*`, `line-height`, `visibility`. Другие нет: `margin`, `padding`, `border`, `width`, `background`. Если нужно принудительно унаследовать — пиши `property: inherit`. Если сбросить к дефолтному значению — `property: initial`. А `unset` — это «inherit, если свойство наследуемое, иначе initial».

### Дефолт браузера

У каждого тега есть встроенные стили: `h1` крупный и жирный, `ul` с отступом и буллетами, `button` с серой рамкой. Это user-agent stylesheet. В начале проекта эти стили принято сбрасывать через готовый reset/normalize или минимальный `* { box-sizing: border-box; margin: 0; }`, чтобы стартовать с чистого листа.

## Пример кода

```css
/* специфичность (0,0,1,1) */
.card h2 { color: blue; }

/* специфичность (0,0,0,1) — слабее, не применится */
h2 { color: red; }

/* inline стиль (1,0,0,0) — победит всех */
/* <h2 style="color: green;">...</h2> */

/* !important — поднимает выше обычных правил */
.card h2 { color: purple !important; }

/* наследование */
body {
  color: #222;
  font-family: Arial, sans-serif;
}
/* .card h2 унаследует font-family, если не перебить */
```

## Частые ошибки

- Использовать `!important` для «быстро пофиксить». Через неделю такой код ничем не отладить, и каждый новый стиль придётся тоже помечать `!important`.
- Писать стили через id (`#header`, `#main`). Специфичность взлетает до потолка, и обычные классы уже не могут ничего перебить.
- Надеяться, что новое правило «поверх старого» автоматически перебьёт, игнорируя специфичность. Самая частая причина «почему не применяется».
- Накапливать глубокие селекторы вроде `.page .content .card .header h2` ради специфичности. Это признак, что архитектуру классов надо переделать, а не добавлять ещё один уровень.
