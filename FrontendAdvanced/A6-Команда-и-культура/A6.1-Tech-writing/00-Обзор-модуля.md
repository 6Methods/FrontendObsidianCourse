# A6.1 — Tech writing

## Зачем модуль

Senior-фронтендер пишет не только код. Он пишет ADR (Architectural Decision Record — запись архитектурного решения), RFC (Request for Comments — запрос на обсуждение), postmortem (разбор инцидента без поиска виноватых), README и changelog (журнал изменений). Всё это — инструменты, которые превращают частное знание одного человека в общее знание команды. Без них каждая следующая итерация проекта повторяет уже сделанные ошибки, а новый человек в команде тратит недели, чтобы понять «почему мы выбрали tRPC, а не GraphQL».

Модуль не про академическое письмо и не про копирайтинг. Он про жанры технических документов, у которых есть устоявшийся формат и ожидания читателя. Каждый жанр решает конкретную задачу: ADR фиксирует решение, RFC собирает feedback (обратную связь) до решения, postmortem учит на инциденте, README объясняет «как запустить», changelog отвечает на вопрос «что изменилось в новой версии».

## Что студент будет уметь

- Написать ADR в формате Michael Nygard (Title / Status / Context / Decision / Consequences) и объяснить, чем он отличается от RFC.
- Оформить RFC с секциями Motivation, Detailed design, Drawbacks, Alternatives — так, чтобы команда могла дать осмысленный review.
- Провести blameless (без поиска виноватых) postmortem: собрать timeline (хронологию), отделить root cause (первопричину) от contributing factors (способствующих факторов), выдать action items с owner'ом и deadline.
- Написать README для npm-пакета так, чтобы hello-world занимал 30 секунд, и подключить typedoc для API-документации.
- Вести changelog по Keep a Changelog, выбрать инструмент автогенерации (Changesets, semantic-release, release-please) и написать migration guide (руководство по миграции) для MAJOR-версии.
