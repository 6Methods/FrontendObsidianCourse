# A6.5.1 — Cursor, Claude Code, Copilot

## Зачем

Инструментов с AI на начало 2026 — много, и они устроены принципиально по-разному. Cursor — форк VS Code с глубоко встроенным агентом. Claude Code — терминальный агент от Anthropic, работающий прямо в репозитории. GitHub Copilot — исторически autocomplete, сейчас — ещё и чат, и агент. Windsurf, Zed AI — конкуренты с собственными акцентами. Если открыть их все и начать использовать одинаково — один как «умный автокомплит», — то половина возможностей пропадает впустую, а половина задач делается медленнее, чем без AI вообще.

Урок — про то, как устроен ландшафт, в чём разница между тремя режимами взаимодействия (autocomplete, chat, agent) и когда какой режим реально ускоряет работу, а когда — вредит.

## Главное

### Ландшафт инструментов — три волны

**Первая волна — autocomplete.** GitHub Copilot (2021), Tabnine, Codeium. Модель предсказывает следующие токены кода на основе файла и нескольких открытых вкладок. Inline (встроенное, в строке) предложение серым текстом — жмёшь Tab, принимаешь. Работает молча, быстро, контекст — узкий.

**Вторая волна — chat-in-IDE.** Copilot Chat, Cursor Chat, Continue. AI получает расширенный контекст: открытые файлы, выделенный фрагмент, часто — индекс всего репозитория (repository index — построенный embeddings-индекс кода для поиска). Формат — диалог: «объясни эту функцию», «напиши тест к этому классу», «почему TypeScript ругается на строку 42». Контекст — средний.

**Третья волна — agent mode.** Claude Code, Cursor Composer / Agent, Windsurf Cascade, Copilot Workspace, Zed Agent. AI получает доступ к tools (инструменты — команды shell, редактирование файлов, git, запуск тестов) и может выполнять многошаговые задачи: «найди все использования deprecated API и замени на новый», «почему падает этот тест — исправь», «добавь фичу X по описанию». Контекст — широкий, вплоть до всего репозитория и history терминала.

### Autocomplete — быстрый рутинный код

Режим, в котором AI силён однозначно: повторяющиеся паттерны, знакомые API, бойлерплейт (boilerplate — шаблонный код). Написал `const [count, set` — autocomplete допишет `Count] = useState(0)`. Написал сигнатуру функции с явными типами — допишет тело по паттерну соседних.

Где autocomplete вреден — в **незнакомом API**. Модель предскажет правдоподобный вызов, который не существует. Метод `array.firstOrDefault()` — в JS нет. `ctx.user.hasRole('admin')` — в вашем проекте `ctx.auth.roles.includes('admin')`. Autocomplete опирается на паттерны, а не на документацию, и в редком API соскальзывает в галлюцинацию.

Правило: в знакомом коде — Tab без оглядки; в незнакомой библиотеке — читать документацию, а не принимать первый вариант.

### Chat-in-IDE — объяснение и локальные правки

Chat уместен, когда **вопрос, а не задача**. «Что делает эта регулярка?», «почему этот useEffect срабатывает дважды?», «как в TypeScript выразить, что поле обязательно только в одном варианте union?». Модель читает выделенный кусок кода плюс связанные файлы, отвечает в panel.

Второй уместный сценарий — **локальная правка под конкретный запрос**: «добавь обработку ошибок в этот handler», «перепиши этот switch в lookup-объект», «извлеки эту логику в отдельную функцию». Chat даёт diff, инженер его применяет или отвергает.

Где chat вреден — когда задача требует **изменений в нескольких файлах одновременно**. Модель увидит один файл, даст совет, который ломается в связанных. Для этого — agent.

### Agent mode — изолированные рефакторинги и миграции

Agent сильнее всего там, где задача **механическая, но многошаговая**: найти все использования, заменить, запустить тесты, исправить оставшееся. Миграция с `useQuery` на `useSuspenseQuery`. Переименование прописного dependency по всему репозиторию. Добавление missing types в legacy-модуль. Очистка неиспользуемых импортов.

Агент плох в задачах, где нужна **архитектурная интуиция** или **интент автора**: «перепиши state management» — получишь правдоподобно выглядящий код, который не учитывает, почему у вас именно так. Агент плох в **чужих доменах**: игровая логика, медицинские формулы, финансовые расчёты — он угадает, а не проверит.

Правило: задачу, которую **сам можешь описать пошагово** — давай агенту. Задачу «я не знаю, как правильно» — не давай.

### Cursor, Claude Code, Copilot — краткие различия

- **Cursor** — IDE, форк VS Code. Встроенный индекс репозитория, tab-to-accept для multi-line edits, chat с `@file` / `@folder` контекстом, Composer — агентный режим внутри редактора. Сильная сторона — UX (пользовательский опыт) редактирования в большом кодовом проекте.
- **Claude Code** — терминальный агент от Anthropic. Запускается `claude` в директории проекта, видит файлы, выполняет shell-команды, делает diff и ждёт подтверждения. Сильная сторона — хорошо интегрируется в существующий workflow terminal + git, минимально навязчив.
- **GitHub Copilot** — autocomplete (исторически) + Copilot Chat (окно в IDE) + Copilot Workspace / Agent (многошаговый режим). Сильная сторона — интеграция с GitHub: issue → PR, автоматические suggestions на PR.
- **Windsurf** — IDE-форк в духе Cursor с акцентом на Cascade — агентный режим с долгой памятью контекста.
- **Zed AI** — Zed (быстрый редактор на Rust) с встроенным AI-чатом и tool use.

Выбор в 2026 — чаще не «один на всех», а стек: Copilot для autocomplete в GitHub-flow, Cursor или Zed как ежедневная IDE, Claude Code для крупных агентных задач в терминале.

### Privacy и compliance

Каждое AI-действие отправляет ваш код в сторонний провайдер. Это важный организационный вопрос, часто проигнорированный разработчиками.

Вопросы, которые стоит знать про выбранный инструмент:
- Код отправляется как prompt — это понятно. Используется ли он для **обучения** модели? У платных планов Copilot Business/Enterprise, Cursor Business и Claude Code — формально нет. У бесплатных версий — читать актуальный ToS (Terms of Service — условия использования).
- Где **hosted** (размещены) серверы? Для европейских компаний с GDPR (General Data Protection Regulation — регламент ЕС о защите данных) — us-only регион может быть проблемой.
- Есть ли **self-hosted** (размещённый на своих серверах) вариант? Для банков и медицинских компаний — часто обязательно.
- Что происходит с **секретами**? Если в открытом файле лежит `.env` с API-ключами, autocomplete его тоже видит. Gitignore для AI — отдельная история.

Практически: в рабочем репозитории — использовать корпоративный план, в pet-проекте — можно что угодно. Не смешивать.

## Пример кода

### Claude Code — изолированный рефакторинг

Диалог в терминале (условно; реальный вывод форматирован иначе):

```
$ claude

> В директории src/components/ найди все использования
  компонента <Button variant="primary">. Замени на
  <PrimaryButton>. Не трогай Button без variant,
  не трогай варианты "secondary" и "ghost". После —
  запусти pnpm typecheck и покажи diff перед коммитом.

● Я просканирую src/components/ на <Button variant="primary">.

  [Tool: Grep pattern="<Button\s+variant=\"primary\""]
  Найдено 14 вхождений в 9 файлах.

● Произведу замену. Применяю Edit к каждому файлу.

  [Tool: Edit] src/components/HeroSection.tsx
  [Tool: Edit] src/components/UserCard.tsx
  ... (9 files)

● Запускаю pnpm typecheck.

  [Tool: Bash pnpm typecheck]
  ✓ No type errors.

● Diff:

  - <Button variant="primary" onClick={handleClick}>
  + <PrimaryButton onClick={handleClick}>
  ... (14 изменений)

  Применить изменения? [y/N]
```

Ключевое — **human gate** перед merge: diff показан, `y/N` — решение инженера. Агент не коммитит самостоятельно.

### Copilot autocomplete — где он силён и где галлюцинирует

```ts
// Силён: знакомый паттерн
function debounce<T extends (...args: unknown[]) => void>(
  fn: T,
  delay: number
): (...args: Parameters<T>) => void {
  let timer: ReturnType<typeof setTimeout> | null = null;
  // Copilot предлагает:
  return (...args: Parameters<T>) => {
    if (timer) clearTimeout(timer);
    timer = setTimeout(() => fn(...args), delay);
  };
}
// Принимай смело — паттерн стандартный.

// Галлюцинирует: редкий API внутреннего SDK компании
import { internalAuth } from '@company/sdk';
// Copilot предлагает:
internalAuth.getCurrentUser().hasRole('admin');
// Метод getCurrentUser().hasRole() не существует.
// Реальный API: internalAuth.user.permissions.includes('admin')
// Autocomplete опирается на паттерны общих SDK и промахивается.
```

## Частые ошибки

1. **Agent без человеческого gate на PR.** Агент сделал 30 файлов правок, инженер жмёт «approve», потому что «тесты прошли». Тесты не проверяют интент. Человек обязан прочесть diff и решить — это минимум.

2. **Доверие autocomplete в незнакомом API.** Tab, Tab, Tab — коммит. Половина вызовов — не существующие методы. В знакомом stack автодополнение точное, в новом — надо открывать docs.

3. **Использование proprietary кода в бесплатных планах без ToS review.** Рабочий репозиторий отправляется в free-tier провайдер, который имеет право использовать данные для обучения. Для компании это может быть нарушение NDA (Non-Disclosure Agreement — соглашение о неразглашении).

4. **Путать chat и agent.** Chat отвечает на вопрос, agent делает задачу. Задавать агенту «что делает эта функция?» — дорого (он ещё запустит tools), задавать чату «отрефактори весь репо» — бесполезно (у него нет tool use).

5. **Включать AI везде одинаково.** Autocomplete в commit message, AI-suggestion в медицинском расчёте, agent на security-критичном коде — это не «продуктивность», это распространение ошибок. Есть зоны, где AI выключают намеренно.

6. **Игнорировать latency (задержка отклика).** Агент на каждый вопрос — 5–30 секунд. Если задача — написать одну строку, это медленнее, чем написать её самому. Выбор инструмента — тоже про время.

## Задания

### 1. Сопоставить режимы с задачами

Перед тобой 6 задач. Распредели каждую по режиму (autocomplete / chat / agent) и обоснуй:

1. Написать `useState` для boolean-флага `isOpen`.
2. Понять, почему тест `user-flow.spec.ts` падает в CI, но не локально.
3. Переименовать prop `onClick` в `onPress` во всех компонентах библиотеки.
4. Объяснить, что делает выделенная регулярка из чужого legacy.
5. Добавить missing return type в функцию, над которой курсор.
6. Мигрировать проект с `react-router-dom@6` на `@7`.

### 2. Проверить галлюцинацию autocomplete

Возьми библиотеку, которой мало пользовался — например, `zod`, `valibot` или `effect`. Напиши комментарием задачу («// валидация email с кастомной ошибкой») и дай Copilot предложить решение. Сравни с документацией. Зафиксируй: где попал, где предложил несуществующий метод.

### 3. Прогнать изолированный рефакторинг через агента

В своём pet-проекте придумай механическую задачу: заменить `console.log` на `logger.debug`, переименовать папку, добавить `data-testid` на кнопки. Дай её Claude Code или Cursor Composer. Прочти diff **полностью** перед принятием. Запиши — что пришлось поправить руками.

### 4. Аудит privacy-политики

Открой ToS инструмента, которым пользуешься каждый день (Copilot, Cursor, Claude Code). Найди ответ на три вопроса: используется ли код для обучения; где hosted серверы; есть ли опция отключить telemetry. Одна запись — один инструмент.
