# A6.3 — Open source contribution

## Зачем модуль

Open source (открытый исходный код — программный продукт, код которого доступен для чтения, изменения и повторного использования) — это не «про альтруизм по выходным». Для senior-фронтендера это один из самых сильных сигналов на собеседовании, канал живого нетворкинга и способ за месяц глубже изучить инструмент, чем за год пассивного использования. Хочешь по-настоящему понять Next.js — почитай его issues и попробуй закрыть одну.

Модуль — не про то, как стать core-контрибьютором React. Он про реалистичный путь: выбрать живой проект, завести первый PR (pull request — запрос на слияние изменений), пройти review loop, не сгореть и не подставить себя юридически. А если однажды сам опубликуешь пакет на npm — как не превратиться в 24/7-саппорт для незнакомых людей.

## Что студент будет уметь

- Объяснить, чем OSS-опыт сильнее LeetCode для senior-позиций и как рассказывать про контрибы в резюме.
- Выбирать проект по проверяемым критериям: активность, метки, CONTRIBUTING.md, tone of voice (тон общения) maintainer'ов (мейнтейнер — человек, отвечающий за развитие и сопровождение проекта).
- Проводить PR от issue до merge: fork (форк — персональная копия репозитория), upstream (исходный репозиторий, от которого сделан форк), ветка, тесты, commit-conventions, review loop.
- Быть maintainer'ом собственного пакета без burnout (выгорание): issue triage (разбор и сортировка входящих задач), release cadence (ритм релизов), делегирование, архивация мёртвых проектов.
- Разбираться в основных лицензиях (MIT, Apache 2.0, GPL, AGPL, ISC, BSD), понимать совместимость и знать разницу между DCO (Developer Certificate of Origin) и CLA (Contributor License Agreement).
