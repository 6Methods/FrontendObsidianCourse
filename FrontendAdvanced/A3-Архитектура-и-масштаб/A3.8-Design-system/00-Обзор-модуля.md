# A3.8 — Design system

## Зачем модуль

Design system (дизайн-система — набор переиспользуемых компонентов, токенов и правил, которые формируют единый UI продукта) — это не Figma-кит и не «наш npm-пакет с кнопкой». Это инженерная граница между дизайн-решениями и кодом. Когда в команде 10+ фронтендеров и несколько продуктов, без design system каждый фичепродуктовый тикет заново решает «какой синий», «какой padding», «какой focus ring» — и каждый раз чуть иначе. Консистентность ломается медленно и незаметно, а a11y (accessibility — доступность, совместимость с ассистивными технологиями) ломается быстро.

Senior-уровень работы с DS — это не «я умею использовать shadcn», а понимание слоёв: токены → primitives → продуктовые компоненты, и где на этих слоях проходят границы ответственности. Токены описывают намерение, а не сырое значение. Primitives решают поведение и a11y. Продуктовые компоненты собирают это в фичу. В модуле разбираем каждый слой отдельно: почему W3C Design Tokens Format — это правильный формат описания токенов, почему headless-библиотеки (Radix, React Aria) победили «готовые» UI-киты, как устроены polymorphic и compound components, как сделать theming без FOUC (flash of unstyled content — вспышка неоформленного контента при загрузке), и зачем Storybook (витрина компонентов и dev-среда для UI) в проекте, где уже есть design-тулы.

Модуль закрывает Track 3 (Архитектура и масштаб): DS — это архитектура UI. Без него масштабирование фронта превращается в копипасту и ре-изобретение кнопки в каждом модуле.

## Что студент будет уметь

- Проектировать трёхуровневую систему токенов (primitive → semantic → component) и генерировать CSS/JS из единого источника через Style Dictionary.
- Выбирать между headless-библиотекой (Radix, React Aria) и собственной реализацией примитивов, понимая trade-off.
- Применять shadcn/ui-подход (copy-paste в репозиторий) осознанно, а не как «модную альтернативу MUI».
- Писать polymorphic и compound components с типобезопасным API на TypeScript.
- Реализовывать theming через CSS custom properties без FOUC при SSR и использовать OKLCH для предсказуемых градаций яркости.
- Настраивать Storybook 8+ с CSF3, controls, play functions и интегрировать visual regression через Chromatic.
- Понимать, когда DS — преждевременная оптимизация, а когда — реальная экономия человеко-месяцев.
