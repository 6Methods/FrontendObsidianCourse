# A5.3.5 — LCA и иерархические меню

## Зачем

LCA (Lowest Common Ancestor — наименьший общий предок) — формально академическая задача («дан корень дерева и два узла, найди самого нижнего общего предка»), которая на фронте встречается ежедневно под другими названиями: «построй breadcrumbs (хлебные крошки — путь по иерархии) для товара», «подсветь активную ветвь бокового меню», «на какой уровень поднять state (состояние), чтобы два компонента видели его через props». Все три задачи сводятся к «найти общего предка двух узлов в дереве». Урок — про наивный алгоритм, оптимизации, и про то, где грань, на которой фронтовый код из O(h) превращается в O(n²) и убивает производительность hot render (горячего рендера — частого перерисования).

## Главное

### Что такое LCA

Дано дерево (не граф — без циклов, у каждого узла один родитель) и два узла `a` и `b`. LCA — самый глубокий (дальний от корня) узел `c`, такой что `a` — потомок `c` (или сам `c`), и `b` — потомок `c` (или сам `c`).

Интуитивно: идём от `a` и от `b` вверх к корню, первое совпадение — LCA.

Свойство, которое важно: LCA в дереве существует всегда (у любых двух узлов хотя бы корень общий). В графе это не так — там может не быть общего предка вообще или их может быть несколько.

### Наивный алгоритм через списки предков

Самый простой способ: от каждого узла подняться до корня, записать путь, найти первое совпадение.

```
ancestors(x) = [x, parent(x), parent(parent(x)), ..., root]
path_a = ancestors(a)
path_b = ancestors(b)

LCA = первый узел из path_a, который встречается в path_b
```

Через `Set` это O(h₁ + h₂), где h — высоты путей. На практике — быстро: дерево категорий в e-commerce редко глубже 5–7 уровней.

Требование: у каждого узла должна быть ссылка на родителя. Если нет — нужно сначала обойти дерево и построить `childId → parentId` карту.

### Альтернатива: спуск из корня

Если родительских ссылок нет, а обходить дерево не хочется дважды, есть элегантный рекурсивный вариант:

```ts
function lca<T>(root: Node<T> | null, a: Node<T>, b: Node<T>): Node<T> | null {
  if (!root) return null;
  if (root === a || root === b) return root;

  let found: Node<T> | null = null;
  for (const child of root.children) {
    const sub = lca(child, a, b);
    if (sub) {
      if (found) return root; // два разных поддерева вернули что-то — это мы
      found = sub;
    }
  }
  return found;
}
```

Сложность O(n) — посещаем каждый узел не более одного раза. Для одного запроса — оптимально.

### Tarjan offline LCA

Когда запросов много (Q штук), каждый раз делать O(n) обход — неэффективно. Алгоритм Тарьяна за O(n + Q) обрабатывает все запросы пакетом, используя DFS и union-find (DSU — disjoint set union, структура «объединение непересекающихся множеств»). Применяется, когда LCA запросы идут батчами: например, пакетная проверка «какие пары страниц в аналитике имеют общего родителя-кластера».

На фронте встречается **редко**. Если заметишь, что в коде делается десятки LCA-запросов за рендер — сперва вопрос, нельзя ли эту информацию предвычислить один раз и класть в мемо/контекст.

### Фронтовые кейсы

**1. Breadcrumbs.** Дано дерево категорий и текущая категория (лист). Построить путь от корня. Формально не LCA, а просто ancestors-list, но задача из того же класса: умение подняться от узла к корню.

**2. Active path в меню.** Sidebar с пунктами типа `Catalog > Electronics > Phones > iPhone 15`. Пользователь на странице iPhone 15 — нужно пометить как «активные» все узлы на пути от корня до iPhone 15. Снова — ancestors-list, один раз на навигацию.

**3. Lift state до LCA.** В React есть правило: если двум компонентам нужно видеть одно состояние, оно должно жить в ближайшем общем предке. Эта формулировка буквально — LCA. На практике делается руками: посмотрел на дерево, понял, что общий предок — `<Page>`, поднял state туда. Автоматизация бывает нужна, только когда дерево генерируется (например, плагинный UI).

**4. Highlight общей ветви.** Визуальный редактор или mind-map: выделил два узла — хочу подсветить ветвь от LCA до каждого из них. Здесь LCA считается реально за запрос — и O(n) допустимо, потому что такие клики редки.

### Когда это всё ломается

Первое — **циклы**. Если кто-то назвал «деревом» структуру, где у ребёнка может быть несколько родителей (или сам становится родителем своего предка) — это граф. LCA на графе — отдельная задача (может не существовать, может быть несколько). Симптом — бесконечный цикл в наивном подъёме.

Второе — **hot render**. Если `findLCA` зовётся на каждом рендере внутри `map` по 1000 элементов — это 1000·O(n). Решение: предвычислить на уровне построения данных (при получении дерева с бэка пройти один раз и проставить `depth` + `path`), потом LCA — это `path.find` между двумя известными массивами.

## Пример кода

```ts
type Category = {
  id: string;
  name: string;
  parentId: string | null;
};

type CategoryNode = {
  id: string;
  name: string;
  children: CategoryNode[];
  parent: CategoryNode | null;
};

// Построить дерево с parent-ссылками из плоского списка.
function buildTree(list: Category[]): Map<string, CategoryNode> {
  const map = new Map<string, CategoryNode>();
  for (const c of list) {
    map.set(c.id, { id: c.id, name: c.name, children: [], parent: null });
  }
  for (const c of list) {
    const node = map.get(c.id)!;
    if (c.parentId) {
      const parent = map.get(c.parentId)!;
      node.parent = parent;
      parent.children.push(node);
    }
  }
  return map;
}

// Ancestors включая сам узел — полезно для breadcrumbs.
function ancestors(node: CategoryNode): CategoryNode[] {
  const path: CategoryNode[] = [];
  let current: CategoryNode | null = node;
  while (current) {
    path.push(current);
    current = current.parent;
  }
  return path.reverse();
}

// LCA через пересечение ancestors.
function findLCA(a: CategoryNode, b: CategoryNode): CategoryNode | null {
  const seen = new Set<string>();
  for (let cur: CategoryNode | null = a; cur; cur = cur.parent) {
    seen.add(cur.id);
  }
  for (let cur: CategoryNode | null = b; cur; cur = cur.parent) {
    if (seen.has(cur.id)) return cur;
  }
  return null;
}

// Breadcrumbs — это просто ancestors(node) от корня до листа.
function breadcrumbs(node: CategoryNode): string {
  return ancestors(node)
    .map((n) => n.name)
    .join(' > ');
}

// Использование
const categories: Category[] = [
  { id: 'root', name: 'Каталог', parentId: null },
  { id: 'elec', name: 'Электроника', parentId: 'root' },
  { id: 'phones', name: 'Телефоны', parentId: 'elec' },
  { id: 'laptops', name: 'Ноутбуки', parentId: 'elec' },
  { id: 'clothes', name: 'Одежда', parentId: 'root' },
];
const tree = buildTree(categories);
const phones = tree.get('phones')!;
const laptops = tree.get('laptops')!;
const clothes = tree.get('clothes')!;

console.log(breadcrumbs(phones)); // "Каталог > Электроника > Телефоны"
console.log(findLCA(phones, laptops)?.name); // "Электроника"
console.log(findLCA(phones, clothes)?.name); // "Каталог"
```

## Частые ошибки

1. **Пытаться искать LCA на графе.** Если у узла может быть несколько родителей (граф категорий с «товар в двух категориях»), наивный алгоритм уйдёт в цикл или выдаст неправильный ответ. Сперва убедиться, что структура — дерево, иначе — специальные алгоритмы графов (см. A5.4).

2. **Делать findLCA в hot render на каждый элемент.** Если `items.map(item => findLCA(item, current))` внутри рендера списка на 1000 элементов и дерево большое — это O(n²). Предвычислить `depth` и `path` на входе, держать в мемо.

3. **Сравнивать узлы по объекту, а не по id.** `node === other` работает, только если у тебя один экземпляр на каждый id. После сериализации (JSON → fetch → JSON.parse) объекты будут разными, `===` вернёт false. Использовать `.id`.

4. **Забывать про случай «сам узел — предок другого».** Если `a` — предок `b`, то LCA это `a`. Наивный алгоритм это корректно ловит, но кастомные реализации иногда забывают обработать.

5. **Считать ancestors при каждом рендере заново.** Дерево меняется редко, а ancestors одного узла — детерминированы. Предвычислить на уровне store/query и использовать как кеш.

## Задания

### 1. Breadcrumbs для e-commerce

Дан плоский список категорий (как в примере). Реализуй `breadcrumbs(productCategoryId)`, которая возвращает путь от корня. Проверь на дереве глубиной 5.

### 2. Active path в меню

Дан tree и активный id. Верни `Set<string>` всех id на пути от корня до активного узла. Используй это в React-компоненте для подсветки active-ветви через `className`.

### 3. LCA двух узлов без parent-ссылок

Реализуй рекурсивный вариант `lca(root, a, b)` из «Главное». Проверь на tree из 1000 узлов. Убедись, что сложность O(n), не O(n·h).

### 4. Найти LCA hot path

Возьми реальный (или синтезированный) tree категорий на 10 000 узлов. Сделай `findLCA(random, random)` 1000 раз. Замерь. Потом предвычисли `path` для каждого узла (массив id от корня) — и найди LCA через два массива: идём от начала, пока пути совпадают. Сравни время.
