# A5.3 — Деревья

## Зачем модуль

Деревья — самая «фронтовая» структура данных. DOM — дерево. Virtual DOM — дерево. Файловая система, роут-таблица Next.js App Router, меню категорий в e-commerce, AST (Abstract Syntax Tree — абстрактное синтаксическое дерево), Fiber-архитектура React, scheduler React — всё это деревья. Если алгоритмы сортировки фронтендеру редко приходится писать вручную, то обходы (traversal — обход) дерева и диффинг (diff — сравнение) двух деревьев встречаются в каждой второй задаче на собеседовании senior-уровня и в каждом сложном UI-баге.

Модуль не про академическую теорию. Цель — дать ровно тот инструментарий, который нужен senior-фронтендеру: как устроен BST и почему его редко используют на фронте, как обходить DOM нативным API без лишней аллокации, как работает reconciliation (согласование) в React и почему `key={index}` ломает список, как устроена `MinHeap` внутри React Scheduler, и как найти LCA (Lowest Common Ancestor — наименьший общий предок) двух узлов в дереве категорий.

## Что студент будет уметь

- Реализовать бинарное дерево поиска (BST) с операциями insert, find, обход in-order — и объяснить, почему на фронте это чаще `Map`.
- Обойти DOM через `TreeWalker`/`NodeIterator` вместо `querySelectorAll('*')` и понимать границы Shadow DOM и iframe.
- Объяснить алгоритм diff (сравнение) в React: эвристики по типу и ключу, сложность O(n), роль Fiber, ловушки `key={index}`.
- Написать `MinHeap` с операциями push/pop/peek и объяснить, зачем он в React Scheduler и в задачах top-K.
- Найти наименьшего общего предка (LCA) двух узлов в дереве категорий за O(n·h) и обсудить, когда имеет смысл Tarjan-offline.
