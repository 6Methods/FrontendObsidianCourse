# A5.4.5 — Cycle detection и dependency graphs

## Зачем

Цикл в зависимостях — это почти всегда баг. `import/no-cycle` в ESLint, `madge --circular`, `dpdm` — всё это инструменты, которые ищут ровно одну вещь: циклы в directed graph (ориентированный граф — рёбра имеют направление) модулей. Turborepo и pnpm падают с ошибкой «cyclic dependency detected», когда в `package.json` пакеты зависят друг от друга по кругу. Redux-reducer, который импортирует селектор, который импортирует сам reducer, — классика циркулярной зависимости, дающей `undefined` в рантайме.

Знать, как работает cycle detection (обнаружение циклов), важно не только чтобы пользоваться готовыми тулами, но и чтобы понимать их вывод, правильно настраивать правила и, когда нужно, написать свою проверку под кастомные нужды.

## Главное

### Directed graph: три цвета DFS

Для directed graph стандартный подход — DFS (depth-first search — поиск в глубину) с тремя состояниями вершины:

- **white** — ещё не посещена.
- **gray** — посещена, но её поддерево ещё обрабатывается (вершина «на стеке» рекурсии).
- **black** — полностью обработана, DFS из неё завершил работу.

Алгоритм: стартуем DFS из любой white-вершины. При входе в вершину красим её gray, по завершении обработки всех потомков — black.

Ключевой момент: если во время DFS мы встречаем ребро в gray-вершину — это back edge (ребро назад — ребро в предка по дереву DFS), и это значит цикл. Нашли путь вперёд, который замыкается назад на уже «открытую» вершину.

Сложность — O(V+E), как и у обычного DFS.

### Directed graph: почему не любой визит в посещённую — цикл

Частая ошибка — считать, что любое ребро в visited-вершину — цикл. Это неверно. Cross edge (пересекающее ребро — в уже обработанную вершину другого поддерева) и forward edge (вперёд — в потомка по дереву DFS, не через прямое ребро) — не циклы. Они просто означают, что мы уже видели эту вершину.

Пример: граф `a → b, a → c, b → c`. DFS из `a`: идём в `b`, оттуда в `c`, возвращаемся, идём в `c` снова — но `c` уже black, не gray. Цикла нет.

Три цвета позволяют отличить back edge от cross/forward: back edge идёт только в gray, остальные — в black.

### Undirected graph: DFS + parent

В undirected graph (неориентированный — рёбра симметричны) три цвета не нужны. Каждое ребро представлено в adjacency list дважды (`a → b` и `b → a`). Если из вершины `v` мы идём в соседа `u`, а из `u` в списке есть `v` — это не цикл, это та же самая связь.

Алгоритм: при обходе в глубину передавать родителя. Если встретили уже visited-вершину, которая не является текущим родителем — это цикл.

```
dfs(v, parent):
  visited[v] = true
  for u in neighbors(v):
    if !visited[u]:
      if dfs(u, v) → cycle found, return true
    else if u != parent:
      return true  // обнаружен цикл
```

### Фронтовые кейсы

- **ESLint `import/no-cycle`** строит граф импортов в проекте и ищет циклы через DFS с тремя цветами. Каждый файл — вершина, каждый `import` — ребро.
- **Turborepo** анализирует `package.json` monorepo. Если `ui → utils → ui`, сборка не запустится.
- **Redux / Zustand circular reducers** — когда два слайса импортируют селекторы друг друга, `undefined` в одном из них гарантирован. Цикл модулей → module.exports ещё не заполнен на момент чтения.
- **Configuration cycles** — в CI-пайплайнах, в Nx task pipeline, в webpack-чанках.

### Инструменты

- **madge** (`npm i -g madge`): `madge --circular src/` выведет список циклов в проекте. Работает поверх AST.
- **dpdm** (`npx dpdm ./src`): показывает warnings по circular dependencies, красиво рисует trail.
- **eslint-plugin-import** с правилом `import/no-cycle`: то же самое, но проверяет прямо в IDE и в CI, падает билд при обнаружении цикла.

Все они внутри реализуют ровно то, что описано выше: строят граф импортов и запускают DFS-проверку.

### Output с trail of cycle

Хороший cycle detector не просто говорит «цикл есть», а показывает конкретную последовательность. Для этого в алгоритме DFS достаточно хранить путь (stack текущего обхода) и при обнаружении back edge вернуть отрезок от gray-вершины до текущей.

```
cycle: a → b → c → a
```

Такой trail — ключ к быстрому фиксу. Просто «есть цикл» в монорепо с 200 пакетами бесполезно: без trail непонятно, где искать.

## Пример кода

```ts
import type { Graph } from './graph';

type Color = 'white' | 'gray' | 'black';

export interface CycleResult<V> {
  hasCycle: boolean;
  trail?: V[]; // последовательность вершин, образующая цикл
}

// Cycle detection для directed graph через DFS с тремя цветами.
export function findCycle<V>(graph: Graph<V>): CycleResult<V> {
  const colors = new Map<V, Color>();
  for (const v of graph.vertices()) colors.set(v, 'white');

  // Путь текущего обхода, чтобы вырезать trail при обнаружении цикла.
  const path: V[] = [];

  for (const start of graph.vertices()) {
    if (colors.get(start) !== 'white') continue;

    const trail = dfsFindCycle(graph, start, colors, path);
    if (trail) return { hasCycle: true, trail };
  }

  return { hasCycle: false };
}

function dfsFindCycle<V>(
  graph: Graph<V>,
  v: V,
  colors: Map<V, Color>,
  path: V[],
): V[] | null {
  colors.set(v, 'gray');
  path.push(v);

  for (const neighbor of graph.neighbors(v)) {
    const color = colors.get(neighbor);

    if (color === 'white') {
      const trail = dfsFindCycle(graph, neighbor, colors, path);
      if (trail) return trail;
    } else if (color === 'gray') {
      // Back edge → цикл. Вырезаем отрезок пути от neighbor до конца.
      const start = path.indexOf(neighbor);
      return [...path.slice(start), neighbor];
    }
    // color === 'black' — cross edge или forward edge, не цикл.
  }

  colors.set(v, 'black');
  path.pop();
  return null;
}

// Пример: цикл в Redux-слайсах.
// authSlice → userSlice, userSlice → notificationsSlice, notificationsSlice → authSlice.
const slices = new Graph<string>(true);
slices.addEdge('authSlice', 'userSlice');
slices.addEdge('userSlice', 'notificationsSlice');
slices.addEdge('notificationsSlice', 'authSlice');

const result = findCycle(slices);
if (result.hasCycle) {
  console.error('Circular dependency detected:', result.trail!.join(' → '));
  // Output: authSlice → userSlice → notificationsSlice → authSlice
}
```

## Частые ошибки

1. **Путать back edge и cross edge.** Любой визит в посещённую вершину ≠ цикл. В directed graph цикл — только back edge, то есть ребро в gray-вершину. Без трёх цветов легко дать false-positive на DAG с пересечениями (нескольких путей к одной вершине).

2. **Давать false-positive на undirected через gray-проверку.** В undirected graph ребро `a → b` и `b → a` — одно и то же. Если применить DFS с тремя цветами к undirected без учёта parent, любое ребро будет «циклом». Использовать DFS + parent для undirected.

3. **Не возвращать trail, только булевый результат.** `hasCycle: true` в большом проекте бесполезно. Возвращать путь цикла — обязательно для полезного output. madge и dpdm делают это по умолчанию.

4. **Не учитывать все компоненты связности.** Если граф несвязный, DFS из одной вершины пропустит целые куски. Нужно перебирать все вершины и запускать DFS из каждой white.

5. **Рекурсивный DFS на огромном графе импортов.** Монорепо с десятками тысяч файлов может дать стек рекурсии глубже Node.js-лимита (~10 000 фреймов). На таких размерах — итеративный DFS с явным стеком. Для большинства проектов рекурсия достаточна.

## Задания

### 1. Реализовать findCycle

Перенеси код примера в свой проект. Протестируй на трёх графах: ациклический DAG, граф с прямым циклом `a → b → a`, граф с непрямым циклом `a → b → c → a`. Убедись, что trail правильный.

### 2. Undirected cycle detector

Реализуй отдельную функцию `findCycleUndirected(graph)` через DFS + parent. Протестируй на цикле в треугольнике `a — b — c — a` и на дереве (без цикла).

### 3. Запустить madge на своём проекте

`npx madge --circular --extensions ts,tsx src/`. Если циклы есть — изучи каждый и реши, как бы ты их убрал (вынести общее в третий модуль, ленивый импорт, перестроить API).

### 4. Написать свой ESLint-правило-lite

Просканируй директорию через `@typescript-eslint/parser`, построй граф импортов, запусти `findCycle`. Для каждого цикла выведи предупреждение с trail. Это — упрощённая версия `import/no-cycle`.
