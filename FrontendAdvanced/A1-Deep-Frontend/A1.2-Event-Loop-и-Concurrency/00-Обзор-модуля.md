# A1.2 — Event Loop и Concurrency

## Зачем

JS однопоточный — знают все. Но что это значит в 2026, когда есть Web Workers, Service Workers, Scheduler API, SharedArrayBuffer, Atomics, микротаски, `scheduler.yield()`? Middle-разработчик пишет `await fetch(...)`, но часто не понимает, **когда** колбэк оттуда исполнится, почему `Promise.resolve().then(...)` запускается раньше `setTimeout(..., 0)`, и что вообще происходит, если на странице анимация + тяжёлый парсинг + React render одновременно.

Этот модуль даёт модель event loop, с которой ты сможешь осознанно выносить работу в workers, правильно использовать микротаски/`requestIdleCallback`/`scheduler.postTask`, и понимать, почему React Concurrent — не «магия», а грамотное использование тех же примитивов.

## Что изучаем

- **Event loop в деталях** — task queue, rendering steps, `scheduler.yield()`.
- **Microtasks vs Macrotasks** — promise-queue, starvation, MutationObserver.
- **Scheduler и idle callback** — `requestAnimationFrame`, `requestIdleCallback`, `scheduler.postTask` с приоритетами.
- **Web Workers** — реальный параллелизм, transferable objects, когда стоит, когда нет.
- **SharedArrayBuffer и Atomics** — shared memory, COOP/COEP, race conditions, WASM threads.
- **Service Workers как runtime** — background-процесс, Cache API, Background Sync, Push.
