# A1.3 — Memory и GC

## Зачем

«У приложения утечка» — фраза, которую middle слышит часто, но редко понимает в деталях. Где эта утечка, кто её держит, как найти, как починить, и вообще почему JS — язык с GC — умудряется «течь»? В этом модуле ты получишь модель памяти V8, поймёшь, как работают GC-алгоритмы Chrome, узнаешь пять типичных паттернов утечек на реальном коде, и научишься использовать Chrome Memory Profiler так, чтобы за 15 минут найти leak в любом продакшн-приложении.

Без этой темы senior-инженер не senior. С ней — ты единственный в команде, кто может починить memory-баг, который «вроде есть, вроде нет».

## Что изучаем

- **Модель памяти JS** — stack vs heap, reference vs value, почему функция — object.
- **V8 hidden classes** — как движок оптимизирует dynamic objects, и почему порядок инициализации важен.
- **GC алгоритмы** — mark-and-sweep, generational GC, minor vs major, incremental marking.
- **Memory leaks** — 6 типичных паттернов: closures, listeners, detached DOM, setInterval, globals, stale closures в React.
- **Chrome Memory Profiler** — Heap snapshot, Allocation timeline, 3-snapshot diff, retainers.
