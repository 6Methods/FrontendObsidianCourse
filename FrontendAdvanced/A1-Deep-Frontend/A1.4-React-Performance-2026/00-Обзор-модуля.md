# A1.4 — React Performance 2026

## Зачем

Три модуля назад ты понял, как работает браузер. Теперь применим это к React — но к React 2026, а не к React, в котором ещё нужно расставлять `useMemo`/`useCallback` руками. В 2026 у тебя:

- **React Compiler** — автоматическая мемоизация, 95% ручного `memo`/`useMemo`/`useCallback` не нужны.
- **Suspense streaming** — HTML приезжает на клиент чанками, параллельно data fetching.
- **Selective hydration** — шабка гидрируется не монолитно, а по Suspense-бэкам, приоритизируя user interaction.
- **Transitions** — `useTransition` и `useDeferredValue` превращают медленные обновления в non-urgent, которые можно прервать.

Этот модуль — финал Track 1. После него ты будешь понимать, что именно компилятор и runtime оптимизируют, когда они ломаются, и как это видно в React Profiler и Chrome Performance panel одновременно.
