# A4.6 — Web Security

## Зачем модуль

Web security 2026 — не спец-тема для бэкендера, это hygiene (базовая гигиена разработки) для любого frontend senior'а. Уязвимости, за которые раньше отвечал «тот парень из security-team», теперь попадают в PR-ревью фронтенд-разработчика: `dangerouslySetInnerHTML` без санитизации, access token в `localStorage`, `SameSite=None` на production-cookie, CSP (Content Security Policy — политика безопасности содержимого) с `'unsafe-inline'` «чтобы заработало».

Современные фреймворки — Next.js, Remix, SvelteKit — закрывают базовые XSS (Cross-Site Scripting — межсайтовый скриптинг) через автоматический escaping (экранирование — замена опасных символов безопасными эквивалентами), но пробиваются на любой user-generated (созданной пользователем) разметке, на неправильном OAuth (Open Authorization — открытый протокол авторизации) flow, на забытых security headers (заголовках безопасности). Этот модуль — про минимально необходимый security-слой, который senior-фронтендер обязан держать в голове при написании и ревью кода.

Модуль не заменяет OWASP-курс и не готовит к penetration testing (тестированию на проникновение — контролируемой атаке на систему). Он фиксирует практический инструментарий: как защититься от XSS и настроить CSP, как работают CSRF-атаки и SameSite-cookies, где хранить JWT, какие headers обязательны на production в 2026 году.

## Что студент будет уметь

- Распознать XSS-уязвимость в React-коде и закрыть её через санитизацию DOMPurify или Trusted Types API.
- Настроить CSP с nonce (одноразовым значением) и `strict-dynamic` в Next.js middleware, включая report-only режим для постепенного rollout (поэтапного развёртывания).
- Объяснить разницу CSRF и CORS, правильно выставить SameSite cookies, не доверять CORS как защите от CSRF.
- Выбрать OAuth flow для SPA vs machine-to-machine, правильно хранить access/refresh токены, применить BFF (Backend-for-Frontend — бэкенд, обслуживающий один конкретный фронтенд) паттерн.
- Собрать обязательный набор security headers 2026 (HSTS, CSP, Referrer-Policy, Permissions-Policy, COOP/COEP) и прогнать production-домен через audit-инструменты.
