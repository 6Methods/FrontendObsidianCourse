# A4.3 — CI/CD глубже

## Зачем

В A3.2.6 мы разобрали CI для monorepo: affected-стратегию, remote cache Turborepo, шардирование. Это был прикладной срез для конкретного инструментария. Здесь задача шире — понять GitHub Actions как платформу и освоить паттерны, которые работают независимо от того, monorepo это или обычный репозиторий.

Команды, у которых «CI просто работает», сталкиваются с одними и теми же проблемами: секреты утекают через логи, прогоны занимают 20 минут из-за отсутствия кэша, preview-деплои требуют ручной работы, релизный процесс — это 30-шаговая вики-страница. CI/CD — это не «настроить один раз и забыть», это инфраструктура, которую нужно проектировать.

## Что студент будет уметь

После модуля студент сможет:

- Объяснять иерархию GitHub Actions: workflow — job — step — event — runner; выбирать правильный тип action для задачи.
- Настраивать build matrix по версиям Node и шардам тестов с управляемым `fail-fast`.
- Применять `actions/cache@v4` с корректными cache key и стратегией инвалидации.
- Реализовать preview deployments на собственном VPS с изоляцией per-PR через docker-compose.
- Управлять секретами через environment-scoped secrets и OIDC-федерацию без долгоживущих ключей.
- Автоматизировать релизы через semantic-release или changesets, включая подпись тегов и npm provenance.
