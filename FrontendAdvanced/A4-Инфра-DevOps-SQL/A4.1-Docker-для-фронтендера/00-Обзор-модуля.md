# A4.1 — Docker для фронтендера

## Зачем модуль

Фронтенд-разработчик 2024–2025 годов редко останавливается на этапе `git push`. Next.js-приложение нужно упаковать, передать в CI (Continuous Integration — непрерывная интеграция, автоматизированная система проверки и сборки кода), развернуть на сервере и убедиться, что оно ведёт себя там так же, как на твоём MacBook. Docker решает именно эту проблему: один и тот же контейнер (изолированная единица запуска — процесс со своей файловой системой, сетью и переменными окружения) работает одинаково везде.

Модуль не про DevOps в широком смысле. Он про минимально необходимый инструментарий для senior-фронтендера: собрать образ (image — неизменяемый слепок файловой системы, из которого запускаются контейнеры) Next.js без лишних мегабайт, запустить локальную Postgres одной командой, правильно завершить процесс под нагрузкой и прогнать сборку в GitHub Actions.

## Что студент будет уметь

- Объяснить, чем контейнер отличается от виртуальной машины, и когда Docker не нужен.
- Написать production-ready Dockerfile для Next.js 15 с multi-stage build (многоэтапная сборка — разделение сборки на несколько независимых шагов, каждый со своим базовым образом) и non-root пользователем.
- Поднять локальное окружение (app + Postgres + Redis) через `compose.yaml` с правильными health check-ами (проверками готовности сервиса).
- Реализовать graceful shutdown (корректное завершение — приём SIGTERM, ожидание inflight-запросов, закрытие соединений) в Next.js App Router.
- Настроить GitHub Actions job с кэшем слоёв и multi-arch (мультиархитектурным) билдом для ARM.
