# A4.7.3 — RTL и Intl API

## Зачем

Если продукт выходит на Ближний Восток, «перевести тексты на арабский» — это 20% работы. Остальные 80% — layout. Арабский, иврит, фарси, урду пишутся справа налево (RTL — right-to-left). Это меняет не только направление текста, но и всю пространственную метафору интерфейса: стрелка «назад» указывает вправо, меню открывается слева (а не справа), прогресс-бар заполняется справа налево, аватарка рядом с именем оказывается с другой стороны. Вёрстка с `margin-left`, `padding-right`, `text-align: left` ломается полностью.

Параллельная боль — форматирование дат, чисел, валют, списков. `new Date().toLocaleString()` возвращает разный результат на сервере (Linux в UTC) и на клиенте (Safari пользователя в Jerusalem). Результат — hydration mismatch (ошибка: серверный HTML не совпадает с клиентским). Если использовать `moment.js` или `date-fns` для форматирования — тянешь 100+ КБ на клиент, когда нативный `Intl.DateTimeFormat` делает то же самое за 0 КБ и работает быстрее.

Этот урок — про две вещи: как сделать RTL-совместимую вёрстку через logical CSS properties, и как использовать `Intl.*` API для форматирования без сторонних библиотек и без hydration-проблем.

## Главное

### `dir="rtl"` — корневой атрибут

RTL включается одним атрибутом на `<html>`:

```html
<html lang="ar" dir="rtl">
```

Браузер автоматически:
- Разворачивает горизонтальное направление текста.
- Разворачивает direction в `flex-direction: row` (элементы идут справа налево).
- Разворачивает порядок в `text-align: start` / `end`.
- Инвертирует значения `margin-inline-start` / `margin-inline-end`.

Критично: `dir` должен устанавливаться на сервере, до гидратации. В App Router — в `layout.tsx` на основе `[locale]` параметра.

```tsx
const rtlLocales = ['ar', 'he', 'fa', 'ur'];
const dir = rtlLocales.includes(locale) ? 'rtl' : 'ltr';

return <html lang={locale} dir={dir}>...</html>;
```

Переключение dir на клиенте после гидратации — плохая идея: flash макета, сбой layout, hydration error.

### Logical CSS properties

Физические свойства (`margin-left`, `padding-right`, `border-left`) привязаны к физическому направлению. В RTL `margin-left: 16px` работает как будто это `margin-right` для LTR — т.е. ломает отступ.

Logical properties работают относительно направления письма:

| Physical | Logical | В RTL превращается в |
|---|---|---|
| `margin-left` | `margin-inline-start` | `margin-right` |
| `margin-right` | `margin-inline-end` | `margin-left` |
| `padding-top` | `padding-block-start` | `padding-top` (не меняется) |
| `padding-bottom` | `padding-block-end` | `padding-bottom` |
| `left: 0` | `inset-inline-start: 0` | `right: 0` |
| `border-left` | `border-inline-start` | `border-right` |
| `text-align: left` | `text-align: start` | `text-align: right` |

Sizing тоже имеет logical-варианты: `inline-size` (width по направлению письма), `block-size` (height). На практике `width`/`height` чаще оставляют, потому что в большинстве дизайнов блок занимает ширину контейнера.

`text-align: start` / `end` работают в обоих направлениях — менять не надо. `text-align: left` ломается в RTL.

### Tailwind logical utilities

Tailwind 3.3+ поддерживает logical properties:

- `ms-4` → `margin-inline-start: 1rem` (вместо `ml-4`).
- `me-4` → `margin-inline-end: 1rem` (вместо `mr-4`).
- `ps-6` → `padding-inline-start: 1.5rem`.
- `pe-6` → `padding-inline-end: 1.5rem`.
- `start-0` → `inset-inline-start: 0`.
- `end-0` → `inset-inline-end: 0`.
- `text-start` / `text-end` — выравнивание.

Для bi-directional проектов — правило: любой `ml-`/`mr-`/`pl-`/`pr-` переводится в `ms-`/`me-`/`ps-`/`pe-`. `left-0` / `right-0` → `start-0` / `end-0`.

Иконки, которые должны зеркалиться (стрелки, направляющие), — через `rtl:-scale-x-100` (инвертировать по X в RTL) или SVG с учётом direction. Иконки, которые не зеркалятся (цифры, логотипы, индикаторы качества) — оставлять как есть.

### `Intl.DateTimeFormat`

Нативное форматирование дат, ноль байт на бандл:

```ts
new Intl.DateTimeFormat('ru-RU', {
  year: 'numeric',
  month: 'long',
  day: 'numeric',
}).format(new Date('2026-04-20'));
// "20 апреля 2026 г."

new Intl.DateTimeFormat('en-US', { dateStyle: 'full' }).format(new Date('2026-04-20'));
// "Monday, April 20, 2026"

new Intl.DateTimeFormat('ar-EG', { dateStyle: 'long' }).format(new Date('2026-04-20'));
// "20 أبريل 2026"
```

Относительные даты («2 часа назад»):

```ts
const rtf = new Intl.RelativeTimeFormat('ru', { numeric: 'auto' });
rtf.format(-2, 'hour'); // "2 часа назад"
rtf.format(1, 'day');   // "завтра" (numeric: 'auto' — особые формы)
rtf.format(-7, 'day');  // "7 дней назад"
```

Для более богатого API (`formatDistance`, `formatRelative`) — `date-fns` с `date-fns/locale`, но для базового форматирования `Intl` достаточно.

### `Intl.NumberFormat`

Числа, валюты, проценты, единицы измерения:

```ts
new Intl.NumberFormat('ru-RU').format(1234567.89); // "1 234 567,89"
new Intl.NumberFormat('en-US').format(1234567.89); // "1,234,567.89"
new Intl.NumberFormat('de-DE').format(1234567.89); // "1.234.567,89"

// Валюта
new Intl.NumberFormat('ru-RU', {
  style: 'currency',
  currency: 'RUB',
}).format(2499); // "2 499,00 ₽"

new Intl.NumberFormat('en-US', {
  style: 'currency',
  currency: 'USD',
}).format(2499); // "$2,499.00"

// Проценты
new Intl.NumberFormat('ru-RU', { style: 'percent' }).format(0.1523); // "15 %"

// Compact — "1.2K", "3.4M"
new Intl.NumberFormat('en-US', { notation: 'compact' }).format(1234); // "1.2K"
new Intl.NumberFormat('ru-RU', { notation: 'compact' }).format(1234); // "1,2 тыс."

// Unit — "100 km/h"
new Intl.NumberFormat('en-US', {
  style: 'unit',
  unit: 'kilometer-per-hour',
}).format(100); // "100 km/h"
```

### `Intl.ListFormat`

Объединение массива в локализованный список:

```ts
const items = ['яблоко', 'груша', 'банан'];

new Intl.ListFormat('en', { style: 'long', type: 'conjunction' }).format(items);
// "яблоко, груша, and банан"

new Intl.ListFormat('ru', { style: 'long', type: 'conjunction' }).format(items);
// "яблоко, груша и банан"

// Disjunction — "или"
new Intl.ListFormat('ru', { type: 'disjunction' }).format(items);
// "яблоко, груша или банан"
```

До появления `ListFormat` разработчики писали `items.join(', ') + ' и ' + last` — ломалось в английском (Oxford comma), не поддерживало арабский.

### `Intl.Collator`

Сортировка строк с учётом локали:

```ts
const strings = ['ёлка', 'енот', 'ель'];

// Без Collator — сортировка по Unicode code points
strings.sort(); // ["ёлка", "ель", "енот"] — ё после я по кодам

// С Collator — правильно для русского
strings.sort(new Intl.Collator('ru').compare);
// ["ель", "енот", "ёлка"] — ё трактуется как е с диакритикой

// В немецком ß сортируется как ss
['Straße', 'Strasse', 'Stress'].sort(new Intl.Collator('de').compare);
```

Опции: `sensitivity` (`base`, `accent`, `case`, `variant`) — чувствительность к регистру и диакритикам. `numeric: true` — числа в строках сортируются численно (`item2` < `item10`).

### Hydration mismatch и фиксация локали

Главная ловушка — `toLocaleString()` без явной локали:

```tsx
// ПЛОХО — hydration mismatch
<span>{new Date().toLocaleString()}</span>
// Сервер (Linux, en-US по умолчанию): "4/20/2026, 12:00:00 PM"
// Клиент (Safari пользователя, ru-RU): "20.04.2026, 12:00:00"
// → React: "Text content did not match"
```

Две причины:
1. **Локаль по умолчанию** зависит от окружения. Сервер использует `LANG` env var (обычно `C` или `en_US.UTF-8`), клиент — пользовательские настройки браузера.
2. **Timezone** зависит от системы. Сервер — UTC, клиент — Europe/Moscow.

Правильно — всегда фиксировать:

```tsx
<span>
  {new Intl.DateTimeFormat('ru-RU', {
    timeZone: 'Europe/Moscow',
    dateStyle: 'long',
  }).format(new Date())}
</span>
```

Или — форматирование только на клиенте через `useEffect` / `'use client'` (но тогда появляется плейсхолдер до гидратации). Или — передать уже форматированную строку с сервера.

### Когда `Intl` недостаточно

- **Парсинг дат** (`Date.parse('20 апреля 2026')`) — `Intl` не умеет, нужен `date-fns/parse` или `chrono-node`.
- **Арифметика дат** (`addDays`, `differenceInBusinessDays`) — `date-fns`, `dayjs`.
- **Таймзоны** за пределами `Intl.DateTimeFormat` (конвертация, DST-aware арифметика) — `date-fns-tz` или `Temporal` API (stage 3, ещё не во всех браузерах).
- **Собственные символы валют** (несколько валют в одной строке с выравниванием) — ручное форматирование.

Для 90% задач Intl достаточно. Выкидывать `moment.js` (70 КБ gzipped) из бандла в пользу `Intl` — классическая оптимизация.

## Пример кода

### Layout с dir на основе локали

```tsx
// app/[locale]/layout.tsx
import { NextIntlClientProvider } from 'next-intl';
import { getMessages } from 'next-intl/server';

const rtlLocales = new Set(['ar', 'he', 'fa', 'ur']);

export default async function LocaleLayout({
  children,
  params,
}: {
  children: React.ReactNode;
  params: Promise<{ locale: string }>;
}) {
  const { locale } = await params;
  const messages = await getMessages();
  const dir = rtlLocales.has(locale) ? 'rtl' : 'ltr';

  return (
    <html lang={locale} dir={dir}>
      <body>
        <NextIntlClientProvider locale={locale} messages={messages}>
          {children}
        </NextIntlClientProvider>
      </body>
    </html>
  );
}
```

### Logical CSS для карточки товара

```tsx
// components/ProductCard.tsx
export function ProductCard({ title, price, imageUrl }: Props) {
  return (
    <article className="flex gap-4 p-4 border rounded">
      {/* ms-0 me-4 — в LTR отступ справа, в RTL — слева */}
      <img src={imageUrl} alt="" className="size-16 rounded ms-0 me-4" />

      <div className="flex-1">
        {/* text-start работает в обоих направлениях */}
        <h3 className="text-lg font-semibold text-start">{title}</h3>

        {/* ps-2 border-s — отступ и граница от начала направления */}
        <p className="ps-2 border-s-2 border-gray-200 text-start">
          {new Intl.NumberFormat('ru-RU', {
            style: 'currency',
            currency: 'RUB',
          }).format(price)}
        </p>
      </div>

      {/* Стрелка: rtl:-scale-x-100 зеркалит в RTL */}
      <button className="ms-auto">
        <ArrowIcon className="size-4 rtl:-scale-x-100" />
      </button>
    </article>
  );
}
```

### Серверный хелпер форматирования с фиксированной таймзоной

```ts
// lib/format.ts
type Locale = 'ru' | 'en' | 'ar';

export function formatDate(
  date: Date | string,
  locale: Locale,
  options: Intl.DateTimeFormatOptions = {},
) {
  return new Intl.DateTimeFormat(`${locale}-${localeToRegion(locale)}`, {
    timeZone: 'UTC',
    dateStyle: 'long',
    ...options,
  }).format(typeof date === 'string' ? new Date(date) : date);
}

export function formatPrice(amount: number, locale: Locale) {
  const currency = { ru: 'RUB', en: 'USD', ar: 'AED' }[locale];
  return new Intl.NumberFormat(`${locale}-${localeToRegion(locale)}`, {
    style: 'currency',
    currency,
  }).format(amount);
}

export function formatList(items: string[], locale: Locale) {
  return new Intl.ListFormat(locale, {
    style: 'long',
    type: 'conjunction',
  }).format(items);
}

function localeToRegion(locale: Locale): string {
  return { ru: 'RU', en: 'US', ar: 'AE' }[locale];
}
```

### Сортировка с Collator в клиентском компоненте

```tsx
'use client';

import { useLocale } from 'next-intl';
import { useMemo } from 'react';

export function SortedList({ items }: { items: string[] }) {
  const locale = useLocale();

  const sorted = useMemo(() => {
    const collator = new Intl.Collator(locale, {
      sensitivity: 'base',
      numeric: true,
    });
    return [...items].sort(collator.compare);
  }, [items, locale]);

  return (
    <ul>
      {sorted.map((item) => <li key={item}>{item}</li>)}
    </ul>
  );
}
```

## Частые ошибки

1. **`toLocaleString()` без явной локали.** Результат разный на сервере и клиенте → hydration mismatch. Всегда `new Intl.DateTimeFormat('ru-RU', {...})` с явной локалью.

2. **`Date` без `timeZone`.** Сервер Linux в UTC, клиент в Europe/Moscow — одна и та же дата рендерится с разницей в 3 часа. Всегда указывать `timeZone` или рендерить только на клиенте.

3. **Физические CSS-свойства в bi-directional проектах.** `margin-left: 16px`, `padding-right: 8px`, `left: 0` — ломают RTL. Переход на `margin-inline-start`, `padding-inline-end`, `inset-inline-start`.

4. **`text-align: left` / `right` вместо `start` / `end`.** `start` автоматически становится `right` в RTL и `left` в LTR. Литералы `left`/`right` — нет.

5. **Зеркаление иконок, которые зеркалить нельзя.** Логотипы, цифры, звёзды рейтинга — не переворачивать в RTL. Стрелки, направляющие, прогресс-бары — переворачивать. Разбираться для каждой иконки отдельно.

6. **`Intl` не поддерживает парсинг.** `new Date('20 апреля 2026')` не работает — `Date` парсит только ISO и несколько legacy-форматов. Для парсинга — `date-fns/parse` или получать из сервера в ISO-формате.

7. **Тянуть `moment.js` ради форматирования.** 70 КБ ради того, что `Intl.DateTimeFormat` делает за 0 КБ. Выкинуть при первой возможности.

## Задания

### 1. RTL-версия существующего интерфейса

Возьми страницу на 5+ компонентов с физическими CSS-свойствами. Поменяй `margin-left/right`, `padding-left/right`, `left`, `right`, `text-align: left` на logical-эквиваленты. Добавь `dir="rtl"` на `<html>`, визуально проверь, что всё зеркалится корректно (панель, карточки, кнопки). Скриншоты до/после.

### 2. Форматирование через `Intl` без библиотек

В проекте с `date-fns` или `moment` перепиши три места: форматирование даты, цены с валютой, относительного времени («2 часа назад»). Удали зависимость (если нет других применений). Измерь разницу в bundle size через `@next/bundle-analyzer`.

### 3. Сортировка с Collator

Создай страницу со списком 20 слов на русском, включая `ёлка`, `енот`, `ель`. Отсортируй через `Array.sort()` — увидишь неправильный порядок. Перепиши через `Intl.Collator('ru').compare`. Добавь переключатель `sensitivity: 'base' | 'variant'` и проверь разницу.

### 4. Hydration mismatch и фикс

Создай компонент, который рендерит `new Date().toLocaleString()` без локали и без timezone. Запусти `next build` + `next start` — наблюдай hydration error в консоли. Исправь через явную локаль и `timeZone: 'UTC'`. Убедись, что ошибки нет.

### 5. Compact notation для статистики

Карточка «2.4M подписчиков» / «1.2K лайков». Используй `Intl.NumberFormat` с `notation: 'compact'` для трёх локалей: `en-US` (2.4M), `ru-RU` (2,4 млн), `de-DE` (2,4 Mio.). Сравни визуально, какие формы получаются.
